Shell Operations

14 June, 2026 Linux Administration Jobaer Hossain

লিনাক্স সিস্টেম অ্যাডমিনিস্ট্রেশনের আসল ক্ষমতা লুকিয়ে আছে এর ব্ল্যাক স্ক্রিন বা টার্মিনালের ভেতরে। আমরা যখন গ্রাফিক্যাল ইন্টারফেস বা GUI ছাড়া কোনো রিমোট ক্লাউড সার্ভার পরিচালনা করি, তখন ওএস কার্নেলকে কমান্ড দেওয়ার জন্য আমাদের একটি শক্তিশালী ইন্টারমিডিয়েট প্রোগ্রামের সাহায্য নিতে হয়, যাকে বলা হয় Shell। লিনাক্স এনভায়রনমেন্টে ডাটা প্রসেসিং, স্ক্রিপ্টিং, ফাইল ম্যানিপুলেশন এবং অটোমেশনের জন্য Shell Operations জানা প্রতিটি সিস্টেম ইঞ্জিনিয়ার এবং সাইবার সিকিউরিটি প্রফেশনালের জন্য একদম প্রাথমিক ও অলঙ্ঘনীয় কারিগরি দক্ষতা।

আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা আলোচনা করবো লিনাক্স শেলের ব্যাকএন্ড মেকানিজম, প্রয়োজনীয় অপারেশনসমূহ, ইনপুট/আউটপুট রিডাইরেকশন এবং কীভাবে একাধিক কমান্ডকে পাইপলাইনের মাধ্যমে যুক্ত করে জটিল সব কাজ এক লাইনে সমাধান করা যায় তা নিয়ে।

১. Linux Shell কী এবং এটি কীভাবে কাজ করে?

Shell হলো একটি বিশেষ ইউজার-স্পেস অ্যাপ্লিকেশন (Command Language Interpreter) যা ব্যবহারকারীর কাছ থেকে ক্যারেক্টার বা টেক্সট ভিত্তিক কমান্ড গ্রহণ করে, সেটির সিনট্যাক্স বা ব্যাকরণ পরীক্ষা করে এবং কার্নেলের (Kernel) বোঝার উপযোগী সিস্টেমে রূপান্তর করে এক্সিকিউট করে।

সহজ কথায়, সেল হলো কার্নেলের ওপর একটি সুরক্ষামূলক মোড়ক বা খোলস। এটি সরাসরি হার্ডওয়্যারে ইউজারকে এক্সেস না দিয়ে একটি নিয়ন্ত্রিত এনভায়রনমেন্ট তৈরি করে দেয়। লিনাক্স জগতে বেশ কয়েক ধরনের শেল রয়েছে:

  • Bash (Bourne Again Shell): এটি লিনাক্সের সবচেয়ে স্ট্যান্ডার্ড এবং ডিফল্ট শেল। প্রায় সব এন্টারপ্রাইজ ডিস্ট্রোতে (যেমন RHEL, Ubuntu) এটি প্রি-ইনস্টলড থাকে।
  • Zsh (Z Shell): এটি বাশের একটি আধুনিক ও কাস্টমাইজড রূপ, যা অটো-কমপ্লিশন এবং প্লাগইন সাপোর্টের জন্য ডেভেলপারদের কাছে অত্যন্ত জনপ্রিয়।

রিয়েল-লাইফ সিনারিও (The Voice Assistant Analogy)

শেল অপারেশনের ভূমিকা সহজে বোঝার জন্য একটি আধুনিক স্মার্টফোনের ভয়েস অ্যাসিস্ট্যান্টের (যেমন Google Assistant বা Siri) কথা চিন্তা করুন:

  • আপনার ফোনের ভেতরের জটিল সার্কিট, ক্যামেরা লেন্স বা ওএস সফটওয়্যার হলো Kernel/Hardware।
  • আপনি যখন মুখে বলেন "Hey Siri, call my manager", তখন এই ভয়েস অ্যাসিস্ট্যান্ট প্রোগ্রামটি আপনার ভাষা প্রসেস করে ফোনের ভেতরের ডায়ালার সার্কিটকে অ্যাক্টিভেট করে দেয়। এখানে অ্যাসিস্ট্যান্টটিই হলো Shell। সে নিজে ফিজিক্যালি কল করছে না, কিন্তু আপনার ইনপুটকে ফোনের ইন্টারনাল মেকানিজমে অনুবাদ করে দিচ্ছে।

২. Core Shell Operations & Navigation

টার্মিনালে কাজ করার সময় শেল কয়েকটি সুনির্দিষ্ট এনভায়রনমেন্ট প্যারামিটার এবং প্রম্পট সাইন (Prompt Signs) প্রদর্শন করে:

  • The Prompt ($ vs #): আপনি যখন টার্মিনালে $ সাইন দেখবেন, তার মানে আপনি একজন সাধারণ ইউজার (Regular User) হিসেবে লগইন আছেন। আর যদি সেখানে # সাইন থাকে, তবে বুঝতে হবে আপনি সিস্টেমের সর্বোচ্চ ক্ষমতাশালী root বা সুপারের ইউজার মোডে আছেন।
  • Command Syntax: শেলের প্রতিটি কমান্ড একটি নির্দিষ্ট ফরম্যাট মেনে চলে: Command [-options] [arguments]। যেমন— ls -lh /var/log। এখানে ls হলো কমান্ড, -lh হলো অপশন বা ফ্ল্যাগ এবং /var/log হলো আর্গুমেন্ট বা নির্দিষ্ট পাথ।

৩. Input, Output, and Error Redirection (I/O Redirection)

লিনাক্স শেলের অন্যতম শক্তিশালী অপারেশন হলো ইনপুট এবং আউটপুট রিডাইরেকশন। ডিফল্টভাবে, একটি কমান্ড তার ইনপুট নেয় কিবোর্ড থেকে এবং আউটপুট ও এরর প্রদর্শন করে স্ক্রিনে বা মনিটরে। শেলের ভাষায় এদের ৩টি স্ট্যান্ডার্ড ডেটা স্ট্রিম (Data Streams) বলা হয়, যাদের নিজস্ব File Descriptor (FD) নাম্বার থাকে:

Stream Name File Descriptor (FD) Default Physical Device Shell Redirection Operator
stdin (Standard Input) 0 Keyboard <
stdout (Standard Output) 1 Screen / Monitor > (Overwrite) বা >> (Append)
stderr (Standard Error) 2 Screen / Monitor 2>
[Image illustration of Standard Input, Output, and Error streams redirection to physical files]

ক্লিপবোর্ড বা ফাইলে ডাটা রিডাইরেক্ট করার প্র্যাকটিক্যাল মেকানিজম:

১. Overwrite Output (`>`): কোনো কমান্ডের স্ক্রিন আউটপুটকে মনিটরে না দেখিয়ে সরাসরি একটি ফাইলে সেভ করার অপারেশন। এটি ফাইলের পুরনো ডাটা মুছে নতুন ডাটা রাইট করে।

$ ls -lh /etc > etc_files_list.txt

২. Append Output (`>>`): এটি পুরনো ডাটা অক্ষুণ্ণ রেখে ফাইলের একদম শেষ লাইনের পর নতুন আউটপুট ডাটা যোগ (Append) করে।

$ date >> uptime_history.txt

৩. Redirecting Errors (`2>`): অনেক সময় কমান্ড রান করার সময় স্ক্রিনে প্রচুর এরর মেসেজ আসে যা আমরা মূল আউটপুট থেকে আলাদা করতে চাই। এররগুলো একটি নির্দিষ্ট লগে জমানোর কমান্ড:

$ find / -name "secret.txt" 2> error_log.txt

৪. Pipelines (The Pipe Operator `|`)

শেল অপারেশনের সবচেয়ে জাদুকরী এবং অ্যাডভান্সড ফিচার হলো Pipelines, যা একটি খাড়া দাগ বা পাইপ ক্যারেক্টার (|) দিয়ে প্রকাশ করা হয়।

কাজের মেকানিজম: এটি প্রথম কমান্ডের আউটপুটকে (stdout) স্ক্রিনে না পাঠিয়ে সরাসরি দ্বিতীয় কমান্ডের ইনপুট (stdin) হিসেবে পাস করে দেয়। এইভাবে আপনি চেইন রিঅ্যাকশনের মতো একের পর এক অসংখ্য কমান্ড যুক্ত করে বিশাল ডেটা ফিল্টারিং করতে পারেন।

[Image schematic diagram of Linux Pipe operation connecting Command 1 Output to Command 2 Input]

প্রফেশনাল রিয়েল-লাইফ পাইপলাইন এক্সাম্পল:

ধরুন আপনার ওয়েব সার্ভারের অ্যাক্সেস লগ ফাইলে (access.log) হাজার হাজার ট্রাফিক ডেটা আছে। আপনি দেখতে চান নির্দিষ্ট একটি আইপি (যেমন 192.168.10.50) কতবার আপনার সার্ভার হিট করেছে:

$ cat /var/log/nginx/access.log | grep "192.168.10.50" | wc -l

স্টেপ বাই স্টেপ টেকনিক্যাল ব্যাখ্যা:

  • cat কমান্ডটি পুরো লগ ফাইলের ডাটা টেক্সট আকারে বের করে পাইপের মুখে দেয়।
  • grep কমান্ডটি সেই আউটপুট থেকে অন্য সব ডাটা ফেলে দিয়ে শুধুমাত্র "192.168.10.50" সম্বলিত লাইনগুলো ফিল্টার করে পরবর্তী পাইপে পাঠায়।
  • wc -l (Word Count Lines) কমান্ডটি সেই ফিল্টার হওয়া লাইনগুলো গুনে স্ক্রিনে একটি সুনির্দিষ্ট সংখ্যা (যেমন 452) প্রদর্শন করে।

৫. Shell Variables and Expansion (এনভায়রনমেন্ট কন্ট্রোল)

শেল তার কাজের সুবিধার্থে মেমরিতে কিছু সাময়িক ভেরিয়েবল বা তথ্য জমা রাখে, যাকে Shell Variables বলা হয়।

  • Global Env Variables: এগুলো ওএস লেভেলে ফিক্সড থাকে। যেমন— $USER (বর্তমানে লগইন থাকা ইউজারের নাম), $HOME (ইউজারের হোম পাথ)।
  • Command Substitution: একটি কমান্ডের ভেতরের আউটপুটকে অন্য কমান্ডের ভেরিয়েবল হিসেবে রিড করানোর অপারেশন। এটি ব্যাকটিক (`) বা $() দিয়ে করা হয়।
# ইউজারের হোম ডিরেক্টরির পাথ স্ক্রিনে প্রিন্ট করা
$ echo "Current user path is $HOME"

# কাস্টম ব্যাকআপ ফাইল তৈরিতে কমান্ড সাবস্টিটিউশন ব্যবহার:
$ tar -czf backup-$(date +%F).tar.gz /home/jobaer/data/

৬. Troubleshooting ও সাধারণ শেল চ্যালেঞ্জসমূহ

  • The "Permission Denied" with Redirection Error: জুনিয়র অ্যাডমিনিস্ট্রেটররা প্রায়ই এই ভুলটি করেন। তারা রুট ডিরেক্টরির কোনো ফাইলে ডাটা রাইট করার জন্য sudo echo "data" > /etc/protected.conf রান করেন এবং "Permission Denied" এরর পান।
    কারণ: sudo শুধুমাত্র echo কমান্ডটিকে রুট প্রিভিলেজ দেয়, কিন্তু শেলের রিডাইরেকশন অপারেটর (>) টি সাধারণ ইউজারের ক্ষমতা দিয়েই ফাইল রাইট করার চেষ্টা করে।
    সমাধান: এই ক্ষেত্রে পাইপ এবং tee কমান্ড ব্যবহার করতে হয়, যা রুট প্রিভিলেজ নিয়ে ফাইল রাইট করতে পারে: 
    $ echo "data" | sudo tee /etc/protected.conf
  • Zsh vs Bash Syntax Mismatch: বাশ শেলের কোনো স্ক্রিপ্ট (যেমন অ্যারে ইন্ডেক্সিং যা ০ থেকে শুরু হয়) হুবহু জশ (Zsh) শেলে রান করলে এরর আসতে পারে, কারণ জশ-এর অ্যারে ইন্ডেক্স ১ থেকে শুরু হয়। সবসময় স্ক্রিপ্টের একদম প্রথম লাইনে Shebang (#!/bin/bash) ব্যবহার করুন, যা ওএস-কে নির্দেশ দেয় স্ক্রিপ্টটি কোন নির্দিষ্ট শেল দিয়ে এক্সিকিউট করতে হবে।
  • Stuck Terminal due to Foreground Process: টার্মিনালে কোনো কমান্ড রান করে স্ক্রিন লক হয়ে গেলে কিবোর্ড থেকে Ctrl + Z প্রেস করে প্রসেসটি সাসপেন্ড করুন এবং bg কমান্ড দিয়ে সেটিকে ব্যাকগ্রাউন্ডে সচল করে টার্মিনাল ফ্রী করুন। অথবা Ctrl + C চেপে প্রসেসটি সম্পূর্ণ কিল বা টার্মিনেট করে দিন।

উপসংহার (Conclusion)

Shell Operations হলো লিনাক্স এবং ইউনিক্স অপারেটিং সিস্টেমের মূল চালিকাশক্তি। শেলের ইনপুট/আউটপুট রিডাইরেকশন মেকানিজম, পাইপলাইনের সাহায্যে ডেটা ফিল্টারিং এবং এনভায়রনমেন্ট ভেরিয়েবলের ব্যবহার বুঝতে পারলে কমান্ড লাইনের ওপর আপনার পূর্ণ নিয়ন্ত্রণ প্রতিষ্ঠা পাবে। এটি আপনার প্রতিদিনের রুটিন কাজকে অটোমেট করার পাশাপাশি সার্ভারের ডাউনটাইম এবং সাইবার সিকিউরিটি অডিটের সময় দ্রুত ডায়াগনস্টিক সিদ্ধান্ত নিতে শতভাগ কারিগরি সহায়তা দেবে। নিয়মিত টার্মিনাল কমান্ড প্র্যাকটিস এবং জটিল চেইন পাইপলাইন তৈরি করার অভ্যাসই আপনাকে একজন প্রো-লেভেল লিনাক্স সিস্টেম ইঞ্জিনিয়ার হিসেবে গড়ে তুলবে।
