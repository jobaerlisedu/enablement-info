Linux Distributions

14 June, 2026 Linux Administration Jobaer Hossain

মাইক্রোসফ্ট উইন্ডোজ বা অ্যাপল ম্যাকওএস-এর মতো ইকোসিস্টেমে আমরা সাধারণত একটি নির্দিষ্ট ওএস সংস্করণ দেখতে পাই। কিন্তু লিনাক্সের দুনিয়ায় পা রাখলেই আমরা শত শত বিভিন্ন নামের অপারেটিং সিস্টেমের মুখোমুখি হই—যেমন Ubuntu, Red Hat, Rocky Linux, বা Kali Linux। এই ভিন্ন ভিন্ন রূপগুলোকে লিনাক্সের ভাষায় বলা হয় Linux Distributions বা সংক্ষেপে Distros। লিনাক্স মূলত ওপেন-সোর্স (Open-Source) হওয়ায় যে কেউ এর কোর অংশকে নিজের মতো কাস্টমাইজ করে নতুন একটি ওএস তৈরি করতে পারে।

একজন এন্টারপ্রাইজ লিনাক্স সিস্টেম অ্যাডমিনিস্ট্রেটর বা আইটি প্রফেশনাল হতে হলে বিভিন্ন লিনাক্স ডিস্ট্রিবিউশনের শ্রেণীবিভাগ, এদের পেছনের Package Manager এবং কোন কাজের জন্য কোন ডিস্ট্রো পারফেক্ট, তা গভীরভাবে জানা অত্যন্ত জরুরি। আজকের এই ডিপ-ডাইভ গাইডে আমরা লিনাক্স ডিস্ট্রিবিউশনের মূল কনসেপ্ট, প্রধান ডিস্ট্রো ফ্যামিলি এবং এদের প্র্যাকটিক্যাল ব্যবহার নিয়ে বিস্তারিত আলোচনা করবো।

১. Linux Distribution কী? (The Core Concept)

সহজ ভাষায়, লিনাক্স নিজে কিন্তু একটি সম্পূর্ণ অপারেটিং সিস্টেম নয়; লিনাক্স হলো একটি Kernel। কার্নেল হলো ওএস-এর সেই ভেতরের ইঞ্জিন যা সরাসরি কম্পিউটারের হার্ডওয়্যারকে নিয়ন্ত্রণ করে। এখন এই ইঞ্জিনের সাথে যখন একটি গোটানো Shell, GNU Core Utilities (যেমন ফাইল ম্যানেজার, কমান্ড টুলস), একটি Package Manager (সফটওয়্যার ইনস্টল করার জন্য), এবং একটি Desktop Environment (গ্রাফিক্যাল ইন্টারফেসের জন্য) একত্রিত করে একটি সম্পূর্ণ ইনস্টল করার উপযোগী ওএস ফাইল (ISO) তৈরি করা হয়, তাকেই বলে Linux Distribution।

রিয়েল-লাইফ সিনারিও (The Automobile Engine Analogy)

লিনাক্স ডিস্ট্রিবিউশনের কনসেপ্টটি প্র্যাকটিক্যালি বোঝার জন্য একটি গাড়ি তৈরির কারখানার কথা চিন্তা করুন:

  • The Linux Kernel: এটি হলো গাড়ির শক্তিশালী মেইন ইঞ্জিন। ইঞ্জিন একা কোনো যাত্রীকে গন্তব্যে নিয়ে যেতে পারে না, এর জন্য বডি ও চাকা প্রয়োজন।
  • Ubuntu / Rocky Linux (Distros): বিভিন্ন গাড়ি নির্মাতা প্রতিষ্ঠান (যেমন Toyota বা BMW) ওই একই ইঞ্জিন বা মেকানিজম ব্যবহার করে নিজস্ব ডিজাইন, সিট, স্টিয়ারিং এবং এসি যোগ করে বাজারে গাড়ি ছাড়ে। একইভাবে বিভিন্ন কমিউনিটি লিনাক্স কার্নেল নিয়ে আলাদা আলাদা ডিস্ট্রো তৈরি করে। কোনো কোম্পানি বানায় রেসিং কার (যেমন Kali Linux ফর সিকিউরিটি), আবার কেউ বানায় ভারী লোড নেওয়ার ট্রাক (যেমন Red Hat ফর এন্টারপ্রাইজ সার্ভার)।

২. প্রধান লিনাক্স ডিস্ট্রিবিউশন ফ্যামিলিসমূহ

পৃথিবীর প্রায় সব লিনাক্স ডিস্ট্রো কোনো না কোনো প্রধান আদি ফ্যামিলির (Parent Lineage) ওপর ভিত্তি করে তৈরি হয়েছে। এদের মূল পার্থক্য থাকে এদের Package Manager এবং রিলিজ সাইকেলের ওপর। প্রধান ৩টি ফ্যামিলি নিচে দেওয়া হলো:

ক. The Debian Family (Debian, Ubuntu, Linux Mint)

এটি লিনাক্স জগতের অন্যতম প্রাচীন, স্থিতিশীল এবং সবচেয়ে জনপ্রিয় ফ্যামিলি।

  • Core Characteristics: এটি ইউজার-ফ্রেন্ডলি এবং এর বিশাল সফটওয়্যার রিপোজিটরি (Repository) রয়েছে। ডেবিয়ান অত্যন্ত স্থিতিশীল (Stable) ওএস হিসেবে পরিচিত।
  • Package Manager: এই ফ্যামিলির প্যাকেজ ফাইলের এক্সটেনশন হয় .deb। ব্যাকএন্ডে সফটওয়্যার ম্যানেজ করার জন্য APT (Advanced Package Tool) বা dpkg ব্যবহৃত হয়।
  • Ubuntu: ডেবিয়ানের ওপর ভিত্তি করে তৈরি উবুন্টু বর্তমান বিশ্বের সবচেয়ে জনপ্রিয় ডিস্ট্রো, যা ডেস্কটপ এবং ক্লাউড সার্ভার উভয় ক্ষেত্রে ব্যাপকভাবে ব্যবহৃত হয়।

খ. The Red Hat Family (RHEL, Rocky Linux, Fedora)

এটি এন্টারপ্রাইজ বিজনেস এবং কর্পোরেট ডাটা সেন্টারের অবিসংবাদিত রাজা।

  • Core Characteristics: এটি চরম লেভেলের সিকিউরিটি (যেমন SE-Linux), পারফরম্যান্স অপ্টিমাইজেশন এবং দীর্ঘমেয়াদী লাইফ-সাপোর্ট (LTS) প্রদান করে।
  • Package Manager: এদের প্যাকেজ ফাইলের এক্সটেনশন হয় .rpm (Red Hat Package Manager)। আধুনিক ডিস্ট্রোগুলোতে কমান্ড লাইনের জন্য DNF (Dandified YUM) বা yum ম্যানেজার ব্যবহৃত হয়।
  • RHEL & Rocky Linux: RHEL (Red Hat Enterprise Linux) একটি পেইড বা বাণিজ্যিক ওএস। RHEL-এর হুবহু ফ্রি এবং ওপেন-সোর্স ক্লোন সংস্করণ হলো Rocky Linux এবং AlmaLinux, যা বর্তমান বিশ্বের টপ ডাটা সেন্টারগুলোতে সার্ভার ওএস হিসেবে ব্যবহৃত হচ্ছে।

গ. The Arch Family (Arch Linux, Manjaro)

অ্যাডভান্সড লিনাক্স ইউজার এবং কাস্টমাইজেশন লাভারদের জন্য এটি একটি স্বর্গ।

  • Core Characteristics: এটি Rolling Release মডেল অনুসরণ করে। অর্থাৎ উইন্ডোজ বা উবুন্টুর মতো এর কোনো ফিক্সড ভার্সন (যেমন উবুন্টু ২৪.০৪) থাকে না; এটি প্রতিদিন প্রতিনিয়ত আপডেট হতে থাকে। আর্চ লিনাক্স ইনস্টল করার সময় কোনো গ্রাফিক্যাল ইন্টারফেস থাকে না, ব্ল্যাক টার্মিনাল থেকে কোড লিখে লিখে ওএস তৈরি করতে হয়।
  • Package Manager: এটি অত্যন্ত ফাস্ট একটি টুল ব্যবহার করে যার নাম Pacman। এছাড়া এর রয়েছে AUR (Arch User Repository), যেখানে দুনিয়ার প্রায় সব কাস্টম সফটওয়্যার পাওয়া যায়।

৩. এক নজরে ডিস্ট্রো ও প্যাকেজ ম্যানেজার রেফারেন্স টেবিল

Distribution Name Parent Family Package Format CLI Package Tool Primary Use Case Target
Ubuntu Server / Desktop Debian .deb apt / apt-get Cloud hosting, developers workstations, web servers.
Rocky Linux / AlmaLinux Red Hat (RHEL) .rpm dnf / yum Enterprise databases, mission-critical data centers.
Kali Linux Debian .deb apt Cybersecurity penetration testing and digital forensics.
Manjaro / Arch Arch Tarball Packages pacman Bleeding-edge software development and deep OS customization.

৪. প্র্যাকটিক্যাল CLI অপারেশন: ওএস এবং প্যাকেজ ভেরিফিকেশন

আপনি যখন কোনো দূরবর্তী ক্লাউড সার্ভারে SSH দিয়ে লগইন করবেন, তখন সেটি কোন ডিস্ট্রিবিউশন এবং সেখানে কোন প্যাকেজ ম্যানেজার কাজ করছে, তা কমান্ড লাইনের মাধ্যমে পরীক্ষা করার স্ট্যান্ডার্ড ধাপগুলো নিচে দেওয়া হলো:

ধাপ ১: ডিস্ট্রিবিউশন এবং ভার্সন আইডেন্টিফিকেশন

যেকোনো মডার্ন লিনাক্স ডিস্ট্রোর অফিশিয়াল রিলিজ তথ্য দেখার গ্লোবাল কমান্ড:

$ cat /etc/os-release

আউটপুট অ্যানালাইসিস: এই কমান্ডটি দিলে স্ক্রিনে NAME="Ubuntu" বা NAME="Rocky Linux" এবং তার সুনির্দিষ্ট VERSION_ID ও ডিস্ট্রো ফ্যামিলির বিবরণ লাইভ দেখতে পাবেন।

ধাপ ২: ডেবিয়ান/উবুন্টু ফ্যামিলিতে সিস্টেম আপডেট ও সফটওয়্যার ইনস্টলেশন

যদি /etc/os-release-এ ডেবিয়ান বা উবুন্টু ফ্যামিলি ডিটেক্ট হয়, তবে নিচের কমান্ড মেকানিজম ব্যবহার করুন:

# সিস্টেমের লোকাল রিপোজিটরি ইনডেক্স আপডেট করা
$ sudo apt update

# একটি নতুন অ্যাপ্লিকেশন (যেমন Nginx Web Server) ইনস্টল করা
$ sudo apt install nginx -y

ধাপ ৩: রেড হ্যাট / রকি লিনাক্স ফ্যামিলিতে সফটওয়্যার ইনস্টলেশন

যদি সার্ভারটি আরপিএম (RPM) ভিত্তিক রকি বা সেন্টওএস (CentOS) হয়ে থাকে, তবে dnf কমান্ড টুল ব্যবহার করুন:

# সিস্টেমের সব প্যাকেজ লেটেস্ট সংস্করণে আপগ্রেড করা
$ sudo dnf upgrade -y

# একই অ্যাপ্লিকেশন DNF এর মাধ্যমে ইনস্টল করা
$ sudo dnf install nginx -y

৫. স্পেশালাইজড ডিস্ট্রিবিউশন: Kali Linux এবং Alpine

সাধারণ ডেস্কটপ বা সার্ভার ছাড়াও কিছু লিনাক্স ডিস্ট্রো তৈরি করা হয়েছে অত্যন্ত সুনির্দিষ্ট ফিজিক্যাল ও লজিক্যাল কাজের জন্য:

  • Kali Linux (Cyber Security): এটি ডেবিয়ান ভিত্তিক একটি ওএস। এর ভেতরে শত শত প্রি-ইনস্টলড সিকিউরিটি এবং হ্যাকিং টুলস (যেমন- Nmap, Wireshark, Burp Suite) বিল্ট-ইন থাকে। এটি মূলত সাইবার সিকিউরিটি প্রফেশনালদের Penetration Testing বা ভেদযোগ্যতা পরীক্ষার জন্য ব্যবহৃত হয়।
  • Alpine Linux (Docker Containers): এটি একটি আল্ট্রা-মিনিমালিস্ট লিনাক্স ডিস্ট্রো যার সাইজ মাত্র ৫ মেগাবাইট। এতে কোনো গ্রাফিক্যাল ইন্টারফেস বা অপ্রয়োজনীয় সার্ভিস থাকে না। আধুনিক Docker Container এবং মাইক্রোসার্ভিস আর্কিটেকচারে ইমেজ সাইজ ছোট রাখার জন্য আলপাইন লিনাক্স বিশ্বব্যাপী এক নম্বর পছন্দ।

৬. Troubleshooting ও সাধারণ ডিস্ট্রো চ্যালেঞ্জসমূহ

  • The "Package Not Found" Error: একটি ডিস্ট্রোর কমান্ড (যেমন apt install) অন্য ডিস্ট্রোতে (যেমন Rocky Linux) রান করলে ওএস এরর দেবে। সবসময় ওএস ফ্যামিলি অনুযায়ী প্যাকেজ টুল ব্যবহার করুন। এছাড়া, যদি সঠিক প্রোভাইডারেও প্যাকেজ না পাওয়া যায়, তবে বুঝতে হবে আপনাকে এক্সটার্নাল রিপোজিটরি এনাবল করতে হবে (যেমন রেড হ্যাট ফ্যামিলির জন্য EPEL - Extra Packages for Enterprise Linux)।
    রকি লিনাক্সে EPEL এড করার কমান্ড: 
    $ sudo dnf install epel-release -y
  • Dependency Hell (প্যাকেজ নির্ভরতা জটিলতা): অনেক সময় ম্যানুয়ালি .rpm বা .deb ফাইল সরাসরি ইনস্টল করতে গেলে ওএস বলে যে এর জন্য আরও ৫টি সাপোর্ট ফাইল বা ডিপেন্ডেন্সি লাগবে। এই সমস্যা এড়াতে সরাসরি অফিশিয়াল প্যাকেজ ম্যানেজার (apt বা dnf) ব্যবহার করা উচিত, কারণ এরা ব্যাকঅ্যান্ডে ইন্টারনেট থেকে স্বয়ংক্রিয়ভাবে সব ডিপেন্ডেন্সি ফাইল ডাউনলোড করে নেয়।
  • GPG Key Verification Failure: থার্ড-পার্টি কোনো সফটওয়্যার সোর্স অ্যাড করার পর সিস্টেম আপডেট দিলে সিকিউরিটি কি (Key) এরর আসতে পারে। এর সমাধান হলো ওই নির্দিষ্ট সফটওয়্যার প্রোভাইডারের ভ্যালিড GPG Public Key-টি রাউটার বা ওএস লেয়ারে ইমপোর্ট করে নেওয়া।

উপসংহার (Conclusion)

Linux Distributions-এর এই বৈচিত্র্যই লিনাক্স ইকোসিস্টেমকে পৃথিবীর সবচেয়ে নমনীয় বা ফ্লেক্সিবল অপারেটিং সিস্টেমে পরিণত করেছে। আপনার যদি ঘরের পুরনো পিসির জন্য হালকা ওএস লাগে তবে উবুন্টু বা মিন্ট, যদি ব্যাংকের ডাটাবেজ সার্ভার চালাতে হয় তবে রকি লিনাক্স, আর যদি সাইবার সিকিউরিটি অডিট করতে হয় তবে কালি লিনাক্স—প্রতিটি কাজের জন্য ডেডিকেটেড ডিস্ট্রো প্রস্তুত রয়েছে। একজন দক্ষ সিস্টেম অ্যাডমিনিস্ট্রেটর হিসেবে এই বিভিন্ন ডিস্ট্রোর আর্কিটেকচার এবং প্যাকেজ ম্যানেজমেন্ট মেকানিজম বুঝতে পারা আপনার আইটি ক্যারিয়ারের পরিধি ও গভীরতা বহুগুণ বাড়িয়ে দেবে। নিয়মিত টার্মিনাল ল্যাব প্র্যাকটিসই আপনাকে এই ভিন্ন ভিন্ন ওএস এনভায়রনমেন্টে মাস্টার করে তুলবে।
