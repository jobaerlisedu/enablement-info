Explain Linux and System Boot Process

14 June, 2026 Linux Administration Jobaer Hossain

আমরা যখন কম্পিউটারের ফিজিক্যাল পাওয়ার বাটনটি প্রেস করি, তখন স্ক্রিনে অপারেটিং সিস্টেমের লগো ভেসে উঠতে মাত্র কয়েক সেকেন্ড সময় লাগে। কিন্তু এই অল্প সময়ের মধ্যে কম্পিউটারের ভেতরের হার্ডওয়্যার এবং সফটওয়্যার লেয়ারে একটি বিশাল ও সুশৃঙ্খল চেইন রিঅ্যাকশন সম্পন্ন হয়। আইটি এবং সিস্টেম অ্যাডমিনিস্ট্রেশনের ভাষায় এই পুরো প্রক্রিয়াটিকে বলা হয় Boot Process বা Bootstrapping।

একজন দক্ষ সিস্টেম ইঞ্জিনিয়ারের জন্য এই বুট প্রসেসের প্রতিটি নিখুঁত ধাপ জানা অত্যন্ত জরুরি। কারণ সিস্টেম ক্র্যাশ করলে বা সার্ভার বুট না হলে, ব্যাকএন্ডের কোন স্তরে সমস্যাটি হয়েছে তা বুট সিকোয়েন্স না বুঝলে সনাক্ত করা অসম্ভব। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা লিনাক্স এবং জেনারেল কম্পিউটার আর্কিটেকচারের বুট প্রসেস নিয়ে একদম স্ক্র্যাচ থেকে অ্যাডভান্সড লেভেলে বিস্তারিত আলোচনা করবো।

১. Boot Process কী?

Boot Process হলো এমন একটি ইনিশিয়াল সিকোয়েন্স বা ধারাবাহিক প্রক্রিয়া যার মাধ্যমে কম্পিউটার তার পাওয়ার অন হওয়ার পর ফিজিক্যাল হার্ডওয়্যার ভেরিফাই করে এবং সেকেন্ডারি স্টোরেজ (HDD/SSD) থেকে অপারেটিং সিস্টেমের কোর ফাইলগুলোকে মেইন মেমরি বা RAM-এ লোড করে সিস্টেমকে সম্পূর্ণ কাজের উপযোগী করে তোলে।

যেহেতু পাওয়ার অন করার সাথে সাথে সরাসরি অপারেটিং সিস্টেম চলতে পারে না, তাই ধাপে ধাপে একটি ছোট প্রোগ্রাম পরবর্তী বড় প্রোগ্রামকে ডেকে সচল করে। এই ছোট থেকে বড় প্রোগ্রামে স্থানান্তরের ধারণাকেই বলা হয় Bootstrapping।

রিয়েল-লাইফ সিনারিও (The Airplane Takeoff Analogy)

কম্পিউটারের বুট প্রসেসকে একটি বড় কমার্শিয়াল এরোপ্লেন উড্ডয়নের (Takeoff) প্রস্তুতির সাথে তুলনা করা যায়:

  • Pre-flight Check: পাইলট বিমানে উঠে ইঞ্জিন চালু করার আগে ককপিটের সব লাইট, ফুয়েল লেভেল এবং উইংসের পার্টসগুলো ঠিক আছে কি না তা একটি একটি করে পরীক্ষা করেন। এটি হলো কম্পিউটারের POST (Power-On Self-Test)।
  • Runway Guidance: ককপিট চেকিং শেষ হলে এয়ার ট্রাফিক কন্ট্রোল থেকে রানওয়ে ধরে এগিয়ে যাওয়ার সিগন্যাল আসে। এটি হলো Bootloader-এর কাজ যা ওএস-কে পথ দেখায়।
  • Cruising Mode: বিমানটি যখন মাটি ছেড়ে আকাশে উড়াল দেয় এবং অটো-পাইলট মোড সচল করে স্বাভাবিক গতিতে চলতে থাকে, তখন এটি অপারেটিং সিস্টেমের Kernel Run Mode-এর মতো কাজ করে।

২. লিনাক্স বুট প্রসেসের ৬টি মূল ধাপ (The 6 Stages)

একটি স্ট্যান্ডার্ড লিনাক্স সার্ভার বা সিস্টেম বুট হওয়ার সময় মূলত নিচের ৬টি প্রধান ধাপ বা স্টেজ পার হয়ে থাকে:

ধাপ ১: BIOS / UEFI (Hardware Initialization)

পাওয়ার বাটন চাপার সাথে সাথে মাদারবোর্ডের ROM (Read-Only Memory) চিপে থাকা একটি ছোট ফার্মওয়্যার সচল হয়, যাকে আমরা BIOS (Basic Input/Output System) বা আধুনিক কম্পিউটারে UEFI (Unified Extensible Firmware Interface) বলি।

  • POST (Power-On Self-Test): এর প্রথম কাজ হলো কম্পিউটারের ফিজিক্যাল হার্ডওয়্যার (যেমন RAM, CPU, Keyboard, Hard Disk) ঠিকমতো সংযুক্ত আছে কি না তা ইলেকট্রিক্যাল পালস দিয়ে চেক করা। কোনো এরর থাকলে মাদারবোর্ড থেকে Beep Code বা শব্দ শোনা যায়।
  • Boot Device Search: POST সফল হলে BIOS/UEFI তার সেটিংসের প্রায়োরিটি লিস্ট অনুযায়ী বুটেবল ডিভাইস (যেমন Hard Drive, SSD, USB Drive, Network) খুঁজে বের করে এবং সেই ড্রাইভের একদম শুরুর সেক্টরে চলে যায়।

ধাপ ২: MBR / EFI Partition (The Entry Point)

BIOS যদি বুটেবল ফিজিক্যাল ড্রাইভ খুঁজে পায়, তবে সেটির পার্টিশন স্টাইল অনুযায়ী প্রথম অংশটি রিড করে।

  • MBR (Master Boot Record): প্রথাগত সিস্টেমে হার্ডডিস্কের একদম প্রথম সেক্টরকে (Sector 0) বলা হয় MBR, যার সাইজ মাত্র ৫১২ বাইট। এই ৫১২ বাইটের ভেতরে ৪৪৬ বাইট থাকে মেইন Primary Bootloader Code-এর জন্য, ৬৪ বাইট থাকে পার্টিশন টেবিলের তথ্যের জন্য এবং ২ বাইট থাকে ম্যাজিক নাম্বারের জন্য।
  • EFI System Partition (ESP): আধুনিক UEFI সিস্টেমে MBR-এর এই সাইজ সীমাবদ্ধতা নেই। এটি সরাসরি ফ্যাট (FAT) ফরম্যাটের একটি আলাদা পার্টিশন থেকে বুট ফাইলের .efi এক্সটেনশন রিড করে।

ধাপ ৩: GRUB / Bootloader (The Guide)

MBR বা EFI থেকে প্রাইমারি কোড লোড হওয়ার পর স্ক্রিনে ওএস সিলেক্ট করার উইন্ডো আসে। লিনাক্স জগতের সবচেয়ে জনপ্রিয় বুটলোডার হলো GRUB2 (Grand Unified Bootloader Version 2)।

  • Configuration Load: GRUB2 তার কনফিগারেশন ফাইল থেকে বুট মেনুর লিস্ট ইউজারকে দেখায়।
  • Kernel Identification: ইউজার যখন কোনো নির্দিষ্ট লিনাক্স কার্নেল সিলেক্ট করেন, GRUB2 তখন হার্ডডিস্কের /boot ডিরেক্টরি থেকে নির্দিষ্ট কার্নেল ইমেজ ফাইল এবং Initramfs ফাইলটি মেমরিতে ডেকে নিয়ে আসে।

ধাপ ৪: Kernel (The Brain)

বুটলোডার যখন কার্নেলকে RAM-এ লোড করে দেয়, তখন থেকে বুটলোডারের দায়িত্ব শেষ এবং কার্নেলের শাসন শুরু হয়। লিনাক্স কার্নেল সাধারণত একটি কমপ্রেসড ফাইল হিসেবে থাকে যাকে vmlinuz বলা হয়। মেমরিতে এসে এটি নিজেকে আন-কমপ্রেস করে নেয়।

  • Hardware Probe: কার্নেল তার নিজস্ব বিল্ট-ইন ড্রাইভার দিয়ে কম্পিউটারের সব ফিজিক্যাল হার্ডওয়্যার পুনরায় গভীরভাবে পরীক্ষা করে এবং নিজের নিয়ন্ত্রণে নেয়।
  • Root Filesystem Mount: কার্নেল তার টেম্পোরারি ফাইল সিস্টেম (Initramfs) ব্যবহার করে মেইন স্টোরেজের রুট ফাইল সিস্টেমকে (/) Read-Only মোডে মাউন্ট করে।

ধাপ ৫: Initramfs / Initrd (Temporary Root)

আজকের যুগের হার্ডডিস্কগুলো বিভিন্ন অ্যাডভান্সড ফরম্যাটে থাকে যেমন—RAID, NVMe বা এনক্রিপ্টেড স্টোরেজ (LUKS)। কার্নেল সাইজ ছোট রাখার জন্য এই জটিল স্টোরেজের ড্রাইভারগুলো কার্নেলের ভেতরে বিল্ট-ইন থাকে না।

  • Loading Storage Drivers: Initramfs (Initial RAM Filesystem) হলো একটি ছোট ভার্চুয়াল ফাইল সিস্টেম যা RAM-এর ভেতরে তৈরি হয়। এর কাজ হলো কার্নেলকে প্রয়োজনীয় স্টোরেজ ড্রাইভার মডিউল সরবরাহ করা যাতে কার্নেল আসল হার্ডডিস্কটি রিড করতে পারে।
  • Pivot Root: আসল হার্ডডিস্ক এক্সেস পাওয়ার সাথে সাথে সিস্টেম এই টেম্পোরারি রুট থেকে মেইন ফিজিক্যাল রুট ডিরেক্টরিতে সুইচ করে।

ধাপ ৬: Systemd / Init (The System Controller)

রুট ফাইল সিস্টেম মাউন্ট করার পর কার্নেল ব্যাকএন্ডের একদম প্রথম এবং আদি প্রসেসটি রান করে। আধুনিক সব লিনাক্স ডিস্ট্রিবিউশনে (যেমন Ubuntu, CentOS, Rocky Linux) এই প্রথম প্রসেসটি হলো systemd। এর PID (Process ID) সব সময় 1 হয়।

  • Starting Services: systemd তার কনফিগারেশন টার্গেট (যেমন- multi-user.target বা graphical.target) অনুযায়ী ব্যাকএন্ডের সমস্ত প্রয়োজনীয় সার্ভিস ও ডেমন (যেমন- SSH Server, Network Manager, Web Server, Display Manager) একে একে প্যারালাল স্পিডে চালু করে।
  • Login Prompt: সব সার্ভিস সফলভাবে চালু হলে systemd স্ক্রিনে ইউজারকে লগইন প্রম্পট বা গ্রাফিক্যাল ডিসপ্লে ম্যানেজার (GDM/LightDM) প্রদর্শন করে বুট প্রসেসের সমাপ্তি ঘটায়।

৩. BIOS বনাম UEFI: আর্কিটেকচারাল পার্থক্য

ফিচার বা বৈশিষ্ট্য (Feature) BIOS (Legacy) UEFI (Modern)
Architecture Mode 16-bit real mode-এ কাজ করে। মাত্র ১ মেগাবাইট মেমরি এক্সেস করতে পারে। 32-bit বা 64-bit প্রোটেক্টেড মোডে চলে। বিশাল মেমরি এক্সেস ক্ষমতা।
Drive Capacity Support সর্বোচ্চ 2.2 TB সাইজের ডিস্ক সাপোর্ট করে (MBR স্টাইল)। সর্বোচ্চ 9.4 ZB (Zettabytes) ডিস্ক সাইজ সাপোর্ট করে (GPT স্টাইল)।
Boot Speed ধীরগতির হার্ডওয়্যার ইনিশিয়ালাইজেশন। অনেক ফাস্ট বুট টাইম এবং অপ্টিমাইজেশন।
Security কোনো সিকিউর বুট মেকানিজম নেই। Secure Boot ফিচার আছে যা ম্যালওয়্যার আক্রান্ত বুট ফাইল ব্লক করে।

৪. বুট প্রসেস ভেরিফিকেশন ও অ্যানালিসিস (Linux CLI Commands)

আপনার লিনাক্স সার্ভারটি শেষবার বুট হতে কত সেকেন্ড সময় নিয়েছে এবং কোন সার্ভিসটির কারণে বুট হতে দেরি হচ্ছে, তা কমান্ড লাইনের সাহায্যে নিখুঁতভাবে মাপা যায়।

১. বুট টাইমের ওভারওভারভিউ দেখা:

$ systemd-analyze

# এই কমান্ডটি দিলে কার্নেল, ইনিটআরডি এবং ইউজারস্পেস বুট হতে মোট কত সেকেন্ড লেগেছে তা দেখাবে।

২. কোন সার্ভিস বুট হতে কত সময় নিয়েছে তার ব্লেম লিস্ট (Blame List) দেখা:

$ systemd-analyze blame

# আউটপুটে সবচেয়ে বেশি সময় নেওয়া সার্ভিসগুলো ক্রমানুসারে (Descending Order) সাজানো থাকবে।
# যেমন: 4.521s network.service

৩. কার্নেলের বুট মেসেজ বা মেমরি লগ চেক করা:

# বুট হওয়ার সময় কার্নেল কী কী হার্ডওয়্যার ডিটেক্ট করেছিল তার লগ দেখা
$ sudo dmesg | grep -i cpu
$ sudo dmesg | less

৫. Common Boot Issues এবং Troubleshooting গাইড

বাস্তব কর্মক্ষেত্রে সার্ভার বুট না হলে নিচে দেওয়া ডায়াগনস্টিক মেথডগুলো ব্যবহার করুন:

  • Problem 1: "No Bootable Device Found" or GRub Rescue Error
    • কারণ: MBR বা EFI পার্টিশনটি ড্যামেজ হয়ে গেছে অথবা বায়োসে হার্ডডিস্কের বুট অর্ডার ওলটপালট হয়ে গেছে।
    • সমাধান: সিস্টেমে একটি Live Linux USB প্লাগ করে বুট করুন এবং grub2-install /dev/sda কমান্ডের মাধ্যমে বুটলোডার কোড পুনরায় ফিক্সড বা রি-ইনস্টল করুন।
  • Problem 2: Stuck at Kernel Panic (Kernel Panic - not syncing)
    • কারণ: কার্নেল ফাইলটি ক্র্যাশ করেছে অথবা Initramfs ফাইলটি মেইন রুট ফাইল সিস্টেম মাউন্ট করতে ব্যর্থ হয়েছে। ওএস আপডেট করার সময় কার্নেল ফাইল ড্যামেজ হলে এটি বেশি হয়।
    • সমাধান: পিসি রিস্টার্ট করে GRUB2 মেনু আসার সাথে সাথে কিবোর্ডের অ্যারো কি চেপে তালিকায় থাকা আগের ওল্ড কার্নেল সংস্করণটি (Old Kernel Version) সিলেক্ট করে বুট করুন। সিস্টেমে ঢুকে নতুন কার্নেলটি ফিক্সড করুন।
  • Problem 3: Emergency Mode / Dracut Shell Prompt
    • কারণ: যদি /etc/fstab ফাইলে কোনো হার্ডডিস্ক বা মাউন্ট পয়েন্টের তথ্য ভুল থাকে, তবে systemd বুট প্রসেস মাঝপথে থামিয়ে ইমার্জেন্সি শেল প্রম্পট দিয়ে দেয়।
    • সমাধান: রুট পাসওয়ার্ড দিয়ে শেলে প্রবেশ করুন এবং mount -o remount,rw / কমান্ড দিয়ে রুট ফাইল সিস্টেম রাইট মোডে এনে nano /etc/fstab ফাইল থেকে ভুল ড্রাইভের লাইনটি মুছে বা কমেন্ট (#) করে দিন।

উপসংহার (Conclusion)

System Boot Process হলো একটি কম্পিউটারের ফিজিক্যাল অস্তিত্ব থেকে লজিক্যাল ওএস স্তরে উন্নীত হওয়ার এক সুনিপুণ মহাকাব্য। BIOS/UEFI-এর হার্ডওয়্যার চেকিং থেকে শুরু করে GRUB-এর পথপ্রদর্শন, কার্নেলের মস্তিস্ক সক্রিয়করণ এবং দিনশেষে systemd-এর সার্ভিস ম্যানেজমেন্ট—প্রতিটি ধাপ একে অপরের পরিপূরক। একজন এন্টারপ্রাইজ সিস্টেম অ্যাডমিনিস্ট্রেটর হিসেবে এই বুট সিকোয়েন্সের মেকানিজম এবং systemd-analyze-এর মতো ডায়াগনস্টিক টুলের ব্যবহার জানা আপনার ক্যারিয়ারের গভীরতা বহুগুণ বাড়িয়ে দেবে, যা আপনাকে যেকোনো ক্রিটিক্যাল ডাউনটাইমে দ্রুত ও নিখুঁত ট্রাবলশুটিং করতে সহায়তা করবে।
