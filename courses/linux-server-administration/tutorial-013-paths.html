Paths

14 June, 2026 Linux Administration Jobaer Hossain

লিনাক্স ফাইল সিস্টেমের কাঠামো বা Filesystem Hierarchy Standard (FHS) নিয়ে স্টাডি করার সময় আমরা ডিরেক্টরিগুলোর বিন্যাস সম্পর্কে অবগত হয়েছি। কিন্তু টার্মিনাল বা Command Line Interface (CLI)-তে ফাইল তৈরি করা, ডিরেক্টরি ব্রাউজ করা কিংবা কোনো স্ক্রিপ্ট রান করার সময় আমাদের ওএস-কে সুনির্দিষ্ট ঠিকানা বা লোকেশন বুঝিয়ে দিতে হয়। লিনাক্সের ভাষায় এই ঠিকানাকেই বলা হয় Path। লিনাক্স অ্যাডমিনিস্ট্রেশনে এবং Bash Scripting-এ ফাইল লোকেটিং নিখুঁতভাবে পরিচালনা করতে Paths-এর গভীর মেকানিজম বুঝতে পারা এক অপরিহার্য কারিগরি চাবিকাঠি।

আপনি যদি একটি ভুল পাথ ডিফাইন করেন, তবে সিস্টেম আপনার ফাইলটি খুঁজে পাবে না অথবা সম্পূর্ণ ভুল ডিরেক্টরিতে ডেটা রাইট করে বড় ধরনের বিপর্যয় ঘটিয়ে ফেলবে। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা লিনাক্সের দুই ধরনের পাথ—Absolute Path এবং Relative Path-এর ইন্টারনাল মেকানিজম, এদের সূক্ষ্ম কারিগরি পার্থক্য এবং প্র্যাকটিক্যাল অপারেশন নিয়ে বিস্তারিত আলোচনা করবো।

১. Linux Path কী? (The Structural Concept)

সহজ কথায়, Path হলো লিনাক্স ডিরেক্টরি ট্রির (Directory Tree) মধ্য দিয়ে একটি সুনির্দিষ্ট ফাইল বা ফোল্ডারে পৌঁছানোর লজিক্যাল রাস্তা বা সিকোয়েন্সিয়াল এড্রেস। লিনাক্সে ডিরেক্টরিগুলোকে আলাদা করার জন্য ফরওয়ার্ড স্ল্যাশ (/) চিহ্ন ব্যবহার করা হয় (মনে রাখবেন, উইন্ডোজে ব্যাকস্ল্যাশ \ ব্যবহৃত হয়)।

যেমন: /var/log/nginx/access.log একটি পাথ। এর অর্থ হলো, মেইন রুট ডিরেক্টরির ভেতরে থাকা var ফোল্ডারের ভেতরের log ফোল্ডারের ভেতরের nginx ডিরেক্টরিতে অবস্থানরত access.log নামক একটি সুনির্দিষ্ট ফাইল।

রিয়েল-লাইফ সিনারিও (The Global Positioning System Analogy)

পাথের কনসেপ্টটি প্র্যাকটিক্যালি বোঝার জন্য বাস্তব জীবনের একটি ঠিকানা বা গ্লোবাল লোকেশনের কথা চিন্তা করুন:

  • Absolute Path (গ্লোবাল জিপিএস কোঅর্ডিনেট): ধরুন আপনি কাউকে আপনার অফিসের ঠিকানা দিলেন: "বাংলাদেশ, ঢাকা, ধানমন্ডি, রোড ৫A, হাউস নম্বর ১২"। এই ঠিকানাটি পৃথিবীর যেকোনো প্রান্ত থেকে একজন মানুষ ট্র্যাক করে সোজা আপনার অফিসে এসে পৌঁছাতে পারবেন। এর জন্য তিনি বর্তমানে কোথায় দাঁড়িয়ে আছেন তা জানা জরুরী নয়।
  • Relative Path (বর্তমান সাপেক্ষ নির্দেশনা): আপনি অলরেডি ধানমন্ডি ৫A রোডের মাথায় দাঁড়িয়ে আছেন আপনার এক বন্ধুর সাথে। আপনি তাকে বললেন, "সামনের দিকে ২ ব্লক এগিয়ে বামের ৩ নম্বর বিল্ডিংয়ে যাও।" এই ঠিকানাটি শুধুমাত্র ওই নির্দিষ্ট মুহূর্তে ওই স্থানে দাঁড়ানো মানুষের জন্যই সত্য। আপনি যদি কুড়িল বিশ্বরোডে দাঁড়িয়ে এই কথা বলেন, তবে আপনার বন্ধু সম্পূর্ণ ভুল জায়গায় চলে যাবে। এটিই হলো রিলেটিভ পাথ।

২. Absolute Path: সর্বজনীন ও সুনির্দিষ্ট ঠিকানা

Absolute Path হলো এমন একটি পাথ যা ফাইল সিস্টেমের বর্তমান ওয়ার্কিং ডিরেক্টরির (Current Working Directory) ওপর নির্ভর করে না। এটি সব সময় এবং সব মোডেই সম্পূর্ণ স্বাধীন।

Absolute Path-এর মূল কারিগরি বৈশিষ্ট্যসমূহ:

  • এই পাথটি সব সময় লিনাক্সের আদি রুট বা স্ল্যাশ (/) চিহ্ন দিয়ে শুরু হয়।
  • টার্মিনালের সেশনে আপনি বর্তমানে যে ফোল্ডারেই অবস্থান করুন না কেন, Absolute Path ব্যবহার করে যেকোনো ফাইলকে সরাসরি ট্রিগার বা অ্যাক্সেস করা সম্ভব।
  • কনফিগারেশন ফাইল ও সিস্টেম ডেমোনের (Systemd Services) ভেতরে সব সময় এই পাথ ব্যবহার করা বাধ্যতামূলক, যাতে ব্যাকঅ্যান্ড প্রসেস কোনো কনফ্লিক্ট ছাড়া ফাইল রিড করতে পারে।

উদাহরণ: যদি আপনি /home/jobaer/projects/app.py ফাইলটি অ্যাক্সেস করতে চান, তবে এটিই হলো এর Absolute Path।

৩. Relative Path: বর্তমান অবস্থান সাপেক্ষ ঠিকানা

Relative Path হলো এমন একটি পাথ যা ব্যবহারকারীর বর্তমান কাজের অবস্থান বা ডিরেক্টরির (Current Working Directory) সাপেক্ষে ফাইল বা ফোল্ডারের ঠিকানা নির্দেশ করে।

Relative Path-এর মূল কারিগরি বৈশিষ্ট্যসমূহ:

  • এই পাথের শুরুতে কখনো ফরওয়ার্ড স্ল্যাশ (/) বসে না।
  • এটি মূলত শর্টকাট নেভিগেশনের জন্য টার্মিনাল ইন্টারেকশনের সময় ইঞ্জিনিয়াররা বেশি ব্যবহার করেন।
  • রিলেটিভ পাথ হ্যান্ডেল করতে লিনাক্স দুটি বিশেষ স্পেশাল ক্যারেক্টার বা ডট-নোটেশন ব্যবহার করে:
    • একটি ডট (.): এটি বর্তমান ডিরেক্টরি (Current Directory) নির্দেশ করে।
    • দুটি ডট (..): এটি ঠিক এক ধাপ উপরের বা প্যারেন্ট ডিরেক্টরি (Parent Directory) নির্দেশ করে।

৪. প্র্যাকটিক্যাল CLI অপারেশন: পাথ নেভিগেশন ও ট্রাভার্সাল

আসুন আমরা টার্মিনাল বা কমান্ড লাইনের মাধ্যমে এই দুই ধরনের পাথের আচরণ ও নেভিগেশন মেকানিজম প্র্যাকটিক্যালি ভেরিফাই করি।

ধাপ ১: বর্তমান কাজের ডিরেক্টরি বা বাউন্ডারি সনাক্ত করা

টার্মিনালে আপনি ফিজিক্যালি কোথায় অবস্থান করছেন তা জানার কমান্ড:

$ pwd
# আউটপুটে আপনার Absolute Path দেখাবে, যেমন: /home/jobaer/downloads

ধাপ ২: Absolute Path ব্যবহার করে ডিরেক্টরি পরিবর্তন

আপনি বর্তমানে যেখানেই থাকুন না কেন, এক কমান্ডে গ্লোবাল কনফিগারেশন ডিরেক্টরিতে যাওয়ার মেথড:

$ cd /etc/nginx/conf.d

ব্যাখ্যা: পাথের শুরুতে / থাকায় ওএস সরাসরি রুট লেভেল থেকে সার্চ শুরু করে সেখানে নিয়ে যাবে।

ধাপ ৩: Relative Path ও ডট-নোটেশন ব্যবহার করে এক ধাপ উপরে যাওয়া

ধরুন আপনি বর্তমানে /var/log/nginx ডিরেক্টরিতে আছেন। আপনি চাচ্ছেন এক লেভেল উপরে অর্থাৎ /var/log ফোল্ডারে ব্যাক করতে:

$ cd ..

এখন যদি আপনি pwd দেন, তবে দেখতে পাবেন আপনার বর্তমান ঠিকানা /var/log।

ধাপ ৪: একই লেভেলের অন্য ডিরেক্টরিতে রিলেটিভলি সুইচ করা

ধরুন আপনি বর্তমানে আছেন /var/log-এ। আপনি এক ধাপ উপরে গিয়ে আবার www ফোল্ডারের ভেতরে ঢুকতে চান (অর্থাৎ /var/www পাথে যেতে চান):

$ cd ../www

মেকানিজম ব্যাখ্যা: .. কমান্ডটিকে /var/log থেকে এক ধাপ উপরে /var-এ নিয়ে যায়, এবং সেখান থেকে ফরওয়ার্ড হয়ে www ডিরেক্টরিতে প্রবেশ করায়।

ধাপ ৫: বর্তমান ডিরেক্টরিতে স্ক্রিপ্ট এক্সিকিউট করা (`./`)

লিনাক্সে কারেন্ট ডিরেক্টরির কোনো স্ক্রিপ্ট রান করার সময় আমরা লিখি ./myscript.sh। এখানে . নির্দেশ করে বর্তমান ফোল্ডার এবং / দিয়ে ফাইলের নাম আলাদা করা হয়। এটি একটি পিওর Relative Path অপারেশন।

৫. দ্য টিল্ডা নোটিফিকেশন (The Tilde Symbol `~`)

পাথ অপারেশনের আরেকটি অ্যাডভান্সড ক্যারেক্টার হলো টিল্ডা (~) সাইন। লিনাক্স শেলের ভেতরে এই ~ চিহ্নটি সরাসরি বর্তমান লগইন থাকা ইউজারের Home Directory-র Absolute Path রিপ্রেজেন্ট করে।

যেমন— আপনি যদি টার্মিনালে লেখেন cd ~/projects, ওএস ব্যাকঅ্যান্ডে এটিকে অনুবাদ করে নেবে cd /home/jobaer/projects হিসেবে। এটি ফাইল পাথের রাইটিং স্পিড বহুগুণ বাড়িয়ে দেয়।

৬. Troubleshooting ও প্রফেশনাল পাথ হ্যান্ডলিং এরর

  • The "No such file or directory" Error in Cron Jobs: এটি ডেভঅপ্স এবং সিস্টেম ইঞ্জিনিয়ারদের ফেস করা সবচেয়ে কমন ট্র্যাপ। আপনি একটি বাশ স্ক্রিপ্ট তৈরি করলেন এবং টার্মিনাল থেকে সেটি রিলেটিভ পাথ (যেমন: cat mydata.txt) ব্যবহার করে রান করলেন, স্ক্রিপ্ট পারফেক্টলি কাজ করলো। কিন্তু যখনই সেটি Cron Job-এ শিডিউল করলেন, সেটি ফেইল করে এই এরর দিল।
    কারণ: ক্রন-জব যখন ব্যাকঅ্যান্ডে রান হয়, তখন তার ডিফল্ট কারেন্ট ডিরেক্টরি থাকে রুটের হোমে। ফলে সে আপনার mydata.txt ফাইলটি সেখানে খুঁজে পায় না।
    সমাধান: অটোমেশন বা ক্রন স্ক্রিপ্টের ভেতরে কখনো রিলেটিভ পাথ ব্যবহার করবেন না। সবসময় ফাইলের পূর্ণ Absolute Path ডিফائن করুন: 
    # স্ক্রিপ্টের ভেতরে সঠিক এন্ট্রি মেথড
cat /home/jobaer/data/mydata.txt
  • Accidental Directory Destruction via Relative Path: ধরুন আপনি /var/www/html/old_app ফোল্ডারের ফাইল ডিলিট করার জন্য একটি ডিলিট কমান্ড রান করতে চান। আপনি টার্মিনালে তাড়াহুড়ো করে ভুল ডিরেক্টরিতে থাকা অবস্থায় rm -rf ./.* বা অনুরূপ রিলেটিভ কমান্ড দিলেন। এটি আপনার বর্তমান ফোল্ডারের প্যারেন্ট ডিরেক্টরির সমস্ত মেইন ফাইল মুছে সার্ভার ক্র্যাশ করিয়ে দিতে পারে।

  • Path Traversal Vulnerability (সাইবার সিকিউরিটি ঝুঁকি): ওয়েব অ্যাপ্লিকেশন ডেভেলপ করার সময় যদি ইউজার ইনপুট পাথের সাথে যুক্ত হয়, তবে হ্যাকাররা রিলেটিভ ডট-নোটেশন ইনজেক্ট করে সিস্টেমের ফাইল বাউন্ডারি ব্রেক করতে পারে। যেমন— ইনপুটে ../../../../etc/passwd দিয়ে আপনার সার্ভারের গোপন পাসওয়ার্ড ফাইল চুরি করা। এটি প্রতিরোধ করতে অ্যাপ্লিকেশন লেয়ারে Path Sanitization বা ইনপুট ভ্যালিডেশন নিশ্চিত করা অত্যন্ত জরুরী।

উপসংহার (Conclusion)

Paths এবং এর ইন্টারনাল ডিরেক্টরি ট্রাভার্সাল মেকানিজম লিনাক্স সিস্টেম অ্যাডমিনিস্ট্রেশনের অত্যন্ত মৌলিক ও শক্তিশালী একটি লজিক্যাল ফ্রেমওয়ার্ক। Absolute Path-এর সর্বজনীন স্থায়ী স্থায়িত্ব এবং Relative Path-এর ডট-নোটেশন ভিত্তিক কুইক নেভিগেশন—উভয় পদ্ধতিরই সিস্টেমে সুনির্দিষ্ট কার্যকারিতা রয়েছে। একজন আইটি প্রফেশনাল হিসেবে এই দুই পাথের কারিগরি সীমানা এবং সেশন কনটেক্সট নিখুঁতভাবে বুঝতে পারলে আপনি প্রোডাকশন সার্ভারে যেকোনো স্ক্রিপ্ট অটোমেশন, ক্রন-জব ডিপ্লয়মেন্ট বা কনফিগারেশন ফাইল ডিক্লেয়ারেশন অত্যন্ত নিরাপদ ও নিখুঁতভাবে পরিচালনা ও ট্রাবলশুট করতে পারবেন। নিয়মিত টার্মিনাল ল্যাব প্র্যাকটিসই আপনাকে ডিরেক্টরি নেভিগেশনে মাস্টার করে তুলবে।
