Configuring DHCP Server and DHCP Client on MikroTik RouterOS

01 September, 2026 Networking with MikroTik Jobaer Hossain

নেটওয়ার্কিং জগতে ডিভাইসগুলোকে একে অপরের সাথে যোগাযোগ করানোর জন্য IP Address হলো প্রধান শর্ত। কিন্তু যখন একটি নেটওয়ার্কে ১০টি, ১০০টি বা হাজার খানেক ডিভাইস থাকে, তখন প্রতিটি ডিভাইসে ম্যানুয়ালি আইপি (Static IP) বসানো প্রায় অসম্ভব এবং ভুল হওয়ার সম্ভাবনা থাকে প্রচুর। এই সমস্যার জাদুকরী সমাধান হলো DHCP (Dynamic Host Configuration Protocol)।

একটি MikroTik রাউটার একই সাথে একটি DHCP Client (যে অন্য কারো কাছ থেকে আইপি নেয়) এবং একটি DHCP Server (যে অন্যদের আইপি দেয়) হিসেবে কাজ করতে পারে। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো কীভাবে RouterOS এ এই দুটি কনফিগার করতে হয়, DORA প্রসেস কী এবং স্ট্যাটিক লিজ (Static Lease) কীভাবে কাজ করে।

DHCP এর মূলনীতি (The DORA Process)

DHCP ব্যাকগ্রাউন্ডে চারটি ধাপে কাজ করে, যাকে সংক্ষেপে DORA বলা হয়:

  • Discover: ক্লায়েন্ট পিসি নেটওয়ার্কে ব্রডকাস্ট করে খোঁজে, "এখানে কি কোনো DHCP Server আছে?"
  • Offer: সার্ভার শুনতে পেয়ে বলে, "হ্যাঁ আমি আছি, তুমি এই আইপিটি নিতে পারো।"
  • Request: ক্লায়েন্ট তখন ওই আইপিটি নেওয়ার জন্য সার্ভারকে রিকোয়েস্ট করে।
  • Acknowledge (ACK): সার্ভার চূড়ান্ত অনুমতি দেয় এবং আইপিটি ক্লায়েন্টকে লিজ (Lease) দিয়ে দেয়।

১. MikroTik রাউটারকে DHCP Client হিসেবে কনফিগার করা

রিয়েল-লাইফ সিনারিও: আপনার ইন্টারনেট সার্ভিস প্রোভাইডার (ISP) আপনাকে একটি ইথারনেট ক্যাবল দিয়েছে এবং বলেছে, "ক্যাবলটি লাগালেই অটোমেটিক আইপি এবং ইন্টারনেট পেয়ে যাবেন।" এক্ষেত্রে আপনার রাউটারকে ISP এর সার্ভার থেকে আইপি গ্রহণ করতে হবে। অর্থাৎ, রাউটার এখানে Client।

Winbox এর মাধ্যমে কনফিগারেশন:

  • ISP এর ক্যাবলটি রাউটারের Ether1 পোর্টে লাগান।
  • Winbox দিয়ে লগইন করে বাম পাশের মেনু থেকে IP > DHCP Client এ যান।
  • + (Add) আইকনে ক্লিক করুন।
  • Interface: ড্রপডাউন থেকে ether1 সিলেক্ট করুন।
  • Use Peer DNS এবং Use Peer NTP অপশনগুলোতে চেকমার্ক রাখুন (যাতে ISP এর DNS অটোমেটিক পেয়ে যান)।
  • Add Default Route: এটি yes রাখুন, যাতে রাউটার অটোমেটিক্যালি ইন্টারনেটের রাস্তা (Gateway) চিনে নিতে পারে।
  • Apply এবং OK দিন।
  • স্ট্যাটাস বারে bound লেখা উঠলে বুঝবেন রাউটার সফলভাবে আইপি পেয়েছে।

CLI কমান্ড (Terminal):

যারা কমান্ড লাইনে কাজ করতে পছন্দ করেন, তারা নিচের কমান্ডটি দিন:

[admin@MikroTik] > /ip dhcp-client add interface=ether1 disabled=no add-default-route=yes use-peer-dns=yes

২. MikroTik রাউটারকে DHCP Server হিসেবে কনফিগার করা

রিয়েল-লাইফ সিনারিও: রাউটারে ইন্টারনেট চলে এসেছে। এবার আপনি চান আপনার অফিসের লোকাল নেটওয়ার্কের (LAN) সব পিসি, ল্যাপটপ এবং মোবাইল ফোন যেন ক্যাবল বা ওয়াই-ফাই দিয়ে কানেক্ট হওয়ামাত্রই অটোমেটিক আইপি পেয়ে যায়। এক্ষেত্রে আপনার রাউটার কাজ করবে একটি Server হিসেবে।

পূর্বপ্রস্তুতি (Prerequisite):

DHCP Server তৈরি করার আগে আপনার ল্যান ইন্টারফেসে (যেমন: Ether2) অবশ্যই একটি আইপি বসানো থাকতে হবে।

[admin@MikroTik] > /ip address add address=192.168.10.1/24 interface=ether2

Winbox এর মাধ্যমে কনফিগারেশন (DHCP Setup Wizard):

MikroTik এ সার্ভার কনফিগার করার জন্য একটি চমৎকার Wizard বা সেটআপ গাইড আছে।

  • বাম দিকের মেনু থেকে IP > DHCP Server এ ক্লিক করুন।
  • DHCP ট্যাবে থাকা অবস্থায় DHCP Setup বাটনে ক্লিক করুন।
  • DHCP Server Interface: এখানে আপনার ল্যান পোর্ট ether2 সিলেক্ট করে Next দিন।
  • DHCP Address Space: নেটওয়ার্ক ব্লকটি (যেমন: 192.168.10.0/24) অটোমেটিক চলে আসবে। Next দিন।
  • Gateway for DHCP Network: পিসিগুলোর গেটওয়ে বা রাউটারের আইপি (192.168.10.1) দেখাবে। Next দিন।
  • Addresses to Give Out (Pool): এটি হলো আইপি রেঞ্জ। ডিফল্টভাবে 192.168.10.2-192.168.10.254 থাকবে। আপনি চাইলে এটি এডিট করে 192.168.10.100-192.168.10.200 করে দিতে পারেন। Next দিন।
  • DNS Servers: রাউটারের ডিফল্ট ডিএনএস বা গুগলের ডিএনএস (8.8.8.8) দিন।
  • Lease Time: আইপিটি কতক্ষণের জন্য লিজ দেওয়া হবে (ডিফল্ট 10m বা 10 মিনিট থাকে)। এটি পরিবর্তন করে 1d (1 দিন) করে দিতে পারেন। Next দিন।

সেটআপ সাকসেসফুল মেসেজ আসবে। আপনার সার্ভার রেডি!

CLI কমান্ড (Terminal):

টার্মিনালে সেটআপ উইজার্ড রান করতে নিচের কমান্ডটি দিন এবং এন্টার চাপতে থাকুন:

[admin@MikroTik] > /ip dhcp-server setup

৩. Advanced Concept: Static Lease (আইপি ফিক্সড করে দেওয়া)

DHCP Server এর একটি প্রধান সমস্যা হলো, এটি পিসি রিস্টার্ট নিলে আইপি পরিবর্তন করে দিতে পারে। কিন্তু অফিসে যদি কোনো Printer বা File Server থাকে, তবে তাদের আইপি ফিক্সড (Fixed) বা অপরিবর্তনশীল হওয়া জরুরি। এটি Static Lease বা MAC Binding এর মাধ্যমে করা হয়।

  • Winbox এর IP > DHCP Server এ গিয়ে Leases ট্যাবে যান।
  • এখানে আপনি আপনার নেটওয়ার্কে কানেক্টেড সব ডিভাইসের আইপি এবং MAC Address এর লিস্ট দেখতে পাবেন। লাইনের শুরুতে D লেখা থাকবে (যার মানে Dynamic)।
  • যে ডিভাইসের (যেমন প্রিন্টার) আইপি আপনি ফিক্সড করতে চান, তার ওপর রাইট-ক্লিক করুন।
  • Make Static অপশনে ক্লিক করুন।
  • এখন D লেখাটি চলে যাবে এবং ওই ডিভাইসটি আজীবন একই আইপি পাবে। আপনি চাইলে ডাবল-ক্লিক করে আইপি অ্যাড্রেসটি ম্যানুয়ালি পরিবর্তনও করে দিতে পারেন।

৪. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

সমস্যা (Problem) কারণ এবং সমাধান (Cause & Solution)
DHCP Client স্ট্যাটাস "Searching..." এ আটকে আছে রাউটার ISP থেকে কোনো ব্রডকাস্ট রেসপন্স পাচ্ছে না। ক্যাবল ঠিক আছে কি না চেক করুন। অনেক সময় ISP পোর্টের MAC Address ব্লক করে রাখে (MAC Binding)। ISP কে কল করে আপনার রাউটারের MAC Address দিন।
DHCP Server লিস্টে লাল (Red) রঙের হয়ে আছে যে ইন্টারফেসে (যেমন ether2) সার্ভারটি কনফিগার করা হয়েছে, সেখানে কোনো ক্যাবল লাগানো নেই (Interface Down), অথবা ওই ইন্টারফেসের আইপি মুছে ফেলা হয়েছে। ক্যাবল লাগালে এটি ঠিক হয়ে যাবে।
পিসি আইপি পাচ্ছে না (169.254.x.x দেখাচ্ছে) রাউটারের IP > Pool এ গিয়ে চেক করুন আইপি শেষ (Exhausted) হয়ে গেছে কি না। যদি পুল শেষ হয়ে যায়, তবে সাবনেট মাস্ক (/24 থেকে /23) পরিবর্তন করে রেঞ্জ বাড়াতে হবে।

উপসংহার (Conclusion)

DHCP (Dynamic Host Configuration Protocol) হলো যেকোনো লোকাল নেটওয়ার্কের প্রাণ। DHCP Client এর মাধ্যমে একটি MikroTik রাউটার নিজে ইন্টারনেট সোর্সের সাথে যুক্ত হয়, আর DHCP Server এর মাধ্যমে সে সেই কানেক্টিভিটি অন্যান্য ডিভাইসে ডিস্ট্রিবিউট করে। DHCP Setup Wizard এর ব্যবহার এবং Static Lease এর কনসেপ্ট ক্লিয়ার থাকলে আপনি খুব সহজেই যেকোনো সাইজের এন্টারপ্রাইজ নেটওয়ার্কে আইপি ম্যানেজমেন্টের মতো জটিল কাজটিকে পুরোপুরি অটোমেট (Automate) করে ফেলতে পারবেন।
