Connecting to the MikroTik Router via Telnet and SSH

05 June, 2026 Networking with MikroTik Jobaer Hossain

MikroTik রাউটার কনফিগার করার জন্য গ্রাফিক্যাল টুল হিসেবে Winbox এবং WebFig চমৎকার কাজ করে। কিন্তু একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে আপনাকে সব সময় গ্রাফিক্যাল ইন্টারফেসের ওপর নির্ভর করলে চলবে না। রিমোট সার্ভার ম্যানেজ করা, অটোমেশন স্ক্রিপ্ট রান করানো, অথবা খুব দ্রুত কনফিগারেশন চেক করার জন্য Command Line Interface (CLI) এর কোনো বিকল্প নেই।

যেকোনো নেটওয়ার্ক ডিভাইসে রিমোটলি CLI-তে প্রবেশ করার জন্য সবচেয়ে জনপ্রিয় দুটি প্রোটোকল হলো Telnet এবং SSH। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো এই দুটি প্রোটোকল কী, এদের মধ্যে পার্থক্য কী এবং কীভাবে এগুলোর মাধ্যমে একটি MikroTik রাউটারের সম্পূর্ণ নিয়ন্ত্রণ নেওয়া যায়।

১. Telnet এবং SSH কী?

Telnet (Telecommunication Network): এটি নেটওয়ার্কিংয়ের অত্যন্ত পুরনো একটি প্রোটোকল। এর সাহায্যে আপনি আপনার পিসি থেকে দূরে থাকা কোনো রাউটার বা সুইচের কমান্ড লাইনে (CLI) প্রবেশ করতে পারবেন। এটি লোকাল নেটওয়ার্কে (LAN) দ্রুত কাজ করার জন্য ভালো হলেও ইন্টারনেটের জন্য মোটেও নিরাপদ নয়।

SSH (Secure Shell): এটি হলো Telnet এর আধুনিক এবং অত্যন্ত সিকিউর (Secure) ভার্সন। Telnet এর মতোই এটি দিয়ে কমান্ড লাইনে প্রবেশ করা যায়, কিন্তু এর সবচেয়ে বড় সুবিধা হলো, এটি ডাটাকে এনক্রিপ্ট (Encrypt) করে পাঠায়। ফলে মাঝপথে কোনো হ্যাকার ডাটা চুরি করলেও তা পড়তে পারে না।

২. Telnet বনাম SSH: কোনটি ব্যবহার করবেন?

আপনার প্রোডাকশন নেটওয়ার্কে (Production Network) কোনটি ব্যবহার করা উচিত, তা বুঝতে নিচের তুলনামূলক টেবিলটি লক্ষ করুন:

বৈশিষ্ট্য (Feature) Telnet SSH (Secure Shell)
ডিফল্ট পোর্ট (Default Port) TCP Port 23 TCP Port 22
সিকিউরিটি (Security) Insecure। ডাটা Plain Text হিসেবে যায়। Highly Secure। ডাটা Encrypted অবস্থায় যায়।
ডাটা চুরি হওয়ার ঝুঁকি হ্যাকার Wireshark দিয়ে প্যাকেট ক্যাপচার করে সহজেই আপনার Username এবং Password দেখে ফেলতে পারে। প্যাকেট ক্যাপচার করলেও হ্যাকার শুধু কিছু এলোমেলো (Garbage) এনক্রিপ্টেড টেক্সট দেখতে পাবে।
রিকমেন্ডেশন (Recommendation) শুধুমাত্র লোকাল ল্যাবে (Local Lab) টেস্টিংয়ের জন্য ব্যবহার করা উচিত। যেকোনো লাইভ নেটওয়ার্ক বা ইন্টারনেটে সব সময় SSH ব্যবহার করা বাধ্যতামূলক।

৩. Telnet এর মাধ্যমে কানেক্ট করার পদ্ধতি

আপনার পিসি যদি উইন্ডোজ (Windows) হয়, তবে ডিফল্টভাবে Telnet ক্লায়েন্ট বন্ধ থাকতে পারে। এটি চালু করে বা থার্ড-পার্টি সফটওয়্যার দিয়ে কানেক্ট করতে পারেন।

  • Command Prompt (CMD) ব্যবহার করে:
    • উইন্ডোজের Start মেনু থেকে cmd লিখে Command Prompt ওপেন করুন।
    • কমান্ড দিন: telnet 192.168.88.1 (এখানে আপনার রাউটারের IP Address বসাবেন)।
    • রাউটার আপনার কাছে Login: চাইবে। ইউজারনেম (যেমন admin) দিয়ে এন্টার দিন।
    • এরপর Password: চাইলে পাসওয়ার্ড দিয়ে এন্টার দিন। ব্যাস, আপনি MikroTik এর CLI তে প্রবেশ করেছেন।
  • PuTTY ব্যবহার করে:
    • ইন্টারনেট থেকে PuTTY নামের ফ্রি সফটওয়্যারটি ডাউনলোড করে ওপেন করুন।
    • Host Name (or IP address) এর ঘরে রাউটারের আইপি দিন।
    • Connection Type এ Telnet রেডিও বাটনটি সিলেক্ট করে Open এ ক্লিক করুন।

৪. SSH এর মাধ্যমে কানেক্ট করার পদ্ধতি (Recommended)

সবচেয়ে সিকিউর পদ্ধতি হলো SSH ব্যবহার করা। বর্তমানে Windows 10/11 এবং Linux/Mac সিস্টেমে ডিফল্টভাবেই SSH ক্লায়েন্ট ইন্সটল করা থাকে।

  • Command Prompt (CMD) বা Terminal ব্যবহার করে:
    • টার্মিনাল ওপেন করুন এবং কমান্ড দিন: ssh admin@192.168.88.1 (এখানে admin হলো আপনার ইউজারনেম এবং 192.168.88.1 হলো রাউটারের আইপি)।
    • প্রথমবার কানেক্ট করার সময় সিস্টেম একটি সিকিউরিটি ওয়ার্নিং দিয়ে জিজ্ঞেস করবে আপনি এই ডিভাইসকে ট্রাস্ট করেন কি না। সেখানে yes লিখে এন্টার দিন।
    • এরপর আপনার পাসওয়ার্ড চাইবে। পাসওয়ার্ড টাইপ করার সময় স্ক্রিনে কিছু দেখা যাবে না (সিকিউরিটির জন্য), টাইপ করা শেষ হলে এন্টার দিন।
  • PuTTY ব্যবহার করে:
    • PuTTY ওপেন করুন।
    • Host Name এর ঘরে আইপি দিন এবং Connection Type এ SSH সিলেক্ট করুন (Port অটোমেটিক 22 হয়ে যাবে)।
    • Open বাটনে ক্লিক করে ইউজারনেম ও পাসওয়ার্ড দিয়ে লগইন করুন।

৫. MikroTik এ Service Management কনফিগারেশন

নিরাপত্তার স্বার্থে রাউটারের অদরকারি সার্ভিসগুলো বন্ধ করে রাখা বা ডিফল্ট পোর্ট পরিবর্তন করা একজন ভালো নেটওয়ার্ক অ্যাডমিনের প্রধান কাজ। চলুন দেখি MikroTik এ এটি কীভাবে করতে হয়:

চলমান সার্ভিসগুলো চেক করা:

Winbox এর New Terminal এ গিয়ে নিচের কমান্ডটি দিন:

[admin@MikroTik] > /ip service print

এখানে আপনি দেখতে পাবেন ftp, ssh, telnet, winbox, www (webfig) কোন কোন পোর্টে চলছে এবং সেগুলো এনাবল (Enable) আছে কি না।

Telnet সার্ভিস চিরতরে বন্ধ (Disable) করা:

যেহেতু Telnet সিকিউর নয়, তাই প্রোডাকশন রাউটারে এটি সবসময় বন্ধ রাখা উচিত:

[admin@MikroTik] > /ip service disable telnet

SSH এর ডিফল্ট পোর্ট পরিবর্তন করা:

হ্যাকাররা সব সময় ডিফল্ট পোর্টে (যেমন 22) অ্যাটাক করে। তাই সিকিউরিটি বাড়ানোর জন্য SSH এর পোর্ট পরিবর্তন করে দেওয়া উচিত (যেমন 2222 বা অন্য যেকোনো আনকমন পোর্ট):

[admin@MikroTik] > /ip service set ssh port=2222

নোট: পোর্ট পরিবর্তন করার পর, পরবর্তীতে SSH করার সময় আপনাকে পোর্ট উল্লেখ করে দিতে হবে। যেমন: ssh admin@192.168.88.1 -p 2222

৬. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

রিমোটলি কানেক্ট করতে গিয়ে কিছু সাধারণ সমস্যার সম্মুখীন হতে পারেন:

  • "Connection Refused" এরর দেখানো: এর মানে হলো রাউটার পর্যন্ত রিকোয়েস্ট পৌঁছাচ্ছে, কিন্তু রাউটারের ওই পোর্টটি বন্ধ আছে। /ip service print দিয়ে চেক করুন SSH বা Telnet সার্ভিসটি Disable করা আছে কি না, অথবা এর পোর্ট পরিবর্তন করা হয়েছে কি না।
  • "Connection Timed Out" এরর দেখানো: এর মানে হলো আপনার পিসি থেকে রাউটার পর্যন্ত কানেকশন পৌঁছাচ্ছে না। এর প্রধান কারণ হতে পারে মাঝখানের কোনো Firewall Rule আপনার রিকোয়েস্ট ড্রপ করে দিচ্ছে, অথবা রাউটারের আইপি অ্যাড্রেস ভুল। রাউটারকে Ping করে চেক করুন।
  • SSH Key Error: যদি আপনি রাউটার রিসেট করেন বা নতুন রাউটারে একই আইপি বসান, তবে আগের সেভ করা SSH Key এর সাথে মিলবে না। তখন WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED! এরর দিতে পারে। উইন্ডোজে .ssh/known_hosts ফাইল থেকে ওই আইপির এন্ট্রিটি মুছে ফেললেই সমাধান হয়ে যাবে।

উপসংহার (Conclusion)

একটি MikroTik Router প্রফেশনাল লেভেলে মেইনটেইন করার জন্য CLI (Command Line Interface) এ দক্ষতা অর্জন করা অত্যন্ত জরুরি। Telnet হয়তো শেখার জন্য সহজ, কিন্তু প্রোডাকশন নেটওয়ার্কে ডাটা লিক হওয়ার মারাত্মক ঝুঁকি থাকায় এর ব্যবহার এখন প্রায় শূন্যের কোঠায়। অন্যদিকে SSH আপনাকে দেয় সর্বোচ্চ সিকিউরিটির নিশ্চয়তা। সার্ভিসগুলোকে সঠিকভাবে ম্যানেজ করা এবং ডিফল্ট পোর্টগুলো পরিবর্তন করে নেওয়ার মাধ্যমে আপনি আপনার নেটওয়ার্ককে ব্রুট-ফোর্স অ্যাটাক (Brute-force attack) থেকে সহজেই রক্ষা করতে পারবেন।
