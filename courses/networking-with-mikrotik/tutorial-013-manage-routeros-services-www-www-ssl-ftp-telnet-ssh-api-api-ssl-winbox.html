Manage RouterOS Services (www, www-ssl, FTP, Telnet, SSH, API, API-SSL, Winbox)

05 August, 2026 Networking with MikroTik Jobaer Hossain

একটি নতুন MikroTik Router বক্স থেকে বের করার পর সেটি অনেকটা এমন একটি বাড়ির মতো, যার সবগুলো দরজা খোলা। আপনি চাইলে Winbox দিয়ে ঢুকতে পারেন, ব্রাউজার (WebFig) দিয়ে ঢুকতে পারেন, আবার কমান্ড লাইন (SSH/Telnet) দিয়েও ঢুকতে পারেন। এই এক্সেস পয়েন্টগুলোকেই RouterOS Services বলা হয়।

সমস্যা হলো, এই দরজাগুলো যদি সবসময় সবার জন্য খোলা থাকে, তবে ইন্টারনেটে থাকা হ্যাকাররাও আপনার রাউটারে প্রবেশ করার চেষ্টা করবে। একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে আপনার প্রথম কাজগুলোর একটি হলো অপ্রয়োজনীয় সার্ভিসগুলো বন্ধ করে দেওয়া এবং প্রয়োজনীয় সার্ভিসগুলোকে সুরক্ষিত (Secure) করা। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা RouterOS-এর প্রতিটি সার্ভিস সম্পর্কে বিস্তারিত জানবো এবং কীভাবে এগুলোকে কনফিগার ও ম্যানেজ করতে হয় তা শিখবো।

Service Management কেন এত গুরুত্বপূর্ণ?

রিয়েল-লাইফ সিনারিও: আপনি একটি কর্পোরেট অফিসে MikroTik রাউটার বসিয়েছেন। রাউটারটি ইন্টারনেটের (Public IP) সাথে যুক্ত। আপনি যদি ডিফল্ট SSH (Port 22) সার্ভিসটি খোলা রাখেন, তবে দেখতে পাবেন প্রতিদিন হাজার হাজার বট (Bot) আপনার রাউটারে ব্রুট-ফোর্স (Brute-force) অ্যাটাক করছে পাসওয়ার্ড ব্রেক করার জন্য। এতে রাউটারের CPU ইউজেস বেড়ে যাবে এবং নেটওয়ার্ক স্লো হয়ে যাবে। সার্ভিস ম্যানেজমেন্টের মাধ্যমে নির্দিষ্ট পোর্টে নির্দিষ্ট আইপি (IP) থেকে এক্সেস সীমাবদ্ধ করে এই ধরনের আক্রমণ ১০০% প্রতিরোধ করা সম্ভব।

১. RouterOS এর Default Service সমূহের তালিকা

MikroTik RouterOS-এ মূলত ৮টি বিল্ট-ইন সার্ভিস থাকে। নিচে এদের নাম, ডিফল্ট পোর্ট এবং কাজের বিবরণ দেওয়া হলো:

সার্ভিসের নাম (Service) ডিফল্ট পোর্ট (Port) কাজের বিবরণ (Description)
winbox TCP 8291 MikroTik এর নিজস্ব গ্রাফিক্যাল টুল Winbox দিয়ে লগইন করার জন্য। এটি সবচেয়ে বেশি ব্যবহৃত হয়।
www (WebFig) TCP 80 যেকোনো ওয়েব ব্রাউজার দিয়ে রাউটারে লগইন করার জন্য। এর ডাটা এনক্রিপ্টেড (Encrypted) থাকে না।
www-ssl TCP 443 ব্রাউজার দিয়ে লগইন করার সিকিউরড মাধ্যম। এটি কাজ করার জন্য রাউটারে SSL Certificate থাকতে হয়।
ssh TCP 22 কমান্ড লাইন ইন্টারফেসে (CLI) লগইন করার সিকিউর মাধ্যম। এর ডাটা সম্পূর্ণ এনক্রিপ্টেড থাকে।
telnet TCP 23 CLI-তে লগইন করার আনসিকিউর মাধ্যম। এর ডাটা প্লেইন টেক্সট (Plain text) হিসেবে ট্রান্সফার হয়।
api TCP 8728 অন্য কোনো থার্ড-পার্টি সফটওয়্যার (যেমন: ISP Billing Software বা Python স্ক্রিপ্ট) রাউটারের সাথে কমিউনিকেট করার জন্য এটি ব্যবহার করে।
api-ssl TCP 8729 API এর সিকিউরড বা এনক্রিপ্টেড ভার্সন।
ftp TCP 21 File Transfer Protocol। রাউটারে কোনো ফাইল (যেমন ব্যাকআপ ফাইল বা .npk আপডেট ফাইল) আপলোড বা ডাউনলোড করার জন্য ব্যবহৃত হয়।

২. Service Management এর বেস্ট প্র্যাকটিস (Security)

রাউটারকে হ্যাকারদের হাত থেকে বাঁচাতে হলে আপনাকে তিনটি মূলনীতি মেনে চলতে হবে:

  1. Disable (বন্ধ করুন): যে সার্ভিসগুলো আপনার একদমই প্রয়োজন নেই, সেগুলো চিরতরে বন্ধ (Disable) করে দিন। যেমন: telnet, ftp, www।
  2. Change Port (পোর্ট পরিবর্তন করুন): যে সার্ভিসগুলো আপনার দরকার (যেমন Winbox বা SSH), সেগুলোর ডিফল্ট পোর্ট পরিবর্তন করে আনকমন কোনো পোর্ট (যেমন 8291 এর বদলে 58291) সেট করুন। হ্যাকাররা সাধারণত ডিফল্ট পোর্টে অ্যাটাক করে।
  3. Available From (আইপি রেস্ট্রিকশন): এটি সবচেয়ে শক্তিশালী সিকিউরিটি ফিচার। আপনি নির্দিষ্ট করে দিতে পারেন যে শুধুমাত্র আপনার আইটি রুমের লোকাল আইপি ব্লক (যেমন 192.168.10.0/24) থেকেই Winbox এ লগইন করা যাবে, অন্য কোনো আইপি থেকে নয়।

৩. Winbox ব্যবহার করে Service কনফিগারেশন

গ্রাফিক্যাল ইন্টারফেস ব্যবহার করে সার্ভিস ম্যানেজ করা খুবই সহজ।

  • Winbox দিয়ে রাউটারে লগইন করুন।
  • বাম পাশের মেনু থেকে IP এ ক্লিক করে Services এ যান।
  • একটি উইন্ডো ওপেন হবে যেখানে আটটি সার্ভিস লিস্ট করা থাকবে।
  • Disable করতে: telnet বা ftp সিলেক্ট করে ওপরের X (লাল ক্রস) আইকনে ক্লিক করুন। সার্ভিসটি ধূসর (Gray) হয়ে যাবে।
  • Port পরিবর্তন করতে: winbox এ ডাবল ক্লিক করুন। Port: এর ঘরে 8291 মুছে আপনার পছন্দের কোনো পোর্ট (যেমন 18291) বসিয়ে Apply দিন।
  • Available From সেট করতে: একই উইন্ডোতে Available From: এর ঘরে আপনার কাঙ্ক্ষিত IP Address বা Subnet (যেমন 192.168.10.0/24) লিখে OK দিন।

৪. CLI (Command Line Interface) ব্যবহার করে কনফিগারেশন

নেটওয়ার্ক প্রফেশনালরা সাধারণত কমান্ড লাইনে কাজ করতে বেশি স্বাচ্ছন্দ্য বোধ করেন। টার্মিনাল ওপেন করে নিচের কমান্ডগুলো প্র্যাকটিস করুন:

A. বর্তমান সার্ভিসগুলোর স্ট্যাটাস দেখা:

[admin@MikroTik] > /ip service print

এই কমান্ড দিলে কোন পোর্ট এনাবল আছে এবং কোনটিতে Available From সেট করা আছে তা দেখতে পাবেন।

B. অপ্রয়োজনীয় সার্ভিস বন্ধ করা (Disable):

Telnet, FTP এবং WWW (সাধারণ ওয়েব) সার্ভিস বন্ধ করতে:

[admin@MikroTik] > /ip service disable telnet,ftp,www

C. পোর্টের নাম্বার পরিবর্তন করা:

Winbox এর ডিফল্ট পোর্ট পরিবর্তন করতে:

[admin@MikroTik] > /ip service set winbox port=18291

(নোট: পোর্ট পরিবর্তন করার পর, পরবর্তীতে Winbox দিয়ে লগইন করার সময় আইপির সাথে পোর্ট উল্লেখ করতে হবে। যেমন: 192.168.88.1:18291)

D. 특정 IP থেকে অ্যাক্সেস দেওয়া (Available From):

SSH সার্ভিসকে শুধুমাত্র একটি নির্দিষ্ট লোকাল ব্লক থেকে অ্যাক্সেস করার অনুমতি দিতে:

[admin@MikroTik] > /ip service set ssh available-from=192.168.10.0/24

৫. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

সার্ভিস ম্যানেজ করতে গিয়ে ভুলবশত কিছু সমস্যা তৈরি হতে পারে:

  • নিজে লক আউট হয়ে যাওয়া (Locked Out): আপনি যদি winbox সার্ভিসের Available From ফিল্ডে ভুল আইপি বসিয়ে সেভ করেন, তবে আপনি সাথে সাথে লগ আউট হয়ে যাবেন। এর সমাধান হলো MAC Address দিয়ে Winbox এ লগইন করা (কারণ MAC কানেকশনে IP রেস্ট্রিকশন কাজ করে না) এবং আইপিটি ঠিক করা।
  • Billing Software কানেক্ট হচ্ছে না: আপনার আইএসপির বিলিং সফটওয়্যার যদি রাউটারের সাথে ডেটা সিঙ্ক করতে না পারে, তবে চেক করুন API সার্ভিসটি চালু আছে কি না। /ip service print দিয়ে দেখুন এটি Disable করা আছে কি না।
  • পোর্ট কনফ্লিক্ট (Port Conflict): আপনি যদি SSH এর পোর্ট পরিবর্তন করে 80 (যা www এর পোর্ট) করে দেন, তবে সার্ভিস কনফ্লিক্ট হবে। সব সময় ৫-ডিজিটের আনকমন পোর্ট (যেমন 10000 থেকে 65000 এর মধ্যে) ব্যবহার করার চেষ্টা করবেন।

উপসংহার (Conclusion)

RouterOS Services গুলো হলো আপনার নেটওয়ার্ক ডিভাইসে প্রবেশ করার মূল দরজা। এই দরজাগুলোর সঠিক ম্যানেজমেন্ট ছাড়া কোনো নেটওয়ার্কই সুরক্ষিত হতে পারে না। অপ্রয়োজনীয় Telnet এবং FTP সার্ভিস বন্ধ রাখা, Winbox ও SSH এর ডিফল্ট পোর্ট পরিবর্তন করা এবং Available From এর মাধ্যমে আইপি রেস্ট্রিকশন দেওয়া—এই বেসিক কনফিগারেশনগুলো প্রতিটি রাউটার সেটআপ করার প্রথম দিনেই সম্পন্ন করা উচিত। এতে আপনার রাউটার ব্রুট-ফোর্স অ্যাটাক থেকে শতভাগ নিরাপদ থাকবে এবং প্রসেসিং পাওয়ার সাশ্রয় হবে।
