Manipulating ARP on MikroTik RouterOS (Enabled, Disabled, Reply-only)

05 September, 2026 Networking with MikroTik Jobaer Hossain

নেটওয়ার্কিং জগতে ARP (Address Resolution Protocol) কে বলা যায় লোকাল নেটওয়ার্কের ফোনবুক বা ডিরেক্টরি। যখন নেটওয়ার্কের কোনো ডিভাইস অন্য ডিভাইসের সাথে যোগাযোগ করতে চায়, তখন তার IP Address (Layer 3) জানা থাকলেও MAC Address (Layer 2) জানার প্রয়োজন হয়। ARP এই IP Address এর বিপরীতে সঠিক MAC Address খুঁজে বের করার কাজটি করে।

সাধারণত রাউটারগুলো স্বয়ংক্রিয়ভাবে ARP ম্যানেজ করে। কিন্তু একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে, নেটওয়ার্কের সিকিউরিটি নিশ্চিত করতে এবং আনঅথরাইজড (Unauthorized) ইন্টারনেট ব্যবহার ঠেকাতে আপনাকে MikroTik RouterOS-এ ARP ম্যানিপুলেট বা নিয়ন্ত্রণ করতে হবে। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো ARP এর বিভিন্ন মোড (Enabled, Disabled, Reply-only) কীভাবে কাজ করে এবং রিয়েল-লাইফ প্রোডাকশনে এগুলো কীভাবে কনফিগার করতে হয়।

কেন ARP ম্যানিপুলেট করা প্রয়োজন? (The Real-life Scenario)

আইপি চুরি বা IP Spoofing: মনে করুন, আপনি একটি অফিসে বা ছোট ISP তে নেটওয়ার্ক অ্যাডমিন। আপনি একজন ইউজারকে 192.168.10.5 আইপিটি দিয়েছেন এবং তার স্পিড লিমিট করেছেন 2 Mbps। এখন সেই ইউজার যদি চালাকি করে তার পিসির আইপি পরিবর্তন করে বসের পিসির আইপি (যেমন 192.168.10.2) বসিয়ে দেয়, তবে সে বসের আনলিমিটেড স্পিড ব্যবহার করতে পারবে। একে IP Spoofing বলে।

রাউটারের ডিফল্ট ARP সেটিংসে এই চুরি ঠেকানো সম্ভব নয়। কিন্তু আমরা যদি রাউটারের ইন্টারফেসে ARP ম্যানিপুলেট করে Reply-only করে দিই, তবে এই ধরনের আইপি চুরি ১০০% ঠেকানো সম্ভব।

১. MikroTik এ ARP এর বিভিন্ন Mode

MikroTik রাউটারে প্রতিটি ইন্টারফেসের (যেমন ether1, ether2, bridge) নিজস্ব ARP সেটিংস থাকে। প্রধানত তিনটি মোড সবচেয়ে বেশি ব্যবহৃত হয়:

ARP Mode কাজের ধরন (Behavior) ব্যবহারের ক্ষেত্র (Use Case)
Enabled (ডিফল্ট) রাউটার স্বয়ংক্রিয়ভাবে নেটওয়ার্কের সব ডিভাইসের IP এবং MAC Address শিখে নেয় (Dynamic Learning) এবং লিস্টে সেভ করে। সাধারণ হোম নেটওয়ার্ক বা ট্রাস্টেড (Trusted) অফিস নেটওয়ার্কের জন্য, যেখানে সিকিউরিটি নিয়ে বেশি ভাবতে হয় না।
Disabled রাউটার কোনো নতুন MAC Address শিখবে না এবং কোনো ARP Request এর রিপ্লাইও দেবে না। অত্যন্ত হাই-সিকিউরিটি নেটওয়ার্কের জন্য। এই মোডে কাজ করতে হলে রাউটার এবং ক্লায়েন্ট পিসি—উভয় জায়গাতেই ম্যানুয়ালি Static ARP এন্ট্রি করতে হয়।
Reply-only রাউটার কোনো নতুন এন্ট্রি ডাইনামিক্যালি শিখবে না। কিন্তু রাউটারের ARP লিস্টে যদি আগে থেকে কোনো IP-MAC সেভ করা (Static) থাকে, শুধু তাকেই রিপ্লাই দেবে। Best Practice: লোকাল নেটওয়ার্কে IP Spoofing বা আইপি চুরি ঠেকাতে DHCP Server এর সাথে মিলিয়ে এটি সবচেয়ে বেশি ব্যবহৃত হয়।

নোট: এছাড়াও proxy-arp এবং local-proxy-arp নামে আরও দুটি মোড আছে, যা মূলত VPN বা স্পেশাল রাউটিংয়ের ক্ষেত্রে ব্যবহৃত হয়।

২. Real-life Configuration: Reply-only এর মাধ্যমে IP চুরি ঠেকানো

চলুন ধাপে ধাপে দেখি কীভাবে আমরা একটি ল্যান (LAN) নেটওয়ার্ককে Reply-only মোড ব্যবহার করে সম্পূর্ণ সুরক্ষিত করতে পারি।

Step 1: DHCP Server এ "Add ARP for Leases" চালু করা

আমরা চাই যে আমাদের DHCP Server যাকে আইপি দেবে, শুধুমাত্র সেই যেন ইন্টারনেট পায়। অন্য কেউ ম্যানুয়ালি আইপি বসালে যেন ইন্টারনেট না পায়।

  • Winbox থেকে IP > DHCP Server এ যান।
  • আপনার তৈরি করা DHCP Server টির ওপর ডাবল-ক্লিক করুন।
  • উইন্ডোর নিচের দিকে Add ARP for Leases নামের একটি চেকবক্স পাবেন। সেটিতে টিক (Check) দিন।
  • Apply এবং OK দিন।

ব্যাখ্যা: এর ফলে DHCP Server কোনো পিসিকে আইপি দেওয়ার সাথে সাথে অটোমেটিক রাউটারের ARP List-এ ওই পিসির IP এবং MAC Address এর একটি ডাইনামিক এন্ট্রি করে দেবে।

Step 2: Interface এর ARP Mode পরিবর্তন করা

এখন আমরা রাউটারের ল্যান ইন্টারফেসকে বলবো, "তোমার লিস্টে যে আইপিগুলো আছে, শুধু তাদেরকেই চিনবে, নতুন কাউকে নয়।"

  • Winbox থেকে Interfaces মেনুতে যান।
  • আপনার লোকাল নেটওয়ার্কের ইন্টারফেসটির (যেমন: ether2 বা bridge1) ওপর ডাবল-ক্লিক করুন।
  • General ট্যাবের নিচে ARP: নামে একটি ড্রপডাউন মেনু আছে।
  • সেখান থেকে ডিফল্ট enabled পরিবর্তন করে reply-only সিলেক্ট করুন।
  • Apply এবং OK দিন।

Step 3: Static ARP Entry তৈরি করা (ঐচ্ছিক)

যদি আপনার নেটওয়ার্কে এমন কোনো ডিভাইস থাকে (যেমন: Printer বা Server) যেগুলোতে ম্যানুয়ালি স্ট্যাটিক আইপি বসানো আছে (DHCP ছাড়া), তবে তারা reply-only মোডের কারণে ইন্টারনেট বা ল্যান কানেকশন হারিয়ে ফেলবে। তাদের জন্য আপনাকে ম্যানুয়ালি ARP এন্ট্রি করতে হবে:

  • IP > ARP মেনুতে যান।
  • + (Add) আইকনে ক্লিক করুন।
  • IP Address: বক্সে প্রিন্টার বা সার্ভারের আইপি দিন (যেমন: 192.168.10.200)।
  • MAC Address: বক্সে ওই ডিভাইসের সঠিক MAC Address দিন।
  • Interface: আপনার ল্যান ইন্টারফেস (যেমন: ether2) সিলেক্ট করে OK দিন।

৩. Command Line (CLI) ব্যবহার করে কনফিগারেশন

প্রফেশনালরা টার্মিনাল ব্যবহার করে খুব দ্রুত এই কাজগুলো করতে পারেন:

ইন্টারফেসকে Reply-only করা:

[admin@MikroTik] > /interface ethernet set ether2 arp=reply-only

DHCP সার্ভারে Add ARP for Leases চালু করা:

[admin@MikroTik] > /ip dhcp-server set dhcp1 add-arp-leases=yes

Static ARP যুক্ত করা:

[admin@MikroTik] > /ip arp add address=192.168.10.200 mac-address=00:11:22:33:44:55 interface=ether2

৪. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

  • Reply-only করার পর সবাই ডিসকানেক্ট হয়ে গেছে: আপনি যদি DHCP সার্ভারে Add ARP for Leases অপশনটি চালু করতে ভুলে যান এবং ইন্টারফেসকে reply-only করে দেন, তবে নেটওয়ার্কের কেউ রাউটারের সাথে কমিউনিকেট করতে পারবে না। দ্রুত ইন্টারফেস সেটিংসে গিয়ে ARP আবার enabled করে দিন এবং কনফিগারেশন পুনরায় চেক করুন।
  • MAC Address স্পুফিং (Spoofing): reply-only মোড আইপি চুরি ঠেকাতে পারলেও, যদি কোনো অ্যাডভান্সড ইউজার আপনার আইপির পাশাপাশি আপনার MAC Address ও কপি বা স্পুফ করে ফেলে, তবে সে নেটওয়ার্ক এক্সেস পেয়ে যাবে। এর হাত থেকে বাঁচতে ম্যানেজেবল সুইচে (Layer 2 Switch) Port Security বা DHCP Snooping কনফিগার করতে হয়।
  • IP > ARP লিস্ট চেক করা: নেটওয়ার্কে কোনো সমস্যা হলে /ip arp print কমান্ড দিয়ে চেক করুন রাউটার ক্লায়েন্টের সঠিক MAC শিখতে পারছে কি না। কোনো এন্ট্রি না থাকলে বা ভুল থাকলে বুঝতে হবে ARP মোডে সমস্যা আছে।

উপসংহার (Conclusion)

নেটওয়ার্ক ম্যানেজমেন্টে ARP Manipulation একটি অত্যন্ত শক্তিশালী টুল। ডিফল্ট Enabled মোড সাধারণ কাজের জন্য ভালো হলেও, অফিস, ক্যাম্পাস বা ISP নেটওয়ার্ককে সুরক্ষিত রাখতে Reply-only মোডের কোনো বিকল্প নেই। DHCP Server এর সাথে মিলিয়ে এই ফিচারটি ব্যবহার করলে আপনি খুব সহজেই আইপি চুরি (IP Spoofing) এবং আনঅথরাইজড ডিভাইসের অনুপ্রবেশ বন্ধ করতে পারবেন, যা আপনার নেটওয়ার্কের পারফরম্যান্স এবং সিকিউরিটি—উভয়ই নিশ্চিত করবে।
