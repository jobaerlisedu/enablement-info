Changing destination address using DST-NAT

20 November, 2026 Networking with MikroTik Jobaer Hossain

নেটওয়ার্কিং জগতে NAT (Network Address Translation) একটি যুগান্তকারী প্রযুক্তি, যা মূলত দুটি ভাগে কাজ করে: Source NAT এবং Destination NAT। আমরা সবাই জানি যে লোকাল নেটওয়ার্ক থেকে ইন্টারনেটে যাওয়ার জন্য Source NAT বা Masquerade ব্যবহার করতে হয়। কিন্তু যখন ইন্টারনেট থেকে কেউ আপনার লোকাল নেটওয়ার্কে প্রবেশ করতে চায়, তখন কী হবে?

লোকাল বা Private IP ইন্টারনেটে রাউট (Route) হতে পারে না। তাই বাইরে থেকে লোকাল নেটওয়ার্কের কোনো সার্ভারকে এক্সেস করার জন্য আমাদের DST-NAT (Destination NAT) ব্যবহার করে প্যাকেটের গন্তব্য বা Destination Address পরিবর্তন করতে হয়। সাধারণ ভাষায় একে Port Forwarding বলা হয়। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা একটি রিয়েল-লাইফ উদাহরণের মাধ্যমে শিখবো কীভাবে MikroTik RouterOS-এ DST-NAT কনফিগার করতে হয়।

রিয়েল-লাইফ সিনারিও (The Web Server Scenario)

ধরুন, আপনার অফিসে একটি Web Server আছে, যার লোকাল বা Private IP হলো 192.168.10.50। এই সার্ভারটি পোর্ট 80 (HTTP) এ কাজ করে। আপনার আইএসপি (ISP) আপনার রাউটারের WAN পোর্টে একটি Public IP দিয়েছে, যা হলো 203.0.113.10।

লক্ষ্য: আপনি চান পৃথিবীর যেকোনো প্রান্ত থেকে কেউ ব্রাউজারে আপনার Public IP (203.0.113.10) লিখে এন্টার দিলে, রাউটার যেন সেই ট্রাফিকটিকে রিসিভ করে লোকাল নেটওয়ার্কের ওয়েব সার্ভারের (192.168.10.50) কাছে পাঠিয়ে দেয়।

১. DST-NAT কীভাবে কাজ করে?

যখন কোনো ইউজার আপনার পাবলিক আইপিতে রিকোয়েস্ট পাঠায়, তখন সেই ডাটা প্যাকেটের (Packet) গায়ে দুটি জিনিস লেখা থাকে:

  • Source IP: ইউজারের নিজের আইপি (যেখান থেকে রিকোয়েস্ট এসেছে)।
  • Destination IP: আপনার রাউটারের পাবলিক আইপি (203.0.113.10)।

ডাটা প্যাকেটটি রাউটারের কাছে আসার পর, ফায়ারওয়ালের dstnat চেইন সেই প্যাকেটের গন্তব্য (Destination IP) পরিবর্তন করে লোকাল সার্ভারের আইপি (192.168.10.50) বসিয়ে দেয়। এরপর রাউটিং টেবিল অনুযায়ী প্যাকেটটি লোকাল সার্ভারের কাছে পৌঁছে যায়।

২. Winbox ব্যবহার করে DST-NAT কনফিগারেশন

আমাদের Web Server সিনারিওটি এখন Winbox এর মাধ্যমে কনফিগার করবো।

Step 1: General Tab কনফিগারেশন

  • Winbox ওপেন করে IP > Firewall এ যান।
  • NAT ট্যাবে ক্লিক করে নতুন একটি রুল তৈরি করতে + (Add) আইকনে ক্লিক করুন।
  • Chain: ড্রপডাউন থেকে dstnat সিলেক্ট করুন।
  • Dst. Address: এর ঘরে আপনার রাউটারের পাবলিক আইপি 203.0.113.10 দিন। (এটি দেওয়া জরুরি, নাহলে রাউটার অন্য সব ট্রাফিককেও ভুল দিকে পাঠিয়ে দিতে পারে)।
  • Protocol: 6 (tcp) সিলেক্ট করুন (যেহেতু ওয়েব সার্ভার TCP তে কাজ করে)।
  • Dst. Port: 80 লিখুন।

Step 2: Action Tab কনফিগারেশন

  • উপরের মেনু থেকে Action ট্যাবে যান।
  • Action: ড্রপডাউন থেকে dst-nat সিলেক্ট করুন।
  • To Addresses: এর ঘরে আপনার লোকাল ওয়েব সার্ভারের আইপি 192.168.10.50 দিন।
  • To Ports: এর ঘরে 80 দিন।
  • Apply এবং OK দিন। কনফিগারেশন সম্পন্ন!

৩. Command Line (CLI) ব্যবহার করে কনফিগারেশন

টার্মিনাল থেকে মাত্র এক লাইনের কমান্ড দিয়ে এই পুরো কাজটি করা সম্ভব:

[admin@MikroTik] > /ip firewall nat add chain=dstnat dst-address=203.0.113.10 protocol=tcp dst-port=80 action=dst-nat to-addresses=192.168.10.50 to-ports=80 comment="Web Server Port Forward"

৪. Advanced Scenario: Port Translation (পোর্ট পরিবর্তন)

সিকিউরিটির কারণে অনেক সময় আমরা বাইরের পোর্ট এবং ভেতরের পোর্ট এক রাখতে চাই না। একে Port Translation বলে।

উদাহরণ: আপনার লোকাল নেটওয়ার্কে একটি পিসি আছে (192.168.10.100) যার Remote Desktop (RDP) পোর্ট হলো 3389। আপনি ইন্টারনেটে সরাসরি 3389 পোর্ট ওপেন করতে চান না (হ্যাকিংয়ের ভয়ে)। আপনি চান বাইরে থেকে কেউ 33899 পোর্টে হিট করলে সেটি লোকাল পিসির 3389 পোর্টে ফরওয়ার্ড হবে।

CLI কমান্ড:

[admin@MikroTik] > /ip firewall nat add chain=dstnat dst-address=203.0.113.10 protocol=tcp dst-port=33899 action=dst-nat to-addresses=192.168.10.100 to-ports=3389 comment="RDP Port Translation"

৫. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

সমস্যা (Problem) কারণ এবং সমাধান (Cause & Solution)
বাইরে থেকে এক্সেস করা যাচ্ছে না (পোর্ট বন্ধ দেখাচ্ছে) প্রথমে চেক করুন আপনার লোকাল সার্ভারে রাউটারের আইপিটি Default Gateway হিসেবে বসানো আছে কি না। গন্তব্যে যাওয়ার রাস্তা (Route) ঠিক না থাকলে সার্ভার রিপ্লাই দিতে পারে না।
Firewall Filter রুল ব্লক করছে MikroTik এর Forward Chain এ যদি সব ট্রাফিক ড্রপ করা থাকে, তবে পোর্ট ফরওয়ার্ডিং কাজ করবে না। আপনাকে Filter Rules এ গিয়ে connection-nat-state=dstnat রুলটি Accept করতে হবে।
লোকাল ইউজার পাবলিক আইপি দিয়ে সার্ভার এক্সেস করতে পারছে না এটি একটি বিখ্যাত সমস্যা, যাকে Hairpin NAT (NAT Loopback) বলা হয়। বাইরে থেকে পাবলিক আইপি দিয়ে সাইট লোড হলেও, ল্যানের ভেতর থেকে পাবলিক আইপি দিয়ে সাইট লোড হয় না। এটি সমাধানের জন্য একটি বিশেষ Source NAT (srcnat) রুল তৈরি করতে হয় যা ল্যানের ট্রাফিককে মাস্ক্যুরেড (Masquerade) করে আবার ল্যানেই পাঠায়।
ISP পোর্ট ব্লক করে রেখেছে অনেক সময় আইএসপি (ISP) ভাইরাস ছড়ানো রোধ করতে পোর্ট 80, 443 বা 25 ব্লক করে রাখে। আপনার রাউটারে প্যাকেট হিট করছে কি না তা দেখার জন্য NAT রুলের Packet Counter চেক করুন। প্যাকেট না আসলে আইএসপির সাথে যোগাযোগ করুন।

উপসংহার (Conclusion)

DST-NAT বা গন্তব্য পরিবর্তন করার এই ফিচারটি নেটওয়ার্ক সিকিউরিটি এবং সার্ভার ম্যানেজমেন্টের জন্য একটি অত্যন্ত শক্তিশালী টুল। এর মাধ্যমে আপনি একটিমাত্র Public IP ব্যবহার করে লোকাল নেটওয়ার্কের ডজন ডজন সার্ভার (যেমন: Web Server, CCTV, Mail Server) ইন্টারনেটে নিরাপদে উন্মুক্ত করতে পারেন। তবে Port Forwarding করার সময় সিকিউরিটির বিষয়টি মাথায় রেখে Port Translation ব্যবহার করা এবং অপ্রয়োজনীয় পোর্টগুলো সবসময় বন্ধ রাখা একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ারের বেস্ট প্র্যাকটিস।
