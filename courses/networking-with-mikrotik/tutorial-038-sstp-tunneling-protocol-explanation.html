SSTP Tunneling Protocol Explanation

01 May, 2026 VPN & Cyber Security Jobaer Hossain

ভিপিএন (VPN) কনফিগার করার সময় নেটওয়ার্ক ইঞ্জিনিয়ারদের সবচেয়ে বড় মাথাব্যথার কারণ হলো Firewall Block। অনেক আইএসপি (ISP) বা কর্পোরেট অফিস নিরাপত্তার কারণে PPTP (Port 1723) বা IPsec (Port 500) এর মতো পরিচিত ভিপিএন পোর্টগুলো ব্লক করে রাখে। এর ফলে রিমোট ইউজাররা কোনোভাবেই অফিসের রাউটারে কানেক্ট হতে পারে না।

এই বিশাল সমস্যার জাদুকরী সমাধান হলো SSTP (Secure Socket Tunneling Protocol)। মাইক্রোসফট (Microsoft) দ্বারা ডেভেলপ করা এই প্রোটোকলটি ভিপিএন ট্রাফিককে সাধারণ ওয়েব ট্রাফিকের ছদ্মবেশে পাঠায়, যা যেকোনো ফায়ারওয়াল বা ন্যাট (NAT) অনায়াসেই বাইপাস করতে পারে। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো SSTP কীভাবে কাজ করে, এর এনক্রিপশন পদ্ধতি কী এবং MikroTik RouterOS-এ এটি কীভাবে কনফিগার করতে হয়।

রিয়েল-লাইফ সিনারিও (The Ultimate Firewall Bypass)

সিনারিও: আপনি একটি থ্রি-স্টার হোটেলের ওয়াই-ফাই ব্যবহার করছেন। সেখানকার আইটি অ্যাডমিন শুধু ব্রাউজিং অ্যালাউ করেছে (Port 80 এবং 443)। বাকি সব পোর্ট কঠোরভাবে ব্লক করা। আপনার ইমার্জেন্সি অফিসের রাউটারে লগইন করতে হবে, কিন্তু PPTP বা L2TP কোনো ভিপিএন-ই কানেক্ট হচ্ছে না।

SSTP এর ম্যাজিক: SSTP প্রোটোকলটি TCP Port 443 (HTTPS) ব্যবহার করে কাজ করে। যেহেতু পোর্ট 443 দিয়ে আমরা ফেসবুক, গুগল বা ব্যাংকের সাইট ব্রাউজ করি, তাই পৃথিবীর কোনো ফায়ারওয়াল এই পোর্টটি ব্লক করে না। আপনার SSTP ভিপিএন ট্রাফিক যখন পোর্ট 443 দিয়ে ফায়ারওয়ালের কাছে যায়, তখন ফায়ারওয়াল ভাবে এটি একটি সাধারণ সিকিউরড ওয়েবসাইট ভিজিটের রিকোয়েস্ট, এবং সে এটিকে কোনো বাধা ছাড়াই পাস হতে দেয়।

১. SSTP কীভাবে কাজ করে? (Technical Overview)

SSTP এর আর্কিটেকচার অন্যান্য সাধারণ প্রোটোকলের চেয়ে একটু আলাদা এবং সিকিউর। এটি মূলত SSL/TLS (Secure Sockets Layer / Transport Layer Security) প্রোটোকল ব্যবহার করে।

ফিচার (Feature) বিস্তারিত বিবরণ (Description)
Transport Layer এটি TCP Protocol ব্যবহার করে, যেখানে অন্যান্য আধুনিক ভিপিএন (যেমন WireGuard) UDP ব্যবহার করে। TCP ব্যবহারের কারণে স্পিড কিছুটা কম হতে পারে, কিন্তু রিলায়েবিলিটি (Reliability) ১০০% থাকে।
Encryption & Security SSTP পুরোপুরি Certificate-based। অর্থাৎ, সার্ভার এবং ক্লায়েন্টের মধ্যে ট্রাস্ট (Trust) তৈরি করার জন্য ডিজিটাল সার্টিফিকেট (SSL Certificate) ব্যবহার করা হয়। এর এনক্রিপশন (AES-256) ব্যাংকিং গ্রেডের সমতুল্য।
OS Support যেহেতু এটি মাইক্রোসফটের তৈরি, তাই Windows 7 থেকে শুরু করে Windows 11 পর্যন্ত সব অপারেটিং সিস্টেমে কোনো থার্ড-পার্টি অ্যাপ ছাড়াই বিল্ট-ইন SSTP ক্লায়েন্ট থাকে।

২. MikroTik রাউটারে SSTP Server কনফিগারেশন

যেহেতু SSTP সার্টিফিকেট (Certificate) ছাড়া কাজ করে না, তাই MikroTik এ এটি কনফিগার করা PPTP বা L2TP এর চেয়ে একটু জটিল। আমাদের প্রথমে নিজেদের একটি Certificate Authority (CA) এবং Server Certificate তৈরি করতে হবে।

Step 1: Certificate তৈরি করা (Self-Signed)

প্রথমে আমরা রাউটারেই একটি সার্টিফিকেট তৈরি করে নেবো। Winbox এর System > Certificates এ গিয়ে এটি করা যায়, তবে টার্মিনাল দিয়ে করা সবচেয়ে সহজ।

(টার্মিনালে নিচের কমান্ডগুলো কপি-পেস্ট করুন):

# Create CA (Certificate Authority)
/certificate add name=ca-template common-name=myCA key-usage=key-cert-sign,crl-sign
/certificate sign ca-template name=CA

# Create Server Certificate
/certificate add name=server-template common-name=your_public_ip_or_domain
/certificate sign server-template ca=CA name=ServerCert

নোট: your_public_ip_or_domain এর জায়গায় আপনার রাউটারের রিয়েল আইপি বা ডোমেইন নাম দেবেন।

Step 2: IP Pool এবং PPP Profile তৈরি করা

ভিপিএন ইউজাররা কানেক্ট হওয়ার পর কোন আইপি পাবে তা নির্ধারণ করতে হবে।

# Create VPN IP Pool
/ip pool add name=sstp-pool ranges=172.16.10.10-172.16.10.50

# Create PPP Profile
/ppp profile add name=sstp-profile local-address=172.16.10.1 remote-address=sstp-pool use-encryption=yes

Step 3: SSTP Server চালু করা এবং Certificate অ্যাসাইন করা

  • Winbox এর PPP মেনুতে যান।
  • Interface ট্যাবে থাকা অবস্থায় উপরের SSTP Server বাটনে ক্লিক করুন।
  • Enabled চেকবক্সে টিক দিন।
  • Default Profile: sstp-profile সিলেক্ট করুন।
  • Certificate: ড্রপডাউন থেকে আমাদের তৈরি করা ServerCert সিলেক্ট করুন।
  • Force AES: টিক দিয়ে দিন (অধিক নিরাপত্তার জন্য)। Apply এবং OK দিন।

Step 4: VPN User (Secret) তৈরি করা

# Create User
/ppp secret add name=remote_user password=SecurePass!23 service=sstp profile=sstp-profile

৩. Firewall কনফিগারেশন

SSTP কাজ করার জন্য রাউটারে TCP Port 443 এক্সেপ্ট করা থাকতে হবে।

# Allow SSTP Port (TCP 443)
/ip firewall filter add action=accept chain=input protocol=tcp dst-port=443 comment="Allow SSTP VPN"

রুলটিকে ড্রপ (Drop) রুলের ওপরে নিয়ে যেতে ভুলবেন না।

৪. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

  • Windows PC কানেক্ট হচ্ছে না (Certificate Error): যেহেতু আমরা Self-Signed (নিজেদের তৈরি করা) সার্টিফিকেট ব্যবহার করছি, তাই উইন্ডোজ পিসি একে "Untrusted" ভেবে রিজেক্ট করতে পারে। এর সমাধান হলো রাউটার থেকে CA সার্টিফিকেটটি এক্সপোর্ট করে পিসির Trusted Root Certification Authorities ফোল্ডারে ইন্সটল করে নেওয়া।
  • Port 443 Conflict: MikroTik রাউটারে যদি WWW-SSL (Webfig HTTPS) পোর্ট হিসেবে 443 ব্যবহার করা থাকে, তবে SSTP এবং Webfig এর মধ্যে কনফ্লিক্ট হবে। IP > Services এ গিয়ে www-ssl এর পোর্টটি পরিবর্তন করে 4433 বা অন্য কিছু করে দিন।
  • Clock/Time Sync ইস্যু: সার্টিফিকেটের একটি নির্দিষ্ট ভ্যালিডিটি টাইম থাকে (Valid From / Valid To)। যদি আপনার রাউটারের সময় এবং তারিখ (Clock) ভুল থাকে, তবে সার্টিফিকেট ইনভ্যালিড দেখাবে এবং SSTP সার্ভার কাজ করবে না। রাউটারে অবশ্যই SNTP Client কনফিগার করে টাইম ঠিক রাখতে হবে।

উপসংহার (Conclusion)

যেখানে অন্যান্য ভিপিএন প্রোটোকলগুলো ফায়ারওয়ালের কাছে এসে আটকে যায়, সেখানে SSTP Tunneling Protocol ওয়েব ট্রাফিকের ছদ্মবেশে অনায়াসে নিজের কাজ করে যায়। Windows এনভায়রনমেন্টের জন্য এটি একটি আশীর্বাদ। যদিও সার্টিফিকেট ম্যানেজমেন্টের কারণে এর কনফিগারেশন একটু এডভান্সড লেভেলের, কিন্তু এর AES-256 এনক্রিপশন এবং Port 443 এর ব্যবহার একে রিমোট এক্সেস ভিপিএন (Remote Access VPN) এর ক্ষেত্রে একটি অত্যন্ত সিকিউর এবং রিলায়েবল অপশনে পরিণত করেছে।
