Connecting the MikroTik Router and the PC to the internet

15 June, 2026 Networking with MikroTik Jobaer Hossain

আপনি একটি নতুন MikroTik Router কিনেছেন। এখন আপনার প্রধান কাজ হলো এটিকে ইন্টারনেটের সাথে যুক্ত করা এবং এরপর আপনার ল্যাপটপ বা পিসিতে সেই ইন্টারনেট পৌঁছে দেওয়া। শুনতে খুব সহজ মনে হলেও, একটি প্রফেশনাল রাউটারে এই কাজটি করতে বেশ কয়েকটি ধাপ (যেমন WAN সেটআপ, LAN আইপি বসানো, NAT এবং DNS কনফিগারেশন) পার হতে হয়।

আজকের এই ইন-ডেপথ টিউটোরিয়ালে আমরা একটি রিয়েল-লাইফ সিনারিও তৈরি করে দেখবো কীভাবে একদম শূন্য থেকে (From Scratch) একটি MikroTik রাউটারকে ইন্টারনেটে কানেক্ট করতে হয় এবং পিসিতে ব্রাউজিং চালু করতে হয়।

ফিজিক্যাল কানেকশন (Physical Setup)

কনফিগারেশন শুরু করার আগে ক্যাবলগুলো ঠিক জায়গায় লাগানো অত্যন্ত জরুরি।

  • WAN Port (ইন্টারনেট): আপনার ISP (ইন্টারনেট সার্ভিস প্রোভাইডার) থেকে যে ক্যাবলটি এসেছে, সেটি রাউটারের Ether1 পোর্টে লাগান।
  • LAN Port (লোকাল নেটওয়ার্ক): রাউটারের Ether2 পোর্ট থেকে একটি ক্যাবল নিয়ে আপনার ল্যাপটপ বা পিসির ল্যান পোর্টে লাগান।

এবার Winbox ওপেন করে MAC Address এর মাধ্যমে রাউটারে লগইন করুন।

১. রাউটারকে ইন্টারনেটে কানেক্ট করা (WAN Setup)

আপনার আইএসপি আপনাকে তিন ধরনের ইন্টারনেট কানেকশন দিতে পারে। এর যেকোনো একটি কনফিগার করলেই রাউটার ইন্টারনেট পেয়ে যাবে:

কানেকশন টাইপ কখন ব্যবহৃত হয়? কনফিগারেশন (CLI Command)
DHCP Client (অটোমেটিক আইপি) আপনার আইএসপি যদি বলে ক্যাবল লাগালেই ইন্টারনেট পেয়ে যাবেন (যেমন ব্রডব্যান্ড মডেম থেকে লাইন নিলে)। /ip dhcp-client add interface=ether1 disabled=no
PPPoE (ইউজার/পাসওয়ার্ড) আইএসপি যদি আপনাকে একটি Username এবং Password দিয়ে থাকে। এটি সবচেয়ে বেশি ব্যবহৃত হয়। /interface pppoe-client add name=pppoe-out1 interface=ether1 user="your_user" password="your_password" disabled=no add-default-route=yes
Static IP (ফিক্সড আইপি) আইএসপি যদি আপনাকে নির্দিষ্ট IP, Subnet Mask এবং Gateway লিখে দেয়। /ip address add address=192.168.x.x/24 interface=ether1
/ip route add dst-address=0.0.0.0/0 gateway=192.168.x.1

২. লোকাল নেটওয়ার্ক তৈরি করা (LAN Setup)

রাউটার তো ইন্টারনেট পেল, এবার পিসিকে ইন্টারনেট দেওয়ার জন্য একটি লোকাল নেটওয়ার্ক তৈরি করতে হবে। আমরা Ether2 পোর্টে একটি প্রাইভেট আইপি ব্লক (যেমন 192.168.10.0/24) কনফিগার করবো।

Ether2 ইন্টারফেসে আইপি বসানোর কমান্ড:

[admin@MikroTik] > /ip address add address=192.168.10.1/24 interface=ether2

৩. DNS (Domain Name System) কনফিগারেশন

আমরা যখন ব্রাউজারে google.com লিখি, তখন রাউটারকে জানতে হয় গুগলের আইপি কত। এই কাজটি করে DNS। আমরা গুগলের পাবলিক ডিএনএস ব্যবহার করবো।

[admin@MikroTik] > /ip dns set servers=8.8.8.8,8.8.4.4 allow-remote-requests=yes

নোট: allow-remote-requests=yes দিলে আপনার রাউটারটি পিসির জন্য একটি লোকাল ডিএনএস সার্ভার হিসেবে কাজ করবে।

৪. পিসির জন্য DHCP Server তৈরি করা

আপনি চাইলে আপনার পিসিতে ম্যানুয়ালি আইপি (যেমন 192.168.10.2) বসাতে পারেন। কিন্তু পিসি যাতে স্বয়ংক্রিয়ভাবে আইপি পায়, তার জন্য রাউটারে একটি DHCP Server চালু করাটা সবচেয়ে ভালো। MikroTik এ এর জন্য একটি দারুণ উইজার্ড (Wizard) আছে।

Winbox এর New Terminal এ নিচের কমান্ডটি দিন এবং এন্টার চাপতে থাকুন (সবকিছু অটোমেটিক নিয়ে নেবে):

[admin@MikroTik] > /ip dhcp-server setup
Select interface to run DHCP server on
dhcp server interface: ether2
Select network for DHCP addresses
dhcp address space: 192.168.10.0/24
Select gateway for given network
gateway for dhcp network: 192.168.10.1
Select pool of ip addresses given out by DHCP server
addresses to give out: 192.168.10.2-192.168.10.254
Select DNS servers
dns servers: 192.168.10.1
Select lease time
lease time: 10m

৫. ইন্টারনেট শেয়ারিং (NAT / Masquerade)

এখন আপনার পিসি আইপি পেয়েছে, রাউটারেও ইন্টারনেট আছে। কিন্তু পিসি থেকে ব্রাউজ করলে ইন্টারনেট চলবে না! কারণ পিসির আইপিটি (192.168.10.2) একটি Private IP, যা ইন্টারনেটে রাউট (Route) হতে পারে না।

এই প্রাইভেট আইপিকে রাউটারের পাবলিক বা আইএসপির আইপিতে কনভার্ট করে ইন্টারনেটে পাঠানোর প্রক্রিয়াই হলো NAT (Network Address Translation)। MikroTik এ একে Masquerade বলা হয়। এটি ছাড়া লোকাল পিসি ইন্টারনেট পাবে না।

NAT রুল তৈরি করার কমান্ড (যদি আপনার WAN পোর্ট ether1 হয়):

[admin@MikroTik] > /ip firewall nat add chain=srcnat out-interface=ether1 action=masquerade

নোট: যদি আপনি PPPoE ব্যবহার করেন, তবে out-interface=pppoe-out1 দিতে হবে।

৬. Troubleshooting Tips (ট্রাবলশুটিং)

সব কনফিগারেশন করার পরও যদি ইন্টারনেট না চলে, তবে নিচের স্টেপগুলো ফলো করুন:

  • রাউটারে ইন্টারনেট আছে কি না চেক করুন: নিউ টার্মিনাল থেকে ping 8.8.8.8 দিন। রিপ্লাই আসলে বুঝবেন রাউটারে ইন্টারনেট আছে।
  • ডিএনএস চেক করুন: রাউটার থেকে ping google.com দিন। যদি invalid value বা unknown host দেখায়, তবে আপনার DNS কনফিগারেশন ভুল হয়েছে।
  • পিসিতে ইন্টারনেট না পেলে: পিসির কমান্ড প্রম্পট (CMD) ওপেন করে ping 192.168.10.1 দিন (রাউটারকে পিং করুন)। যদি পিং পায়, কিন্তু ping 8.8.8.8 না পায়, তবে নিশ্চিত থাকুন আপনার NAT (Masquerade) রুলে কোনো ভুল হয়েছে। রুলটি চেক করুন।
  • IP Conflict: পিসিতে যদি আগে থেকে কোনো স্ট্যাটিক আইপি বসানো থাকে, সেটি মুছে দিয়ে Obtain an IP address automatically সিলেক্ট করে দিন।

উপসংহার (Conclusion)

একটি MikroTik Router-কে ইন্টারনেটে কানেক্ট করা এবং লোকাল নেটওয়ার্কে সেই ইন্টারনেট ডিস্ট্রিবিউট করা হলো যেকোনো নেটওয়ার্ক ইঞ্জিনিয়ারের প্রথম এবং সবচেয়ে গুরুত্বপূর্ণ কাজ। WAN Configuration দিয়ে আইএসপির সাথে যুক্ত হওয়া, DHCP Server দিয়ে পিসিগুলোকে আইপি দেওয়া এবং সবশেষে NAT (Masquerade) এর মাধ্যমে ইন্টারনেট শেয়ার করা—এই ফ্লো (Flow) বা ধারাবাহিকতাটি মনে রাখলে আপনি পৃথিবীর যেকোনো ব্র্যান্ডের রাউটার খুব সহজেই কনফিগার করতে পারবেন।
