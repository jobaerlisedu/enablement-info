Wireless Access List: Bandwidth, Password and Time

05 November, 2026 Networking with MikroTik Jobaer Hossain

MikroTik রাউটারের Wireless কনফিগারেশনের ক্ষেত্রে Access List হলো এক জাদুকরী টুল। সাধারণ রাউটারগুলোতে একটি SSID এর জন্য একটিমাত্র পাসওয়ার্ড থাকে এবং সেই ওয়াই-ফাইয়ে যারা কানেক্ট হয়, তারা সবাই সমান সুবিধা পায়। কিন্তু এন্টারপ্রাইজ লেভেলে এটি কোনোভাবেই কার্যকরী নয়।

আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো কীভাবে একটিমাত্র Wireless Network (SSID) ব্যবহার করে Access List এর মাধ্যমে বিভিন্ন ইউজারের (MAC Address এর ওপর ভিত্তি করে) Bandwidth Limit, আলাদা Password এবং ইন্টারনেট ব্যবহারের নির্দিষ্ট Time সেট করা যায়।

রিয়েল-লাইফ সিনারিও (The Setup)

ধরুন, আপনার অফিসে Office_WiFi নামের একটি Access Point আছে। আপনি চান:

  • Manager: তিনি আনলিমিটেড স্পিড পাবেন এবং তার পাসওয়ার্ড হবে AdminPass123।
  • Staff: তারা সর্বোচ্চ 2 Mbps স্পিড পাবে, তাদের পাসওয়ার্ড হবে StaffPass456 এবং তারা শুধুমাত্র সকাল ৯টা থেকে বিকেল ৫টা (Office Hours) পর্যন্ত ওয়াই-ফাইয়ে কানেক্ট হতে পারবে।

এই সম্পূর্ণ কাজটি আমরা কোনো Hotspot বা Queue তৈরি না করেই সরাসরি Wireless Access List দিয়ে করতে পারি।

১. Private Password কনফিগারেশন (ভিন্ন ইউজারের জন্য ভিন্ন পাসওয়ার্ড)

সাধারণত আমরা Security Profiles এ গিয়ে একটি কমন পাসওয়ার্ড সেট করি। কিন্তু Access List ব্যবহার করে আমরা নির্দিষ্ট MAC Address এর জন্য আলাদা Private Pre-shared Key সেট করতে পারি।

Winbox এর মাধ্যমে:

  • প্রথমে Winbox থেকে Wireless মেনুতে যান।
  • Access List ট্যাবে ক্লিক করুন এবং নতুন একটি রুল তৈরি করতে + (Add) আইকনে ক্লিক করুন।
  • MAC Address: এর ঘরে আপনার কাঙ্ক্ষিত ইউজারের MAC Address দিন (যেমন: 00:11:22:33:44:55)।
  • Interface: আপনার ওয়্যারলেস ইন্টারফেস (যেমন: wlan1) সিলেক্ট করুন।
  • একটু নিচের দিকে Private Pre-shared Key নামের একটি ঘর পাবেন। এখানে ওই নির্দিষ্ট ইউজারের জন্য আলাদা পাসওয়ার্ড দিন (যেমন: AdminPass123)।
  • Authentication: বক্সে টিক (Check) দিন, যাতে সে কানেক্ট হতে পারে।
  • Apply এবং OK দিন।

২. Bandwidth Limit সেট করা (AP Tx Limit / AP Rx Limit)

MikroTik এ স্পিড লিমিট করার জন্য আমরা সাধারণত Simple Queue ব্যবহার করি। কিন্তু আপনি চাইলে সরাসরি ওয়্যারলেস ইন্টারফেস থেকেই MAC Address অনুযায়ী স্পিড নিয়ন্ত্রণ করতে পারেন।

  • Access List এ গিয়ে আপনার তৈরি করা রুলটিতে ডাবল-ক্লিক করুন।
  • নিচের দিকে AP Tx Limit এবং AP Rx Limit নামের দুটি ফিল্ড দেখতে পাবেন।
  • AP Tx Limit (Download Speed): রাউটার থেকে ক্লায়েন্টে ডাটা যাওয়ার স্পিড। এখানে 2M (2 Mbps) লিখে দিন।
  • AP Rx Limit (Upload Speed): ক্লায়েন্ট থেকে রাউটারে ডাটা আসার স্পিড। এখানেও 2M লিখে দিন।
  • Apply এবং OK দিন।

নোট: ভ্যালুগুলো অবশ্যই bits per second হিসেবে লিখতে হয়। 1 Mbps বোঝাতে 1M বা 1000k ব্যবহার করবেন।

৩. Time Restriction কনফিগারেশন (নির্দিষ্ট সময়ে অ্যাক্সেস)

আপনি যদি চান কোনো ইউজার শুধু নির্দিষ্ট একটি সময়ে (যেমন অফিস চলাকালীন) ওয়াই-ফাই ব্যবহার করতে পারবে, তবে Access List এর Time ফিচারটি দারুণ কাজ করে।

  • একইভাবে Access List এর রুলটি ওপেন করুন।
  • নিচের দিকে Time নামের একটি অপশন পাবেন।
  • প্রথম বক্সে শুরুর সময় দিন: 09:00:00 (সকাল ৯টা)।
  • দ্বিতীয় বক্সে শেষের সময় দিন: 17:00:00 (বিকেল ৫টা)।
  • ডানপাশে সপ্তাহের দিনগুলো (Sun, Mon, Tue...) সিলেক্ট করার অপশন আছে। চাইলে ছুটির দিন আনচেক (Uncheck) করে রাখতে পারেন।
  • Apply এবং OK দিন।

এখন এই ইউজারটি বিকেল ৫টার পর আর কোনোভাবেই রাউটারের সাথে কানেক্ট হতে পারবে না।

৪. Command Line (CLI) ব্যবহার করে কনফিগারেশন

টার্মিনাল ব্যবহার করে উপরের ৩টি কাজ (Password, Bandwidth, Time) একটিমাত্র কমান্ড দিয়ে করা সম্ভব।

স্টাফের জন্য রুল (2Mbps Speed, Time Limit, Private Password):

[admin@MikroTik] > /interface wireless access-list add mac-address=11:22:33:44:55:66 interface=wlan1 authentication=yes private-pre-shared-key="StaffPass456" ap-tx-limit=2M ap-rx-limit=2M time=09:00:00-17:00:00,sun,mon,tue,wed,thu

ম্যানেজারের জন্য রুল (Unlimited Speed, No Time Limit, Private Password):

[admin@MikroTik] > /interface wireless access-list add mac-address=00:11:22:33:44:55 interface=wlan1 authentication=yes private-pre-shared-key="AdminPass123"

৫. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

সমস্যা (Problem) কারণ এবং সমাধান (Cause & Solution)
Time Restriction কাজ করছে না রাউটারের System > Clock চেক করুন। রাউটারের টাইম জোন বা সময় ভুল থাকলে এই রুল কাজ করবে না। রাউটারে অবশ্যই SNTP Client কনফিগার করে সময় সিঙ্ক (Sync) করে নিতে হবে।
Private Password কাজ করছে না Access List এ পাসওয়ার্ড দেওয়ার পর ক্লায়েন্টের মোবাইল বা পিসি থেকে আগের পাসওয়ার্ডটি অবশ্যই Forget/Remove করে নতুন করে কানেক্ট করতে হবে। এছাড়া, Security Profile এ WPA2 PSK এনাবল থাকতে হবে।
AP Tx/Rx Limit কাজ করছে না লিমিট বসানোর সময় ফরমেট ভুল হতে পারে (যেমন 2m এর বদলে শুধু 2 লিখলে তা 2 bps হিসেবে কাউন্ট হবে)। অবশ্যই M বা k সঠিকভাবে ব্যবহার করতে হবে। এছাড়া যদি Simple Queue তে ওই আইপির জন্য আলাদা লিমিট থাকে, তবে সেটি ওভাররাইড (Override) হতে পারে।
লিস্টের বাইরের সবাই কানেক্ট হয়ে যাচ্ছে Access List এর মূল সুবিধা পেতে হলে Wireless ইন্টারফেসের Default Authenticate অপশনটি বন্ধ (Uncheck) করে রাখা উচিত। তাহলে লিস্টের বাইরের কেউ পাসওয়ার্ড জানলেও কানেক্ট হতে পারবে না।

উপসংহার (Conclusion)

Wireless Access List ব্যবহার করে আপনি একটি সাধারণ Access Point কেই শক্তিশালী Bandwidth Manager এবং Access Controller এ পরিণত করতে পারেন। আলাদা আলাদা SSID বা Hotspot Server কনফিগার করার ঝামেলায় না গিয়ে, শুধুমাত্র MAC Address এর ওপর ভিত্তি করে Password, Time এবং Bandwidth নিয়ন্ত্রণ করার এই পদ্ধতিটি কর্পোরেট অফিস বা ছোট হোস্টেল নেটওয়ার্কের জন্য একটি স্মার্ট এবং অত্যন্ত কার্যকরী সমাধান।
