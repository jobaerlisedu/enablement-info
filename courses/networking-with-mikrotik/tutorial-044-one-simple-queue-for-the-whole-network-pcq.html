One Simple Queue for the whole network (PCQ)

05 June, 2026 Networking with MikroTik Jobaer Hossain

যেকোনো নেটওয়ার্ক অ্যাডমিনিস্ট্রেটরের জন্য সবচেয়ে বিরক্তিকর কাজগুলোর একটি হলো ஒவ்வொரு ইউজারের জন্য আলাদা আলাদা Bandwidth Control রুল তৈরি করা। আপনার নেটওয়ার্কে যদি ২০০ জন ইউজার থাকে, তবে ২০০টি Simple Queue তৈরি করা শুধু সময়সাপেক্ষই নয়, এটি রাউটারের প্রসেসরের (CPU) ওপরও প্রচণ্ড চাপ ফেলে।

MikroTik RouterOS এই সমস্যার একটি চমৎকার এবং ডায়নামিক সমাধান নিয়ে এসেছে, যার নাম PCQ (Per Connection Queue)। PCQ ব্যবহার করে আপনি মাত্র একটি Simple Queue তৈরি করেই পুরো নেটওয়ার্কের শত শত ইউজারের মধ্যে সমানভাবে ব্যান্ডউইথ ভাগ করে দিতে পারবেন। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো PCQ এর আর্কিটেকচার এবং কীভাবে এটি কনফিগার করতে হয়।

রিয়েল-লাইফ সিনারিও (The Internet Download Manager Problem)

সমস্যা: আপনার অফিসের মোট ইন্টারনেট ব্যান্ডউইথ হলো 50 Mbps। আপনি একটি Simple Queue তৈরি করে 192.168.10.0/24 সাবনেটে 50 Mbps লিমিট করে দিলেন। এখন অফিসে থাকা ৫০ জন ইউজারের মধ্যে একজন ইউজার যদি IDM (Internet Download Manager) দিয়ে কোনো মুভি ডাউনলোড শুরু করে, তবে IDM তার মাল্টিপল কানেকশন ব্যবহার করে একাই 48 Mbps স্পিড টেনে নেবে। বাকি ৪৯ জন ইউজার মিলে পাবে মাত্র 2 Mbps। পুরো অফিস ব্রাউজিং করতে গিয়ে চরম ভোগান্তিতে পড়বে।

PCQ এর সমাধান: আমরা যদি ওই Queue টিতে PCQ মেকানিজম যুক্ত করে দিই, তবে রাউটার দেখবে কয়জন ইউজার (IP) বর্তমানে ডাটা টানছে। যদি ২ জন ইউজার ডাটা টানে, রাউটার 50 Mbps কে ২ ভাগ করে দুজনকে 25 Mbps করে দেবে। যদি ৫০ জন ইউজার একসাথে ডাটা টানে, তবে সবাইকে সমানভাবে 1 Mbps করে স্পিড ডিস্ট্রিবিউট করে দেবে। কেউ চাইলেও অন্য ইউজারের ভাগের স্পিড দখল করতে পারবে না।

১. PCQ (Per Connection Queue) কীভাবে কাজ করে?

PCQ মূলত দুটি প্যারামিটারের ওপর ভিত্তি করে ডাটা ট্রাফিককে ডায়নামিকভাবে ভাগ করে:

প্যারামিটার (Parameter) কাজের বিবরণ (Description)
Rate (স্পিড লিমিট) এখানে দুটি অপশন থাকে। আপনি যদি Rate = 0 দেন, তবে মোট ব্যান্ডউইথ উপস্থিত ইউজারদের মধ্যে সমানভাবে ভাগ হবে (Dynamic Division)। আর যদি Rate = 5M দেন, তবে প্রতি ইউজার সর্বোচ্চ 5 Mbps স্পিড পাবে, কিন্তু সব ইউজার মিলে মূল Simple Queue এর Max Limit (যেমন 50 Mbps) অতিক্রম করতে পারবে না।
Classifier (শনাক্তকরণ) রাউটার কীভাবে বুঝবে কে আলাদা ইউজার? এর জন্য Classifier ব্যবহার করা হয়। Upload ভাগ করার জন্য Src. Address (ইউজারের নিজস্ব আইপি) এবং Download ভাগ করার জন্য Dst. Address (ডাটা যার কাছে যাচ্ছে) সিলেক্ট করতে হয়।

২. Step-by-Step Configuration (Winbox এর মাধ্যমে)

চলুন আমাদের 50 Mbps এর নেটওয়ার্কের জন্য PCQ সেটআপ করি, যেখানে প্রতিটি ইউজার সর্বোচ্চ 5 Mbps করে পাবে।

Step 1: PCQ Queue Type তৈরি করা

আমাদের প্রথমে আপলোড এবং ডাউনলোডের জন্য দুটি আলাদা ডাইনামিক টেমপ্লেট (Queue Type) বানাতে হবে।

  • Winbox এর Queues মেনুতে গিয়ে Queue Types ট্যাবে ক্লিক করুন।
  • + (Add) আইকনে ক্লিক করে ডাউনলোডের টেমপ্লেট তৈরি করুন:
    • Type Name: PCQ-Download-5M
    • Kind: pcq
    • Rate: 5M (প্রতি ইউজার ম্যাক্সিমাম 5 Mbps পাবে)।
    • Classifier: Dst. Address বক্সে টিক (Check) দিন। অন্যগুলো আনচেক থাকবে। OK করুন।
  • আবার + (Add) আইকনে ক্লিক করে আপলোডের টেমপ্লেট তৈরি করুন:
    • Type Name: PCQ-Upload-5M
    • Kind: pcq
    • Rate: 5M
    • Classifier: Src. Address বক্সে টিক (Check) দিন। OK করুন।

Step 2: একটিমাত্র Simple Queue তৈরি করা

এবার আমরা পুরো নেটওয়ার্কের জন্য একটি মাস্টার Queue তৈরি করবো এবং তাতে আমাদের পিসি কিউ (PCQ) টেমপ্লেটগুলো যুক্ত করে দেবো।

  • একই উইন্ডো থেকে Simple Queues ট্যাবে যান এবং + (Add) ক্লিক করুন।
  • General Tab:
    • Name: Total-LAN-Queue
    • Target: আপনার অফিসের ল্যান ব্লকটি দিন, যেমন 192.168.10.0/24।
    • Max Limit (Target Upload): 50M (আইএসপি থেকে পাওয়া মোট আপলোড)।
    • Max Limit (Target Download): 50M (আইএসপি থেকে পাওয়া মোট ডাউনলোড)।
  • Advanced Tab (The Magic Happens Here):
    • Queue Type (Target Upload): ড্রপডাউন থেকে একটু আগে তৈরি করা PCQ-Upload-5M সিলেক্ট করুন।
    • Queue Type (Target Download): ড্রপডাউন থেকে PCQ-Download-5M সিলেক্ট করুন।
  • Apply এবং OK দিন।

৩. Command Line (CLI) ব্যবহার করে কনফিগারেশন

যারা টার্মিনাল ভালোবাসেন, তাদের জন্য পুরো প্রসেসটি মাত্র ৩ লাইনের কমান্ড:

# Create PCQ Type for Download
/queue type add name="PCQ-Download-5M" kind=pcq pcq-rate=5M pcq-classifier=dst-address

# Create PCQ Type for Upload
/queue type add name="PCQ-Upload-5M" kind=pcq pcq-rate=5M pcq-classifier=src-address

# Create the Single Simple Queue for the whole subnet
/queue simple add name="Total-LAN-Queue" target=192.168.10.0/24 max-limit=50M/50M queue=PCQ-Upload-5M/PCQ-Download-5M

৪. Troubleshooting Tips (সাধারণ সমস্যা ও সতর্কতা)

  • FastTrack কনফ্লিক্ট: MikroTik রাউটারে যদি FastTrack (IP > Firewall > Filter Rules) এনাবল থাকে, তবে কোনো Queue-ই কাজ করবে না। প্যাকেটগুলো স্পিড লিমিট চেকিং বাইপাস করে চলে যাবে। PCQ ব্যবহার করতে হলে অবশ্যই ফায়ারওয়াল থেকে FastTrack রুলটি ডিজেবল বা রিমুভ করে নিতে হবে।
  • PCQ Rate 0 (Zero) এর ব্যবহার: আপনি যদি চান কোনো নির্দিষ্ট লিমিট (যেমন 5M) না দিয়ে আইএসপির দেওয়া 50M স্পিডটিই উপস্থিত ইউজারদের মাঝে সমানভাবে ভাগ হোক, তবে PCQ Type তৈরি করার সময় Rate = 0 দিয়ে রাখবেন। এটি ডায়নামিকভাবে ক্যালকুলেট হবে।
  • Memory / RAM Issue: অনেক বড় নেটওয়ার্কে (যেমন ৫০০০+ ইউজার) PCQ ব্যবহার করলে রাউটারের র‍্যামের (RAM) ওপর কিছুটা চাপ পড়ে, কারণ রাউটারকে প্রতিটি কানেকশনের হিসাব আলাদাভাবে মেমোরিতে রাখতে হয়। এর জন্য pcq-limit এবং pcq-total-limit প্যারামিটারগুলো অ্যাডজাস্ট করার প্রয়োজন হতে পারে। তবে সাধারণ অফিস বা ছোট আইএসপি নেটওয়ার্কের জন্য ডিফল্ট সেটিং-ই যথেষ্ট।

উপসংহার (Conclusion)

PCQ (Per Connection Queue) হলো MikroTik RouterOS এর এমন একটি মাস্টারপিস ফিচার, যা নেটওয়ার্ক অ্যাডমিনিস্ট্রেশনের সময় এবং শ্রম দুটিই বাঁচিয়ে দেয়। শ'য়ে শ'য়ে Simple Queue তৈরি করে রাউটারের কনফিগারেশন ভারী করার কোনো মানেই হয় না, যখন একটিমাত্র রুল দিয়ে পুরো Subnet এর ট্রাফিক সমানভাবে এবং সুশৃঙ্খলভাবে বণ্টন করা যায়। স্পিড হগিং (Speed Hogging) রোধ করতে এবং প্রতিটি ইউজারকে একটি স্মুথ ব্রাউজিং এক্সপেরিয়েন্স দিতে PCQ এর বিকল্প নেই।
