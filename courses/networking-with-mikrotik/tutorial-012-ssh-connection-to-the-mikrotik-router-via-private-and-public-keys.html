SSH connection to the MikroTik Router via Private and Public Keys

25 July, 2026 Networking with MikroTik Jobaer Hossain

নেটওয়ার্ক ডিভাইসে রিমোটলি লগইন করার জন্য SSH (Secure Shell) হলো সবচেয়ে নিরাপদ এবং জনপ্রিয় প্রোটোকল। সাধারণত আমরা Username এবং Password দিয়ে SSH এ লগইন করে থাকি। কিন্তু হ্যাকাররা প্রতিনিয়ত Brute-force Attack এর মাধ্যমে পাসওয়ার্ড ব্রেক করার চেষ্টা করে। এই সমস্যার চূড়ান্ত এবং প্রফেশনাল সমাধান হলো পাসওয়ার্ডের বদলে SSH Key-Based Authentication ব্যবহার করা।

আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো Public এবং Private Key কীভাবে কাজ করে এবং আপনার পিসি থেকে পাসওয়ার্ড ছাড়াই অত্যন্ত সুরক্ষিত উপায়ে একটি MikroTik Router-এ কীভাবে SSH লগইন করতে হয়।

১. Public এবং Private Key কীভাবে কাজ করে?

SSH Key-Based Authentication মূলত Asymmetric Cryptography এর ওপর ভিত্তি করে কাজ করে। এখানে একটি চাবির বদলে দুটি চাবি বা Key ব্যবহার করা হয়।

  • Public Key: এটিকে আপনি একটি তালার (Padlock) সাথে তুলনা করতে পারেন। এই চাবিটি আপনি আপনার রাউটার বা যেকোনো সার্ভারে আপলোড করে রাখতে পারেন। এটি যে কারো কাছে থাকলেও কোনো সমস্যা নেই।
  • Private Key: এটি হলো সেই তালার আসল চাবি (Key)। এটি শুধুমাত্র আপনার পিসিতে বা ল্যাপটপে অত্যন্ত গোপনীয়ভাবে সংরক্ষিত থাকবে। এটি ভুলেও কাউকে দেওয়া যাবে না।

কীভাবে কানেক্ট হয়? আপনি যখন রাউটারে লগইন করার চেষ্টা করেন, রাউটার আপনার পিসিকে একটি ম্যাথমেটিক্যাল পাজল (Puzzle) দেয়, যা রাউটারে থাকা Public Key দিয়ে লক করা থাকে। আপনার পিসি তার কাছে থাকা Private Key দিয়ে সেই পাজলটি সলভ করে রাউটারকে ফেরত পাঠায়। রাউটার তখন বুঝতে পারে যে আপনিই সঠিক ইউজার এবং আপনাকে পাসওয়ার্ড ছাড়াই লগইন করার অনুমতি দেয়।

২. Step-by-Step: পিসিতে SSH Key তৈরি করা

Windows 10/11, Linux, বা Mac—যেকোনো অপারেটিং সিস্টেমে ডিফল্টভাবেই SSH ক্লায়েন্ট ইন্সটল করা থাকে। প্রথমে আমাদের একটি Key Pair (Public এবং Private) তৈরি করতে হবে।

আপনার পিসির Command Prompt (CMD) বা Terminal ওপেন করে নিচের কমান্ডটি দিন: 

C:\> ssh-keygen -t rsa -b 4096

এই কমান্ড দেওয়ার পর আপনার কাছে কিছু ইনফরমেশন জানতে চাইবে:

  • File in which to save the key: কোথায় ফাইলটি সেভ হবে তা জানতে চাইবে। ডিফল্ট লোকেশনে সেভ করার জন্য শুধু Enter চাপুন।
  • Passphrase: Private Key-কে আরও সুরক্ষিত করতে আপনি একটি পাসওয়ার্ড (Passphrase) দিতে পারেন। আপনি চাইলে এটি খালিও রাখতে পারেন (শুধু Enter চাপুন)।

কাজটি সফল হলে আপনার পিসির C:\Users\YourUser\.ssh\ ফোল্ডারে দুটি ফাইল তৈরি হবে:

  • id_rsa (এটি আপনার Private Key, যা পিসিতেই থাকবে)।
  • id_rsa.pub (এটি আপনার Public Key, যা রাউটারে পাঠাতে হবে)।

৩. MikroTik রাউটারে Public Key আপলোড করা

এখন আমাদের তৈরি করা id_rsa.pub ফাইলটিকে MikroTik রাউটারে আপলোড করতে হবে।

  • Winbox পদ্ধতি: Winbox ওপেন করে রাউটারে লগইন করুন। বাম পাশের মেনু থেকে Files এ ক্লিক করুন। আপনার পিসি থেকে id_rsa.pub ফাইলটি মাউস দিয়ে ড্র্যাগ (Drag) করে Winbox-এর Files উইন্ডোর ভেতরে ড্রপ (Drop) করে দিন।
  • SCP পদ্ধতি (Terminal থেকে): আপনি চাইলে কমান্ড লাইন দিয়েও ফাইলটি পাঠাতে পারেন: 
    C:\> scp C:\Users\YourUser\.ssh\id_rsa.pub admin@192.168.88.1:/

৪. RouterOS এ Public Key ইম্পোর্ট করা

ফাইলটি আপলোড করার পর রাউটারকে বলে দিতে হবে যে এই Public Key-টি কোন ইউজারের জন্য কাজ করবে। Winbox এর New Terminal ওপেন করে নিচের কমান্ডটি দিন (ধরি, ইউজারনেম admin):

[admin@MikroTik] > /user ssh-keys import public-key-file=id_rsa.pub user=admin

ইম্পোর্ট সফল হয়েছে কি না, তা চেক করতে নিচের কমান্ডটি দিন:

[admin@MikroTik] > /user ssh-keys print

আউটপুটে আপনি আপনার পিসির নাম এবং ইউজারের নাম দেখতে পাবেন। এর মানে রাউটার এখন আপনার পিসিকে চেনে।

৫. পাসওয়ার্ড ছাড়া SSH লগইন করা

এখন ম্যাজিক দেখার পালা! আপনার পিসির Command Prompt (CMD) বা Terminal ওপেন করে রাউটারে SSH করার কমান্ড দিন: 

C:\> ssh admin@192.168.88.1

লক্ষ্য করুন, রাউটার এবার আপনার কাছে কোনো পাসওয়ার্ড (Password) চাইবে না। Private Key ভেরিফাই করে সরাসরি আপনাকে RouterOS এর CLI-তে লগইন করিয়ে দেবে। এটি অটোমেশন (Automation) এবং স্ক্রিপ্টিংয়ের জন্য অত্যন্ত কার্যকরী।

৬. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

SSH Key দিয়ে লগইন করতে গিয়ে কিছু সাধারণ এরর আসতে পারে। নিচে সেগুলোর সমাধান দেওয়া হলো:

সমস্যা বা Error Message কারণ এবং সমাধান (Cause & Solution)
"Permission denied (publickey,password)" রাউটার আপনার Private Key চিনতে পারছে না। কমান্ড প্রম্পটে ssh -i C:\Users\YourUser\.ssh\id_rsa admin@192.168.88.1 কমান্ড দিয়ে নির্দিষ্ট করে Private Key-এর লোকেশন চিনিয়ে দিন।
"Server refused our key" (PuTTY এর ক্ষেত্রে) PuTTY সরাসরি OpenSSH এর id_rsa ফরম্যাট বুঝতে পারে না। PuTTY ব্যবহার করলে আপনাকে PuTTYgen সফটওয়্যার দিয়ে .ppk ফরম্যাটে Private Key কনভার্ট করে নিতে হবে।
Unprotected Private Key File Linux বা Mac সিস্টেমে Private Key ফাইলের পারমিশন বেশি উন্মুক্ত থাকলে (যেমন 777) সিকিউরিটির কারণে SSH কানেক্ট করতে দেয় না। টার্মিনালে chmod 600 ~/.ssh/id_rsa কমান্ড দিয়ে ফাইলের পারমিশন ঠিক করে নিন।

উপসংহার (Conclusion)

SSH Key-Based Authentication কনফিগার করাটা আধুনিক নেটওয়ার্ক ইঞ্জিনিয়ারিংয়ের একটি বেস্ট প্র্যাকটিস (Best Practice)। এর ফলে আপনার MikroTik Router-এর সিকিউরিটি কয়েকগুণ বেড়ে যায়, কারণ হ্যাকাররা পাসওয়ার্ড ব্রুট-ফোর্স করে আর লগইন করতে পারে না। এছাড়া, Ansible বা Python এর মতো অটোমেশন টুলস ব্যবহার করে রাউটার ম্যানেজ করার জন্য Public এবং Private Key এর মাধ্যমে এই পাসওয়ার্ডলেস লগইন মেকানিজম কনফিগার করাটা অপরিহার্য।
