Configuring SSTP Client-to-Site VPN on MikroTik and Windows

10 May, 2026 VPN & Cyber Security Jobaer Hossain

যেকোনো প্রতিষ্ঠানের এমপ্লয়িদের রিমোটলি (যেমন বাসা বা হোটেল থেকে) অফিসের নেটওয়ার্কে যুক্ত করার জন্য Client-to-Site VPN বা Remote Access VPN ব্যবহার করা হয়। কিন্তু সাধারণ ভিপিএন (যেমন PPTP বা L2TP) অনেক সময় আইএসপি (ISP) বা কর্পোরেট ফায়ারওয়াল দ্বারা ব্লক হয়ে যায়। এই ফায়ারওয়াল বাইপাস করার সবচেয়ে মোক্ষম হাতিয়ার হলো SSTP (Secure Socket Tunneling Protocol), কারণ এটি সাধারণ ওয়েব ট্রাফিকের মতো TCP Port 443 (HTTPS) ব্যবহার করে কাজ করে।

মাইক্রোসফট (Microsoft) দ্বারা তৈরি হওয়ায় উইন্ডোজ অপারেটিং সিস্টেমে কোনো থার্ড-পার্টি অ্যাপ ছাড়াই SSTP কানেক্ট করা যায়। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো কীভাবে MikroTik RouterOS-এ একটি SSTP সার্ভার তৈরি করতে হয় এবং একটি Windows PC থেকে সার্টিফিকেট ইমপোর্ট করে সেই সার্ভারে নিরাপদে যুক্ত হতে হয়।

রিয়েল-লাইফ সিনারিও (The Remote Worker)

লক্ষ্য: আপনার অফিসের রাউটারের রিয়েল আইপি (Public IP) হলো 203.0.113.10। আপনার একজন কর্মী দেশের বাইরে থেকে তার Windows 11 ল্যাপটপ দিয়ে অফিসের লোকাল ফাইল সার্ভারটি এক্সেস করতে চায়। আমরা রাউটারে SSTP Server তৈরি করবো এবং কর্মীকে 172.16.10.x ব্লকের একটি ভিপিএন আইপি দেবো।

প্রধান চ্যালেঞ্জ: SSTP কাজ করার জন্য Certificate (SSL) বাধ্যতামূলক। আমরা রাউটারেই একটি Self-Signed Certificate তৈরি করবো এবং সেটি এক্সপোর্ট করে ওই কর্মীর ল্যাপটপে ইন্সটল করবো, যাতে উইন্ডোজ পিসি আমাদের রাউটারকে বিশ্বাস (Trust) করতে পারে।

১. MikroTik রাউটার কনফিগারেশন (SSTP Server)

রাউটারের কনফিগারেশনটি আমরা কয়েকটি ধাপে ভাগ করবো। কাজের সুবিধার জন্য টার্মিনাল বা কমান্ড লাইন ব্যবহার করা সবচেয়ে ভালো।

Step 1: Certificate তৈরি এবং সাইন (Sign) করা

প্রথমে আমাদের একটি Certificate Authority (CA) এবং একটি Server Certificate তৈরি করতে হবে।

# CA (Certificate Authority) তৈরি করা
/certificate add name=ca-template common-name=myOfficeCA key-usage=key-cert-sign,crl-sign
/certificate sign ca-template name=OfficeCA

# Server Certificate তৈরি করা (common-name এ অবশ্যই Public IP বা Domain Name দিতে হবে)
/certificate add name=server-template common-name=203.0.113.10
/certificate sign server-template ca=OfficeCA name=SSTP-ServerCert

ব্যাখ্যা: এখানে common-name=203.0.113.10 অত্যন্ত গুরুত্বপূর্ণ। ক্লায়েন্ট পিসি যখন ভিপিএন কানেক্ট করবে, তখন সে হুবহু এই আইপি বা ডোমেইন অ্যাড্রেসটিতেই হিট করতে হবে, নাহলে উইন্ডোজ এরর দেবে।

Step 2: CA Certificate এক্সপোর্ট করা

উইন্ডোজ পিসিতে ইন্সটল করার জন্য আমাদের মূল OfficeCA সার্টিফিকেটটি রাউটার থেকে বের করে নিতে হবে।

/certificate export-certificate OfficeCA export-passphrase=""

কমান্ড দেওয়ার পর Winbox এর Files মেনুতে যান। সেখানে cert_export_OfficeCA.crt নামের একটি ফাইল তৈরি হবে। ফাইলটি ড্র্যাগ করে (Drag and drop) আপনার পিসিতে কপি করে নিন। এই ফাইলটিই রিমোট কর্মীর ল্যাপটপে পাঠাতে হবে।

Step 3: IP Pool, Profile এবং User তৈরি করা

# IP Pool তৈরি
/ip pool add name=sstp-pool ranges=172.16.10.10-172.16.10.50

# PPP Profile তৈরি
/ppp profile add name=sstp-profile local-address=172.16.10.1 remote-address=sstp-pool use-encryption=yes

# VPN User (Secret) তৈরি
/ppp secret add name=remote_user password=SecretPass!23 service=sstp profile=sstp-profile

Step 4: SSTP Server চালু করা এবং Firewall রুল অ্যাড করা

# Enable SSTP Server and Attach Certificate
/interface sstp-server server set enabled=yes default-profile=sstp-profile certificate=SSTP-ServerCert force-aes=yes

# Allow Port 443 in Firewall (অবশ্যই Drop রুলের ওপরে রাখতে হবে)
/ip firewall filter add action=accept chain=input protocol=tcp dst-port=443 comment="Allow SSTP VPN"

২. Windows PC কনফিগারেশন (SSTP Client)

রাউটারের কাজ শেষ। এখন রিমোট কর্মীর ল্যাপটপে কনফিগার করতে হবে। এই অংশটি দুটি প্রধান ধাপে বিভক্ত।

Step 1: Certificate ইন্সটল করা (Trusted Root Authority)

যেহেতু আমাদের সার্টিফিকেটটি Self-Signed, তাই উইন্ডোজ ডিফল্টভাবে এটিকে ট্রাস্ট করবে না। আমাদের ম্যানুয়ালি এটিকে ট্রাস্ট করাতে হবে।

  1. রাউটার থেকে যে cert_export_OfficeCA.crt ফাইলটি এনেছিলেন, তার ওপর ডাবল-ক্লিক করুন।
  2. Install Certificate... বাটনে ক্লিক করুন।
  3. Store Location হিসেবে Local Machine সিলেক্ট করে Next দিন।
  4. "Place all certificates in the following store" অপশনটি বেছে নিন এবং Browse এ ক্লিক করুন।
  5. লিস্ট থেকে Trusted Root Certification Authorities ফোল্ডারটি সিলেক্ট করে OK দিন এবং ফিনিশ (Finish) করুন। একটি সাকসেস মেসেজ আসবে।

Step 2: Windows VPN Connection তৈরি করা

  1. উইন্ডোজের Settings > Network & internet > VPN এ যান।
  2. Add VPN বাটনে ক্লিক করুন।
  3. নিচের তথ্যগুলো পূরণ করুন:
    • VPN provider: Windows (built-in)
    • Connection name: Office SSTP VPN (যেকোনো নাম)
    • Server name or address: 203.0.113.10 (এখানে হুবহু সেই আইপি বা ডোমেইন দেবেন, যা সার্টিফিকেটের common-name এ দিয়েছিলেন)।
    • VPN type: Secure Socket Tunneling Protocol (SSTP)
    • Type of sign-in info: User name and password
    • User name: remote_user
    • Password: SecretPass!23
  4. Save বাটনে ক্লিক করুন।
  5. তৈরি হওয়া ভিপিএন প্রোফাইলের পাশে Connect বাটনে ক্লিক করুন। কয়েক সেকেন্ডের মধ্যেই আপনি নিরাপদে অফিসের নেটওয়ার্কের সাথে যুক্ত হয়ে যাবেন।

৩. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

সমস্যা (Error / Problem) কারণ এবং সমাধান (Cause & Solution)
Error 0x80092013 (Revocation Check Failed) উইন্ডোজ বাই-ডিফল্ট চেক করে সার্টিফিকেটটি বাতিল (Revoke) করা হয়েছে কি না। যেহেতু আমাদের Self-Signed সার্টিফিকেটের কোনো CRL লিস্ট নেই, তাই এটি ফেইল করে। সমাধান: উইন্ডোজের Registry Editor (regedit) এ গিয়ে HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\SstpSvc\Parameters পাথে NoCertRevocationCheck নামে একটি DWORD তৈরি করে এর ভ্যালু 1 করে দিন।
Certificate Common Name Mismatch আপনি রাউটারে সার্টিফিকেটের common-name দিয়েছেন আইপি অ্যাড্রেস, কিন্তু উইন্ডোজে Server Address এর ঘরে দিয়েছেন ডোমেইন নেম (বা উল্টো)। এই দুটি নাম হুবহু এক হতে হবে।
Port 443 Conflict (Webfig) আপনার রাউটারে যদি WWW-SSL (Webfig HTTPS) সার্ভিস পোর্ট 443 এ চালু থাকে, তবে SSTP কাজ করবে না। Winbox থেকে IP > Services এ গিয়ে www-ssl এর পোর্টটি পরিবর্তন করে (যেমন 4433) দিন, অথবা সার্ভিসটি ডিজেবল করে দিন।
Clock Sync Issue (Date and Time) রাউটারের বা পিসির তারিখ এবং সময় যদি সঠিক না থাকে, তবে সার্টিফিকেট "Expired" বা "Not Valid Yet" দেখাবে এবং ভিপিএন কানেক্ট হবে না। উভয় ডিভাইসের সময় সিঙ্ক (Sync) করে নিন।

উপসংহার (Conclusion)

SSTP (Secure Socket Tunneling Protocol) হলো রিমোট ওয়ার্কারদের জন্য অন্যতম সেরা একটি সমাধান, বিশেষ করে যখন তারা কোনো রেস্ট্রিক্টেড (Restricted) নেটওয়ার্ক বা পাবলিক ওয়াই-ফাইয়ের পেছনে থাকে। MikroTik এ সার্টিফিকেট জেনারেট করে Windows পিসিতে ট্রাস্ট করানোর এই পদ্ধতিটি একবার আয়ত্ত করতে পারলে, ফায়ারওয়াল বাইপাস করে অত্যন্ত সিকিউরড (AES-256) একটি Client-to-Site VPN নেটওয়ার্ক তৈরি করা আপনার জন্য একদম সহজ হয়ে যাবে।
