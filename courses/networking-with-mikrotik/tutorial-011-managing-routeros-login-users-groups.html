Managing RouterOS Login – Users & Groups

10 July, 2026 Networking with MikroTik Jobaer Hossain

একটি নতুন MikroTik Router কিনে বক্স থেকে বের করার পর, আপনি যখন প্রথমবার Winbox বা ব্রাউজার দিয়ে লগইন করেন, তখন ডিফল্ট Username থাকে admin এবং Password এর ঘর থাকে সম্পূর্ণ ফাঁকা (Blank)। এটি কনফিগারেশন শুরু করার জন্য সুবিধাজনক হলেও, একটি লাইভ নেটওয়ার্কের জন্য এটি এক ভয়াবহ সিকিউরিটি রিস্ক (Security Risk)।

আপনার রাউটারে কে লগইন করতে পারবে, কে কী পরিবর্তন করতে পারবে এবং কে শুধু দেখতে পারবে—এই পুরো অ্যাক্সেস কন্ট্রোল সিস্টেমটি মেইনটেইন করা হয় Users এবং Groups এর মাধ্যমে। আজকের এই ডিপ-ডাইভ গাইডে আমরা শিখবো কীভাবে RouterOS এ ইউজার ম্যানেজমেন্ট করতে হয়, গ্রুপের Policies কীভাবে কাজ করে এবং সিকিউরিটি বেস্ট প্র্যাকটিসগুলো কী কী।

User এবং Group এর প্রয়োজনীয়তা

রিয়েল-লাইফ সিনারিও: আপনি একটি বড় কর্পোরেট অফিসের আইটি ম্যানেজার। আপনার আন্ডারে কয়েকজন জুনিয়র সাপোর্ট ইঞ্জিনিয়ার কাজ করেন। আপনার রাউটারে যদি শুধুমাত্র একটি admin অ্যাকাউন্ট থাকে এবং আপনি সবাইকে সেই পাসওয়ার্ড দিয়ে দেন, তবে কেউ ভুলে কোনো ফায়ারওয়াল রুল মুছে ফেললে আপনি ধরতে পারবেন না কে এই কাজটি করেছে।

সমাধান: এই সমস্যার সমাধান হলো Role-Based Access Control (RBAC)। আপনি সাপোর্ট টিমের জন্য আলাদা User তৈরি করবেন এবং তাদের এমন একটি Group এ রাখবেন, যাদের শুধু রাউটারের কনফিগারেশন দেখার (Read-only) বা বেসিক কাজ করার অনুমতি থাকবে। আর আপনার জন্য থাকবে সর্বোচ্চ ক্ষমতার (Full access) অ্যাকাউন্ট।

১. RouterOS এর Default Groups এবং Policies

RouterOS এ ইউজার তৈরি করার আগে আমাদের বুঝতে হবে Group কীভাবে কাজ করে। প্রতিটি Group মূলত কিছু Policy বা রুলের সমষ্টি। ডিফল্টভাবে MikroTik এ তিনটি গ্রুপ তৈরি করা থাকে:

গ্রুপের নাম (Group Name) অনুমতি বা পলিসি (Policies) ব্যবহারের ক্ষেত্র (Use Case)
Full রাউটারের সবকিছু করার ক্ষমতা রাখে (Read, Write, Policy change, Reboot, FTP, SSH ইত্যাদি)। শুধুমাত্র প্রধান নেটওয়ার্ক অ্যাডমিনিস্ট্রেটর বা আইটি হেড-এর অ্যাকাউন্টের জন্য।
Write রাউটারে কনফিগারেশন পরিবর্তন করতে পারে (Read এবং Write)। কিন্তু নতুন ইউজার তৈরি করা বা পলিসি পরিবর্তন করার ক্ষমতা নেই। সিনিয়র সিস্টেম ইঞ্জিনিয়ার বা কনফিগারেশন টিমের জন্য।
Read রাউটারের কোনো কনফিগারেশন পরিবর্তন করতে পারে না। শুধুমাত্র দেখতে পারে এবং পিং (Ping) বা ট্রেস (Traceroute) করতে পারে। জুনিয়র সাপোর্ট টিম, অডিটর বা মনিটরিং সিস্টেমের (যেমন Zabbix/PRTG) জন্য।

২. Winbox ব্যবহার করে User এবং Group কনফিগারেশন

চলুন প্র্যাকটিক্যালি দেখি কীভাবে Winbox এর মাধ্যমে ইউজার ম্যানেজমেন্ট করতে হয়।

Step 1: নতুন Group তৈরি করা (Custom Group)

ডিফল্ট গ্রুপের বাইরে আপনি চাইলে আপনার প্রয়োজনমতো পলিসি সিলেক্ট করে নতুন গ্রুপ তৈরি করতে পারেন। ধরি, আমরা "Support-Team" নামে একটি গ্রুপ বানাবো, যারা শুধু কনফিগারেশন দেখতে পারবে এবং রাউটার রিবুট (Reboot) করতে পারবে।

  • Winbox থেকে System > Users মেনুতে ক্লিক করুন।
  • Groups ট্যাবে যান এবং + (Add) আইকনে ক্লিক করুন।
  • Name: এর ঘরে লিখুন Support-Team।
  • নিচে অনেকগুলো চেকবক্স (Policies) দেখতে পাবেন। সেখান থেকে read, winbox, web, api, এবং reboot চেক করুন। বাকিগুলোর আনচেক (Uncheck) রাখুন।
  • Apply এবং OK দিন।

Step 2: নতুন User তৈরি করা

এবার আমরা নতুন সাপোর্ট ইঞ্জিনিয়ারের জন্য একটি অ্যাকাউন্ট তৈরি করবো।

  • System > Users মেনুর Users ট্যাবে ফিরে যান এবং + আইকনে ক্লিক করুন।
  • Name: এর ঘরে ইউজারের নাম দিন (যেমন: rafiq)।
  • Group: এর ড্রপডাউন থেকে আমাদের বানানো Support-Team সিলেক্ট করুন।
  • Password এবং Confirm Password এর ঘরে একটি শক্তিশালী পাসওয়ার্ড দিন।
  • Apply এবং OK বাটনে ক্লিক করুন।

Step 3: Allowed Address সেট করা (অ্যাডভান্সড সিকিউরিটি)

User তৈরি করার সময় একটি অত্যন্ত গুরুত্বপূর্ণ সিকিউরিটি ফিচার হলো Allowed Address। আপনি যদি চান যে 'rafiq' নামের ইউজারটি শুধুমাত্র অফিসের একটি নির্দিষ্ট আইপি (যেমন: 192.168.10.50) বা ব্লক থেকে লগইন করতে পারবে, তবে User প্রোপার্টিজ উইন্ডোর Allowed Address বক্সে ওই আইপিটি বসিয়ে দিন। ইন্টারনেটের অন্য কোনো জায়গা থেকে সে সঠিক পাসওয়ার্ড দিলেও লগইন করতে পারবে না।

Step 4: Default Admin অ্যাকাউন্ট ডিসেবল বা ডিলিট করা

আপনার নিজের জন্য একটি ফুল অ্যাক্সেস (Full group) ইউজার তৈরি করার পর, সবচেয়ে গুরুত্বপূর্ণ কাজ হলো ডিফল্ট admin অ্যাকাউন্টটি ডিসেবল (Disable) করে দেওয়া। কারণ হ্যাকাররা সব সময় admin ইউজারনেম দিয়ে ব্রুট-ফোর্স (Brute-force) অ্যাটাক করার চেষ্টা করে। admin সিলেক্ট করে X (Disable) আইকনে ক্লিক করুন।

৩. Command Line (CLI) ব্যবহার করে ইউজার ম্যানেজমেন্ট

প্রফেশনালরা সাধারণত CLI ব্যবহার করে দ্রুত এই কাজগুলো করেন। নিচে প্রয়োজনীয় কমান্ডগুলো দেওয়া হলো:

নতুন Custom Group তৈরি করা:

[admin@MikroTik] > /user group add name=Support-Team policy=read,winbox,web,api,reboot,!write,!policy,!sensitive

(নোট: পলিসির আগে ! চিহ্ন থাকার মানে হলো ওই অনুমতিটি দেওয়া হয়নি।)

নতুন User তৈরি এবং Group অ্যাসাইন করা:

[admin@MikroTik] > /user add name=rafiq group=Support-Team password="SecurePass123" allowed-address=192.168.10.0/24

বর্তমান ইউজারদের লিস্ট দেখা:

[admin@MikroTik] > /user print

ডিফল্ট Admin অ্যাকাউন্ট বন্ধ করা:

[admin@MikroTik] > /user disable admin

৪. Troubleshooting Tips (সাধারণ ভুল ও সমাধান)

ইউজার ম্যানেজমেন্ট করতে গিয়ে কিছু অনাকাঙ্ক্ষিত পরিস্থিতি তৈরি হতে পারে:

  • নিজে লক আউট হয়ে যাওয়া (Locked Out): আপনি যদি Allowed Address এ ভুল আইপি বসিয়ে সেভ করেন, তবে আপনি সাথে সাথে লগ আউট হয়ে যাবেন এবং আর ঢুকতে পারবেন না। এর একমাত্র সমাধান হলো রাউটারের Reset Button ব্যবহার করে ফ্যাক্টরি রিসেট করা, অথবা MAC Address দিয়ে লগইন করার চেষ্টা করা (যদি MAC Server এ রেস্ট্রিকশন না থাকে)।
  • লগইন ফেইল হওয়ার মেসেজ: আপনি যদি /log print কমান্ড দিয়ে দেখেন যে বারবার "login failure for user admin from IP..." মেসেজ আসছে, তবে বুঝতে হবে বাইরে থেকে কেউ অ্যাটাক করছে। দ্রুত System > Users এ গিয়ে ডিফল্ট admin অ্যাকাউন্টটি বন্ধ করুন এবং আপনার কাস্টম ইউজারের জন্য একটি জটিল পাসওয়ার্ড সেট করুন।
  • পাসওয়ার্ড রিকভারি: MikroTik এ একবার পাসওয়ার্ড সেট করলে তা এনক্রিপ্টেড থাকে এবং রিকভার করার কোনো উপায় নেই। পাসওয়ার্ড ভুলে গেলে রাউটার ফিজিক্যালি রিসেট করা অথবা Netinstall করা ছাড়া কোনো বিকল্প থাকে না।

উপসংহার (Conclusion)

নেটওয়ার্ক ডিভাইসের সুরক্ষার প্রথম ধাপই হলো সঠিক User Management। একটি রাউটারে শুধু একটি ডিফল্ট admin অ্যাকাউন্ট ব্যবহার করাটা প্রফেশনালিজমের অভাব এবং বিশাল সিকিউরিটি লুপহোল (Loophole) হিসেবে গণ্য হয়। RouterOS এর শক্তিশালী Groups এবং Policies ব্যবহার করে আপনি অত্যন্ত নিখুঁতভাবে নির্ধারণ করতে পারেন কে রাউটারের কোন অংশে কাজ করবে। Allowed Address এর মতো ফিচার ব্যবহার করে আপনার রাউটারের ম্যানেজমেন্ট ইন্টারফেসকে হ্যাকারদের ধরাছোঁয়ার বাইরে রাখুন এবং নিজের নেটওয়ার্কের নিরাপত্তা নিশ্চিত করুন।
