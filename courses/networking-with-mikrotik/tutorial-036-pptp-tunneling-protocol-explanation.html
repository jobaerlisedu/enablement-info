PPTP Tunneling Protocol Explanation

24 April, 2026 VPN & Cyber Security Jobaer Hossain

নেটওয়ার্কিং এবং ভিপিএন (VPN) এর দুনিয়ায় PPTP (Point-to-Point Tunneling Protocol) হলো অন্যতম প্রাচীন এবং সর্বাধিক পরিচিত একটি প্রোটোকল। নব্বইয়ের দশকে মাইক্রোসফট (Microsoft) এবং আরও কয়েকটি কোম্পানি মিলে ডাটা আদান-প্রদানের জন্য এই প্রোটোকলটি তৈরি করেছিল। যদিও বর্তমানে এটি সিকিউরিটির দিক থেকে কিছুটা দুর্বল হিসেবে বিবেচিত হয়, তবুও কনফিগারেশনের সহজলভ্যতা এবং প্রায় সব ডিভাইসে ডিফল্ট সাপোর্ট থাকার কারণে এটি আজও প্রচুর ব্যবহৃত হয়।

আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা জানবো PPTP কীভাবে কাজ করে, এর এনক্রিপশন মেকানিজম কী এবং একটি MikroTik Router-এ কীভাবে ধাপে ধাপে PPTP Server কনফিগার করতে হয়।

রিয়েল-লাইফ সিনারিও (Remote Access VPN)

সিনারিও: আপনি হয়তো দেশের বাইরে বা কোনো কফি শপে বসে আছেন। হুট করে আপনার অফিসের লোকাল সার্ভারে (যাতে কোনো পাবলিক আইপি নেই) ঢুকে একটি গুরুত্বপূর্ণ ফাইল আপডেট করতে হবে।

সমাধান: আপনার অফিসের রাউটারে যদি একটি PPTP Server কনফিগার করা থাকে, তবে আপনি আপনার ল্যাপটপের বিল্ট-ইন ভিপিএন অপশন থেকে একটি ইউজারনেম ও পাসওয়ার্ড দিয়ে সরাসরি অফিসের রাউটারে কানেক্ট হতে পারবেন। কানেক্ট হওয়ার পর আপনি অফিসের লোকাল নেটওয়ার্কের (LAN) একটি IP Address পাবেন এবং দূর থেকে বসেই এমনভাবে কাজ করতে পারবেন যেন আপনি অফিসেই বসে আছেন।

১. PPTP কীভাবে কাজ করে? (Technical Architecture)

PPTP প্রোটোকলটি কাজ করার জন্য মূলত দুটি আলাদা নেটওয়ার্ক সেশন বা চ্যানেল ব্যবহার করে। এটি বোঝা ট্রাবলশুটিংয়ের জন্য অত্যন্ত জরুরি:

  • Control Channel (TCP Port 1723): ক্লায়েন্ট যখন সার্ভারের সাথে কানেক্ট হতে চায়, তখন প্রথমে TCP Port 1723 ব্যবহার করে একটি কন্ট্রোল কানেকশন তৈরি হয়। এই চ্যানেল দিয়ে ইউজারনেম, পাসওয়ার্ড এবং ভিপিএন ম্যানেজমেন্টের তথ্য আদান-প্রদান হয়।
  • Data Channel (GRE - Protocol 47): কন্ট্রোল চ্যানেল সফলভাবে তৈরি হওয়ার পর, আসল ডাটা (যেমন ফাইল ট্রান্সফার বা ব্রাউজিং) পাঠানোর জন্য GRE (Generic Routing Encapsulation) প্রোটোকল ব্যবহার করা হয়। GRE হলো Protocol 47 (এটি কোনো পোর্ট নয়, এটি TCP বা UDP এর মতোই একটি প্রোটোকল)।

২. Security এবং Encryption (নিরাপত্তা ব্যবস্থা)

ভিপিএন টানেলের ভেতর দিয়ে যাওয়া ডাটাগুলো যেন কেউ মাঝপথ থেকে পড়ে ফেলতে না পারে, তার জন্য PPTP কিছু স্পেসিফিক মেকানিজম ব্যবহার করে:

মেকানিজম (Mechanism) কাজের বিবরণ (Description)
Authentication (প্রমাণীকরণ) ইউজারনেম এবং পাসওয়ার্ড চেক করার জন্য PPTP সাধারণত MS-CHAPv2 (Microsoft Challenge Handshake Authentication Protocol) ব্যবহার করে। এটি প্লেইন টেক্সটের বদলে হ্যাশ (Hash) করে পাসওয়ার্ড পাঠায়।
Encryption (এনক্রিপশন) ডাটা লুকিয়ে ফেলার জন্য এটি MPPE (Microsoft Point-to-Point Encryption) ব্যবহার করে, যা সাধারণত 128-bit এনক্রিপশন সাপোর্ট করে।
দুর্বলতা (Vulnerability) PPTP এর MS-CHAPv2 অথেন্টিকেশন প্রসেসটি আধুনিক হ্যাকিং টুলের (যেমন: Dictionary Attack) মাধ্যমে ক্র্যাক করা সম্ভব। তাই এন্টারপ্রাইজ লেভেলে হাই-সিকিউরিটি ডাটার জন্য PPTP ব্যবহার করা রিকমেন্ডেড নয়।

৩. MikroTik রাউটারে PPTP Server কনফিগারেশন

চলুন একটি রিয়েল-লাইফ এক্সাম্পল দেখি। আমরা রাউটারে PPTP Server চালু করবো এবং ভিপিএন ইউজারদের জন্য 10.10.10.0/24 ব্লকের আইপি দেবো।

Step 1: IP Pool তৈরি করা

ভিপিএন ক্লায়েন্টরা কানেক্ট হওয়ার পর যে আইপিগুলো পাবে, তার একটি পুল (Pool) তৈরি করতে হবে।

  • IP > Pool এ যান এবং + (Add) আইকনে ক্লিক করুন।
  • Name: vpn-pool দিন।
  • Addresses: 10.10.10.10-10.10.10.50 দিন। OK করুন।

Step 2: PPP Profile তৈরি করা

এখানে আমরা নির্ধারণ করবো ক্লায়েন্টরা কোন আইপি পাবে এবং কোন এনক্রিপশন ব্যবহার করবে।

  • PPP মেনুতে গিয়ে Profiles ট্যাবে যান এবং + (Add) এ ক্লিক করুন।
  • Name: pptp-profile দিন।
  • Local Address: রাউটারের লোকাল আইপি (যেমন 10.10.10.1) দিন।
  • Remote Address: ড্রপডাউন থেকে vpn-pool সিলেক্ট করুন।
  • Protocols ট্যাবে গিয়ে Use Encryption অপশনটি yes করে দিন। OK করুন।

Step 3: VPN User (Secret) তৈরি করা

  • PPP মেনুর Secrets ট্যাবে যান এবং + (Add) এ ক্লিক করুন।
  • Name: ইউজারের নাম দিন (যেমন jobaer)।
  • Password: একটি শক্তিশালী পাসওয়ার্ড দিন (যেমন P@ssw0rd123)।
  • Service: pptp সিলেক্ট করুন।
  • Profile: pptp-profile সিলেক্ট করুন। OK করুন।

Step 4: PPTP Server চালু করা

  • PPP মেনুর Interface ট্যাবে যান।
  • উপরের PPTP Server বাটনে ক্লিক করুন।
  • Enabled চেকবক্সে টিক দিন।
  • Default Profile: pptp-profile সিলেক্ট করুন এবং Apply / OK দিন।

৪. Command Line (CLI) কনফিগারেশন

টার্মিনাল ব্যবহার করে উপরের ধাপগুলো খুব দ্রুত করা যায়:

# Create IP Pool
/ip pool add name=vpn-pool ranges=10.10.10.10-10.10.10.50

# Create PPP Profile
/ppp profile add name=pptp-profile local-address=10.10.10.1 remote-address=vpn-pool use-encryption=yes

# Create VPN User
/ppp secret add name=jobaer password=P@ssw0rd123 service=pptp profile=pptp-profile

# Enable PPTP Server
/interface pptp-server server set enabled=yes default-profile=pptp-profile

৫. Firewall কনফিগারেশন (অত্যন্ত গুরুত্বপূর্ণ)

রাউটারের ফায়ারওয়ালে যদি PPTP এর পোর্ট এবং প্রোটোকল অ্যালাউ করা না থাকে, তবে ক্লায়েন্ট কানেক্ট হতে পারবে না। 

# Allow TCP Port 1723 (Control Channel)
/ip firewall filter add action=accept chain=input protocol=tcp dst-port=1723 comment="Allow PPTP"

# Allow GRE Protocol 47 (Data Channel)
/ip firewall filter add action=accept chain=input protocol=gre comment="Allow GRE for PPTP"

নোট: এই রুল দুটিকে ফায়ারওয়ালের ওপরের দিকে (Drop রুলগুলোর ওপরে) নিয়ে যেতে হবে।

৬. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

  • Error 800 বা Timeout: যদি উইন্ডোজ পিসি থেকে কানেক্ট করার সময় কানেকশন টাইমআউট হয়, তবে বুঝতে হবে আপনার রাউটারের পাবলিক আইপিতে TCP 1723 পোর্ট ব্লক আছে। ফায়ারওয়াল রুল চেক করুন অথবা আইএসপি (ISP) পোর্ট ব্লক করেছে কি না তা নিশ্চিত করুন।
  • Error 619 বা "Verifying Username and Password" এ আটকে থাকা: এটি PPTP এর সবচেয়ে কমন সমস্যা। এর মানে হলো Control Channel (TCP 1723) কাজ করছে, কিন্তু Data Channel (GRE Protocol 47) ব্লক হয়ে আছে। ফায়ারওয়ালে GRE প্রোটোকল এক্সেপ্ট করা আছে কি না তা যাচাই করুন। অনেক সময় মধ্যবর্তী রাউটারগুলো GRE প্রোটোকল পাস করতে দেয় না (যাকে GRE Pass-through সমস্যা বলে)।
  • কানেক্ট হচ্ছে কিন্তু ইন্টারনেট বা লোকাল পিসি পিং পাচ্ছে না: ভিপিএন ইন্টারফেসে NAT (Masquerade) করা না থাকলে এই সমস্যা হয়। IP > Firewall > NAT এ গিয়ে srcnat চেইনে Out. Interface হিসেবে আপনার ইন্টারনেট পোর্ট সিলেক্ট করে Masquerade করে দিন অথবা ভিপিএন ব্লকের জন্য রাউটিং কনফিগার করুন।

উপসংহার (Conclusion)

PPTP Tunneling Protocol সেটআপ করা অত্যন্ত সহজ এবং এর স্পিডও অনেক ভালো, কারণ এর এনক্রিপশন ওভারহেড (Overhead) তুলনামূলক কম। যদি আপনার লক্ষ্য হয় শুধুমাত্র জিও-রেস্ট্রিকশন (Geo-restriction) বাইপাস করা বা খুব বেসিক লেভেলের রিমোট এক্সেস নেওয়া, তবে PPTP এখনো একটি কার্যকরী প্রোটোকল। কিন্তু কর্পোরেট নেটওয়ার্ক, ব্যাংকিং সেক্টর বা হাইলি কনফিডেন্সিয়াল ডাটা আদান-প্রদানের ক্ষেত্রে PPTP এড়িয়ে L2TP/IPsec, OpenVPN বা আধুনিক WireGuard প্রোটোকল ব্যবহার করা একজন দক্ষ নেটওয়ার্ক ইঞ্জিনিয়ারের বেস্ট প্র্যাকটিস।
