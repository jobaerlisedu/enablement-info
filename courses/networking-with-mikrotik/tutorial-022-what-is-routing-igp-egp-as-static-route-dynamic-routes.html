What is Routing (IGP, EGP, AS, Static Route, Dynamic Routes)

20 September, 2026 Network Engineering Jobaer Hossain

ইন্টারনেট বা যেকোনো বড় কম্পিউটার নেটওয়ার্কের মেরুদণ্ড হলো Routing। আপনার ল্যাপটপ থেকে যখন আপনি ব্রাউজারে google.com লিখে এন্টার চাপেন, তখন আপনার রিকোয়েস্ট (Packet) হাজার হাজার মাইল দূরের গুগলের সার্ভার পর্যন্ত কীভাবে পৌঁছায়? এর পেছনের জাদুকরী প্রক্রিয়াটিই হলো রাউটিং।

আজকের এই ইন-ডেপথ টিউটোরিয়ালে আমরা রাউটিংয়ের আদ্যোপান্ত জানবো। আমরা শিখবো Autonomous System (AS) কী, IGP এবং EGP এর মধ্যে পার্থক্য কী, এবং Static ও Dynamic Routing কীভাবে কাজ করে।

Routing কী? (What is Routing?)

Routing হলো একটি নেটওয়ার্ক থেকে অন্য একটি নেটওয়ার্কে ডাটা প্যাকেট (Data Packet) পাঠানোর জন্য সবচেয়ে সেরা এবং সংক্ষিপ্ত পথ (Best Path) খুঁজে বের করার প্রক্রিয়া। OSI Model এর Network Layer (Layer 3) এ এই কাজটি সম্পন্ন হয় এবং যে ডিভাইস এই কাজটি করে তাকে Router বলা হয়।

রিয়েল-লাইফ উদাহরণ: মনে করুন, আপনি ঢাকা থেকে কক্সবাজার যেতে চান। আপনার হাতে অনেকগুলো পথ আছে—আপনি কুমিল্লা হয়ে যেতে পারেন, আবার চাঁদপুর হয়েও যেতে পারেন। আপনি গুগল ম্যাপ (Google Maps) চেক করে দেখলেন কোন রাস্তায় জ্যাম কম এবং দূরত্ব কম। এরপর আপনি সেরা রাস্তাটি বেছে নিলেন। নেটওয়ার্কিংয়ের দুনিয়ায় Router ঠিক এই গুগল ম্যাপের মতোই কাজ করে। সে তার Routing Table চেক করে সিদ্ধান্ত নেয় প্যাকেটটি কোন ইন্টারফেস দিয়ে পাঠালে সবচেয়ে দ্রুত গন্তব্যে পৌঁছাবে।

১. Autonomous System (AS) কী?

রাউটিংয়ের গভীরে যাওয়ার আগে Autonomous System (AS) কনসেপ্টটি পরিষ্কার হওয়া অত্যন্ত জরুরি।

AS হলো এমন একটি বড় নেটওয়ার্ক বা নেটওয়ার্কের সমষ্টি, যা একটি মাত্র প্রতিষ্ঠান বা অ্যাডমিনিস্ট্রেশনের সম্পূর্ণ নিয়ন্ত্রণে থাকে এবং যাদের রাউটিং পলিসি একই। প্রতিটি AS এর একটি ইউনিক নাম্বার থাকে, যাকে ASN (Autonomous System Number) বলা হয়। এটি অনেকটা একটি দেশের পিন কোডের মতো।

  • উদাহরণ: বাংলাদেশে গ্রামীণফোন (Grameenphone) এর নিজস্ব একটি বিশাল নেটওয়ার্ক আছে। এটি একটি AS। আবার বাংলালিংক (Banglalink) এর নিজস্ব নেটওয়ার্ক আছে, এটি আরেকটি AS। যখন এই দুটি ভিন্ন AS একে অপরের সাথে ডাটা আদান-প্রদান করতে চায়, তখন তাদের বিশেষ রাউটিং প্রোটোকল ব্যবহার করতে হয়। গ্লোবাল ইন্টারনেট মূলত হাজার হাজার AS এর একটি বিশাল সংযোগস্থল।

২. Routing Protocol এর প্রকারভেদ (IGP বনাম EGP)

এই AS এর কনসেপ্টের ওপর ভিত্তি করেই রাউটিং প্রোটোকলগুলোকে প্রধানত দুটি ভাগে ভাগ করা হয়েছে:

বৈশিষ্ট্য (Feature) IGP (Interior Gateway Protocol) EGP (Exterior Gateway Protocol)
সংজ্ঞা ও কাজের ক্ষেত্র যে প্রোটোকলগুলো একটি একক AS (Autonomous System) এর ভেতরে রাউটিংয়ের কাজ করে। যে প্রোটোকল দুটি ভিন্ন AS এর মধ্যে রাউটিং বা ডাটা আদান-প্রদানের কাজ করে।
প্রধান কাজ লোকাল নেটওয়ার্কের ভেতরে সবচেয়ে দ্রুত (Best Path) পথ খুঁজে বের করা। সারা বিশ্বের ইন্টারনেটে এক নেটওয়ার্ক থেকে অন্য নেটওয়ার্কে ডাটা পাঠানো।
উদাহরণ (Protocols) OSPF (Open Shortest Path First), RIP, IS-IS, EIGRP। BGP (Border Gateway Protocol)। বর্তমান ইন্টারনেট পুরোপুরি BGP এর ওপর চলে।
রিয়েল-লাইফ সিনারিও গ্রামীণফোনের ভেতরে ঢাকা অফিসের রাউটার থেকে চট্টগ্রাম অফিসের রাউটারে ডাটা পাঠানোর জন্য। গ্রামীণফোনের নেটওয়ার্ক থেকে গুগলের বা ফেসবুকের নেটওয়ার্কে ডাটা পাঠানোর জন্য।

৩. Static Route বনাম Dynamic Route

রাউটার কীভাবে পথ চিনবে, তার ওপর ভিত্তি করে রাউটিংকে আবার দুটি পদ্ধতিতে ভাগ করা যায়: Static এবং Dynamic।

A. Static Routing (স্ট্যাটিক রাউটিং)

স্ট্যাটিক রাউটিং হলো এমন একটি পদ্ধতি, যেখানে নেটওয়ার্ক অ্যাডমিনিস্ট্রেটর নিজে ম্যানুয়ালি রাউটারে কমান্ড দিয়ে বলে দেন যে, "কোন আইপি ব্লকে যেতে হলে কোন রাস্তা (Gateway) ব্যবহার করতে হবে।"

  • সুবিধা: এটি অত্যন্ত সুরক্ষিত (Secure), কারণ রাউটার নিজে থেকে কোনো রুট আপডেট নেয় না। এতে রাউটারের প্রসেসর (CPU) এবং মেমোরির ওপর কোনো বাড়তি চাপ পড়ে না।
  • অসুবিধা: যদি মাঝখানের কোনো ক্যাবল বা লিংক ডাউন হয়ে যায়, তবে রাউটার নিজে নিজে অন্য কোনো বিকল্প পথ খুঁজে নিতে পারে না। অ্যাডমিনকে ম্যানুয়ালি এসে আবার নতুন রুট বসাতে হয়। বড় নেটওয়ার্কে এটি মেইনটেইন করা অসম্ভব।
  • ব্যবহার: ছোট অফিস নেটওয়ার্কে (যেখানে শুধু আইএসপির একটি সংযোগ আছে), যাকে Stub Network বলা হয়। সেখানে আমরা সাধারণত Default Static Route (0.0.0.0/0) ব্যবহার করি।

B. Dynamic Routing (ডাইনামিক রাউটিং)

ডাইনামিক রাউটিংয়ে অ্যাডমিনকে ম্যানুয়ালি সব রুট বসাতে হয় না। রাউটারগুলো একে অপরের সাথে কথা বলে (প্রোটোকল ব্যবহার করে) নিজে নিজেই নেটওয়ার্কের একটি ম্যাপ তৈরি করে নেয়।

  • সুবিধা: যদি কোনো ক্যাবল কাটা পড়ে বা লিংক ডাউন হয়, তবে ডাইনামিক রাউটিং প্রোটোকল (যেমন OSPF) মুহূর্তের মধ্যে ক্যালকুলেশন করে বিকল্প পথ (Backup path) খুঁজে বের করে এবং ট্রাফিক ঘুরিয়ে দেয়। এটি বড় এবং জটিল নেটওয়ার্কের জন্য অপরিহার্য।
  • অসুবিধা: এটি কনফিগার করা বেশ জটিল। রাউটারগুলোকে সবসময় একে অপরের সাথে আপডেট আদান-প্রদান করতে হয়, তাই প্রচুর Bandwidth, CPU এবং মেমোরি খরচ হয়।

৪. MikroTik CLI: Static Route কনফিগারেশন

চলুন দেখি একটি MikroTik রাউটারে কীভাবে Static Route বসাতে হয়। মনে করুন আপনার আইএসপি (ISP) আপনাকে 10.10.10.1 গেটওয়ে দিয়েছে এবং আপনি চান আপনার লোকাল ইউজাররা ইন্টারনেটে যাক।

Default Static Route বসানোর কমান্ড:

[admin@MikroTik] > /ip route add dst-address=0.0.0.0/0 gateway=10.10.10.1

ব্যাখ্যা: 0.0.0.0/0 মানে হলো পৃথিবীর যেকোনো আইপি (ইন্টারনেট)। অর্থাৎ, রাউটারকে বলা হচ্ছে, "তোমার কাছে অচেনা যেকোনো আইপির রিকোয়েস্ট আসলে তুমি তা 10.10.10.1 গেটওয়েতে পাঠিয়ে দেবে।"

৫. Troubleshooting Tips (রাউটিং ট্রাবলশুটিং)

নেটওয়ার্কে ডাটা পাস না হলে একজন প্রফেশনাল হিসেবে আপনাকে নিচের ধাপগুলো ফলো করতে হবে:

  • Routing Table চেক করা: রাউটারটি গন্তব্যের পথ চেনে কি না তা দেখতে টার্মিনালে কমান্ড দিন: 
    [admin@MikroTik] > /ip route print
    লিস্টে যদি দেখেন A S (Active, Static) বা A O (Active, OSPF) লেখা আছে, তার মানে রুটটি ঠিক আছে। যদি unreachable লেখা থাকে, তবে বুঝবেন ওই গন্তব্যে যাওয়ার ক্যাবল বা গেটওয়ে ডাউন আছে।
  • Traceroute (ট্রেসরাউট) ব্যবহার করা: প্যাকেটটি কোন রাউটার পর্যন্ত গিয়ে আটকে যাচ্ছে তা দেখার জন্য ট্রেসরাউট অত্যন্ত কার্যকরী: 
    [admin@MikroTik] > /tool traceroute 8.8.8.8
  • Distance (Administrative Distance) চেক করা: যদি একই গন্তব্যে যাওয়ার দুটি পথ (Link 1 এবং Link 2) থাকে, তবে আপনি প্রাইমারি লিংকের Distance 1 এবং ব্যাকআপ লিংকের Distance 2 দিয়ে খুব সহজেই Failover কনফিগার করতে পারবেন। রাউটার সবসময় কম Distance এর পথটি বেছে নেয়।

উপসংহার (Conclusion)

Routing হলো নেটওয়ার্ক ইঞ্জিনিয়ারিংয়ের সবচেয়ে রোমাঞ্চকর এবং গুরুত্বপূর্ণ অধ্যায়। একটি ছোট অফিস থেকে ইন্টারনেটে যাওয়ার জন্য Static Route যেমন অপরিহার্য, তেমনি একটি বিশাল Autonomous System (AS) এর ভেতরের রাউটারগুলোকে ম্যানেজ করার জন্য IGP (যেমন OSPF) এবং পুরো বিশ্বকে যুক্ত করার জন্য EGP (যেমন BGP) এর কোনো বিকল্প নেই। রাউটিং টেবিল কীভাবে কাজ করে এবং ডাইনামিক প্রোটোকলগুলো কীভাবে বেস্ট পাথ (Best Path) নির্বাচন করে—তা পরিষ্কারভাবে বুঝতে পারলে আপনি যেকোনো এন্টারপ্রাইজ নেটওয়ার্ক আত্মবিশ্বাসের সাথে ডিজাইন ও ট্রাবলশুট করতে পারবেন।
