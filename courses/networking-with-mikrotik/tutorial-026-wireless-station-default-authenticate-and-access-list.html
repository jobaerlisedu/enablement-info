Wireless station, Default Authenticate and Access list

25 October, 2026 Networking with MikroTik Jobaer Hossain

ওয়্যারলেস নেটওয়ার্ক কনফিগার করার সময় আমরা সাধারণত একটি পাসওয়ার্ড (Security Profile) দিয়েই কাজ শেষ মনে করি। কিন্তু প্রফেশনাল নেটওয়ার্ক বা WISP (Wireless ISP) এর ক্ষেত্রে শুধু পাসওয়ার্ড দিয়ে নেটওয়ার্ক সুরক্ষিত রাখা সম্ভব নয়। পাসওয়ার্ড লিক হয়ে গেলে যে কেউ আপনার Access Point (AP) এর সাথে যুক্ত হয়ে ব্যান্ডউইথ (Bandwidth) নষ্ট করতে পারে।

MikroTik RouterOS এ ওয়্যারলেস কানেকশনগুলোকে আরও কঠোরভাবে নিয়ন্ত্রণ করার জন্য Default Authenticate এবং Access List নামের দুটি অত্যন্ত শক্তিশালী ফিচার রয়েছে। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো Wireless Station কী, MAC Address এর ওপর ভিত্তি করে কীভাবে সিকিউরিটি পলিসি তৈরি করতে হয় এবং সিগন্যাল স্ট্রেন্থ (Signal Strength) অনুযায়ী ক্লায়েন্ট ফিল্টারিং কীভাবে কাজ করে।

Real-life Scenario: সিগন্যাল ড্রপ এবং আনঅথরাইজড এক্সেস ঠেকানো

সমস্যা: ধরুন, আপনার একটি ওয়্যারলেস টাওয়ার আছে (Access Point)। আপনার ১০ জন ক্লায়েন্ট (Station) সেই টাওয়ার থেকে ইন্টারনেট নেয়। হঠাৎ একজন ক্লায়েন্ট আপনার পাসওয়ার্ড জেনে গিয়ে অনেক দূর থেকে একটি দুর্বল ডিভাইস দিয়ে কানেক্ট করার চেষ্টা করলো। তার সিগন্যাল খুবই দুর্বল (যেমন -85 dBm)। ওয়্যারলেস নেটওয়ার্কের নিয়ম হলো, কোনো একজন ক্লায়েন্টের সিগন্যাল খারাপ হলে পুরো Access Point এর পারফরম্যান্স এবং স্পিড কমে যায়।

সমাধান: এই পরিস্থিতি এড়ানোর জন্য আমরা রাউটারের Default Authenticate বন্ধ করে দেবো এবং Access List এ একটি রুল তৈরি করবো। রুলটি এমন হবে যে, শুধুমাত্র রেজিস্টার্ড MAC Address গুলোই কানেক্ট হতে পারবে এবং তাদের সিগন্যাল যদি -75 dBm এর নিচে নেমে যায়, তবে রাউটার অটোমেটিক্যালি তাকে কিক (Disconnect) করে দেবে।

১. Wireless Station মোড কী?

MikroTik এ ওয়্যারলেস ইন্টারফেসের বিভিন্ন মোড থাকে। Station মোড মানে হলো "ক্লায়েন্ট" বা "রিসিভার"।

  • যখন আপনি একটি MikroTik রাউটারকে অন্য কোনো Access Point (AP) এর সাথে যুক্ত করতে চান (যেমন মোবাইল ফোন ওয়াই-ফাই এর সাথে যুক্ত হয়), তখন রাউটারের ওয়্যারলেস মোড station সিলেক্ট করতে হয়।
  • Point-to-Point (PTP) লিংকের ক্ষেত্রে এক প্রান্ত থাকে ap bridge মোডে এবং অপর প্রান্ত থাকে station (বা station bridge) মোডে।

২. Default Authenticate এর কাজ কী?

যেকোনো MikroTik ওয়্যারলেস ইন্টারফেসে ডাবল-ক্লিক করলে Wireless ট্যাবের নিচের দিকে Default Authenticate নামের একটি চেকবক্স দেখতে পাবেন। এটি ডিফল্টভাবে টিক (Check) দেওয়া থাকে।

অবস্থা (State) AP Mode এ এর প্রভাব Station Mode এ এর প্রভাব
Checked (টিক দেওয়া) পাসওয়ার্ড জানলে পৃথিবীর যেকোনো ক্লায়েন্ট এই AP এর সাথে কানেক্ট হতে পারবে। রাউটারটি আশেপাশের যেকোনো AP (যার SSID এবং পাসওয়ার্ড মিলবে) এর সাথে কানেক্ট হওয়ার চেষ্টা করবে।
Unchecked (টিক তুলে নেওয়া) কাউকেই কানেক্ট হতে দেবে না, যদি না তার MAC Address রাউটারের Access List এ সেভ করা থাকে। কোনো AP এর সাথেই কানেক্ট হবে না, যদি না সেই AP এর MAC Address রাউটারের Connect List এ থাকে।

৩. Access List কনফিগারেশন (The Ultimate Gatekeeper)

Access List হলো আপনার রাউটারের ভিআইপি (VIP) লিস্ট। Default Authenticate বন্ধ করে দেওয়ার পর, যাদেরকে আপনি এই লিস্টে অ্যাড করবেন, শুধুমাত্র তারাই আপনার নেটওয়ার্কে প্রবেশ করতে পারবে। এটি মূলত MAC Address Filtering এর একটি অ্যাডভান্সড রূপ।

Step 1: Default Authenticate বন্ধ করা

  • Winbox থেকে Wireless মেনুতে যান।
  • WiFi Interfaces ট্যাবে আপনার ইন্টারফেসটির (যেমন: wlan1) ওপর ডাবল-ক্লিক করুন।
  • Wireless ট্যাবে গিয়ে Default Authenticate এর চেকবক্স থেকে টিক চিহ্নটি তুলে দিন।
  • Apply এবং OK দিন। এর ফলে আপনার বর্তমান সব ক্লায়েন্ট সাথে সাথে ডিসকানেক্ট হয়ে যাবে।

Step 2: Access List এ ক্লায়েন্ট অ্যাড করা

  • একই Wireless উইন্ডো থেকে Access List ট্যাবে যান।
  • + (Add) আইকনে ক্লিক করুন।
  • MAC Address: এর ঘরে আপনার বৈধ ক্লায়েন্টের MAC Address দিন (যেমন: 00:11:22:33:44:55)।
  • Interface: আপনার ওয়্যারলেস ইন্টারফেস (wlan1) সিলেক্ট করুন।
  • Signal Strength Range: এটি অত্যন্ত গুরুত্বপূর্ণ। এখানে ডিফল্ট -120..120 থাকে। আপনি এটি পরিবর্তন করে -75..120 করে দিন। এর মানে হলো, ক্লায়েন্টের সিগন্যাল -75 dBm এর চেয়ে খারাপ হলে (যেমন -80) তাকে কানেক্ট হতে দেওয়া হবে না।
  • Authentication: এর চেকবক্সে টিক (Check) দিন। এর মানে আপনি তাকে কানেক্ট হওয়ার অনুমতি দিচ্ছেন।
  • Apply এবং OK দিন।

Step 3: Private Security Profile অ্যাসাইন করা (ঐচ্ছিক)

Access List এর আরেকটি দারুণ ফিচার হলো, আপনি চাইলে একই AP তে ভিন্ন ভিন্ন ক্লায়েন্টকে ভিন্ন ভিন্ন পাসওয়ার্ড দিতে পারেন। Access List এ MAC অ্যাড করার সময় Private Key বা Security Profile অপশন থেকে ওই নির্দিষ্ট ক্লায়েন্টের জন্য আলাদা একটি পাসওয়ার্ড প্রোফাইল সিলেক্ট করে দেওয়া যায়।

৪. Command Line (CLI) ব্যবহার করে কনফিগারেশন

প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়াররা সাধারণত টার্মিনাল দিয়ে এই কাজগুলো দ্রুত করে ফেলেন।

Default Authenticate বন্ধ করা:

[admin@MikroTik] > /interface wireless set wlan1 default-authentication=no

Access List এ নির্দিষ্ট সিগন্যাল রেঞ্জ সহ MAC অ্যাড করা:

[admin@MikroTik] > /interface wireless access-list add mac-address=AA:BB:CC:DD:EE:FF interface=wlan1 signal-range=-75..120 authentication=yes forwarding=no

নোট: forwarding=no মানে হলো এই ক্লায়েন্ট অন্য কোনো ওয়্যারলেস ক্লায়েন্টের সাথে সরাসরি ল্যানে ডাটা আদান-প্রদান করতে পারবে না (Client Isolation), যা WISP এর জন্য ভালো একটি সিকিউরিটি প্র্যাকটিস।

৫. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

  • ক্লায়েন্ট বারবার কানেক্ট এবং ডিসকানেক্ট হচ্ছে: আপনি যদি Access List এ Signal Strength Range সেট করে থাকেন (যেমন -75..120), এবং ক্লায়েন্টের সিগন্যাল যদি বারবার -74 এবং -76 এর মধ্যে ওঠানামা করে, তবে রাউটার তাকে বারবার কিক করবে। এই ক্ষেত্রে সিগন্যাল রেঞ্জটি একটু বাড়িয়ে দিন (যেমন -80..120) অথবা ফিজিক্যাল এলাইনমেন্ট (Alignment) ঠিক করুন।
  • "Authentication timeout" এরর মেসেজ: রাউটারের লগে (Log) যদি এই মেসেজ দেখেন, তবে এর দুটি অর্থ হতে পারে। এক, ক্লায়েন্ট ভুল পাসওয়ার্ড দিচ্ছে। দুই, আপনি Default Authenticate বন্ধ করেছেন কিন্তু ক্লায়েন্টের MAC Address টি Access List এ অ্যাড করতে ভুলে গেছেন অথবা টাইপিংয়ে ভুল করেছেন।
  • Connect List বনাম Access List: মনে রাখবেন, Access List কাজ করে AP Mode এ (যাতে ক্লায়েন্টরা কানেক্ট হতে পারে)। আর Connect List কাজ করে Station Mode এ (যাতে আপনার রাউটারটি নির্দিষ্ট কোনো AP তেই শুধু কানেক্ট হয়)। দুটোর কাজ গুলিয়ে ফেলা যাবে না।

উপসংহার (Conclusion)

একটি প্রফেশনাল এবং স্টেবল Wireless Network তৈরি করতে শুধুমাত্র পাসওয়ার্ডের ওপর নির্ভর করা বোকামি। Default Authenticate বন্ধ করে Access List এর মাধ্যমে হার্ডওয়্যার লেভেলের (MAC Address) সিকিউরিটি নিশ্চিত করা একজন নেটওয়ার্ক অ্যাডমিনিস্ট্রেটর-এর বেস্ট প্র্যাকটিস। বিশেষ করে Signal Strength Range ফিল্টারিং ব্যবহার করে দুর্বল সিগন্যালের ক্লায়েন্টদের নেটওয়ার্ক থেকে দূরে রাখলে আপনার পুরো Access Point এর ওভারঅল স্পিড এবং থ্রুপুট (Throughput) অনেকগুণ বেড়ে যায়।
