Ping, Traceroute and Profile tools on MikroTik

10 June, 2026 Networking with MikroTik Jobaer Hossain

নেটওয়ার্কিংয়ের জগতে একটি প্রবাদ আছে: "Where there is a network, there is a problem." একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ারের প্রতিদিনের কাজের একটা বড় অংশই হলো Troubleshooting বা সমস্যা খুঁজে বের করা। নেটওয়ার্কে কখন কোথায় প্যাকেট ড্রপ হচ্ছে, ইন্টারনেট কেন স্লো, কিংবা রাউটার কেন হ্যাং করছে—এই প্রশ্নের উত্তরগুলো খুঁজে বের করার জন্য আমাদের কিছু ডায়াগনস্টিক টুলস ব্যবহার করতে হয়।

MikroTik RouterOS এ অসংখ্য ট্রাবলশুটিং টুলস থাকলেও, সবচেয়ে বেসিক এবং শক্তিশালী তিনটি টুল হলো Ping, Traceroute এবং Profile। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা এই তিনটি টুলের কাজ, রিয়েল-লাইফ সিনারিও এবং অ্যাডভান্সড লেভেলের ব্যবহার শিখবো।

১. Ping Tool (The Connectivity Checker)

Ping হলো নেটওয়ার্ক দুনিয়ার সবচেয়ে পরিচিত টুল। এটি মূলত ICMP (Internet Control Message Protocol) ব্যবহার করে কাজ করে। এর কাজ হলো একটি নির্দিষ্ট গন্তব্যে (Destination IP) একটি Echo Request প্যাকেট পাঠানো এবং সেখান থেকে Echo Reply আসার সময়টুকু (Latency) হিসাব করা।

রিয়েল-লাইফ সিনারিও (Ping)

সমস্যা: কাস্টমার কল করে বললো, "আমার ইন্টারনেট চলছে না।"

ট্রাবলশুটিং: আপনি প্রথমে রাউটার থেকে 8.8.8.8 (Google DNS) এ পিং করবেন।

  • যদি Timeout আসে, তার মানে আপনার আইএসপি (ISP) থেকে লাইন ডাউন।
  • যদি রিপ্লাই আসে, কিন্তু Time (Latency) অনেক বেশি (যেমন 300ms) হয়, তবে বুঝতে হবে লিংকে প্রচুর জ্যাম (Congestion) আছে বা রাউটার ওভারলোডেড।
  • যদি রিপ্লাই ঠিক থাকে, তখন Advanced Ping ব্যবহার করে রাউটারের src-address হিসেবে কাস্টমারের লোকাল আইপি দিয়ে পিং করতে হবে। এতে বোঝা যাবে কাস্টমারের ল্যান থেকে ডাটা ইন্টারনেটে যেতে পারছে কি না (NAT বা Firewall ইস্যু আছে কি না)।

Advanced Ping প্যারামিটারসমূহ:

  • Size: ডিফল্ট প্যাকেট সাইজ ৫৬ বাইট থাকে। MTU সমস্যা (যেমন পিপিইও কানেকশনে ওয়েবসাইট লোড না হওয়া) চেক করার জন্য বড় সাইজের (যেমন 1400 বা 1500) প্যাকেট দিয়ে পিং করতে হয়।
  • Src. Address: রাউটারে একাধিক আইএসপি (Dual WAN) থাকলে, নির্দিষ্ট কোনো আইএসপির লাইন চেক করতে ওই পোর্টের আইপিটি সোর্স হিসেবে ব্যবহার করতে হয়।

CLI কমান্ড:

[admin@MikroTik] > /ping 8.8.8.8 count=5
[admin@MikroTik] > /ping 8.8.8.8 src-address=192.168.10.1 size=1500

২. Traceroute Tool (The Path Finder)

Ping শুধু বলে গন্তব্যে পৌঁছানো যাচ্ছে কি না। কিন্তু Traceroute বলে দেয় সেই গন্তব্যে পৌঁছাতে রাস্তার মাঝে কতগুলো রাউটার (Hop) পার হতে হয়েছে এবং কোন রাউটারে গিয়ে প্যাকেট আটকে যাচ্ছে বা দেরি হচ্ছে। এটি প্যাকেটের TTL (Time To Live) ভ্যালু পরিবর্তন করে করে এই ম্যাপটি তৈরি করে।

রিয়েল-লাইফ সিনারিও (Traceroute)

সমস্যা: গেমিং ইউজাররা বলছে সিঙ্গাপুর সার্ভারে পিং অনেক বেশি (High Latency) বা প্যাকেট লস হচ্ছে।

ট্রাবলশুটিং: আপনি সিঙ্গাপুর গেম সার্ভারের আইপিতে Traceroute চালালেন। রেজাল্টে দেখলেন আপনার রাউটার থেকে আইএসপি পর্যন্ত পিং (1ms) ঠিক আছে। আইএসপি থেকে আইআইজি (IIG) পর্যন্ত (5ms) ঠিক আছে। কিন্তু আইআইজি থেকে যখন সাবমেরিন ক্যাবল (IIG to Upstream) হয়ে যাচ্ছে, তখন ৪ নম্বর হপ (Hop)-এ পিং 150ms হয়ে যাচ্ছে বা প্যাকেট ড্রপ (Timeout) দেখাচ্ছে। এর মানে সমস্যা আপনার রাউটারে নয়, সমস্যা দেশের বাইরের লিংকে।

CLI কমান্ড:

[admin@MikroTik] > /tool traceroute 8.8.8.8

নোট: Winbox এ Tools > Traceroute মেনু থেকে এটি ব্যবহার করা আরও সহজ।

৩. Profile Tool (The CPU Detective)

নেটওয়ার্কের সমস্যা সব সময় তারের বা আইএসপির হয় না। অনেক সময় রাউটার নিজেই নিজের লোড নিতে পারে না। রাউটারের CPU 100% হয়ে গেলে পুরো নেটওয়ার্ক হ্যাং করে। কিন্তু কোন প্রসেসের কারণে CPU 100% হলো তা বের করার একমাত্র উপায় হলো Profile Tool।

এটি উইন্ডোজের Task Manager এর মতো কাজ করে। এটি রিয়েল-টাইমে দেখায় রাউটারের কোন ফিচারটি (যেমন: Firewall, Routing, Management) কতটুকু CPU ব্যবহার করছে।

Profile Process Name কখন লোড বেশি হয়? (Reason for High Load) সমাধান (Solution)
Firewall রাউটারে যদি প্রচুর Filter Rule, Mangle Rule বা Layer 7 Protocol থাকে। অথবা হ্যাকাররা যদি DDoS Attack করে। FastTrack এনাবল করুন। Connection Tracking এর Timeout কমান। RAW টেবিল ব্যবহার করে অ্যাটাকার আইপি ব্লক করুন।
Networking / Ethernet ল্যানে যদি কোনো லুপ (Network Loop) তৈরি হয় বা প্রচুর ব্রডকাস্ট স্টর্ম (Broadcast Storm) হয়। ল্যানের সুইচে লুপ চেক করুন। Bridge সেটিংসে গিয়ে (R)STP (Rapid Spanning Tree Protocol) চালু করুন।
Queuing অনেক বেশি Simple Queue থাকলে বা পিসি কিউ (PCQ) মেমোরি ফুল হয়ে গেলে। অপ্রয়োজনীয় Queue মুছে ফেলুন। মাল্টিপল ইউজারের জন্য PCQ ব্যবহার করে Queue এর সংখ্যা কমিয়ে আনুন।
Management কেউ যদি Winbox, SSH বা API দিয়ে ক্রমাগত লগইন করার চেষ্টা করে (Brute-force Attack)। Firewall দিয়ে নির্দিষ্ট আইপি ছাড়া বাকি সবার জন্য Winbox/SSH পোর্ট ব্লক করে দিন।

CLI কমান্ড:

[admin@MikroTik] > /tool profile

(এটি রান করার পর q চেপে বের হতে হবে)।

৪. Troubleshooting Tips (সাধারণ সমস্যা ও সতর্কতা)

  • Ping কাজ না করা (Request Timed Out): অনেক ওয়েবসাইট বা সার্ভার (এমনকি কিছু আইএসপি রাউটার) নিরাপত্তার স্বার্থে তাদের ফায়ারওয়ালে ICMP (Ping) ব্লক করে রাখে। তাই কোনো সার্ভারে পিং না পাওয়ার মানেই এই নয় যে সার্ভারটি ডাউন। এক্ষেত্রে TCP Traceroute বা নির্দিষ্ট পোর্টে টেলনেট (Telnet) করে চেক করা ভালো।
  • Traceroute এ * * * (Asterisks) দেখা: Traceroute করার সময় মাঝপথে কোনো হপ (Hop) যদি * * * দেখায়, তার মানে হলো ওই রাউটারটি ICMP TTL Expired মেসেজটি ব্লক করে রেখেছে। এটি একটি সাধারণ সিকিউরিটি প্র্যাকটিস, এতে ভয়ের কিছু নেই। যদি এর পরের হপগুলোতে রিপ্লাই আসে, তবে বুঝতে হবে ডাটা ঠিকই পাস হচ্ছে।
  • Profile টুলে Unclassified লোড: মাঝে মাঝে Profile টুলে unclassified প্রসেসে অনেক লোড দেখা যায়। এটি সাধারণত রাউটারওএস এর কোনো বাগ (Bug) বা হার্ডওয়্যার ফল্টের কারণে হয়। রাউটার রিবুট (Reboot) বা ফার্মওয়্যার আপগ্রেড (Upgrade) করলে এটি ঠিক হয়ে যায়।

উপসংহার (Conclusion)

একজন এন্ট্রি-লেভেল টেকনিশিয়ান এবং একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ারের মধ্যে প্রধান পার্থক্য হলো Troubleshooting Skill। যখন নেটওয়ার্ক ডাউন হয়, তখন অনুমান করে কাজ না করে Ping দিয়ে কানেক্টিভিটি চেক করা, Traceroute দিয়ে ফল্ট পয়েন্ট (Fault Point) খুঁজে বের করা এবং Profile টুল দিয়ে রাউটারের ভেতরের স্বাস্থ্য (Health) পর্যবেক্ষণ করাটাই হলো সঠিক পদ্ধতি। এই তিনটি টুলের সঠিক ব্যবহার আপনাকে যেকোনো জটিল নেটওয়ার্ক সমস্যা খুব দ্রুত এবং নির্ভুলভাবে সমাধান করতে সাহায্য করবে।
