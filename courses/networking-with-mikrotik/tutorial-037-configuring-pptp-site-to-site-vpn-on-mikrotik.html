Configuring PPTP Site-to-Site VPN on MikroTik

10 December, 2026 VPN & Cyber Security Jobaer Hossain

যখন কোনো কোম্পানির একাধিক শাখা বা অফিস থাকে, তখন এক অফিসের ইউজারদের অন্য অফিসের ফাইল সার্ভার, প্রিন্টার বা ইআরপি (ERP) সফটওয়্যারে এক্সেস করার প্রয়োজন হয়। ইন্টারনেট ব্যবহার করে দুটি ভিন্ন ভৌগলিক অবস্থানে থাকা লোকাল নেটওয়ার্ককে (LAN) একটি সিঙ্গেল প্রাইভেট নেটওয়ার্কে পরিণত করার পদ্ধতিকেই Site-to-Site VPN বলা হয়।

MikroTik RouterOS ব্যবহার করে খুব সহজেই PPTP (Point-to-Point Tunneling Protocol) এর মাধ্যমে Site-to-Site VPN তৈরি করা যায়। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা একটি রিয়েল-লাইফ সিনারিও ধরে হেড অফিস (Head Office) এবং ব্রাঞ্চ অফিসের (Branch Office) মধ্যে PPTP টানেল এবং রাউটিং কনফিগার করা শিখবো।

রিয়েল-লাইফ সিনারিও এবং আইপি প্ল্যানিং

আমাদের নেটওয়ার্কের আর্কিটেকচারটি হবে নিচের মতো:

  • Head Office (HQ):
    • Public IP (WAN): 203.0.113.1 (অবশ্যই একটি স্ট্যাটিক রিয়েল আইপি থাকতে হবে)।
    • Local Network (LAN): 192.168.10.0/24
    • PPTP Tunnel IP: 10.10.10.1
  • Branch Office:
    • Public IP (WAN): রিয়েল আইপি থাকার প্রয়োজন নেই (যেকোনো শেয়ার্ড ইন্টারনেট হলেই হবে)।
    • Local Network (LAN): 192.168.20.0/24
    • PPTP Tunnel IP: 10.10.10.2

আমাদের লক্ষ্য: ব্রাঞ্চ অফিসের 192.168.20.x ব্লকের পিসিগুলো যেন সরাসরি হেড অফিসের 192.168.10.x ব্লকের সার্ভারকে পিং (Ping) করতে পারে।

১. Head Office রাউটার কনফিগারেশন (PPTP Server)

প্রথমে আমাদের হেড অফিসের রাউটারটিকে একটি PPTP Server হিসেবে তৈরি করতে হবে, যাতে ব্রাঞ্চ অফিস তার সাথে কানেক্ট হতে পারে।

Step 1: PPTP Server চালু করা

  • Winbox থেকে PPP মেনুতে যান।
  • Interface ট্যাবে থাকা অবস্থায় উপরের PPTP Server বাটনে ক্লিক করুন।
  • Enabled চেকবক্সে টিক দিন এবং OK করুন।

Step 2: PPP Secret (ইউজার) তৈরি করা

  • একই PPP উইন্ডো থেকে Secrets ট্যাবে যান এবং + (Add) আইকনে ক্লিক করুন।
  • Name: branch1 দিন (এটি ব্রাঞ্চ অফিসের ইউজারনেম)।
  • Password: একটি শক্তিশালী পাসওয়ার্ড দিন (যেমন: Secret@Branch!)।
  • Service: pptp সিলেক্ট করুন।
  • Local Address: 10.10.10.1 দিন (হেড অফিসের টানেল আইপি)।
  • Remote Address: 10.10.10.2 দিন (ব্রাঞ্চ অফিসের টানেল আইপি)।
  • Apply এবং OK দিন।

Step 3: Branch LAN এর জন্য Static Route তৈরি করা

হেড অফিসকে চিনিয়ে দিতে হবে যে, ব্রাঞ্চের লোকাল নেটওয়ার্কে যাওয়ার রাস্তা হলো এই PPTP টানেলটি।

  • IP > Routes এ যান এবং + (Add) ক্লিক করুন।
  • Dst. Address: 192.168.20.0/24 (ব্রাঞ্চের ল্যান ব্লক) দিন।
  • Gateway: 10.10.10.2 (ব্রাঞ্চের টানেল আইপি) দিন।
  • Apply এবং OK দিন।

২. Branch Office রাউটার কনফিগারেশন (PPTP Client)

এবার আমরা ব্রাঞ্চ অফিসের রাউটারে লগইন করবো এবং হেড অফিসের সাথে কানেক্ট হবো।

Step 1: PPTP Client তৈরি করা

  • Winbox থেকে PPP মেনুতে যান।
  • + (Add) আইকনে ক্লিক করে ড্রপডাউন থেকে PPTP Client সিলেক্ট করুন।
  • General ট্যাবে Name: pptp-to-hq দিন।
  • Dial Out ট্যাবে যান।
  • Connect To: 203.0.113.1 (হেড অফিসের পাবলিক আইপি) দিন।
  • User: branch1 দিন (যা হেড অফিসে তৈরি করেছেন)।
  • Password: Secret@Branch! দিন।
  • Apply দিন। স্ট্যাটাস বারে connected লেখা উঠলে বুঝবেন টানেল সফলভাবে তৈরি হয়েছে।

Step 2: Head Office LAN এর জন্য Static Route তৈরি করা

ব্রাঞ্চ রাউটারকেও চিনিয়ে দিতে হবে যে হেড অফিসে যাওয়ার রাস্তা কোনটি।

  • IP > Routes এ যান এবং + (Add) ক্লিক করুন।
  • Dst. Address: 192.168.10.0/24 (হেড অফিসের ল্যান ব্লক) দিন।
  • Gateway: 10.10.10.1 (অথবা pptp-to-hq ইন্টারফেস) দিন।
  • Apply এবং OK দিন।

৩. Firewall কনফিগারেশন (অত্যন্ত গুরুত্বপূর্ণ)

হেড অফিসের রাউটারে যদি ফায়ারওয়াল কনফিগার করা থাকে, তবে বাই-ডিফল্ট PPTP কানেকশন ব্লক হয়ে যাবে। হেড অফিসের IP > Firewall > Filter Rules এ গিয়ে নিচের রুলগুলো সবার ওপরে অ্যাড করতে হবে:

[admin@HQ-Router] > /ip firewall filter add chain=input action=accept protocol=tcp dst-port=1723 comment="Allow PPTP Control Channel"
[admin@HQ-Router] > /ip firewall filter add chain=input action=accept protocol=gre comment="Allow GRE (Protocol 47) for PPTP"

৪. Command Line (CLI) সামারি

পুরো কনফিগারেশনটি টার্মিনাল দিয়ে খুব দ্রুত করা যায়:

Head Office (HQ) Commands:

/interface pptp-server server set enabled=yes
/ppp secret add name=branch1 password=Secret@Branch! service=pptp local-address=10.10.10.1 remote-address=10.10.10.2
/ip route add dst-address=192.168.20.0/24 gateway=10.10.10.2

Branch Office Commands:

/interface pptp-client add name=pptp-to-hq connect-to=203.0.113.1 user=branch1 password=Secret@Branch! disabled=no
/ip route add dst-address=192.168.10.0/24 gateway=10.10.10.1

৫. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

সমস্যা (Problem) কারণ এবং সমাধান (Cause & Solution)
টানেল Connected, কিন্তু ল্যান পিসি পিং পাচ্ছে না প্রথমত, দুই রাউটারের Static Route চেক করুন। দ্বিতীয়ত, পিসির Windows Firewall ডিফল্টভাবে অন্য সাবনেটের (Subnet) পিং ব্লক করে রাখে। উইন্ডোজ ফায়ারওয়াল সাময়িকভাবে বন্ধ করে চেক করুন।
Branch থেকে Head Office এ কানেক্ট হচ্ছে না (Timeout) হেড অফিসের আইএসপি (ISP) হয়তো পোর্ট 1723 অথবা GRE প্রোটোকল ব্লক করে রেখেছে। আইএসপির সাথে কথা বলে পোর্ট ওপেন করে নিন।
NAT (Masquerade) কনফ্লিক্ট যদি আপনার রাউটারে srcnat চেইনে Out. Interface নির্দিষ্ট করা না থাকে, তবে ভিপিএন ট্রাফিকও মাস্ক্যুরেড হয়ে যায়। IP > Firewall > NAT এ গিয়ে আপনার ইন্টারনেট মাস্ক্যুরেড রুলে Out. Interface = ether1 (WAN) নির্দিষ্ট করে দিন।
টানেল বারবার ড্রপ করছে এটি সাধারণত দুটি রাউটারের মধ্যে ইন্টারনেটের প্যাকেট লস (Packet Loss) বা MTU (Maximum Transmission Unit) সমস্যার কারণে হয়। PPP Profile এ গিয়ে MTU 1400 করে দেখতে পারেন।

উপসংহার (Conclusion)

Site-to-Site PPTP VPN কনফিগারেশন করা অত্যন্ত সহজ এবং এটি ছোট বা মাঝারি সাইজের কর্পোরেট নেটওয়ার্কের জন্য একটি চমৎকার সমাধান। PPTP Server, PPP Secret এবং Static Routing এর সঠিক সমন্বয় ঘটিয়ে আপনি মুহূর্তের মধ্যেই দুটি আলাদা অফিসকে এক সুতায় বাঁধতে পারেন। তবে মনে রাখবেন, PPTP এর এনক্রিপশন আধুনিক হ্যাকিং স্ট্যান্ডার্ডে কিছুটা দুর্বল। যদি আপনার কোম্পানির ডাটা অত্যন্ত কনফিডেন্সিয়াল হয়, তবে PPTP এর বদলে IPsec বা WireGuard ব্যবহার করা একটি প্রফেশনাল বেস্ট প্র্যাকটিস।
