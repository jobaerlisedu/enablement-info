Interface Monitor, Torch tool and Graphing on MikroTik

24 April, 2026 Networking with MikroTik Jobaer Hossain

আপনার নেটওয়ার্কের স্পিড স্লো হয়ে গেলে প্রথম প্রশ্নটি কী আসে? নিশ্চয়ই—"ব্যান্ডউইথ কে খাচ্ছে?" অথবা "কোন পোর্ট দিয়ে কতটুকু ট্রাফিক যাচ্ছে?"। MikroTik RouterOS এই প্রশ্নের উত্তর দেওয়ার জন্য অত্যন্ত শক্তিশালী তিনটি মনিটরিং ফিচার দিয়ে রেখেছে: Interface Monitor, Torch Tool এবং Graphing।

এই টুলগুলো ব্যবহার করে আপনি শুধুমাত্র রিয়েল-টাইম ট্রাফিকই দেখতে পাবেন না, বরং দীর্ঘমেয়াদী ট্রাফিক হিস্ট্রি পর্যবেক্ষণ করে নেটওয়ার্কের ভবিষ্যতের পরিকল্পনাও করতে পারবেন। আজকের এই বিস্তারিত গাইডে আমরা শিখবো কীভাবে এই টুলগুলো প্রফেশনালি ব্যবহার করতে হয়।

১. Interface Monitor (Real-time Speed Tracker)

Interface Monitor হলো রাউটারের সবচেয়ে বেসিক মনিটরিং সিস্টেম। এটি আপনাকে রিয়েল-টাইমে দেখায় যে আপনার রাউটারের ফিজিক্যাল পোর্ট (যেমন ether1) বা ভার্চুয়াল ইন্টারফেস (যেমন VLAN বা PPPoE) দিয়ে ঠিক এই মুহূর্তে কত Mbps ডাটা আপলোড এবং ডাউনলোড হচ্ছে।

কিভাবে ব্যবহার করবেন (Winbox):

  • Winbox-এর বাম দিকের মেনু থেকে Interfaces এ ক্লিক করুন।
  • ইন্টারফেস লিস্ট থেকে আপনার কাঙ্ক্ষিত ইন্টারফেসে ডাবল-ক্লিক করুন।
  • General ট্যাবে থাকা অবস্থায় আপনি নিচে Tx (Transmit/Upload) এবং Rx (Receive/Download) রেট দেখতে পাবেন।
  • আরও বিস্তারিত দেখতে Status ট্যাবে ক্লিক করুন। এখানে আপনি রানিং স্পিড ছাড়াও মোট কত ডাটা (Bytes) আদান-প্রদান হয়েছে তা দেখতে পাবেন।

CLI কমান্ড:

/interface monitor-traffic ether1

ব্যাখ্যা: এটি টার্মিনালে ইন্টারফেসের ট্রাফিক আপডেট রিয়েল-টাইমে দেখাতে থাকবে।

২. Torch Tool (The Packet Investigator)

Interface Monitor শুধু বলে কতটুকু স্পিড যাচ্ছে, কিন্তু Torch Tool বলে দেয় সেই স্পিডটি ঠিক কারা ব্যবহার করছে। এটি মূলত একটি রিয়েল-টাইম প্যাকেট অ্যানালাইজার। এর মাধ্যমে আপনি দেখতে পাবেন কোন Source IP থেকে কোন Destination IP-তে ট্রাফিক যাচ্ছে এবং তারা কোন Protocol বা Port ব্যবহার করছে।

রিয়েল-লাইফ সিনারিও (The Bandwidth Hog)

সমস্যা: আপনার ইন্টারনেটে 50 Mbps লাইন আছে। ইন্টারফেস মনিটরে দেখছেন 49 Mbps ট্রাফিক ব্যবহার হচ্ছে। আপনি জানতে চান কে এই ব্যান্ডউইথ টেনে নিচ্ছে।

সমাধান: আপনি ল্যান পোর্টের ওপর Torch চালালেন। রেজাল্টে দেখলেন 192.168.10.55 আইপিটি ইউটিউব (Port 443) থেকে একা 40 Mbps ডাটা নিচ্ছে। এখন আপনি চাইলে তাকে ব্লক বা লিমিট করতে পারেন।

Torch-এর প্রধান ফিল্টারসমূহ:

  • Entry Timeout: কতক্ষণ নিষ্ক্রিয় থাকলে একটি এন্ট্রি লিস্ট থেকে মুছে যাবে।
  • Src. Address: নির্দিষ্ট কোনো ইউজারের ট্রাফিক ট্র্যাক করার জন্য।
  • Protocol: শুধু TCP, UDP বা ICMP ট্রাফিক ফিল্টার করার জন্য।
  • Port: নির্দিষ্ট কোনো সার্ভিস (যেমন DNS বা HTTP) মনিটর করার জন্য।

CLI কমান্ড:

/tool torch interface=ether1 port=80,443 src-address=192.168.88.0/24

৩. Graphing Tool (The History Keeper)

Interface Monitor এবং Torch হলো রিয়েল-টাইম। অর্থাৎ আপনি স্ক্রিন থেকে সরে গেলে আপনি জানবেন না ২ ঘণ্টা আগে কী ঘটেছিল। Graphing Tool হলো রাউটারের সিসিটিভি ক্যামেরার মতো। এটি নির্দিষ্ট সময় পর পর ট্রাফিক এবং রিসোর্স (CPU/RAM) ডেটা রেকর্ড করে গ্রাফ আকারে সাজিয়ে রাখে, যা আপনি ওয়েব ব্রাউজার দিয়ে দেখতে পারেন।

কিভাবে কনফিগার করবেন:

  1. Winbox থেকে Tools > Graphing এ যান।
  2. Interface Rules ট্যাবে গিয়ে + (Add) ক্লিক করুন।
  3. Interface: all বা নির্দিষ্ট ইন্টারফেস সিলেক্ট করুন। Store on Disk এ টিক দিন (যাতে রাউটার রিবুট হলেও ডেটা না হারায়)। OK দিন।
  4. Resource Rules ট্যাবে গিয়ে একইভাবে CPU এবং RAM মনিটরিং অ্যাড করুন।

কিভাবে গ্রাফ দেখবেন:

আপনার ব্রাউজারে গিয়ে রাউটারের আইপি লিখুন এবং শেষে /graphs যোগ করুন। উদাহরণ: http://192.168.88.1/graphs। এখানে আপনি ৫ মিনিট, দিন, সপ্তাহ এবং মাসের ট্রাফিক চার্ট দেখতে পাবেন।

টুল (Tool) কখন ব্যবহার করবেন (Best Usage)
Interface Monitor যখন শুধু ইন্টারনেটের মোট স্পিড চেক করা প্রয়োজন।
Torch যখন নির্দিষ্ট আইপি বা অ্যাপ্লিকেশন ট্রাবলশুট করা প্রয়োজন।
Graphing যখন মাসিক ব্যান্ডউইথ রিপোর্ট বা হিস্ট্রি দেখা প্রয়োজন।

৪. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

  • Torch-এ কোনো ডেটা দেখাচ্ছে না: আপনি যদি ভুল ইন্টারফেসে Torch চালান (যেমন ল্যান পোর্টে না চালিয়ে ওয়ান পোর্টে চালানো), তবে লোকাল আইপিগুলো সহজে দেখা যায় না। সবসময় যেখান থেকে ট্রাফিক জেনারেট হচ্ছে সেই ইন্টারফেসে Torch চালান।
  • Graphing চালু করলে CPU বেড়ে যাওয়া: আপনার রাউটার যদি খুব পুরনো বা লো-এন্ড মডেলের হয়, তবে সব ইন্টারফেসের জন্য গ্রাফিং চালু করলে CPU লোড বেড়ে যেতে পারে। শুধু গুরুত্বপূর্ণ ওয়ান (WAN) পোর্টের জন্য গ্রাফিং অন রাখুন।
  • Graphing-এ পুরনো ডেটা নেই: যদি আপনি Store on Disk এনাবল না করেন, তবে রাউটার রিবুট হলে সব পুরনো গ্রাফ ডেটা মুছে যাবে।

উপসংহার (Conclusion)

একজন দক্ষ নেটওয়ার্ক ইঞ্জিনিয়ারের চোখ সব সময় ডেটার ওপর থাকে। Interface Monitor আপনাকে বর্তমান অবস্থা জানায়, Torch আপনাকে সমস্যার মূলে নিয়ে যায় এবং Graphing আপনাকে ভবিষ্যতের পরিকল্পনা করতে সাহায্য করে। এই তিনটি টুলের সঠিক ব্যবহার আপনার MikroTik ম্যানেজমেন্টকে অনেক বেশি প্রফেশনাল এবং সহজ করে তুলবে।
