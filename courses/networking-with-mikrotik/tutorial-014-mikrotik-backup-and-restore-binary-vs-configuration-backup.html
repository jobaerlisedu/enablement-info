MikroTik Backup and Restore (Binary vs Configuration backup)

10 August, 2026 Networking with MikroTik Jobaer Hossain

নেটওয়ার্ক ইঞ্জিনিয়ারিংয়ে একটি প্রবাদ আছে, "যে ডাটার কোনো ব্যাকআপ নেই, সেই ডাটার অস্তিত্বই নেই।" একটি কর্পোরেট অফিস বা ডেটা সেন্টারের মূল MikroTik Router যদি কোনো কারণে পুড়ে যায় বা কনফিগারেশন ক্র্যাশ করে, তবে পুরো নেটওয়ার্ক রিস্টোর করতে কয়েক দিন সময় লেগে যেতে পারে—যদি আপনার কাছে সঠিক ব্যাকআপ না থাকে।

MikroTik RouterOS-এ কনফিগারেশন সেভ করে রাখার জন্য মূলত দুটি পদ্ধতি রয়েছে: Binary Backup (.backup) এবং Configuration Export (.rsc)। এই দুটি পদ্ধতি সম্পূর্ণ ভিন্নভাবে কাজ করে এবং এদের ব্যবহারের ক্ষেত্রও আলাদা। আজকের এই ডিপ-ডাইভ গাইডে আমরা শিখবো এই দুটি পদ্ধতির পার্থক্য, কীভাবে এগুলো তৈরি করতে হয় এবং কোন পরিস্থিতিতে কোনটি ব্যবহার করতে হয়।

Binary Backup বনাম Configuration Export: মূল পার্থক্য

Binary Backup: এটি হলো রাউটারের একটি কমপ্লিট স্ন্যাপশট (Snapshot)। এর মধ্যে রাউটারের পাসওয়ার্ড, ইউজারনেম এবং ফিজিক্যাল পোর্টের MAC Address সহ সবকিছু এনক্রিপ্টেড (Encrypted) অবস্থায় সেভ হয়। এটি শুধু একই মডেলের রাউটারে রিস্টোর করার জন্য ডিজাইন করা।

Configuration Export: এটি একটি হিউম্যান-রিডেবল টেক্সট ফাইল (Text file)। এটিতে শুধু আপনার দেওয়া কমান্ডগুলো (যেমন আইপি বসানো, রুল তৈরি করা) স্ক্রিপ্ট বা কোড আকারে সেভ থাকে। এটি যেকোনো টেক্সট এডিটর (যেমন Notepad) দিয়ে পড়া যায় এবং অন্য যেকোনো মডেলের MikroTik রাউটারে ইম্পোর্ট করা যায়।

১. Binary Backup (.backup) তৈরি এবং রিস্টোর করা

যখন আপনি নিশ্চিত যে রাউটার নষ্ট হলে আপনি ঠিক একই মডেলের (Same hardware) আরেকটি রাউটার কিনবেন, তখন Binary Backup সবচেয়ে কার্যকর।

Backup তৈরি করার পদ্ধতি (Winbox ও CLI):

  • Winbox দিয়ে: বাম পাশের মেনু থেকে Files এ ক্লিক করুন। উপরের Backup বাটনে ক্লিক করুন। Name এর ঘরে একটি নাম দিন এবং প্রয়োজনে একটি Password দিন (ফাইলটি এনক্রিপ্ট করার জন্য)। সবশেষে Backup এ ক্লিক করুন। ফাইলটি লিস্টে .backup এক্সটেনশন নিয়ে সেভ হবে। এরপর ফাইলটি ড্র্যাগ করে (Drag and Drop) আপনার পিসিতে সেভ করে রাখুন।
  • CLI দিয়ে: টার্মিনালে নিচের কমান্ডটি দিন: 
    [admin@MikroTik] > /system backup save name="Office_Backup_10Aug" password="securepass"

Restore করার পদ্ধতি:

  • Winbox এর Files মেনুতে গিয়ে আপনার পিসি থেকে ব্যাকআপ ফাইলটি ড্র্যাগ করে আপলোড করুন।
  • ফাইলটি সিলেক্ট করে উপরের Restore বাটনে ক্লিক করুন। পাসওয়ার্ড দেওয়া থাকলে সেটি দিন এবং Restore এ ক্লিক করুন।
  • রাউটার অটোমেটিক রিবুট (Reboot) নেবে এবং আগের অবস্থায় ফিরে আসবে।
  • CLI কমান্ড: /system backup load name="Office_Backup_10Aug.backup" password="securepass"

২. Configuration Export (.rsc) তৈরি এবং রিস্টোর করা

আপনি যদি একটি পুরনো ছোট রাউটার (যেমন RB750Gr3) থেকে আপনার নেটওয়ার্কটিকে একটি বড় শক্তিশালী রাউটারে (যেমন CCR2004) শিফট বা আপগ্রেড করতে চান, তবে আপনাকে অবশ্যই Export বা .rsc ফাইল ব্যবহার করতে হবে।

Export তৈরি করার পদ্ধতি:

Export সাধারণত টার্মিনাল থেকেই করতে হয়। Winbox এর New Terminal ওপেন করুন।

  • পুরো রাউটারের কনফিগারেশন এক্সপোর্ট করতে: 
    [admin@MikroTik] > /export file="Full_Config_10Aug"
  • নির্দিষ্ট কোনো অংশের কনফিগারেশন এক্সপোর্ট করতে: (ধরুন আপনি শুধু ফায়ারওয়াল রুলগুলো নিতে চাচ্ছেন) 
    [admin@MikroTik] > /ip firewall export file="Firewall_Rules"

কমান্ড দেওয়ার পর Files মেনুতে গিয়ে দেখবেন Full_Config_10Aug.rsc নামে একটি ফাইল তৈরি হয়েছে। ফাইলটি আপনার পিসিতে কপি করে নিন। এটি Notepad দিয়ে ওপেন করলে সব কোড দেখতে পাবেন।

Import বা রিস্টোর করার পদ্ধতি:

  • নতুন রাউটারের Files মেনুতে .rsc ফাইলটি আপলোড করুন।
  • New Terminal ওপেন করে নিচের কমান্ডটি দিন: 
    [admin@MikroTik] > /import file="Full_Config_10Aug.rsc"
  • স্ক্রিনে Script file loaded and executed successfully দেখালে বুঝবেন কনফিগারেশন সফলভাবে রিস্টোর হয়েছে।

৩. Binary vs Export: তুলনামূলক পার্থক্য

বৈশিষ্ট্য (Feature) Binary Backup (.backup) Configuration Export (.rsc)
ফাইলের ধরন এনক্রিপ্টেড বাইনারি ফাইল (যা মানুষ পড়তে পারে না)। প্লেইন টেক্সট স্ক্রিপ্ট (যা Notepad দিয়ে পড়া ও এডিট করা যায়)।
MAC Address আগের রাউটারের MAC Address সহ সেভ হয়। MAC Address সেভ হয় না (যা অন্য রাউটারে নেওয়ার জন্য ভালো)।
Passwords & Users সমস্ত ইউজারের পাসওয়ার্ড সেভ করে। ইউজার একাউন্টের পাসওয়ার্ড সেভ করে না (নিরাপত্তার স্বার্থে)।
হার্ডওয়্যার ডিপেন্ডেন্সি শুধুমাত্র একই মডেলের রাউটারে রিস্টোর করা উচিত। যেকোনো মডেলের MikroTik রাউটারে ইম্পোর্ট করা যায়।

৪. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

ব্যাকআপ ও রিস্টোর করার সময় কিছু পরিচিত এরর আসতে পারে:

  • ভিন্ন মডেলে Binary Restore করা (MAC Conflict): আপনি যদি একটি RB3011 এর .backup ফাইল একটি RB4011 এ রিস্টোর করেন, তবে নতুন রাউটারের পোর্টগুলো আগের রাউটারের MAC Address ধারণ করবে। ফলে নেটওয়ার্কে MAC Conflict বা লুপ তৈরি হবে। তাই ভিন্ন মডেলে কখনোই Binary Restore করবেন না।
  • .rsc ফাইল Import এ ফেইল করা: অনেক সময় /import করার সময় স্ক্রিপ্ট মাঝপথে আটকে যায় (যেমন লাইন নাম্বার ৩৫ এ এরর)। এটি হয় যদি ওই কমান্ডের সাথে সম্পর্কিত কোনো ইন্টারফেস (যেমন pppoe-out1) নতুন রাউটারে না থাকে। সমাধান: পিসির Notepad দিয়ে .rsc ফাইলটি ওপেন করুন, এরর হওয়া লাইনটি খুঁজে এডিট করুন এবং তারপর টার্মিনালে কোডগুলো কপি-পেস্ট করুন।
  • Passwords Missing in Export: .rsc ফাইলে নিরাপত্তার কারণে রাউটারের ইউজার পাসওয়ার্ড এক্সপোর্ট হয় না। তাই নতুন রাউটারে স্ক্রিপ্ট ইম্পোর্ট করার পর আপনাকে নিজে থেকে নতুন করে অ্যাডমিন পাসওয়ার্ড এবং ইউজার গ্রুপ তৈরি করে নিতে হবে।

উপসংহার (Conclusion)

আপনার নেটওয়ার্কের সাইজ যেমনই হোক না কেন, Backup একটি আবশ্যিক রুটিন হওয়া উচিত। একই ডিভাইসের রেগুলার ব্যাকআপ বা হার্ডওয়্যার রিপ্লেসমেন্টের জন্য Binary Backup (.backup) ব্যবহার করুন। অন্যদিকে, আপনি যদি রাউটার আপগ্রেড করেন, অন্য মডেলে শিফট করেন, অথবা কনফিগারেশন টেমপ্লেট হিসেবে অন্য ব্রাঞ্চে ব্যবহার করতে চান, তবে আপনার একমাত্র ভরসা হলো Configuration Export (.rsc)। এই দুটি মেকানিজমের সঠিক ব্যবহার আপনাকে যেকোনো হার্ডওয়্যার ফেইলিওর বা নেটওয়ার্ক ক্র্যাশ থেকে দ্রুততম সময়ে রিকভার করতে সাহায্য করবে।
