Make MikroTik router a DNS Forwarder and redirect DNS queries

15 November, 2026 Networking with MikroTik Jobaer Hossain

যেকোনো নেটওয়ার্কে যখন আমরা কোনো ওয়েবসাইট ব্রাউজ করি, তখন ব্রাউজার সেই সাইটের নামকে (Domain Name) আইপি অ্যাড্রেসে (IP Address) কনভার্ট করার জন্য DNS (Domain Name System) সার্ভারের সাহায্য নেয়। আপনি যদি একটি অফিস বা আইএসপি (ISP) নেটওয়ার্ক চালান, তবে ইউজারদের ইন্টারনেট ব্রাউজিং এক্সপেরিয়েন্স দ্রুত করার জন্য লোকাল ডিএনএস সার্ভারের ভূমিকা অপরিসীম।

MikroTik RouterOS দিয়ে আপনি খুব সহজেই আপনার রাউটারকে একটি DNS Forwarder বা ক্যাশ (Cache) সার্ভার হিসেবে তৈরি করতে পারেন। শুধু তাই নয়, ইউজাররা যাতে চালাকি করে আপনার নেটওয়ার্কের ডিএনএস বাইপাস না করতে পারে, তার জন্য DNS Query Redirect বা Force DNS করাটাও শিখতে হবে। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা এই দুটি কাজ প্র্যাকটিক্যালি করে দেখবো।

১. DNS Forwarder কী এবং এর সুবিধা কী?

DNS Forwarder হলো এমন একটি মেকানিজম যেখানে আপনার রাউটার নিজেই ইউজারদের জন্য লোকাল ডিএনএস সার্ভার হিসেবে কাজ করে।

  • কীভাবে কাজ করে? যখন কোনো ইউজার facebook.com লিখে এন্টার দেয়, রিকোয়েস্টটি প্রথমে আপনার MikroTik রাউটারের কাছে আসে। রাউটার তখন মেইন ডিএনএস সার্ভার (যেমন Google বা Cloudflare) থেকে আইপিটি জেনে এনে ইউজারকে দেয় এবং নিজের মেমোরিতে তা Cache (জমা) করে রাখে।
  • সুবিধা: পরবর্তীতে অন্য কোনো ইউজার যখন আবার facebook.com এ যেতে চাইবে, রাউটার তখন ইন্টারনেট থেকে আইপি না খুঁজে নিজের ক্যাশ থেকেই সাথে সাথে রিপ্লাই দিয়ে দেবে। এতে ব্রাউজিং স্পিড বাড়ে এবং ব্যান্ডউইথ (Bandwidth) সাশ্রয় হয়।

২. MikroTik রাউটারকে DNS Forwarder হিসেবে কনফিগার করা

রাউটারকে ফরোয়ার্ডার বানানোর জন্য আমাদের শুধু Allow Remote Requests অপশনটি চালু করতে হবে।

Winbox এর মাধ্যমে কনফিগারেশন:

  • বাম দিকের মেনু থেকে IP > DNS এ ক্লিক করুন।
  • Servers: এর ঘরে আপনার আপস্ট্রিম আইএসপির ডিএনএস অথবা গুগলের ডিএনএস (8.8.8.8 এবং 1.1.1.1) বসিয়ে দিন।
  • সবচেয়ে গুরুত্বপূর্ণ ধাপ হলো Allow Remote Requests চেকবক্সে টিক (Check) দেওয়া। এটি রাউটারকে লোকাল ডিএনএস সার্ভার হিসেবে কাজ করার অনুমতি দেয়।
  • Cache Size: এর ঘরে ডিফল্ট 2048 KiB থাকে। বড় নেটওয়ার্কের জন্য আপনি এটি বাড়িয়ে 4096 KiB বা 8192 KiB করে দিতে পারেন।
  • Apply এবং OK দিন।

CLI কমান্ড (Terminal):

[admin@MikroTik] > /ip dns set servers=8.8.8.8,1.1.1.1 allow-remote-requests=yes cache-size=4096

এখন আপনার লোকাল নেটওয়ার্কের (LAN) ইউজারদের DHCP সার্ভারের মাধ্যমে ডিএনএস আইপি হিসেবে আপনার রাউটারের লোকাল আইপি (যেমন: 192.168.10.1) দিতে হবে।

৩. রিয়েল-লাইফ সিনারিও: DNS Query Redirect কেন প্রয়োজন?

সমস্যা: আপনি হয়তো আপনার অফিসে একটি ফিল্টার্ড ডিএনএস (যেমন: OpenDNS) ব্যবহার করছেন, যাতে পর্নোগ্রাফি বা ম্যালওয়্যার সাইট ব্লক থাকে। কিন্তু আপনার অফিসের কিছু চতুর ইউজার তাদের পিসি বা মোবাইল ফোনের সেটিংসে গিয়ে ম্যানুয়ালি 8.8.8.8 বসিয়ে দিলো। এর ফলে তারা আপনার রাউটারের ডিএনএস বাইপাস করে সরাসরি ইন্টারনেটে চলে যাবে এবং সমস্ত ব্লক করা সাইট ভিজিট করতে পারবে।

সমাধান: এই বাইপাস ঠেকানোর জন্য আমরা ফায়ারওয়ালে NAT Redirect (Force DNS) ব্যবহার করবো। আমরা রাউটারকে বলবো, "নেটওয়ার্কের ভেতর থেকে যদি কেউ পোর্ট 53 (DNS Port) ব্যবহার করে বাইরের কোনো সার্ভারের কাছে যেতে চায়, তবে তাকে জোর করে ঘুরিয়ে রাউটারের নিজের কাছে (Local DNS) নিয়ে আসো।"

৪. DNS Query Redirect (Force DNS) কনফিগার করার পদ্ধতি

DNS ট্রাফিক সাধারণত UDP Port 53 এবং ক্ষেত্রবিশেষে TCP Port 53 ব্যবহার করে। আমাদের ফায়ারওয়ালের Destination NAT (dstnat) চেইন ব্যবহার করে এই দুই ধরনের ট্রাফিককেই রিডাইরেক্ট করতে হবে।

Winbox এর মাধ্যমে কনফিগারেশন:

  • IP > Firewall > NAT ট্যাবে যান।
  • + (Add) আইকনে ক্লিক করে নতুন রুল তৈরি করুন।
  • General Tab:
    • Chain: dstnat সিলেক্ট করুন।
    • Protocol: 17 (udp) সিলেক্ট করুন।
    • Dst. Port: 53 লিখুন।
    • In. Interface: আপনার ল্যান ইন্টারফেস (যেমন: ether2 বা bridge-lan) সিলেক্ট করুন।
  • Action Tab:
    • Action: ড্রপডাউন থেকে redirect সিলেক্ট করুন।
    • To Ports: 53 লিখুন।
  • Apply এবং OK দিন।

ঠিক একই নিয়মে Protocol এর জায়গায় 6 (tcp) সিলেক্ট করে আরেকটি রুল তৈরি করুন।

CLI কমান্ড (Terminal):

টার্মিনাল ব্যবহার করে খুব দ্রুত রুল দুটি অ্যাড করার জন্য নিচের কমান্ডগুলো ব্যবহার করুন:

UDP Port 53 Redirect:

[admin@MikroTik] > /ip firewall nat add chain=dstnat action=redirect to-ports=53 protocol=udp dst-port=53 in-interface=bridge-lan comment="Force DNS UDP"

TCP Port 53 Redirect:

[admin@MikroTik] > /ip firewall nat add chain=dstnat action=redirect to-ports=53 protocol=tcp dst-port=53 in-interface=bridge-lan comment="Force DNS TCP"

৫. Troubleshooting Tips (সাধারণ সমস্যা ও সিকিউরিটি সতর্কতা)

সমস্যা (Problem) / সতর্কতা কারণ এবং সমাধান (Cause & Solution)
Security Alert: DNS Amplification Attack অত্যন্ত গুরুত্বপূর্ণ: Allow Remote Requests চালু করলে ইন্টারনেট (WAN) থেকেও হ্যাকাররা আপনার রাউটারকে ডিএনএস সার্ভার হিসেবে ব্যবহার করে স্প্যাম রিকোয়েস্ট পাঠাতে পারে। রাউটারের CPU 100% হয়ে যাবে। এর সমাধান হলো Firewall > Filter Rules এ গিয়ে WAN পোর্ট থেকে আসা পোর্ট 53 এর ট্রাফিক ড্রপ (Drop) করে দেওয়া। কমান্ড: /ip firewall filter add action=drop chain=input in-interface=ether1 dst-port=53 protocol=udp
নতুন আইপি আপডেট হচ্ছে না মাঝে মাঝে কোনো সাইটের আইপি পরিবর্তন হলে ক্যাশের কারণে পুরনো আইপি লোড হয়। এই সমস্যা সমাধানের জন্য রাউটারের ডিএনএস ক্যাশ ফ্লাশ (Flush) করতে হয়। কমান্ড: /ip dns cache flush
Redirection কাজ করছে কি না চেক করা এটি কাজ করছে কি না তা বোঝার জন্য উইন্ডোজ পিসির CMD ওপেন করে nslookup facebook.com 8.8.8.8 লিখে এন্টার দিন। যদি রেজাল্টে আপনার রাউটারের লোকাল আইপি (যেমন 192.168.10.1) থেকে নন-অথরিটেটিভ আনসার আসে, তবে বুঝবেন আপনার Force DNS রুল ঠিকমতো কাজ করছে।

উপসংহার (Conclusion)

আপনার লোকাল নেটওয়ার্কে DNS Forwarder কনফিগার করা ব্রাউজিং স্পিড বাড়ানোর জন্য একটি চমৎকার উপায়। কিন্তু নেটওয়ার্কের ওপর আপনার সম্পূর্ণ নিয়ন্ত্রণ বা পলিসি ধরে রাখার জন্য DNS Query Redirect (NAT) করাটা অত্যাবশ্যক। এই দুটি ফিচারের সমন্বয় আপনার MikroTik Router-কে শুধু একটি প্যাকেট ফরোয়ার্ডার থেকে একটি স্মার্ট এবং সিকিউরড লোকাল গেটওয়েতে পরিণত করে। তবে মনে রাখবেন, ডিএনএস সার্ভিস চালু করার সাথে সাথে WAN পোর্ট থেকে আসা ট্রাফিক ড্রপ করার সিকিউরিটি রুলটি কখনোই ভুলে যাওয়া যাবে না।
