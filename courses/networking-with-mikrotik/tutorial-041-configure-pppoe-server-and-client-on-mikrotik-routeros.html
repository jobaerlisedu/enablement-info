Configure PPPoE Server and Client on MikroTik RouterOS

20 May, 2026 Networking with MikroTik Jobaer Hossain

ব্রডব্যান্ড ইন্টারনেট ডিস্ট্রিবিউশনের ক্ষেত্রে PPPoE (Point-to-Point Protocol over Ethernet) হলো সবচেয়ে বিশ্বস্ত এবং বহুল ব্যবহৃত একটি প্রযুক্তি। পূর্ববর্তী আর্টিকেলে আমরা PPPoE এর থিওরি এবং এর পেছনের মেকানিজম নিয়ে আলোচনা করেছি। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা সম্পূর্ণ প্র্যাকটিক্যাল সেটআপ দেখবো।

আমরা শিখবো কীভাবে একটি MikroTik রাউটারকে আইএসপি (ISP) এর মতো PPPoE Server হিসেবে তৈরি করতে হয় এবং অপর একটি রাউটার বা ডিভাইসকে কাস্টমার হিসেবে PPPoE Client কনফিগার করে ইন্টারনেট সংযোগ নিতে হয়।

রিয়েল-লাইফ সিনারিও (The Setup)

আমাদের সেটআপটিতে দুটি অংশ থাকবে:

  • ISP Router (Server): এই রাউটারে ইন্টারনেট কানেকশন আছে। এর ether2 পোর্ট দিয়ে আমরা কাস্টমারদের লাইন দেবো। আমরা কাস্টমারদের জন্য 10.10.10.0/24 ব্লকের আইপি ব্যবহার করবো এবং তাদের স্পিড 10 Mbps এ লিমিট করে দেবো।
  • Customer Router (Client): এটি কাস্টমারের বাসার রাউটার। এর ether1 পোর্টে আইএসপির ক্যাবল লাগানো থাকবে। এই রাউটারে আমরা ইউজারনেম এবং পাসওয়ার্ড বসিয়ে ইন্টারনেট চালু করবো।

পর্ব ১: ISP রাউটারে PPPoE Server কনফিগারেশন

একটি প্রফেশনাল PPPoE Server তৈরি করার জন্য আমাদের প্রধান ৪টি ধাপ অনুসরণ করতে হবে।

Step 1: IP Pool তৈরি করা

কাস্টমাররা যখন কানেক্ট হবে, তখন তারা কোন আইপি পাবে, তা নির্ধারণ করার জন্য একটি IP Pool তৈরি করতে হবে।

  • Winbox থেকে IP > Pool এ যান এবং + (Add) আইকনে ক্লিক করুন।
  • Name: pppoe-pool দিন।
  • Addresses: 10.10.10.10-10.10.10.250 দিন। OK করুন।
[admin@ISP-Router] > /ip pool add name=pppoe-pool ranges=10.10.10.10-10.10.10.250

Step 2: PPP Profile তৈরি করা (Speed Limit সহ)

Profile এর মাধ্যমে আমরা কাস্টমারের স্পিড লিমিট এবং আইপি পুল অ্যাসাইন করবো।

  • PPP মেনুতে গিয়ে Profiles ট্যাবে যান এবং + (Add) এ ক্লিক করুন।
  • Name: 10Mbps-Package দিন।
  • Local Address: 10.10.10.1 দিন (এটি হবে সার্ভারের নিজস্ব আইপি বা Gateway)।
  • Remote Address: ড্রপডাউন থেকে আমাদের তৈরি করা pppoe-pool সিলেক্ট করুন।
  • Limits ট্যাবে গিয়ে Rate Limit (rx/tx): এর ঘরে 10M/10M দিন। OK করুন।
[admin@ISP-Router] > /ppp profile add name=10Mbps-Package local-address=10.10.10.1 remote-address=pppoe-pool rate-limit=10M/10M

Step 3: PPPoE Server চালু করা

এখন আমরা নির্দিষ্ট ইন্টারফেসে সার্ভিসটি চালু করবো।

  • একই PPP উইন্ডো থেকে PPPoE Servers ট্যাবে যান এবং + (Add) ক্লিক করুন।
  • Service Name: My_ISP_Service দিন।
  • Interface: ether2 সিলেক্ট করুন (যে পোর্ট দিয়ে কাস্টমার ক্যাবল যুক্ত হবে)।
  • Default Profile: 10Mbps-Package সিলেক্ট করুন। OK দিন। 
[admin@ISP-Router] > /interface pppoe-server server add service-name=My_ISP_Service interface=ether2 default-profile=10Mbps-Package disabled=no

Step 4: Customer User (Secret) তৈরি করা

কাস্টমারের লগইন ক্রেডেনশিয়াল তৈরি করতে হবে।

  • PPP মেনুর Secrets ট্যাবে যান এবং + (Add) ক্লিক করুন।
  • Name: user1 দিন।
  • Password: pass123 দিন।
  • Service: pppoe সিলেক্ট করুন।
  • Profile: 10Mbps-Package সিলেক্ট করুন। OK দিন।
[admin@ISP-Router] > /ppp secret add name=user1 password=pass123 service=pppoe profile=10Mbps-Package

Step 5: NAT কনফিগারেশন (অত্যন্ত গুরুত্বপূর্ণ)

কাস্টমার আইপি পেলেও ইন্টারনেট পাবে না যদি আপনি সার্ভার রাউটারে NAT (Masquerade) না করেন।

  • IP > Firewall > NAT এ গিয়ে + (Add) করুন।
  • Chain: srcnat, Src. Address: 10.10.10.0/24 (কাস্টমার আইপি ব্লক)।
  • Action: masquerade সিলেক্ট করে OK দিন।
[admin@ISP-Router] > /ip firewall nat add chain=srcnat src-address=10.10.10.0/24 action=masquerade

পর্ব ২: Customer রাউটারে PPPoE Client কনফিগারেশন

এখন আমরা কাস্টমার রাউটারে লগইন করে ইন্টারনেট চালু করবো।

Step 1: PPPoE Client ইন্টারফেস তৈরি করা

  • Winbox থেকে Interfaces মেনুতে যান।
  • + (Add) আইকনে ক্লিক করে ড্রপডাউন থেকে PPPoE Client সিলেক্ট করুন।
  • General ট্যাবে Interfaces: ether1 সিলেক্ট করুন (যেখানে আইএসপির লাইন লাগানো)।
  • Dial Out ট্যাবে যান।
  • User: user1 দিন (যা আইএসপি থেকে দিয়েছে)।
  • Password: pass123 দিন।
  • Use Peer DNS: টিক (Check) দিন (যাতে আইএসপির ডিএনএস অটোমেটিক পায়)।
  • Add Default Route: টিক (Check) দিন (যাতে ইন্টারনেটের রুট অটোমেটিক সেট হয়)।
  • Apply এবং OK দিন। স্ট্যাটাস বারে connected লেখা উঠলেই বুঝতে পারবেন কনফিগারেশন সফল।

Step 2: Customer LAN এর জন্য NAT কনফিগারেশন

যেহেতু এটি কাস্টমারের রাউটার, তাই ভেতরের লোকাল ডিভাইসে (মোবাইল/ল্যাপটপ) ইন্টারনেট দেওয়ার জন্য ক্লায়েন্ট রাউটারেও NAT করতে হবে।

  • IP > Firewall > NAT এ গিয়ে + (Add) করুন।
  • Chain: srcnat সিলেক্ট করুন।
  • Out. Interface: pppoe-out1 (আপনার তৈরি করা PPPoE ইন্টারফেস) সিলেক্ট করুন।
  • Action: masquerade সিলেক্ট করে OK দিন।
[admin@Customer-Router] > /ip firewall nat add chain=srcnat out-interface=pppoe-out1 action=masquerade

৩. Troubleshooting Tips (সাধারণ সমস্যা ও সমাধান)

সমস্যা (Problem) কারণ এবং সমাধান (Cause & Solution)
Status: Dialing এ আটকে থাকা রাউটার সার্ভার খুঁজে পাচ্ছে না। ক্যাবল ঠিকমতো লাগানো আছে কি না চেক করুন। যদি ISP VLAN ব্যবহার করে, তবে PPPoE Client টি ফিজিক্যাল পোর্টের বদলে (যেমন ether1) নির্দিষ্ট VLAN ইন্টারফেসের ওপর তৈরি করতে হবে।
Authentication Failed মেসেজ ইউজারনেম অথবা পাসওয়ার্ড ভুল হয়েছে। অনেক সময় পাসওয়ার্ডের শেষে স্পেস (Space) পড়ে গেলে এই এরর আসে। পুনরায় টাইপ করে চেষ্টা করুন।
কিছু ওয়েবসাইট (যেমন Facebook, Netflix) লোড হচ্ছে না এটি PPPoE এর সবচেয়ে কমন MTU (Maximum Transmission Unit) ইস্যু। PPPoE হেডার সাইজের কারণে সাধারণ 1500 MTU কাজ করে না। ISP রাউটারের PPP Profile এ গিয়ে MTU পরিবর্তন করে 1480 বা 1492 করে দিন।
কানেক্ট হচ্ছে কিন্তু ইন্টারনেট চলছে না ক্লায়েন্ট রাউটারে IP > Routes চেক করুন 0.0.0.0/0 ডাইনামিক রুটটি এসেছে কি না। না আসলে PPPoE Client সেটিংসে Add Default Route অপশনটি চেক করা আছে কি না তা নিশ্চিত করুন।

উপসংহার (Conclusion)

PPPoE Server এবং Client সেটআপ করা ব্রডব্যান্ড নেটওয়ার্কিংয়ের একদম বেসিক কিন্তু সবচেয়ে শক্তিশালী ধাপ। এই আর্টিকেলের মাধ্যমে আপনি একটি কমপ্লিট আইএসপি সলিউশনের ডেমো দেখতে পেলেন, যেখানে আমরা IP Pool, Profile, Secret এবং NAT কনফিগার করার মাধ্যমে কাস্টমারকে ইন্টারনেট দিলাম। এই বেসিক সেটআপটি বুঝতে পারলে, পরবর্তীতে RADIUS Server (User Manager) এর সাথে PPPoE ইন্টিগ্রেট করে হাজার হাজার কাস্টমারকে অটোমেটিক্যালি বিলিং এবং ম্যানেজ করা আপনার জন্য একদম সহজ হয়ে যাবে।
