Basic Command Line Syntax on MikroTik RouterOS

10 June, 2026 Networking with MikroTik Jobaer Hossain

MikroTik কনফিগারেশনের ক্ষেত্রে Winbox এর গ্রাফিক্যাল ইউজার ইন্টারফেস (GUI) সবার কাছে জনপ্রিয় হলেও, একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে আপনাকে অবশ্যই Command Line Interface (CLI) এ দক্ষ হতে হবে। বড় কোনো ডেটা সেন্টারে রিমোটলি SSH করে কাজ করা, অটোমেশনের জন্য Scripting করা, বা দ্রুত কোনো কনফিগারেশন চেক করার জন্য CLI এর বিকল্প নেই।

Cisco বা Juniper এর মতো MikroTik এরও নিজস্ব একটি কমান্ড লাইন সিনট্যাক্স (Syntax) বা কাঠামো রয়েছে। আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা MikroTik RouterOS এর কমান্ড লাইনের বেসিক স্ট্রাকচার, নেভিগেশন এবং সবচেয়ে বেশি ব্যবহৃত কমান্ডগুলো সম্পর্কে বিস্তারিত জানবো।

RouterOS CLI এর হায়ারার্কি (Hierarchical Structure)

MikroTik এর CLI অনেকটা একটি কম্পিউটারের ফাইল সিস্টেম বা ফোল্ডার স্ট্রাকচারের মতো কাজ করে। প্রতিটি মেইন ফিচারের জন্য একটি রুট (Root) মেনু থাকে এবং তার ভেতরে সাব-মেনু (Sub-menu) থাকে। যেমন, আইপি সম্পর্কিত সব কাজ /ip মেনুর ভেতরে থাকে। এর ভেতরে আবার address, route, firewall ইত্যাদি সাব-মেনু থাকে।

১. কমান্ড লাইনে নেভিগেশন (Navigation)

লগইন করার পর আপনি ডিফল্টভাবে রুট বা মেইন ডিরেক্টরিতে ([admin@MikroTik] >) থাকেন। এখান থেকে বিভিন্ন মেনুতে যাওয়ার জন্য নিচের নিয়মগুলো অনুসরণ করতে হয়:

  • কোনো নির্দিষ্ট মেনুতে প্রবেশ করতে: সরাসরি মেনুর নাম লিখুন। যেমন, IP Address মেনুতে যেতে চাইলে লিখুন:
    [admin@MikroTik] > ip address
    এন্টার দেওয়ার পর আপনার প্রম্পট পরিবর্তন হয়ে [admin@MikroTik] /ip address> হয়ে যাবে।
  • এক ধাপ পেছনে (Up) যেতে: দুটি ডট .. টাইপ করে এন্টার দিন।
  • সরাসরি রুট (Root) মেনুতে ফিরে যেতে: একটি ফরোয়ার্ড স্ল্যাশ / টাইপ করে এন্টার দিন।

২. কমান্ড লাইনের জাদুকরী শর্টকাট (Keyboard Shortcuts)

কমান্ড লাইনে কাজ করার গতি বাড়ানোর জন্য RouterOS কিছু চমৎকার শর্টকাট প্রদান করে, যা আপনার ভুল করার সম্ভাবনাও কমিয়ে দেয়:

শর্টকাট (Key) কাজের বিবরণ (Description)
Tab (ট্যাব কীবোর্ড) Auto-complete: কোনো কমান্ডের প্রথম কয়েকটি অক্ষর লিখে Tab চাপলে RouterOS অটোমেটিকভাবে পুরো কমান্ডটি লিখে দেয়। যেমন ip add লিখে ট্যাব চাপলে ip address হয়ে যাবে।
? (প্রশ্নবোধক চিহ্ন) Help Menu: আপনি যদি ভুলে যান যে এরপর কী কমান্ড দিতে হবে, তবে ? চাপুন। ওই নির্দিষ্ট জায়গায় আর কী কী কমান্ড দেওয়া সম্ভব, তার একটি লিস্ট স্ক্রিনে চলে আসবে।
Up / Down Arrow Command History: আগে টাইপ করা কমান্ডগুলো পুনরায় ব্যবহার করতে উপরের বা নিচের অ্যারো কী (Arrow Key) চাপুন।

৩. সবচেয়ে বেশি ব্যবহৃত বেসিক কমান্ডসমূহ

যেকোনো মেনুর ভেতরে গিয়ে আপনি নির্দিষ্ট কিছু অ্যাকশন বা কাজ করতে পারেন। নিচে সবচেয়ে প্রয়োজনীয় অ্যাকশন কমান্ডগুলো দেওয়া হলো:

  • print: বর্তমান মেনুর কনফিগারেশনগুলো স্ক্রিনে দেখার জন্য। যেমন: /ip address print
  • add: নতুন কোনো রুল বা আইটেম যোগ করার জন্য। যেমন: /ip address add...
  • set: আগে থেকেই তৈরি করা কোনো কনফিগারেশন পরিবর্তন (Modify) বা আপডেট করার জন্য।
  • remove: কোনো রুল বা কনফিগারেশন ডিলিট করার জন্য। এই কমান্ডের পর রুলের ID Number দিতে হয়।
  • enable / disable: কোনো রুলকে সাময়িকভাবে চালু বা বন্ধ করার জন্য।
  • export: কনফিগারেশনের ব্যাকআপ (Script) তৈরি করার জন্য। এটি হুবহু কমান্ড ফরমেটে কনফিগারেশন দেখায়, যা অন্য রাউটারে সরাসরি পেস্ট করা যায়।

৪. Real-life Scenario: CLI দিয়ে একটি IP Address কনফিগার করা

ধরি, আপনি আপনার রাউটারের ether2 ইন্টারফেসে একটি লোকাল আইপি (192.168.1.1/24) বসাতে চান। চলুন দেখি এটি কমান্ড লাইনে কীভাবে করতে হয়:

ধাপ ১: সরাসরি কমান্ড দেওয়া (Absolute Path)

আপনি চাইলে রুট মেনু থেকে এক লাইনেই পুরো কমান্ডটি দিয়ে দিতে পারেন:

[admin@MikroTik] > /ip address add address=192.168.1.1/24 interface=ether2

ধাপ ২: মেনুতে প্রবেশ করে কমান্ড দেওয়া (Relative Path)

আপনি চাইলে ধাপে ধাপে মেনুর ভেতরে প্রবেশ করেও কাজটি করতে পারেন:

[admin@MikroTik] > ip address
[admin@MikroTik] /ip address> add address=192.168.1.1/24 interface=ether2

ধাপ ৩: কনফিগারেশন চেক করা (Print)

আইপি ঠিকমতো বসেছে কি না, তা দেখার জন্য:

[admin@MikroTik] /ip address> print
Flags: X - disabled, I - invalid, D - dynamic 
 #   ADDRESS            NETWORK         INTERFACE                                 
 0   192.168.1.1/24     192.168.1.0     ether2

ধাপ ৪: কনফিগারেশন রিমুভ করা

যদি আইপিটি ভুল হয়ে থাকে এবং আপনি এটি মুছতে চান, তবে print কমান্ডের আউটপুটে থাকা ID Number (এখানে 0) ব্যবহার করতে হবে:

[admin@MikroTik] /ip address> remove 0

৫. Troubleshooting Tips (সাধারণ ভুল ও সমাধান)

CLI তে কাজ করার সময় নতুনদের কিছু সাধারণ ভুল হয়ে থাকে:

  • Syntax Error বা Red Text: কমান্ড টাইপ করার সময় যদি দেখেন কোনো টেক্সটের কালার লাল (Red) হয়ে গেছে, তার মানে কমান্ডটি ভুল বা ওই স্থানে ওই কমান্ডটি প্রযোজ্য নয়। Tab বা ? চেপে সঠিক কমান্ডটি খুঁজে নিন।
  • ভুল মেনুতে কমান্ড দেওয়া: আপনি হয়তো /ip route মেনুতে আছেন, কিন্তু কমান্ড দিচ্ছেন add address...। এটি কাজ করবে না। সবসময় খেয়াল রাখবেন আপনার প্রম্পট কোথায় আছে। অন্য মেনুর কমান্ড দিতে হলে শুরুতে একটি / ব্যবহার করে অ্যাবসলিউট পাথ দিন (যেমন /ip address add...)।
  • Safe Mode এর ব্যবহার: রিমোটলি (SSH) কাজ করার সময় ভুল কমান্ড দিলে আপনার কানেকশন বিচ্ছিন্ন হয়ে যেতে পারে। তাই কোনো ক্রিটিক্যাল কনফিগারেশনের আগে [CTRL] + [X] চেপে Safe Mode অন করে নিন। যদি কানেকশন ড্রপ করে, তবে রাউটার সাথে সাথে আগের অবস্থায় ফিরে যাবে। কাজ শেষ হলে আবার [CTRL] + [X] চেপে সেফ মোড অফ করে দিন।

উপসংহার (Conclusion)

Command Line Interface (CLI) প্রথম দিকে কিছুটা কঠিন মনে হলেও, একবার এর সিনট্যাক্স এবং Tab বাটনের ব্যবহার আয়ত্ত করতে পারলে Winbox এর চেয়ে অনেক দ্রুত কাজ করা সম্ভব। প্রফেশনাল লেভেলে রাউটারের ব্যাকআপ নেওয়া (Export), বড় কোনো ফায়ারওয়াল রুল সেটআপ করা, অথবা স্ক্রিপ্টিং করে অটোমেশন করার জন্য RouterOS এর CLI কমান্ড জানাটা একজন নেটওয়ার্ক ইঞ্জিনিয়ারের জন্য অপরিহার্য।
