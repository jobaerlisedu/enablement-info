Data Link Layer: Detailed Guide

January 15, 2026 Networking Jobaer Hossain

Data Link Layer কী?

Data Link Layer হলো OSI Model-এর নিচ থেকে দ্বিতীয় স্তর। এটি মূলত একই স্থানীয় Network-এর মধ্যে এক Node থেকে অন্য Node-এ Data পৌঁছানোর Responsibility পালন করে। একে OSI Model-এর সবচেয়ে জটিল Layer বলা হয় কারণ এটি Next উপরের Layer-গুলো থেকে Hardware-এর Complexity-গুলো আড়াল করে রাখে।

  • Node-to-Node Delivery: একই local network-এর মধ্যে data স্থানান্তর নিশ্চিত করে।
  • Reliable Transmission: Information-এর Accurate তা এবং Error হীন আদান-প্রদান নিশ্চিত করা এর মূল কাজ।
  • Encoding & Decoding: আগত ও বহির্গত data-কে সাজানো এবং সংকেতে রূপান্তর করা।

Data Link Layer-এর Sub-layer-সমূহ

Data Link Layer-কে আরও দুটি গুরুত্বপূর্ণ উপ-স্তরে বা Sub-layer-এ ভাগ করা হয়েছে:

1. Logical Link Control (LLC)

LLC Sub-layer-টি Network Layer এবং MAC Sub-layer-এর মধ্যে সেতুবন্ধন হিসেবে কাজ করে। এর কাজ-গুলো হলো:

  • Application-গুলোর মধ্যে data-র প্রবাহ (Flow) এবং Multiplexing Operation করা।
  • Error Message প্রদান এবং প্রাপ্তি স্বীকার (Acknowledgment) নিশ্চিত করা।
  • Device-এর Physical বা MAC Address ব্যবহার করে Frame তৈরি করা।
  • Network Layer থেকে পাওয়া Packet-গুলোকে 'Frame'-এ বিভক্ত করে bit by bit physical Layer-এ পাঠানো।
  • Physical Media Access নিয়ন্ত্রণ করা।

Data Link Layer-এর প্রধান কাজ সমূহ (Functions)

Data Link Layer-এর প্রধান কাজ-গুলো নিচে Detailed Discussion করা হলো:

  • Framing: Packet-গুলোকে Frame-এ সাজানো যাতে Receiver Frame-এর শুরু এবং শেষ বুঝতে পারে।
  • Physical Addressing: প্রতিটি Frame-এর Header-এ Sender ও Receiver-এর MAC Address যুক্ত করা।
  • Flow Control: একটি নির্দিষ্ট Speed তে data পাঠানো নিশ্চিত করা যাতে Receiver device-টি অতিরিক্ত data-র চাপে না পড়ে।
  • Error Control: Frame Damage বা হারিয়ে গেলে তা শনাক্ত করা এবং পুনরায় পাঠানোর ব্যবস্থা করা। এটি trailer-এ redundant data যুক্ত করে করা হয়।
  • Access Control: যখন Multiple device একই link share করে, তখন কোন device-টি কখন data পাঠাবে তা নিয়ন্ত্রণ করা।

Data Link Layer-এর Protocol-সমূহ

এই Layer-এ নিরবিচ্ছিন্ন যোগাযোগের জন্য বিভিন্ন protocol ব্যবহৃত হয়:

  • SDLC (Synchronous Data Link Control): IBM দ্বারা তৈরি প্রাচীন Protocol।
  • HDLC (বেশি-Level Data Link Control): এটি bit-oriented protocol যা data transmission-এ ব্যবহৃত হয়।
  • SLIP (Serial Line Interface Protocol): Serial port এবং Modem connection-এর জন্য ব্যবহৃত হয়।
  • PPP (Point-to-Point Protocol): সরাসরি দুটি Computer-এর মধ্যে IP Traffic transmit করতে ব্যবহৃত হয়।
  • LAP (Link Access Procedure): X.25 Frame Structure-এর protocol।
  • LCP & NCP: Link Control এবং Network Control protocol যা PPP-এর সাথে কাজ করে।

Data Link Layer-এ ব্যবহৃত Device-সমূহ

এই Layout-এ Information-এর সঠিক আদান-প্রদান নিশ্চিত করতে বেশ কিছু Hardware Device কাজ করে:

1. Switch

MAC Address Table ব্যবহার করে Frame-গুলোকে সঠিক Device-এ forward করে।

2. Bridge

দুটি LAN-কে সংযুক্ত করে একটি network segment তৈরি করে এবং traffic কমায়।

3. NIC (Network Interface Card)

Computer-এর physical যোগাযোগ unit যা MAC address ব্যবহার করে frame পাঠায় ও গ্রহণ করে।

4. Wireless Access Point (WAP)

Wireless device-গুলোকে wired network-এর সাথে সংযুক্ত করে Wi-Fi protocol ব্যবহার করে।

Security Risk: Data Link Layer অনেক সময় MAC Spoofing বা ARP Poisoning-এর মতো Cyber Attack-এর শিকার হতে পারে। এই Attack-গুলো সম্পর্কে জ্ঞান থাকা network সুরক্ষার জন্য অপরিহার্য।

সীমাবদ্ধতা ও প্রয়োগ (Applications)

সীমাবদ্ধতা:

  • সীমিত Scope: শুধুমাত্র স্থানীয় network-এর মধ্যে কাজ করতে পারে।
  • Overhead বৃদ্ধি: Header ও trailer যুক্ত করার ফলে data-র size বেড়ে যায়।
  • Routing অ Capacity: এটি network-এর বাইরে routing করতে পারে না।

প্রয়োগ:

  • Ethernet এবং LAN Operation।
  • Wi-Fi (IEEE 802.11) communication।
  • Point-to-point protocol-এর মাধ্যমে সরাসরি সংযোগ।
  • Switching operation Operation।

Summary

Data Link Layer Networking-এর অত্যন্ত গুরুত্বপূর্ণ একটি স্তর যা bit-গুলোকে frame-এ রূপান্তর করে data-র Reliable তা নিশ্চিত করে। এর flow control, error handling এবং addressing ব্যবস্থা ছাড়া Internet-এর মতো বিশাল network-এ Information-এর আদান-প্রদান করা অসম্ভব ছিল।
