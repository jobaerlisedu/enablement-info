Data Link Layer & Media Access Control: Detailed Guide

03 May, 2026 Network Engineering Jobaer Hossain

OSI Model-এর প্রথম লেয়ার বা Physical Layer আমাদের ফিজিক্যাল ট্রান্সমিশন মিডিয়া (ক্যাবল, রেডিও ওয়েভ) প্রদান করে। কিন্তু সেই মিডিয়ার ওপর দিয়ে ডাটা কখন যাবে, কার পর কে ডাটা পাঠাবে এবং মিডিয়াটি কীভাবে নিয়ন্ত্রিত হবে—তার সম্পূর্ণ দায়িত্ব হলো Data Link Layer (Layer 2) এর।

আজকের এই ডিপ-ডাইভ আর্টিকেলে আমরা Data Link Layer কীভাবে ফিজিক্যাল মিডিয়াকে নিয়ন্ত্রণ করে, Framing কী, MAC Address কীভাবে কাজ করে এবং Switch-এর বেসিক অপারেশন সম্পর্কে বিস্তারিত জানবো।

Data Link Layer কী?

Data Link Layer হলো OSI মডেলের দ্বিতীয় স্তর। এটি মূলত Network Layer (Layer 3) থেকে ডাটা প্যাকেট গ্রহণ করে এবং সেগুলোকে ফিজিক্যাল মিডিয়ামের ওপর দিয়ে ট্রান্সমিট করার জন্য প্রস্তুত করে। এই লেয়ারে ডাটাকে Frame বলা হয়। এটি নিশ্চিত করে যে একটি ল্যান (LAN) বা লোকাল নেটওয়ার্কের এক ডিভাইস থেকে অন্য ডিভাইসে ডাটা যেন কোনো ভুল (Error) ছাড়া নিরাপদে পৌঁছায়।

১. Data Link Layer এর সাব-লেয়ারসমূহ (Sub-layers)

IEEE 802 স্ট্যান্ডার্ড অনুযায়ী, Data Link Layer কে তার কাজের সুবিধার্থে দুটি উপ-স্তরে (Sub-layers) ভাগ করা হয়েছে:

  • Logical Link Control (LLC): এটি ওপরের লেয়ার অর্থাৎ Network Layer এর সাথে যোগাযোগ রক্ষা করে। এটি ডাটা ফ্রেমে প্রোটোকল ইনফরমেশন যুক্ত করে, যাতে রিসিভার বুঝতে পারে এই ডাটাটি IPv4 নাকি IPv6 এর জন্য এসেছে।
  • Media Access Control (MAC): এটি নিচের লেয়ার অর্থাৎ Physical Layer এর সাথে সরাসরি যুক্ত। ক্যাবল বা ওয়্যারলেস মিডিয়ার ওপর কে কখন ডাটা পাঠাবে এবং ফিজিক্যাল অ্যাড্রেসিং (MAC Address) কীভাবে হবে, তা এই সাব-লেয়ার নিয়ন্ত্রণ করে।

২. Media Access Control (মিডিয়া কন্ট্রোল মেকানিজম)

একই ক্যাবল বা নেটওয়ার্কে যদি একাধিক ডিভাইস একসাথে ডাটা পাঠায়, তবে ডাটার সংঘর্ষ বা Collision হয়। এই সংঘর্ষ এড়ানোর জন্য Data Link Layer কিছু নিয়ম বা প্রোটোকল মেনে চলে।

A. CSMA/CD (Carrier Sense Multiple Access with Collision Detection)

পুরনো Half-Duplex ইথারনেট নেটওয়ার্কে (যেমন Hub যুক্ত নেটওয়ার্ক) ডাটা সংঘর্ষ এড়ানোর জন্য এটি ব্যবহৃত হতো।

  • Carrier Sense: ডিভাইস ডাটা পাঠানোর আগে প্রথমে মিডিয়া বা ক্যাবলটি চেক করে (শোনে) যে কেউ কথা বলছে কি না। যদি ক্যাবল খালি থাকে, তবে সে ডাটা পাঠায়।
  • Collision Detection: যদি ভুলে দুজন একসাথে ডাটা পাঠিয়ে দেয়, তবে তারা সংঘর্ষ ডিটেক্ট করে এবং ডাটা পাঠানো বন্ধ করে দেয়। এরপর একটি র‍্যান্ডম সময় (Backoff timer) অপেক্ষা করে আবার পাঠানোর চেষ্টা করে।

B. CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance)

এটি Wireless (Wi-Fi) নেটওয়ার্কে ব্যবহৃত হয়। ওয়্যারলেসে Collision ডিটেক্ট করা খুব কঠিন, তাই এখানে আগে থেকেই সতর্কতা অবলম্বন করা হয়, যাতে কলিশন না হয়। ডিভাইস ডাটা পাঠানোর আগে একটি Request to Send (RTS) সিগন্যাল পাঠায়।

৩. Data Framing এবং Error Detection

Network Layer থেকে ডাটা আসার পর Data Link Layer তার সাথে একটি Header এবং একটি Trailer যুক্ত করে। এই পুরো প্যাকেজটিকে Frame বলা হয়।

Frame অংশ বিস্তারিত (Description)
Header এখানে সোর্স (Source) এবং ডেস্টিনেশনের (Destination) MAC Address থাকে।
Payload (Data) এটি হলো মূল ডাটা বা Packet যা Network Layer থেকে এসেছে।
Trailer (FCS) এখানে Frame Check Sequence (FCS) বা CRC (Cyclic Redundancy Check) থাকে। এটি একটি গাণিতিক মান যা ডাটাতে কোনো Error বা ভুল আছে কি না তা যাচাই করে।
রিয়েল-লাইফ উদাহরণ: আপনি যখন কুরিয়ারে কোনো পার্সেল পাঠান, তখন পার্সেলের ওপর প্রেরক ও প্রাপকের ঠিকানা লিখেন (Header) এবং প্যাকেটের মুখ শক্ত করে সিলগালা করেন (Trailer)। প্রাপক সিলগালা অক্ষত আছে কি না তা দেখে বুঝতে পারে ভেতরে জিনিস ঠিক আছে কি না (Error Detection)। ফ্রেম এভাবেই কাজ করে।

৪. MAC Address এবং Switch এর ভূমিকা

Data Link Layer-এর সবচেয়ে জনপ্রিয় ডিভাইস হলো Switch। এটি একটি খাঁটি Layer 2 ডিভাইস, যা MAC Address চিনে কাজ করে।

  • MAC Address Learning: যখন কোনো পিসি সুইচের পোর্টে ডাটা পাঠায়, সুইচ সাথে সাথে সেই ফ্রেমের Source MAC Address টি পড়ে নেয় এবং তার MAC Address Table (যাকে CAM Table-ও বলা হয়)-এ সেভ করে রাখে।
  • Forwarding: সুইচ যখন জানতে পারে কোন পোর্টে কোন ডিভাইস আছে, তখন সে ব্রডকাস্ট করার বদলে সরাসরি ওই নির্দিষ্ট পোর্টে ডাটা ফরোয়ার্ড করে দেয়। এর ফলে নেটওয়ার্কের Bandwidth বেঁচে যায় এবং পারফরম্যান্স অনেক বাড়ে।

৫. Cisco CLI: MAC Address Table কনফিগারেশন এবং ট্রাবলশুটিং

একজন নেটওয়ার্ক অ্যাডমিনিস্ট্রেটর হিসেবে আপনাকে প্রায়ই সুইচের MAC Address Table চেক করতে হবে।

সুইচের MAC Address Table দেখা:

নেটওয়ার্কে কোন পোর্টে কোন ডিভাইস কানেক্টেড আছে তা দেখার জন্য এই কমান্ডটি ব্যবহার করা হয়:

Switch# show mac address-table

আউটপুট এনালাইসিস: আপনি দেখতে পাবেন একটি নির্দিষ্ট VLAN, তার MAC Address, এটি স্ট্যাটিক নাকি DYNAMIC, এবং সেটি কোন Ports (যেমন: Fa0/1) এ কানেক্টেড আছে।

একটি নির্দিষ্ট MAC Address খুঁজে বের করা:

যদি আপনার নেটওয়ার্কে শত শত ডিভাইস থাকে এবং আপনি একটি নির্দিষ্ট MAC Address কোন পোর্টে আছে তা জানতে চান:

Switch# show mac address-table address aaaa.bbbb.cccc

MAC Table ক্লিয়ার করা:

কখনো কখনো নেটওয়ার্ক ট্রাবলশুট করার জন্য সুইচের ডাইনামিক এন্ট্রিগুলো মুছে ফেলার প্রয়োজন হয় (সুইচ আবার নতুন করে লার্ন করে নেবে):

Switch# clear mac address-table dynamic

ট্রাবলশুটিং টিপস: MAC Spoofing & Port Security

হ্যাকাররা অনেক সময় ফেক MAC Address ব্যবহার করে নেটওয়ার্কে ঢোকার চেষ্টা করে। এটি ঠেকাতে Layer 2 তে Port Security ব্যবহার করা হয়। এর মাধ্যমে আপনি নির্ধারণ করে দিতে পারেন একটি পোর্টে সর্বোচ্চ কয়টি এবং কোন নির্দিষ্ট MAC Address কানেক্ট হতে পারবে। আনঅথরাইজড কেউ ক্যাবল লাগালে পোর্টটি সাথে সাথে err-disabled স্টেটে চলে যাবে।

উপসংহার (Conclusion)

Data Link Layer ছাড়া শুধুমাত্র ফিজিক্যাল ক্যাবল দিয়ে কোনো সুশৃঙ্খল নেটওয়ার্ক তৈরি করা অসম্ভব। মিডিয়া এক্সেস কন্ট্রোল (MAC), ডাটাকে ফ্রেমে রূপান্তর, Error Detection এবং সুইচের বুদ্ধিদীপ্ত ডাটা ফরোয়ার্ডিং—এই সবই Layer 2 এর অবদান। নেটওয়ার্ক ট্রাবলশুটিংয়ের ক্ষেত্রে MAC Address Table কীভাবে পড়তে হয় এবং কাজ করে তা বোঝা একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ারের অন্যতম মৌলিক স্কিল।
