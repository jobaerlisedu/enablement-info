Calculate Network, Broadcast and Host Addresses: A Step-by-Step Guide

27 April, 2026 Networking Concepts Jobaer Hossain

কম্পিউটার নেটওয়ার্কিং নিয়ে কাজ করতে গেলে আমাদের প্রায়ই একটি আইপি অ্যাড্রেস এবং সাবনেট মাস্ক দেখে বলতে হয় ওই নেটওয়ার্কের সীমানা কতটুকু। অর্থাৎ, ওই নেটওয়ার্কের প্রথম ঠিকানা কোনটি, শেষ ঠিকানা কোনটি এবং আমরা আমাদের পিসি বা সার্ভারে কোন আইপিগুলো বসাতে পারবো।

আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো কীভাবে অত্যন্ত সহজ উপায়ে Network Address, Broadcast Address এবং Valid Host Range ক্যালকুলেশন করতে হয়। এটি নেটওয়ার্কিং ক্যারিয়ারের জন্য একটি অত্যন্ত বেসিক কিন্তু পাওয়ারফুল স্কিল।

১. প্রধান তিনটি এড্রেসের সংজ্ঞা

ক্যালকুলেশনে যাওয়ার আগে আমাদের জানতে হবে আমরা আসলে কী বের করছি:

  • Network Address: এটি একটি সাবনেটের প্রথম আইপি। এটি পুরো নেটওয়ার্কের পরিচয় হিসেবে কাজ করে। এই আইপিটি কোনো ডিভাইসে এসাইন করা যায় না।
  • Broadcast Address: এটি একটি সাবনেটের সর্বশেষ আইপি। এই ঠিকানায় কোনো প্যাকেট পাঠালে তা ওই নেটওয়ার্কের সব ডিভাইসের কাছে পৌঁছে যায়। এটিও কোনো ডিভাইসে বসানো যায় না।
  • Valid Host Range: নেটওয়ার্ক এবং ব্রডকাস্ট অ্যাড্রেসের মাঝখানের আইপিগুলো। এই আইপিগুলোই আমরা রাউটার, পিসি বা সার্ভারে ব্যবহার করি।

রিয়েল-লাইফ সিনারিও (The Apartment Analogy)

ধরুন একটি অ্যাপার্টমেন্ট বিল্ডিংয়ের ১০টি ফ্ল্যাট আছে (নম্বর ০ থেকে ৯)।

  • Network ID: বিল্ডিংয়ের মেইন গেট বা নাম (নম্বর ০)।
  • Broadcast ID: বিল্ডিংয়ের নোটিশ বোর্ড (নম্বর ৯)। যেখানে নোটিশ দিলে সবাই দেখতে পায়।
  • Host Range: ফ্ল্যাট নম্বর ১ থেকে ৮ পর্যন্ত। যেখানে মানুষ বসবাস করে।

২. ক্যালকুলেশন মেথড: The Magic Number Method

বাইনারি ক্যালকুলেশন ছাড়াই আমরা "ম্যাজিক নম্বর" ব্যবহার করে খুব দ্রুত এই অ্যাড্রেসগুলো বের করতে পারি। চলুন একটি উদাহরণের মাধ্যমে শিখি।

উদাহরণ আইপি: 192.168.10.45/26

ধাপ ১: সাবনেট মাস্ক বের করা

এখানে /26 মানে হলো প্রথম ২৬টি বিট 1। আমরা জানি একটি অকটেটে ৮টি বিট থাকে।

  • ১ম অকটেট: 8 bits (255)
  • ২য় অকটেট: 8 bits (255)
  • ৩য় অকটেট: 8 bits (255)
  • ৪র্থ অকটেট: বাকি থাকে ২ bits (128+64 = 192)

তাহলে সাবনেট মাস্ক হলো: 255.255.255.192

ধাপ ২: ম্যাজিক নম্বর বের করা

ম্যাজিক নম্বর বের করার সূত্র হলো: 256 - (আক্রান্ত অকটেটের মাস্ক ভ্যালু)।

এখানে আমাদের আক্রান্ত অকটেট হলো ৪র্থ অকটেট (যেখানে ১৯২ আছে)।

ম্যাজিক নম্বর = 256 - 192 = 64।

ধাপ ৩: Network Address বের করা

এখন আমরা দেখবো আইপির শেষ অকটেট (৪৫) আমাদের ম্যাজিক নম্বর (৬৪) এর কোন গুণিতক বা ব্লকের ভেতরে আছে।

  • ব্লকগুলো হলো: ০, ৬৪, ১২৮, ১৯২...
  • আমাদের আইপি ৪৫ হলো ০ এবং ৬৪ এর মাঝখানে।

সুতরাং, Network Address: 192.168.10.0

ধাপ ৪: Broadcast Address বের করা

পরবর্তী ব্লকের শুরুর আগের আইপিটিই হলো বর্তমান ব্লকের ব্রডকাস্ট অ্যাড্রেস।

পরবর্তী ব্লক শুরু হয়েছে ৬৪-তে। তাহলে এর আগের আইপি হলো ৬৩।

সুতরাং, Broadcast Address: 192.168.10.63

ধাপ ৫: Valid Host Range বের করা

নেটওয়ার্ক অ্যাড্রেসের পরবর্তী আইপি থেকে শুরু করে ব্রডকাস্ট অ্যাড্রেসের আগের আইপি পর্যন্ত।

  • First Host: 192.168.10.1
  • Last Host: 192.168.10.62

৩. কুইক রেফারেন্স টেবিল (Common Subnets)

নিচে সচরাচর ব্যবহৃত কিছু সাবনেটের তথ্য দেওয়া হলো:

CIDR Prefix Magic Number Total IPs Usable Hosts
/24 256 256 254
/25 128 128 126
/26 64 64 62
/27 32 32 30
/30 4 4 2

৪. প্রাকটিক্যাল কনফিগারেশন এবং ভেরিফিকেশন

আপনি যখন আপনার লিনাক্স পিসি বা MikroTik রাউটারে আইপি বসাবেন, তখন আপনি নিজেই এটি ভেরিফাই করতে পারবেন।

MikroTik এ ভেরিফাই করা:

/ip address add address=192.168.10.45/26 interface=ether1
/ip address print

ব্যাখ্যা: এখানে অ্যাড্রেস প্রিন্ট করলে আপনি দেখবেন রাউটার অটোমেটিক 'Network' কলামে 192.168.10.0 বসিয়ে নিয়েছে।

Linux এ চেক করা:

$ ip addr show

ব্যাখ্যা: এখানে 'brd' বা Broadcast ফিল্ডে আপনি আপনার ক্যালকুলেট করা ব্রডকাস্ট অ্যাড্রেসটি দেখতে পাবেন।

৫. Troubleshooting টিপস

  • আইপি কনফিগার হচ্ছে না: যদি আপনি ভুলবশত Network Address (যেমন .0) বা Broadcast Address (যেমন .63) পিসিতে বসাতে চান, তবে অপারেটিং সিস্টেম "Invalid IP Address" বা "Subnet mask and IP mismatch" এরর দেবে।
  • কানেক্টিভিটি নেই: যদি একই সাবনেটের ভেতর দুটি পিসি একে অপরকে পিং না পায়, তবে চেক করুন তাদের সাবনেট মাস্ক (CIDR) একই কি না। পিসি-১ যদি /২৪ এবং পিসি-২ যদি /২৬ হয়, তবে তাদের নেটওয়ার্ক সীমানা আলাদা হয়ে যাবে।
  • গেটওয়ে সমস্যা: রাউটারের আইপি বা গেটওয়ে সবসময় ওই Valid Host Range-এর মধ্যেই হতে হবে। সাধারণত রেঞ্জের প্রথম বা শেষ আইপিটি গেটওয়ে হিসেবে ব্যবহার করা প্রফেশনাল প্র্যাকটিস।

উপসংহার (Conclusion)

Network, Broadcast এবং Host addresses ক্যালকুলেশন করা নেটওয়ার্কিংয়ের একটি আবশ্যিক কারিগরি দক্ষতা। ম্যাজিক নম্বর মেথড ব্যবহার করে আপনি খুব সহজেই যেকোনো বড় বা ছোট নেটওয়ার্কের সীমানা নির্ধারণ করতে পারেন। এটি আপনাকে আইপি অপচয় রোধ করতে এবং নেটওয়ার্ক ট্রাবলশুটিং দ্রুত করতে সাহায্য করবে। নিয়মিত প্র্যাকটিস করলে আপনি যেকোনো CIDR নোটেশন দেখে মুহূর্তেই তার ডিটেইলস বলে দিতে পারবেন।
