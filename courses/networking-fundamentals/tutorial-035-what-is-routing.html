What is Routing? A Comprehensive Deep Dive

27 April, 2026 Networking Concepts Jobaer Hossain

কম্পিউটার নেটওয়ার্কিংয়ের দুনিয়ায় Routing হলো সবচেয়ে মৌলিক এবং গুরুত্বপূর্ণ ধারণা। আপনি যখন ইন্টারনেটে কোনো ওয়েবসাইটে ঢোকেন বা বন্ধুকে ইমেইল পাঠান, তখন আপনার ডাটা হাজার হাজার মাইল পাড়ি দিয়ে সঠিক গন্তব্যে পৌঁছায়। এই ডাটা প্যাকেটগুলোকে এক নেটওয়ার্ক থেকে অন্য নেটওয়ার্কে পাঠানোর সঠিক পথ খুঁজে বের করার প্রক্রিয়াটিই হলো রাউটিং।

আজকের এই টিউটোরিয়ালে আমরা রাউটিং এর একদম শুরু থেকে শেষ পর্যন্ত বিস্তারিত আলোচনা করবো। আমরা জানবো এটি কীভাবে কাজ করে এবং এর বিভিন্ন প্রকারভেদ সম্পর্কে।

১. Routing কী?

Routing হলো নেটওয়ার্ক লেয়ারে (OSI Model-এর Layer 3) ডাটা প্যাকেট পাঠানোর জন্য সবচেয়ে কার্যকর পথ নির্বাচন করার প্রক্রিয়া। একটি Router বিভিন্ন নেটওয়ার্কের মধ্যে সংযোগ স্থাপন করে এবং সিদ্ধান্ত নেয় যে কোন পথ দিয়ে ডাটা পাঠালে তা দ্রুত এবং নির্ভুলভাবে পৌঁছাবে।

রিয়েল-লাইফ সিনারিও (The GPS Analogy)

রাউটিংকে আপনি গুগল ম্যাপ বা জিপিএস (GPS) সিস্টেমের সাথে তুলনা করতে পারেন।

  • আপনার বর্তমান অবস্থান হলো Source।
  • আপনার গন্তব্য হলো Destination।
  • রাস্তাগুলো হলো Network Paths।
  • গুগল ম্যাপ হলো Routing Algorithm যা জ্যাম, রাস্তার দূরত্ব এবং গতি বিবেচনা করে আপনাকে সবচেয়ে ভালো পথটি দেখিয়ে দেয়।

২. Routing কীভাবে কাজ করে?

যখন একটি রাউটার কোনো ডাটা প্যাকেট পায়, সে প্যাকেটের গন্তব্য আইপি অ্যাড্রেস (Destination IP) চেক করে। এরপর রাউটার তার নিজস্ব একটি ডিজিটাল ডায়েরি বা Routing Table পরীক্ষা করে। এই টেবিলে লেখা থাকে কোন নেটওয়ার্কে যাওয়ার জন্য কোন পথ (Next Hop) ব্যবহার করতে হবে।

Routing Table-এর প্রধান উপাদানসমূহ:

  • Destination Network: যেখানে ডাটা পৌঁছাতে হবে।
  • Next Hop: পরবর্তী যে রাউটারে ডাটা পাঠাতে হবে।
  • Interface: রাউটারের কোন পোর্ট (Ethernet/Fiber) দিয়ে ডাটা বের হবে।
  • Metric/Cost: পথের দূরত্ব বা গুণমান নির্দেশক ভ্যালু।

৩. Routing-এর প্রকারভেদ (Types of Routing)

নেটওয়ার্কিংয়ে প্রধানত তিন ধরনের রাউটিং দেখা যায়:

Routing Type Description Pros & Cons
Static Routing অ্যাডমিনিস্ট্রেটর নিজে হাতে রুট বসিয়ে দেন। সুবিধা: সিকিউর এবং কম রিসোর্স লাগে। অসুবিধা: বড় নেটওয়ার্কে মেনটেইন করা কঠিন।
Dynamic Routing প্রোটোকল (OSPF, BGP) ব্যবহার করে রাউটার নিজেই পথ খুঁজে নেয়। সুবিধা: অটোমেটিক এবং স্কেলেবল। অসুবিধা: রাউটারের CPU/RAM বেশি খরচ হয়।
Default Routing অচেনা সব গন্তব্যের জন্য একটি নির্দিষ্ট পথ ব্যবহার করা। সুবিধা: কনফিগারেশন সহজ। সাধারণত ইন্টারনেট গেটওয়ের জন্য ব্যবহৃত হয়।

৪. Dynamic Routing Protocols

ডায়নামিক রাউটিং এর ক্ষেত্রে কিছু বিশেষ প্রোটোকল ব্যবহৃত হয় যা একে অপরের সাথে তথ্য বিনিময় করে নেটওয়ার্ক ম্যাপ তৈরি করে:

  • RIP (Routing Information Protocol): সবচেয়ে সহজ এবং পুরনো প্রোটোকল। এটি কতগুলো রাউটার (Hops) পার হতে হবে তার ওপর ভিত্তি করে পথ ঠিক করে।
  • OSPF (Open Shortest Path First): এটি লিংকের ব্যান্ডউইথ এবং অবস্থা দেখে সবচেয়ে দ্রুত পথটি বেছে নেয়।
  • BGP (Border Gateway Protocol): এটি ইন্টারনেটের প্রোটোকল। এক দেশ থেকে অন্য দেশে ডাটা পাঠানোর জন্য এটি ব্যবহৃত হয়।

৫. Practical CLI Commands ও কনফিগারেশন

রাউটার বা লিনাক্স সার্ভারে রাউটিং কনফিগারেশন এবং চেক করার কিছু সাধারণ কমান্ড নিচে দেওয়া হলো।

১. লিনাক্স বা উইন্ডোজ এ Routing Table দেখা:

# Windows
C:\> route print

# Linux
$ netstat -rn  অথবা  $ ip route show

২. MikroTik Router-এ Static Route যোগ করা:

/ip route add dst-address=192.168.20.0/24 gateway=192.168.10.1

ব্যাখ্যা: এখানে ১৯২.১৬৮.২০.০ নেটওয়ার্কে যাওয়ার জন্য ১০.১ গেটওয়েকে পথ হিসেবে সেট করা হয়েছে।

৬. Troubleshooting ও কমন সমস্যাসমূহ

  • Routing Loop: যখন ডাটা প্যাকেট গোলকধাঁধায় পড়ে যায় এবং গন্তব্যে না পৌঁছে রাউটারগুলোর মাঝে ঘুরতে থাকে। এটি প্রতিরোধ করতে TTL (Time to Live) ভ্যালু কাজ করে।
  • Destination Unreachable: এর মানে রাউটিং টেবিলে ওই গন্তব্যে যাওয়ার কোনো পথ নেই।
  • Asymmetric Routing: ডাটা এক পথ দিয়ে যায় কিন্তু ফিরে আসে অন্য পথ দিয়ে। এটি অনেক সময় ফায়ারওয়াল সমস্যা তৈরি করে।

উপসংহার (Conclusion)

Routing হলো ইন্টারনেটের অদৃশ্য ট্রাফিক পুলিশ। এটি ছাড়া আজকের বিশাল গ্লোবাল নেটওয়ার্ক পরিচালনা করা অসম্ভব ছিল। আপনি যখন একটি ছোট অফিসের ল্যান (LAN) তৈরি করেন বা একটি বিশাল টেলিকম নেটওয়ার্ক ডিজাইন করেন, রাউটিং এর সঠিক জ্ঞানই আপনাকে একজন দক্ষ নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে প্রতিষ্ঠিত করবে। স্ট্যাটিক রাউটিং দিয়ে শুরু করলেও বর্তমানের জটিল নেটওয়ার্কে ডায়নামিক রাউটিং এর কোনো বিকল্প নেই।
