Static vs Dynamic Routing: A Technical Deep Dive

27 April, 2026 Networking Concepts Jobaer Hossain

নেটওয়ার্কিং এর দুনিয়ায় একটি প্যাকেটকে তার গন্তব্যে পৌঁছানোর জন্য রাউটারকে অবশ্যই জানতে হয় কোন পথে যেতে হবে। এই পথটি রাউটারকে দুইভাবে শেখানো যায়: হয় আপনি নিজে হাতে পথটি দেখিয়ে দেবেন, নয়তো রাউটার নিজে থেকেই অন্য রাউটারদের সাথে কথা বলে পথ খুঁজে নেবে। এই দুই পদ্ধতিকেই আমরা যথাক্রমে Static Routing এবং Dynamic Routing বলি।

আজকের এই বিস্তারিত টিউটোরিয়ালে আমরা এই দুই পদ্ধতির কারিগরি পার্থক্য, সুবিধা, অসুবিধা এবং প্র্যাকটিক্যাল ব্যবহার নিয়ে গভীর আলোচনা করবো।

১. Static Routing কী?

Static Routing হলো এমন একটি প্রক্রিয়া যেখানে নেটওয়ার্ক অ্যাডমিনিস্ট্রেটর ম্যানুয়ালি রাউটারের Routing Table-এ গন্তব্য নেটওয়ার্ক এবং সেখানে যাওয়ার পথ (Next Hop) সেট করে দেন। রাউটার নিজে থেকে এই রুট পরিবর্তন বা আপডেট করতে পারে না।

রিয়েল-লাইফ সিনারিও (The Guided Tour)

ধরুন আপনি একটি নতুন শহরে বেড়াতে গেছেন। আপনার গাইড আপনাকে একটি ম্যাপ দিয়ে বলল, "আপনি যদি নীলক্ষেত যেতে চান, তবে অবশ্যই সায়েন্স ল্যাব হয়ে যাবেন।" এটি হলো Static Routing। রাস্তায় যদি জ্যাম থাকে বা রাস্তা বন্ধ থাকে, তবুও আপনি ওই ম্যাপ অনুযায়ীই যাওয়ার চেষ্টা করবেন যতক্ষণ না গাইড আপনাকে নতুন নির্দেশনা দিচ্ছে।

সুবিধাসমূহ:

  • Low Resource Consumption: রাউটারের CPU এবং RAM অনেক কম খরচ হয় কারণ রাউটারকে কোনো জটিল অ্যালগরিদম চালাতে হয় না।
  • High Security: অ্যাডমিনিস্ট্রেটর সম্পূর্ণ কন্ট্রোল করতে পারেন ডাটা কোন পথ দিয়ে যাবে।
  • Bandwidth Saving: রাউটারগুলোর মধ্যে কোনো রুট আপডেট বিনিময় হয় না, ফলে নেটওয়ার্ক ব্যান্ডউইথ সাশ্রয় হয়।

অসুবিধাসমূহ:

  • Lack of Scalability: বিশাল নেটওয়ার্কে শ'য়ে শ'য়ে রুট ম্যানুয়ালি বসানো প্রায় অসম্ভব।
  • Manual Intervention: কোনো লিংক ডাউন হলে নেটওয়ার্ক নিজে থেকে রিকভার করতে পারে না; অ্যাডমিনিস্ট্রেটরকে নিজে এসে রুট পরিবর্তন করতে হয়।

২. Dynamic Routing কী?

Dynamic Routing এমন একটি পদ্ধতি যেখানে রাউটারগুলো বিশেষ প্রোটোকল (যেমন: OSPF, EIGRP, BGP) ব্যবহার করে একে অপরের সাথে নেটওয়ার্ক তথ্য শেয়ার করে। নেটওয়ার্কের ম্যাপে কোনো পরিবর্তন হলে রাউটারগুলো স্বয়ংক্রিয়ভাবে তাদের Routing Table আপডেট করে নেয়।

রিয়েল-লাইফ সিনারিও (Google Maps)

আপনি নীলক্ষেত যাওয়ার জন্য গুগল ম্যাপ চালু করলেন। ম্যাপ আপনাকে দেখাল সায়েন্স ল্যাবে জ্যাম আছে, তাই আপনি ধানমন্ডি ২ নম্বর দিয়ে ঘুরে যান। এটিই হলো Dynamic Routing। ম্যাপ নিজেই জ্যাম বা রাস্তার অবস্থা দেখে রিয়েল-টাইমে পথ পরিবর্তন করে দিচ্ছে।

সুবিধাসমূহ:

  • Auto-Discovery: রাউটার নিজে থেকেই নতুন নেটওয়ার্ক খুঁজে পায়।
  • Fault Tolerance: কোনো লিংক ডাউন হলে প্রোটোকল নিজে থেকেই অল্টারনেটিভ বা ব্যাকআপ পথ খুঁজে বের করে ডাটা পাঠিয়ে দেয়।
  • Scalability: বড় বড় এন্টারপ্রাইজ নেটওয়ার্ক বা ইন্টারনেটের জন্য এটি অপরিহার্য।

অসুবিধাসমূহ:

  • Resource Intensive: রাউটিং প্রোটোকল এবং অ্যালগরিদম চালানোর জন্য রাউটারের CPU এবং RAM বেশি প্রয়োজন হয়।
  • Complexity: কনফিগারেশন এবং ট্রাবলশুটিং স্ট্যাটিক রাউটিং এর চেয়ে কিছুটা জটিল।

৩. Static vs Dynamic Routing: তুলনামূলক আলোচনা

ফিচার (Feature) Static Routing Dynamic Routing
Configuration ম্যানুয়ালি করতে হয় (Manual) স্বয়ংক্রিয়ভাবে হয় (Automatic)
Topology Changes অ্যাডমিনিস্ট্রেটরকে রুট আপডেট করতে হয়। নিজে থেকেই আপডেট হয়ে যায়।
Algorithm নেই। OSPF, BGP, RIP ইত্যাদি ব্যবহৃত হয়।
Network Size ছোট নেটওয়ার্কের জন্য ভালো। বড় এবং জটিল নেটওয়ার্কের জন্য আদর্শ।
Security অধিক নিরাপদ (Higher Security)। কম নিরাপদ (নিরাপত্তা নিশ্চিত করতে এক্সট্রা কনফিগারেশন লাগে)।

৪. প্র্যাকটিক্যাল CLI কনফিগারেশন

Static Routing কনফিগারেশন (MikroTik):

ধরুন ১৯২.১৬৮.২০.০/২৪ নেটওয়ার্কে যাওয়ার জন্য আমাদের ১০.১ গেটওয়ে ব্যবহার করতে হবে।

/ip route add dst-address=192.168.20.0/24 gateway=192.168.10.1

Dynamic Routing (OSPF) কনফিগারেশন (MikroTik):

রাউটারকে ওএসপিএফ-এ যুক্ত করা এবং নিজের ল্যান নেটওয়ার্ক পাবলিশ করা:

/routing ospf instance add name=default router-id=1.1.1.1
/routing ospf interface-template add networks=192.168.10.0/24 area=main

৫. Troubleshooting টিপস

  • Administrative Distance (AD): মনে রাখবেন, যদি একই গন্তব্যের জন্য স্ট্যাটিক এবং ডায়নামিক উভয় রুট থাকে, রাউটার স্ট্যাটিক রুটকেই পছন্দ করবে কারণ স্ট্যাটিক রুটের AD ভ্যালু কম (সাধারণত 1)।
  • Connectivity Check: স্ট্যাটিক রুটে গেটওয়ে আইপি পিং (Ping) পাওয়া যাচ্ছে কি না চেক করুন। গেটওয়ে ডাউন থাকলে রুটটি ইনঅ্যাক্টিভ (Inactive) হয়ে যাবে।
  • Routing Loop: স্ট্যাটিক রাউটিং এ ভুল কনফিগারেশনের কারণে লুপ হতে পারে। ডায়নামিক রাউটিং এ এই সমস্যা সাধারণত প্রোটোকল নিজেই হ্যান্ডেল করে।
  • Convergence Time: ডায়নামিক রাউটিং এ লিংক ডাউনের পর কত দ্রুত ব্যাকআপ রুট সচল হচ্ছে তা নজর রাখুন। একে Convergence Time বলে।

উপসংহার (Conclusion)

Static Routing এবং Dynamic Routing—কোনটি সেরা তার উত্তর নির্ভর করে আপনার নেটওয়ার্ক ডিজাইনের ওপর। যদি নেটওয়ার্ক ছোট হয় এবং খুব একটা পরিবর্তন না হয়, তবে স্ট্যাটিক রাউটিং সেরা। কিন্তু আপনার নেটওয়ার্ক যদি ক্রমবর্ধমান হয় এবং নিরবিচ্ছিন্ন সংযোগের প্রয়োজন থাকে, তবে Dynamic Routing এর কোনো বিকল্প নেই। একজন প্রফেশনাল ইঞ্জিনিয়ার হিসেবে আপনাকে উভয় পদ্ধতিরই গভীর জ্ঞান রাখতে হবে যাতে প্রয়োজনে আপনি Static-Dynamic Hybrid নেটওয়ার্ক তৈরি করতে পারেন।
