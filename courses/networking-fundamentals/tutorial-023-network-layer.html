Network Layer in OSI Model: A Deep Dive Guide

27 April, 2026 Networking Concepts Jobaer Hossain

কম্পিউটার নেটওয়ার্কিংয়ের জগতটি সাতটি স্তরে বিভক্ত, যাকে আমরা OSI (Open Systems Interconnection) Model হিসেবে জানি। এই সাতটি স্তরের মধ্যে তিন নম্বর স্তর বা Network Layer হলো ইন্টারনেটের মেরুদণ্ড। আপনি যখন আপনার ব্রাউজারে কোনো ওয়েবসাইটের নাম লিখে এন্টার দেন, তখন সেই ডাটা কীভাবে হাজার মাইল দূরে থাকা একটি সার্ভারে পৌঁছাবে, তার ম্যাপ তৈরি করে এই লেয়ারটি।

আজকের এই টিউটোরিয়ালে আমরা Network Layer-এর কার্যাবলী, প্রোটোকল এবং এর ভেতরের মেকানিজম নিয়ে বিস্তারিত আলোচনা করবো।

[Image of Network Layer in OSI Model diagram]

১. Network Layer কী?

Network Layer হলো OSI Model-এর ৩ নম্বর লেয়ার। এর প্রধান কাজ হলো ডাটা প্যাকেটগুলোকে (Packets) সোর্স (Source) থেকে ডেস্টিনেশন (Destination) পর্যন্ত পৌঁছে দেওয়ার জন্য সবচেয়ে ছোট এবং কার্যকর পথ খুঁজে বের করা। এই প্রক্রিয়াটিকে বলা হয় Routing।

ডাটা লিংক লেয়ার (Layer 2) যেখানে শুধুমাত্র একই লোকাল নেটওয়ার্কের মধ্যে ডাটা পাঠাতে পারে, সেখানে Network Layer ভিন্ন ভিন্ন নেটওয়ার্কের মধ্যে যোগাযোগ স্থাপন করে।

রিয়েল-লাইফ সিনারিও (The Postal Service Analogy)

ধরুন, আপনি ঢাকা থেকে চট্টগ্রামে একটি চিঠি পাঠাবেন।

  • Layer 2 (Data Link): এটি আপনার বাসার পাশের পোস্ট অফিসের মতো, যারা শুধু আপনার এলাকা চিনে।
  • Layer 3 (Network Layer): এটি কেন্দ্রীয় পোস্টাল সিস্টেমের মতো। সে জানে ঢাকা থেকে চট্টগ্রামে যাওয়ার জন্য কোন রাস্তাটি খোলা আছে, কোথায় জ্যাম কম এবং কোন ট্রান্সপোর্ট ব্যবহার করলে দ্রুত পৌঁছানো যাবে। সে চিঠির ওপর একটি জিপ কোড (IP Address) ব্যবহার করে গন্তব্য নিশ্চিত করে।

২. Network Layer-এর প্রধান কার্যাবলী (Key Functions)

Network Layer মূলত দুটি গুরুত্বপূর্ণ কাজ সম্পাদন করে:

A. Logical Addressing (IP Addressing)

প্রতিটি ডিভাইসের একটি ইউনিক পরিচয় থাকতে হয় যাতে নেটওয়ার্কে তাকে খুঁজে পাওয়া যায়। Network Layer প্রতিটি ডাটা প্যাকেটের সাথে সোর্স এবং ডেস্টিনেশন IP Address যুক্ত করে দেয়। একে বলা হয় Logical Addressing। এটি Physical Address বা MAC Address থেকে আলাদা কারণ আইপি অ্যাড্রেস নেটওয়ার্ক পরিবর্তনের সাথে সাথে পরিবর্তিত হতে পারে।

B. Routing

নেটওয়ার্কের এক প্রান্ত থেকে অন্য প্রান্তে যাওয়ার একাধিক রাস্তা থাকতে পারে। Network Layer-এর কাজ হলো Routing Algorithms ব্যবহার করে শ্রেষ্ঠ পথটি (Best Path) নির্বাচন করা। এই কাজটি করার জন্য Router নামক হার্ডওয়্যার ডিভাইস ব্যবহৃত হয়।

৩. Network Layer Protocols

Network Layer-এ বিভিন্ন ধরনের প্রোটোকল কাজ করে, তবে নিচে উল্লিখিত প্রোটোকলগুলো সবচেয়ে বেশি জনপ্রিয়:

প্রোটোকল (Protocol) কাজের বিবরণ (Description)
IP (IPv4 & IPv6) ইন্টারনেট প্রোটোকল। এটি প্যাকেটের অ্যাড্রেসিং এবং ডেলিভারি নিশ্চিত করে। বর্তমানে আমরা IPv4 এবং IPv6 উভয়ই ব্যবহার করি।
ICMP Internet Control Message Protocol. এটি মূলত নেটওয়ার্কের ভুল বা এরর মেসেজ এবং ডায়াগনস্টিকস (যেমন Ping) এর জন্য ব্যবহৃত হয়।
IGMP Internet Group Management Protocol. এটি মাল্টিকাস্টিং (Multicasting) ট্রাফিক ম্যানেজ করতে ব্যবহৃত হয়।
IPsec ডাটাকে সুরক্ষিত এবং এনক্রিপ্ট করার জন্য ব্যবহৃত প্রোটোকল স্যুট, যা সাধারণত VPN-এ ব্যবহৃত হয়।

৪. Routing Algorithms ও Path Determination

রাউটার কীভাবে সিদ্ধান্ত নেয় কোন পথে ডাটা যাবে? রাউটার তার Routing Table চেক করে এবং কিছু অ্যালগরিদম ব্যবহার করে:

  • Distance Vector Routing: এটি প্যাকেটের গন্তব্যে পৌঁছাতে কতগুলো রাউটার (Hops) পার হতে হবে তার ওপর ভিত্তি করে পথ ঠিক করে (যেমন: RIP)।
  • Link State Routing: এটি নেটওয়ার্কের বর্তমান অবস্থা, জ্যাম এবং স্পিড চেক করে শ্রেষ্ঠ পথটি বেছে নেয় (যেমন: OSPF)।

৫. Troubleshooting ও CLI কমান্ড

Network Layer-এর সমস্যা ট্রাবলশুট করার জন্য কিছু কমান্ড নেটওয়ার্ক অ্যাডমিনদের নিয়মিত ব্যবহার করতে হয়।

১. Connectivity চেক করা (Ping):

C:\> ping 8.8.8.8

এটি ICMP প্রোটোকল ব্যবহার করে সোর্স এবং ডেস্টিনেশনের মধ্যে কানেক্টিভিটি চেক করে।

২. পথের ম্যাপ দেখা (Traceroute):

C:\> tracert google.com

এই কমান্ডটি দেখায় আপনার ডাটা প্যাকেটটি গন্তব্যে পৌঁছাতে মাঝপথে কোন কোন রাউটার অতিক্রম করছে।

কমন সমস্যা ও সমাধান:

  • Request Timed Out: এর মানে গন্তব্য রাউটারটি ডাউন অথবা ফায়ারওয়াল আপনার প্যাকেট ব্লক করছে।
  • Network Unreachable: এর মানে রাউটার গন্তব্যে যাওয়ার কোনো পথ খুঁজে পাচ্ছে না। রাউটিং টেবিল চেক করা প্রয়োজন।
  • IP Conflict: যখন একই নেটওয়ার্কে দুটি ডিভাইসে একই আইপি দেওয়া হয়। DHCP ব্যবহার করলে এই সমস্যা এড়ানো যায়।

উপসংহার (Conclusion)

Network Layer ছাড়া আজকের ইন্টারনেট কল্পনা করা অসম্ভব। এটি শুধুমাত্র ডাটা এক জায়গা থেকে অন্য জায়গায় পাঠায় না, বরং নিশ্চিত করে যে ডাটাটি সবচেয়ে নিরাপদ এবং দ্রুততম পথে পৌঁছাচ্ছে। একজন নেটওয়ার্ক ইঞ্জিনিয়ার বা আইটি প্রফেশনাল হিসেবে এই লেয়ারটির মেকানিজম এবং আইপি অ্যাড্রেসিং সম্পর্কে স্বচ্ছ ধারণা থাকা অত্যন্ত জরুরি।
