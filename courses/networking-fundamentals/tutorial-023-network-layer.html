Network Layer in OSI Model

February 27, 2026 Networking Jobaer Hossain

Network Layer হলো ওএসআই (OSI) Model তৃতীয় Layer যা Logical Addressing, Routing এবং Interconnected Network-গুলোর মাধ্যমে End-to-End Packet Delivery-র জন্য দায়ী।

  • Device-এর Logical (IP) Addressing Operation করে।
  • Network-গুলোর মধ্যে সর্বোত্তম Routing Path নির্ধারণ করে।
  • Multiple Network জুড়ে Host-to-Host যোগাযোগ সক্ষম করে।

যেকোনো Data Link Layer-এর বিপরীতে, যা শুধুমাত্র একটি Single Network Segment-এর মধ্যে Node-to-Node Delivery-তে Focus করে, Network Layer নিশ্চিত করে যে Data Source Host থেকে Destination Host-এ পৌঁছাবে, এমনকি তারা ভিন্ন ভিন্ন Network-এ থাকলেও।

Network Layer-এর প্রধান Responsibility সমূহ

  • Logical Addressing: প্রতিটি Device-এ একটি Unique IP Address প্রদান করে, যা সঠিক যোগাযোগ নিশ্চিত করে।
  • Packetization: Transport Layer থেকে আসা Segment-গুলোকে Packet-এ রূপান্তর করে।
  • Host-to-Host Delivery: বিভিন্ন Network-এর মাধ্যমে Sender থেকে Receiver কাছে Packet-এর Reliable Delivery নিশ্চিত করে।
  • Forwarding: Destination IP-এর ভিত্তিতে Router-এর Input Interface থেকে Output Interface-এ Packet পাঠায়।
  • Routing: Routing Algorithm ব্যবহার করে Packet-এর জন্য সবচেয়ে ভালো Path নির্ধারণ করে।
  • Fragmentation and Reassembly: বড় Packet-গুলোকে ছোট ছোট Chunk করে এবং Destination এ আবার Pair লাগায়।
  • Subnetting: বড় Network-কে ছোট ছোট অংশে ভাগ করে Traffic Management সহজ করে।
  • NAT (Network Address Translation): Private IP-কে Public IP-তে রূপান্তর করে Address Space সাশ্রয় করে।

Network Layer কীভাবে কাজ করে?

  1. প্রতিটি Device-কে একটি Unique Logical অ্যাড্রেস (IP Address) দেওয়া হয়।
  2. Transport Layer-এর Data-কে Packet-এ রূপান্তর করা হয়, যেখানে Source এবং Destination IP যুক্ত থাকে।
  3. Router Destination Address বিশ্লেষণ করে এবং সেরা Path নির্ধারণ করে।
  4. Packet-গুলো এক Router থেকে অন্য Router-এ (hop-by-hop) Destination এ পৌঁছানো পর্যন্ত চলতে থাকে।
  5. Destination এ Packet-গুলো পুনরায় Pair লাগানো হয় এবং উপরের Layer-এ পাঠানো হয়।

Network Layer-এর Protocol

প্রধান Protocol

  • IP (v4/v6): Logical Addressing এবং Packet Delivery নিশ্চিত করে।
  • ICMP: Error Report এবং Diagnostic Message পাঠায় (যেমন: Ping)।
  • ARP: IP Address-কে MAC Address-এ রূপান্তর করে।
  • RARP: MAC Address ব্যবহার করে Device-এর IP Address খুঁজে বের করে।
  • IPSec: Encryption-এর মাধ্যমে IP যোগাযোগ সুরক্ষিত করে।
  • MPLS: Label ব্যবহারের মাধ্যমে Packet Forwarding সহজ ও Traffic Management Advanced করে।

Routing Protocol

  • RIP: Distance বা Hop Count-এর ওপর ভিত্তি করে Path Selection করে।
  • OSPF: Network Topology ব্যবহার করে সংক্ষিপ্ততম Path খুঁজে বের করে।
  • BGP: Internet-এ বিভিন্ন Autonomous System-এর মধ্যে Data Route করে।

Network Layer-এর সুবিধা

  • ভিন্ন ভিন্ন Network-এর মধ্যে End-to-End যোগাযোগ সম্ভব করে।
  • Subnetting এবং Hierarchical Addressing-এর মাধ্যমে Scalability Support করে।
  • Dynamic Routing Algorithm ব্যবহার করে কার্যকরভাবে Packet পাঠায়।

Network Layer-এর সীমাবদ্ধতা

  • সরাসরি কোনো Flow Control মেকানিজম নেই।
  • Reliable তার জন্য মূলত উপরের Layer-গুলোর (যেমন TCP) ওপর নির্ভর করে।
  • অতিরিক্ত Load-এর সময় Router Packet Drop করতে পারে, যা Data Loss ঘটাতে পারে।

Routing vs Flooding

Feature রাউটিং (Routing) ফ্লাডিং (Flooding)
Routing Table Table প্রয়োজন হয়। Table প্রয়োজন হয় না।
সংক্ষিপ্ত Path সবসময় হতে পারে না। সবসময় সংক্ষিপ্ত Path নিশ্চিত করে।
Reliable তা Relatively low। অত্যন্ত Reliable।
Traffic Traffic কম থাকে। অত্যধিক Traffic সৃষ্টি হয়।
Duplicate Packet থাকে না। অনেক Duplicate Packet থাকে।

Summary

Network Layer হলো Modern Internet-এর মেরুদণ্ড। এটি ছাড়া বিভিন্ন Network-এর মধ্যে যোগাযোগ অসম্ভব হতো। এটি Data Packet-গুলোকে সঠিক Destination এ পৌঁছানোর জন্য Logical Address এবং কার্যকরি Routing Path নিশ্চিত করে।
