Variable Length Subnet Masking (VLSM): A Deep Dive Guide

27 April, 2026 Networking Concepts Jobaer Hossain

সাবনেটিংয়ের দুনিয়ায় VLSM (Variable Length Subnet Masking) হলো সবচেয়ে শক্তিশালী এবং আধুনিক পদ্ধতি। আপনি যদি সাধারণ সাবনেটিং জেনে থাকেন, তবে আপনি জানেন যে সেখানে সবকটি সাবনেটের সাইজ সমান হয় (Fixed Length)। কিন্তু বাস্তবে একটি অফিসের সব ডিপার্টমেন্টে সমান সংখ্যক পিসি থাকে না। এখানেই আমাদের ত্রাণকর্তা হিসেবে কাজ করে VLSM।

সহজ কথায়, একটি বড় নেটওয়ার্ক ব্লককে প্রয়োজন অনুযায়ী বিভিন্ন সাইজের ছোট ছোট সাবনেটে ভাগ করার পদ্ধতিই হলো VLSM। আজকের এই বিস্তারিত ব্লগে আমরা VLSM এর লজিক এবং ক্যালকুলেশন নিয়ে গভীর আলোচনা করবো।

১. VLSM কী এবং কেন এটি প্রয়োজন?

Variable Length Subnet Masking বা VLSM হলো এমন একটি মেকানিজম যা একই নেটওয়ার্কে একাধিক Subnet Mask ব্যবহার করার অনুমতি দেয়। এটি Classless Inter-Domain Routing (CIDR) এর একটি অংশ।

রিয়েল-লাইফ সিনারিও (The Apartment Partition Analogy)

ধরুন আপনার কাছে একটি বড় ফ্লোর আছে। এখন আপনার রিকোয়ারমেন্ট হলো:

  • Sales Team: ১০০ জন মানুষ (বড় রুম প্রয়োজন)।
  • IT Team: ৫০ জন মানুষ (মাঝারি রুম প্রয়োজন)।
  • HR Team: ২০ জন মানুষ (ছোট রুম প্রয়োজন)।

আপনি যদি সব রুম সমান সাইজের (Fixed Length) করেন এবং ১০০ জনের জন্য বড় সাইজ বেছে নেন, তবে HR টিমের ২০ জনের জন্য বিশাল রুম দেওয়া হবে, যা জায়গার প্রচণ্ড অপচয়। VLSM আপনাকে সেই স্বাধীনতা দেয় যেখানে আপনি Sales এর জন্য বড়, IT এর জন্য মাঝারি এবং HR এর জন্য ছোট রুম কাটতে পারবেন। নেটওয়ার্কিংয়ে এটি IP Address এর অপচয় রোধ করে।

২. FLSM বনাম VLSM

এই দুটির মূল পার্থক্য বুঝতে নিচের টেবিলটি লক্ষ্য করুন:

বৈশিষ্ট্য (Feature) FLSM (Fixed Length) VLSM (Variable Length)
Subnet Mask সব সাবনেটে একই মাস্ক থাকে। প্রয়োজন অনুযায়ী ভিন্ন ভিন্ন মাস্ক থাকে।
IP Efficiency আইপি অপচয় বেশি হয়। আইপি অপচয় সর্বনিম্ন হয়।
Implementation সহজ। কিছুটা জটিল ও পরিকল্পিত হতে হয়।
Routing Protocols RIP v1 সাপোর্ট করে। OSPF, EIGRP, RIP v2 সাপোর্ট করে।

৩. VLSM ক্যালকুলেশন করার গোল্ডেন রুলস

VLSM করার সময় এই নিয়মগুলো অবশ্যই মানতে হবে:

  • বৃহত্তম থেকে ক্ষুদ্রতম: সবসময় সবচেয়ে বড় রিকোয়ারমেন্ট (যেখানে বেশি আইপি লাগবে) দিয়ে শুরু করতে হয় এবং ক্রমানুসারে ছোটর দিকে যেতে হয়।
  • ম্যাজিক নম্বর মেথড: প্রতিটি সাবনেটের জন্য আলাদা ম্যাজিক নম্বর (Block Size) বের করতে হয়।
  • ব্যাবহারযোগ্য হোস্ট (Usable Hosts): প্রতিটি সাবনেটে নেটওয়ার্ক এবং ব্রডকাস্ট অ্যাড্রেস বাদ দিয়ে হিসাব করতে হয় ($2^n - 2$)।

৪. প্র্যাকটিক্যাল উদাহরণ ও সমাধান

ধরি আমাদের মাস্টার আইপি ব্লক হলো 192.168.1.0/24। আমাদের নিচের তিনটি নেটওয়ার্ক প্রয়োজন:

  1. Network A (Sales): ১০০টি হোস্ট প্রয়োজন।
  2. Network B (IT): ৫০টি হোস্ট প্রয়োজন।
  3. Network C (Point-to-Point): ২টি হোস্ট প্রয়োজন।

ধাপ ১: Network A (Sales - ১০০ হোস্ট)

  • ১০০ হোস্টের জন্য বিট প্রয়োজন ৭টি ($2^7 = 128$)।
  • CIDR হবে: $32 - 7 = /25$।
  • নেটওয়ার্ক অ্যাড্রেস: 192.168.1.0/25।
  • ব্রডকাস্ট অ্যাড্রেস: 192.168.1.127।

ধাপ ২: Network B (IT - ৫০ হোস্ট)

  • ৫০ হোস্টের জন্য বিট প্রয়োজন ৬টি ($2^6 = 64$)।
  • পরবর্তী এভেলেবল আইপি 128 থেকে শুরু হবে।
  • CIDR হবে: $32 - 6 = /26$।
  • নেটওয়ার্ক অ্যাড্রেস: 192.168.1.128/26।
  • ব্রডকাস্ট অ্যাড্রেস: 192.168.1.191।

ধাপ ৩: Network C (P2P - ২ হোস্ট)

  • ২ হোস্টের জন্য বিট প্রয়োজন ২টি ($2^2 = 4$)।
  • পরবর্তী এভেলেবল আইপি 192 থেকে শুরু হবে।
  • CIDR হবে: $32 - 2 = /30$।
  • নেটওয়ার্ক অ্যাড্রেস: 192.168.1.192/30।
  • ব্রডকাস্ট অ্যাড্রেস: 192.168.1.195।

৫. CLI কনফিগারেশন এবং ভেরিফিকেশন

রাউটারে এই ভিন্ন ভিন্ন সাবনেট মাস্কের আইপি বসানো অত্যন্ত সহজ।

MikroTik CLI Command:

/ip address add address=192.168.1.1/25 interface=ether1 comment="Sales"
/ip address add address=192.168.1.129/26 interface=ether2 comment="IT"
/ip address add address=192.168.1.193/30 interface=ether3 comment="WAN-Link"

৬. Troubleshooting ও সচরাচর ভুলসমূহ

  • Overlapping Subnets: এটি VLSM এর সবচেয়ে কমন ভুল। যদি আপনি বড় রিকোয়ারমেন্ট দিয়ে শুরু না করেন, তবে একটি সাবনেট অন্যটির আইপি সীমানার ভেতর ঢুকে যেতে পারে। এর ফলে রাউটিং কাজ করবে না।
  • Wrong CIDR Usage: অনেক সময় ভুলবশত ভুল স্ল্যাশ নোটেশন দিলে হোস্ট সংখ্যা কমে যায়, ফলে পিসি কানেক্ট হতে পারে না।
  • Static Route: VLSM নেটওয়ার্কে স্ট্যাটিক রুট দেওয়ার সময় অবশ্যই সাবনেট মাস্ক সঠিকভাবে বসাতে হবে, অন্যথায় ট্রাফিক ভুল দিকে চলে যাবে।

উপসংহার (Conclusion)

Variable Length Subnet Masking (VLSM) হলো একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ারের দক্ষতাকে প্রমাণ করার হাতিয়ার। এটি আইপি অ্যাড্রেসিংকে করে তোলে নমনীয় এবং অত্যন্ত সাশ্রয়ী। VLSM এর মাধ্যমে একটি আইপি ব্লককে মাস্টারপিস ডিজাইনে রূপান্তর করা সম্ভব যা বড় বড় এন্টারপ্রাইজ নেটওয়ার্ককে করে তোলে অনেক বেশি দক্ষ এবং সিকিউর। নিয়মিত প্র্যাকটিস এবং বড় থেকে ছোট এই নিয়মটি মেনে চললে VLSM আপনার কাছে পানির মতো সহজ মনে হবে।
