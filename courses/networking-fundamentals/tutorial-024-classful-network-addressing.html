Classful IP Addressing: A Comprehensive Deep Dive

27 April, 2026 Networking Concepts Jobaer Hossain

ইন্টারনেটে সংযুক্ত প্রতিটি ডিভাইসের একটি অনন্য ঠিকানা প্রয়োজন, যাকে আমরা IP Address বলি। আদিতে যখন ইন্টারনেট প্রোটোকল ভার্সন ৪ (IPv4) ডিজাইন করা হয়েছিল, তখন নেটওয়ার্কগুলোকে সুশৃঙ্খলভাবে সাজানোর জন্য একটি আর্কিটেকচার ব্যবহার করা হতো যার নাম Classful Addressing। যদিও বর্তমানে আমরা Classless Addressing (CIDR) বেশি ব্যবহার করি, কিন্তু নেটওয়ার্কিংয়ের ভিত্তি বোঝার জন্য এবং সাবনেটিং (Subnetting) শেখার জন্য Classful IP Addressing জানা অত্যন্ত জরুরি।

আজকের এই টিউটোরিয়ালে আমরা IPv4-এর ক্লাসগুলো নিয়ে বিস্তারিত আলোচনা করবো এবং জানবো কীভাবে একটি আইপি দেখে তার নেটওয়ার্ক ও হোস্ট বের করা যায়।

১. Classful Addressing কী?

Classful Addressing হলো এমন একটি পদ্ধতি যেখানে ৩২-বিটের (32-bit) IPv4 অ্যাড্রেসকে পাঁচটি সুনির্দিষ্ট ক্লাসে ভাগ করা হয় (Class A, B, C, D, এবং E)। প্রতিটি ক্লাসের একটি নির্দিষ্ট Subnet Mask থাকে যা দিয়ে বোঝা যায় আইপির কতটুকু অংশ নেটওয়ার্কের জন্য এবং কতটুকু হোস্টের জন্য।

রিয়েল-লাইফ সিনারিও (The City Analogy)

ধরুন একটি শহরের ঠিকানা ব্যবস্থা:

  • Class A: এটি বড় বড় দেশের মতো, যেখানে অল্প কয়েকটি বড় বড় এলাকা (Network) আছে কিন্তু প্রতি এলাকায় কোটি কোটি বাড়ি (Host) থাকতে পারে।
  • Class B: এটি মাঝারি শহরের মতো, যেখানে এলাকা এবং বাড়ির সংখ্যা একটি ভারসাম্যপূর্ণ অবস্থায় থাকে।
  • Class C: এটি ছোট পাড়া বা অফিসের মতো, যেখানে এলাকার সংখ্যা অনেক বেশি হতে পারে কিন্তু প্রতি এলাকায় অল্প কয়েকটি বাড়ি থাকে।

২. IP Address-এর গঠন (IP Structure)

একটি IPv4 অ্যাড্রেসে ৪টি অকটেট (Octet) থাকে এবং প্রতিটি অকটেট ৮ বিট করে। মোট ৩২ বিট।

  • Network ID (NID): এটি নির্দেশ করে ডাটা প্যাকেটটি কোন নেটওয়ার্কে যাবে।
  • Host ID (HID): এটি নির্দেশ করে ওই নেটওয়ার্কের ভেতর কোন নির্দিষ্ট ডিভাইসে ডাটা যাবে।

৩. Classful IP-র প্রকারভেদ

নিচে প্রতিটি ক্লাসের রেঞ্জ এবং তাদের গঠন বিশ্লেষণ করা হলো:

Class First Octet Range Network / Host Bits Default Subnet Mask
Class A 1 – 126 8 Net / 24 Host 255.0.0.0
Class B 128 – 191 16 Net / 16 Host 255.255.0.0
Class C 192 – 223 24 Net / 8 Host 255.255.255.0
Class D 224 – 239 Multicast Address N/A
Class E 240 – 255 Research/Experimental N/A

Class A: বিশাল নেটওয়ার্কের জন্য

প্রথম বিট সবসময় 0 থাকে। এর প্রথম অকটেট নেটওয়ার্কের জন্য এবং বাকি ৩টি অকটেট হোস্টের জন্য। এটি বিশাল কোম্পানি বা সরকারি প্রতিষ্ঠানের জন্য ব্যবহৃত হতো। এখানে মোট $2^{24} - 2$ বা ১৬,৭৭৭,২১৪ টি হোস্ট সম্ভব।

নোট: ১২৭ (127.x.x.x) রেঞ্জটি লুপব্যাক (Loopback) বা নিজের পিসি টেস্ট করার জন্য সংরক্ষিত।

Class B: মাঝারি নেটওয়ার্কের জন্য

প্রথম দুই বিট সবসময় 10 থাকে। প্রথম ২ অকটেট নেটওয়ার্ক এবং বাকি ২ অকটেট হোস্ট। এখানে প্রতিটি নেটওয়ার্কে ৬৫,৫৩৪ টি হোস্ট সম্ভব।

Class C: ছোট নেটওয়ার্কের জন্য

প্রথম তিন বিট সবসময় 110 থাকে। প্রথম ৩ অকটেট নেটওয়ার্ক এবং শেষ ১ অকটেট হোস্ট। এটি ছোট অফিস বা বাসার জন্য সবচেয়ে জনপ্রিয়। এখানে প্রতিটি নেটওয়ার্কে মাত্র ২৫৪ টি হোস্ট থাকতে পারে।

Class D এবং E: বিশেষ উদ্দেশ্যে

  • Class D: এটি Multicast এর জন্য ব্যবহৃত হয় (যেমন রাউটিং প্রোটোকল আপডেট বা স্ট্রিমিং)। এর কোনো নির্দিষ্ট হোস্ট আইডি নেই।
  • Class E: এটি বৈজ্ঞানিক গবেষণা এবং পরীক্ষামূলক কাজের জন্য সংরক্ষিত। সাধারণ ইউজাররা এটি ব্যবহার করতে পারেন না।

৪. Classful Addressing-এর সীমাবদ্ধতা

Classful সিস্টেমটি আগে বেশ কার্যকর থাকলেও ইন্টারনেটের প্রসারের সাথে সাথে এতে বড় সমস্যা দেখা দেয়:

  • IP Waste (অপচয়): ধরুন আপনার একটি নেটওয়ার্কে ৫০০টি আইপি প্রয়োজন। আপনি Class C নিলে হবে না (কারণ মাত্র ২৫৪টি সম্ভব), তাই আপনাকে Class B নিতে হবে যেখানে ৬৫,৫৩৪টি আইপি থাকে। এখানে প্রায় ৬৫,০০০ আইপি অপচয় হচ্ছে।
  • Rigidity: ফিক্সড মাস্ক থাকার কারণে নেটওয়ার্ক ডিজাইন ফ্লেক্সিবল ছিল না।

এই সমস্যা সমাধানের জন্যই পরবর্তীতে Classless Inter-Domain Routing (CIDR) বা স্ল্যাশ নোটেশন (যেমন /25) চালু করা হয়।

৫. Troubleshooting ও প্র্যাকটিক্যাল টিপস

নেটওয়ার্ক ট্রাবলশুটিং করার সময় আইপি দেখে ক্লাস শনাক্ত করা জরুরি:

  • যদি আইপি হয় 10.0.0.5, প্রথম অকটেট 10, তাই এটি Class A। সাবনেট মাস্ক হবে 255.0.0.0।
  • যদি আইপি হয় 192.168.1.1, প্রথম অকটেট 192, তাই এটি Class C। সাবনেট মাস্ক হবে 255.255.255.0।

কিভাবে বুঝবেন আপনার পিসির আইপি ক্লাস কত?

আপনার টার্মিনাল বা কমান্ড প্রম্পটে গিয়ে লিখুন:

C:\> ipconfig (Windows)
$ ifconfig (Linux/Mac)

সেখানে আপনার IPv4 Address এবং Subnet Mask দেখে সহজেই ক্লাস মিলিয়ে নিতে পারবেন।

উপসংহার (Conclusion)

Classful IP Addressing আধুনিক নেটওয়ার্কিংয়ে সরাসরি ব্যবহৃত না হলেও এটি শেখা অপরিহার্য। এটি আমাদের শেখায় কীভাবে নেটওয়ার্ক ও হোস্ট বিভক্ত হয় এবং কীভাবে ডাটা প্যাকেট সঠিক গন্তব্যে পৌঁছানোর জন্য লজিক্যাল ঠিকানা ব্যবহার করে। এই ধারণাটি পরিষ্কার থাকলে Subnetting এবং VLSM (Variable Length Subnet Masking) শেখা আপনার জন্য অনেক সহজ হবে।
