Framing in Data Link Layer: Detailed Guide

February 01, 2026 Networking Jobaer Hossain

Framing কী?

একটি Frame হলো Data Link Layer-এ Data Transmission-এর মৌলিক Single। এটি Bit-এর একটি সংগঠিত গোষ্ঠী যেখানে মূল Information ছাড়াও Address ও Error Detection-এর মতো Control Information থাকে।

Standard Ethernet Frame Structure

Preamble7 Bytes
SFD1 Byte
Dest MAC6 Bytes
Src MAC6 Bytes
Type/Len2 Bytes
Payload (Data & Padding)46 - 1500 Bytes
FCS (CRC)4 Bytes
  • Frame-এ Address ও Error Detection-এর মতো Control Information থাকে।
  • এটি Data Link লেয়ার (DLL)-এ সম্পাদিত হয়।
  • Bit stream-কে ছোট ও অর্থবহ Single ে বিভক্ত করে।
  • Sender ও Receiver-এর মধ্যে Synchronization নিশ্চিত করে।
  • Receiver-এর কাছে Reliable Transmission ও সঠিক Processing নিশ্চিত করে।
উল্লেখ্য: Framing Time Division Multiplexing (TDM)-এর মতো Technique-এও অত্যন্ত গুরুত্বপূর্ণ, যেখানে Data নির্দিষ্ট Time Slot-এ পাঠানো হয়। Ethernet, Token Ring এবং Frame Relay-এর মতো প্রযুক্তি সুনির্দিষ্ট Frame Format ব্যবহার করে।

Framing-এর Purpose (Purpose of Framing)

  • Frame Boundary নির্ধারণ: স্পষ্ট সীমানা নির্ধারণ করে যাতে Receiver প্রতিটি Frame সঠিকভাবে Symbol িত করতে পারে।
  • Address সংযোজন: Source ও Destination Address বহন করে সঠিক Point-to-Point যোগাযোগ নিশ্চিত করে।
  • Error Detection & Correction: Data Integrity বজায় রাখতে সহায়তা করে।
  • Reliable Transmission: সুসংগঠিত ও Reliable Data পাঠানো নিশ্চিত করে।
মনে রাখুন: Framing ও Reassembly প্রক্রিয়া সম্পূর্ণভাবে Data Link Layer দ্বারা পরিচালিত হয় এবং ব্যবহারকারীর কাছে এটি সম্পূর্ণ Transparent থাকে।

Framing-এর Classification (Types of Framing)

1. Fixed-Size Framing

এই Method তে প্রতিটি Frame-এর একটি পূর্ব Fixed স্থায়ী Size থাকে।

  • Frame-এর Length ই সীমানা নির্ধারণ করে বলে Separate Delimiter-এর প্রয়োজন হয় না।
  • অসুবিধা: যখন Data Frame-এর আকারের চেয়ে ছোট হয়, তখন Internal Fragmentation হয়, যা Bandwidth-এর অপচয় করে।
  • সমাধান: অব্যবহৃত স্থান পূরণ করতে Padding ব্যবহার করা হয়।

2. Variable-Size Framing

এই Method তে Frame-এর শুরু ও শেষ Symbol িত করার জন্য স্পষ্ট সীমানা Instruction ক ব্যবহার করা হয়। দুটি ুপায়ে এটি করা হয়:

Length Field:

Frame-এর মধ্যে একটি Field থাকে যা Frame-এর Size Instruction করে (IEEE 802.3 Ethernet-এ ব্যবহৃত)। এই Field নষ্ট হলে Receiver Frame-টি সঠিকভাবে বুঝতে পারে না।

End Delimiter:

একটি অনন্য Pattern ব্যবহার করে Frame-এর শেষ Symbol িত করা হয় (Token Ring-এ ব্যবহৃত)। তবে এই Pattern Data-র ভেতরেও থাকতে পারে, যা সমস্যা তৈরি করে।

এই সমস্যা সমাধানের জন্য Stuffing Technique ব্যবহার করা হয়:

(ক) Character/Byte Stuffing

  • Character-ভিত্তিক Data-র Field ে ব্যবহৃত হয়।
  • যদি Data-র মধ্যে বিশেষ Delimiter Character (যেমন $) থাকে, তাহলে তার আগে একটি Escape Character (যেমন \O) যোগ করা হয়।
Example:
ED (End Delimiter) = $
Original Data: Hello $ World
After Stuffing: Hello \O$ World

Disadvantage: Processing overhead বেশি এবং Modern সিস্টে মেএটি পুরনো।

(খ) Bit Stuffing

  • End Delimiter (ED) হলো একটি Bit Sequence, যেমন 01111।
  • যখনই Data-য় এই Pattern দেখা যায়, বিভ্রান্তি এড়াতে একটি অতিরিক্ত 0 Bit প্রবেশ করানো হয়।
Example 1:
Original Data: 011100011110
ED Pattern: 0111
After Bit Stuffing: 011010001101100

Example 2:
Original Data: 110001001
ED Pattern: 1000
After Bit Stuffing: 11001010011

Framing-এর সমস্যাসমূহ (Problems in Framing)

শুরু ও শেষ শনাক্তকরণ:

Receiver অবশ্যই delimiter বা flag ব্যবহার করে Frame সীমানা সঠিকভাবে Symbol িত করতে হবে।

Synchronization:

বেশি-Speed-র Network-এ Sender ও Receiver-কে Frame-এর Timing-এ একমত থাকতে হবে।

Error Handling:

Noise Data বা Delimiter নষ্ট করতে পারে। CRC বা Checksum-এর মতো Method ব্যবহার করা হয়।

Overhead:

Header ও Trailer Control Information বহন করে কিন্তু ব্যবহারযোগ্য Bandwidth কমিয়ে দেয়।

Efficiency:

Overhead ও Processing বিলম্ব কমিয়ে Payload সর্বাধিক করাই লক্ষ্য।

Compatibility:

ভিন্ন Device ভিন্ন Framing Method ব্যবহার করতে পারে, যা Data-র ভুল ব্যাখ্যা করার কারণ হতে পারে।

Summary

Framing হলো Data Link Layer-এর একটি মৌলিক প্রক্রিয়া যা Bit-এর অবিচ্ছিন্ন প্রবাহকে অর্থবহ Single ে বিভক্ত করে। Fixed-size Framing সহজ হলেও Padding-এর প্রয়োজন হয়, আর Variable-size Framing Flexible হলেও Stuffing Technique-এর মাধ্যমে Delimiter-এর সমস্যা সামলাতে হয়। সঠিক Framing ছাড়া Reliable Network যোগাযোগ সম্ভব নয়।
