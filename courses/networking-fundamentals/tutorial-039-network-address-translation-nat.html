Network Address Translation (NAT): A Deep Dive Guide

27 April, 2026 Networking with MikroTik Jobaer Hossain

কম্পিউটার নেটওয়ার্কিংয়ের সবচেয়ে গুরুত্বপূর্ণ এবং বহুল ব্যবহৃত একটি প্রযুক্তির নাম হলো Network Address Translation বা সংক্ষেপে NAT। আপনি বর্তমানে যে ডিভাইসটি দিয়ে এই ব্লগটি পড়ছেন, সেটিও ইন্টারনেটের সাথে যুক্ত হওয়ার জন্য কোনো না কোনোভাবে NAT ব্যবহার করছে। এটি মূলত আমাদের লোকাল নেটওয়ার্কের Private IP-কে ইন্টারনেটে ব্যবহারযোগ্য Public IP-তে রূপান্তর করার কাজ করে।

আজকের এই টিউটোরিয়ালে আমরা NAT-এর খুঁটিনাটি, এর প্রয়োজনীয়তা এবং বিভিন্ন ধরনের NAT কনফিগারেশন নিয়ে বিস্তারিত আলোচনা করবো।

১. NAT কী এবং কেন এটি প্রয়োজন?

IPv4 অ্যাড্রেস সিস্টেমে মাত্র ৪.৩ বিলিয়ন আইপি সম্ভব, যা বর্তমান বিশ্বের ডিভাইসের সংখ্যার তুলনায় অত্যন্ত কম। এই আইপি সংকট দূর করার জন্য Private IP (যেমন: 192.168.1.0/24) তৈরি করা হয়েছে। কিন্তু Private IP সরাসরি ইন্টারনেটে কাজ করে না।

রাউটার যখন লোকাল নেটওয়ার্ক থেকে আসা কোনো ডাটা প্যাকেট ইন্টারনেটে পাঠায়, তখন সে প্যাকেটের সোর্স হিসেবে আপনার লোকাল আইপি মুছে দিয়ে নিজের Public IP বসিয়ে দেয়। এই প্রক্রিয়াটিই হলো NAT।

রিয়েল-লাইফ সিনারিও (The Office Intercom Analogy)

একটি অফিসের কথা চিন্তা করুন যেখানে অনেকগুলো ডেস্ক আছে এবং প্রতিটি ডেস্কে একটি করে Intercom (Private IP) আছে। অফিসের বাইরের কেউ সরাসরি ইন্টারকম নাম্বারে কল দিতে পারে না। তাদের অফিসের মেইন ল্যান্ডলাইন নাম্বারে (Public IP) কল দিতে হয়। অফিসের রিসেপশনিস্ট (Router) তখন কলটি বুঝে নিয়ে সঠিক ডেস্কে পাঠিয়ে দেয়। NAT ঠিক এই রিসেপশনিস্টের মতোই কাজ করে।

২. NAT-এর প্রকারভেদ (Types of NAT)

নেটওয়ার্কের প্রয়োজন অনুযায়ী NAT-কে প্রধানত তিনটি ভাগে ভাগ করা যায়:

NAT Type Description Use Case
Static NAT একটি Private IP-র বিপরীতে একটি নির্দিষ্ট Public IP ম্যাপ করা হয় (1-to-1)। সার্ভার হোস্টিংয়ের ক্ষেত্রে ব্যবহৃত হয়।
Dynamic NAT একটি পাবলিক আইপি পুল (Pool) থেকে ফ্রি আইপি নিয়ে ডাইনামিকভাবে ম্যাপ করা হয়। লিমিটেড পাবলিক আইপি থাকলে বড় নেটওয়ার্কে ব্যবহৃত হয়।
PAT (Port Address Translation) একটি মাত্র Public IP ব্যবহার করে হাজার হাজার লোকাল ইউজার ইন্টারনেট চালায় (Many-to-1)। বাসা বা অফিসের সাধারণ ইন্টারনেট কানেকশনের জন্য।

৩. PAT বা NAT Masquerade কীভাবে কাজ করে?

আজকাল আমরা যে NAT সবচেয়ে বেশি ব্যবহার করি তা হলো PAT। যখন একাধিক ইউজার একই পাবলিক আইপি ব্যবহার করে ইন্টারনেটে যায়, রাউটার তখন প্রতিটি ইউজারের রিকোয়েস্ট আলাদা করার জন্য Port Number ব্যবহার করে। রাউটার একটি NAT Translation Table তৈরি করে যেখানে লেখা থাকে কোন লোকাল আইপি কোন পোর্টের মাধ্যমে কোন ওয়েবসাইট ভিজিট করছে।

৪. কনফিগারেশন ধাপসমূহ (MikroTik CLI)

MikroTik রাউটারে NAT কনফিগার করা অত্যন্ত সহজ। নিচে সাধারণ ইন্টারনেট শেয়ারিং বা Masquerade করার কমান্ড দেওয়া হলো:

১. সাধারণ ইন্টারনেট শেয়ারিং (Source NAT):

/ip firewall nat add chain=srcnat out-interface=ether1 action=masquerade

ব্যাখ্যা: এখানে ether1 হলো আপনার WAN বা ইন্টারনেট পোর্ট। এই কমান্ডের মাধ্যমে ল্যানের সব ইউজার ইন্টারনেটে যেতে পারবে।

২. পোর্ট ফরওয়ার্ডিং (Destination NAT):

যদি আপনার লোকাল নেটওয়ার্কে কোনো ওয়েব সার্ভার (Port 80) থাকে এবং আপনি চান বাইরের মানুষ আপনার পাবলিক আইপি দিয়ে সেই সার্ভারটি দেখুক:

/ip firewall nat add chain=dstnat dst-address=1.1.1.1 protocol=tcp dst-port=80 action=dst-nat to-addresses=192.168.1.10 to-ports=80

ব্যাখ্যা: এখানে 1.1.1.1 হলো আপনার পাবলিক আইপি এবং 192.168.1.10 হলো আপনার লোকাল সার্ভার।

৫. Troubleshooting এবং কমন সমস্যাসমূহ

  • Double NAT Issue: যদি আপনার আইএসপি থেকে একটি রাউটার দেওয়া থাকে এবং তার পেছনে আপনি নিজের রাউটার লাগান, তবে অনেক সময় গেম বা ভিপিএন-এ সমস্যা হয়। একে Double NAT বলে। সমাধান হলো প্রথম রাউটারকে Bridge Mode-এ চালানো।
  • Port Conflict: যদি একাধিক সার্ভিস একই পোর্টে চালাতে চান, তবে আপনাকে Custom Ports ব্যবহার করতে হবে।
  • NAT Table Full: রাউটারের র‍্যাম কম হলে অনেক সময় NAT টেবিল ফুল হয়ে যায়, ফলে ইন্টারনেট স্লো হয়ে যায় বা ডিসকানেক্ট হয়। সেক্ষেত্রে Timeout পিরিয়ড কমিয়ে দিতে হয়।

ভেরিফিকেশন কমান্ড (MikroTik):

/ip firewall nat print
/ip firewall connection print

ব্যাখ্যা: এই কমান্ডের মাধ্যমে আপনি বর্তমানে রাউটারে কতগুলো NAT সেশন সচল আছে তা দেখতে পাবেন।

উপসংহার (Conclusion)

Network Address Translation (NAT) শুধুমাত্র আইপি সংরক্ষণের একটি পদ্ধতি নয়, এটি নেটওয়ার্ক সিকিউরিটির একটি স্তর হিসেবেও কাজ করে কারণ এটি আপনার লোকাল আইপিকে সরাসরি ইন্টারনেটে প্রকাশ করে না। বড় প্রতিষ্ঠানের নেটওয়ার্ক ডিজাইন থেকে শুরু করে বাসার ওয়াইফাই রাউটার—সবখানেই NAT অপরিহার্য। একজন নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে NAT-এর সঠিক মেকানিজম জানা থাকলে আপনি ট্রাবলশুটিং এবং সার্ভার পাবলিশিংয়ের কাজগুলো অত্যন্ত নিখুঁতভাবে করতে পারবেন।
