Distance Vector Routing: Concepts, Mechanisms and Deep Dive

27 April, 2026 Networking Concepts Jobaer Hossain

ডায়নামিক রাউটিং-এর জগতে সবচেয়ে পুরনো এবং সহজ মেকানিজম হলো Distance Vector Routing। আপনি যদি নেটওয়ার্কিং নিয়ে পড়াশোনা শুরু করেন, তবে RIP (Routing Information Protocol) এর নাম অবশ্যই শুনে থাকবেন, যা এই Distance Vector মেথডের ওপর ভিত্তি করেই কাজ করে। যদিও আধুনিক এন্টারপ্রাইজ নেটওয়ার্কে Link State প্রোটোকল বেশি জনপ্রিয়, কিন্তু ছোট নেটওয়ার্ক এবং রাউটিং-এর মূল ভিত্তি বোঝার জন্য এটি জানা অত্যন্ত জরুরি।

আজকের এই টিউটোরিয়ালে আমরা শিখবো Distance Vector Routing কীভাবে কাজ করে, এর পেছনে থাকা গাণিতিক অ্যালগরিদম এবং এর সীমাবদ্ধতাগুলো কী কী।

১. Distance Vector Routing কী?

Distance Vector Routing হলো এমন একটি পদ্ধতি যেখানে প্রতিটি রাউটার তার পুরো নেটওয়ার্ক টপোলজি সম্পর্কে জানে না। সে শুধুমাত্র জানে একটি নির্দিষ্ট গন্তব্যে পৌঁছাতে হলে তাকে কোন দিক দিয়ে যেতে হবে (Vector) এবং গন্তব্যটি কত দূরে (Distance)।

এই পদ্ধতিতে প্রতিটি রাউটার নিয়মিত বিরতিতে তার নিজস্ব Routing Table এর একটি কপি তার সরাসরি যুক্ত প্রতিবেশীদের (Neighbors) কাছে পাঠিয়ে দেয়। এভাবেই ধীরে ধীরে পুরো নেটওয়ার্কে রুট ইনফরমেশন ছড়িয়ে পড়ে।

রিয়েল-লাইফ সিনারিও (The Rumor-Based Routing)

Distance Vector কে অনেক সময় "Routing by Rumor" বলা হয়। ধরুন আপনি একটি নতুন রাস্তায় দাঁড়িয়ে আছেন এবং আপনি জানেন না গন্তব্য কোথায়।

  • আপনি আপনার পাশের বাড়ির একজনকে জিজ্ঞেস করলেন। সে বলল, "ডান দিকে ৩ কিলোমিটার গেলে আপনি গন্তব্য পাবেন।"
  • আপনি ওই ব্যক্তির কথা বিশ্বাস করে এগিয়ে গেলেন। আপনি নিজে কিন্তু পুরো শহর বা রাস্তা দেখেননি, আপনি শুধুমাত্র পাশের মানুষের দেওয়া তথ্যের ওপর ভিত্তি করে সিদ্ধান্ত নিয়েছেন। এটিই হলো Distance Vector।

২. এটি কীভাবে কাজ করে? (Mechanism)

Distance Vector Routing মূলত নিচের তিনটি বিষয়ের ওপর ভিত্তি করে কাজ করে:

  • Distance: গন্তব্যে পৌঁছাতে কতগুলো রাউটার পার হতে হবে, যাকে আমরা Hop Count বলি।
  • Vector: কোন ইন্টারফেস বা কোন প্রতিবেশী রাউটার হয়ে যেতে হবে।
  • Periodic Updates: নির্দিষ্ট সময় পর পর (যেমন RIP-এর ক্ষেত্রে ৩০ সেকেন্ড) রাউটার তার পুরো টেবিল প্রতিবেশীকে দেয়।

Bellman-Ford Algorithm

Distance Vector Routing এর মূল ভিত্তি হলো Bellman-Ford Algorithm। এটি নিচের সূত্রের মাধ্যমে কাজ করে:

dx(y) = minv{c(x,v) + dv(y)}

সহজ ভাষায়: রাউটার X থেকে গন্তব্য Y তে যাওয়ার খরচ হলো—রাউটার X থেকে তার প্রতিবেশী V তে যাওয়ার খরচ যোগ প্রতিবেশী V থেকে গন্তব্য Y তে যাওয়ার খরচের সমষ্টির সর্বনিম্ন মান।

৩. Distance Vector Routing এর সমস্যাসমূহ

এই প্রোটোকলটি অনেক সহজ হলেও এতে কিছু কারিগরি ত্রুটি রয়েছে যা নেটওয়ার্ক অ্যাডমিনদের জন্য মাথাব্যথার কারণ হয়ে দাঁড়ায়:

A. Slow Convergence

নেটওয়ার্কে কোনো পরিবর্তন হলে সব রাউটারের কাছে সেই তথ্য পৌঁছাতে অনেক সময় লাগে। একে বলা হয় Slow Convergence।

B. Count to Infinity Problem

যদি কোনো লিঙ্ক ডাউন হয়ে যায়, তবে রাউটারগুলো অনেক সময় নিজেদের মধ্যে ভুল তথ্য শেয়ার করে একটি লুপ তৈরি করে ফেলে। তারা মনে করে গন্তব্যটি অন্য কোনো রাউটার হয়ে এখনও পাওয়া যাচ্ছে। এভাবে Hop Count বাড়তে বাড়তে অসীম (Infinity) পর্যন্ত চলে যেতে পারে।

৪. সমাধান: Loop Prevention Mechanisms

উপরের সমস্যাগুলো সমাধানের জন্য রাউটার কিছু মেকানিজম ব্যবহার করে:

  • Maximum Hop Count: যেমন RIP-এ সর্বোচ্চ ১৫ হপ পর্যন্ত যাওয়া যায়। ১৬ নম্বর হপকে অকেজো (Unreachable) ধরা হয়।
  • Split Horizon: যে ইন্টারফেস দিয়ে একটি রুট সম্পর্কে জানা গেছে, সেই ইন্টারফেস দিয়ে ওই রুটের আপডেট আর পাঠানো যাবে না।
  • Poison Reverse: কোনো রুট ডাউন হলে সেটিকে সাথে সাথে অসীম ভ্যালু (যেমন ১৬) দিয়ে পাবলিশ করা হয় যাতে অন্যরা ওই রুটটি ডিলিট করে দেয়।

৫. সুবিধা ও অসুবিধা

সুবিধা (Pros) অসুবিধা (Cons)
Simple Configuration এটি কনফিগার করা অত্যন্ত সহজ।
Low Hardware Demand রাউটারের CPU এবং RAM অনেক কম ব্যবহার করে কারণ এর অ্যালগরিদম অনেক সহজ।
Network Loops রাউটিং লুপ হওয়ার সম্ভাবনা অনেক বেশি।
Bandwidth Waste পুরো রাউটিং টেবিল নিয়মিত বিরতিতে পাঠানোর ফলে ব্যান্ডউইথ অপচয় হয়।

৬. Practical CLI Configuration (RIP on MikroTik)

Distance Vector প্রোটোকল হিসেবে RIP কনফিগার করার পদ্ধতি নিচে দেওয়া হলো।

১. RIP ইন্টারফেস এবং নেটওয়ার্ক যুক্ত করা:

/routing rip instance add name=rip-instance
/routing rip interface-template add instance=rip-instance interfaces=all
/routing rip network add network=192.168.10.0/24

ব্যাখ্যা: এখানে রাউটার তার ১৯২.১৬৮.১০.০ নেটওয়ার্কটি তার প্রতিবেশীদের সাথে শেয়ার করবে।

৭. Troubleshooting টিপস

  • Metric Check: যদি দেখেন গন্তব্যে পৌঁছানো যাচ্ছে না, তবে /routing rip route print কমান্ড দিয়ে দেখুন হপ কাউন্ট কত। যদি ১৬ দেখায়, তবে বুঝতে হবে রুটটি আনরিচেবল।
  • Update Interval: রাউটার ৩০ সেকেন্ড পর পর আপডেট পাঠাচ্ছে কি না তা লগে চেক করুন।
  • Version Mismatch: RIP Version 1 এবং Version 2 এর মধ্যে পার্থক্য আছে। সবসময় Version 2 ব্যবহার করার চেষ্টা করুন কারণ এটি সাবনেট মাস্ক সাপোর্ট করে।

উপসংহার (Conclusion)

Distance Vector Routing যদিও অনেক পুরনো পদ্ধতি, কিন্তু নেটওয়ার্কিং-এর মূল ভিত্তি বুঝতে এর কোনো বিকল্প নেই। এর সহজ ইমপ্লিমেন্টেশন এবং লো-রিসোর্স ব্যবহার একে ছোট নেটওয়ার্কের জন্য আজও উপযোগী করে রেখেছে। তবে লুপ হওয়া এবং স্লো কনভারজেন্স-এর কারণে বড় কর্পোরেট নেটওয়ার্কে এখন OSPF বা EIGRP এর মতো অ্যাডভান্সড প্রোটোকল ব্যবহার করা হয়। একজন নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে এই পদ্ধতির গাণিতিক লজিক বোঝা আপনার ক্যারিয়ারের জন্য একটি বড় প্লাস পয়েন্ট।
