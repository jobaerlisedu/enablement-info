What is TCP/IP Model? - Layers of TCP/IP Model

26 April, 2026 Network Engineering Jobaer Hossain

আমরা আগেই জেনেছি যে, OSI Model হলো নেটওয়ার্কিং বোঝার জন্য একটি থিওরেটিক্যাল বা লজিক্যাল ফ্রেমওয়ার্ক। কিন্তু বাস্তবে, আমাদের বর্তমান ইন্টারনেট ব্যবস্থা সম্পূর্ণভাবে যে আর্কিটেকচারের ওপর দাঁড়িয়ে আছে, তা হলো TCP/IP Model। আপনি যখন মোবাইল বা পিসি থেকে ইন্টারনেটে কোনো কিছু ব্রাউজ করেন, তখন আপনার ডিভাইস ব্যাকগ্রাউন্ডে এই TCP/IP প্রোটোকলগুলোই ব্যবহার করে।

আজকের এই ডিপ-ডাইভ আর্টিকেলে আমরা শিখবো TCP/IP Model কী, এটি কীভাবে কাজ করে এবং এর ৪টি Layer-এর বিস্তারিত কার্যাবলী।

TCP/IP Model কী?

TCP/IP (Transmission Control Protocol/Internet Protocol) Model হলো একটি প্র্যাকটিক্যাল নেটওয়ার্কিং মডেল এবং প্রোটোকল স্যুট (Protocol Suite)। এটি ১৯৭০-এর দশকে আমেরিকার ডিপার্টমেন্ট অফ ডিফেন্স (DoD) এর প্রজেক্ট ARPANET এর অংশ হিসেবে তৈরি করা হয়েছিল।

এই মডেলের প্রধান উদ্দেশ্য ছিল এমন একটি নেটওয়ার্ক তৈরি করা, যার কোনো একটি অংশ (যেমন একটি রাউটার বা ক্যাবল) নষ্ট হয়ে গেলেও যেন পুরো নেটওয়ার্ক ডাউন না হয়ে যায় এবং বিকল্প রাস্তায় ডাটা পাঠানো সম্ভব হয়। এই আর্কিটেকচারটিই আজকের আধুনিক ইন্টারনেটের ভিত্তি।

১. OSI Model বনাম TCP/IP Model

OSI মডেলের ৭টি লেয়ারকে একত্রিত করে TCP/IP মডেলে মাত্র ৪টি লেয়ারে (Layer) ভাগ করা হয়েছে। চলুন একটি তুলনামূলক ছক দেখে নিই:

[Image of TCP/IP model layers compared to OSI model]
OSI Model (7 Layers) TCP/IP Model (4 Layers) Function (কাজ)
Application, Presentation, Session Application Layer ইউজার ইন্টারফেস এবং ডাটা ফরম্যাটিংয়ের কাজ করে।
Transport Layer Transport Layer এন্ড-টু-এন্ড ডাটা ডেলিভারি এবং Error Connection নিশ্চিত করে।
Network Layer Internet Layer রাউটিং এবং IP Addressing নিয়ে কাজ করে।
Data Link, Physical Layer Network Access Layer হার্ডওয়্যার, ক্যাবল এবং MAC Address নিয়ে কাজ করে।

২. TCP/IP Model এর ৪টি Layer এর বিস্তারিত বর্ণনা

এখন আমরা প্রতিটি Layer এর কাজ এবং এখানে ব্যবহৃত প্রোটোকলগুলো সম্পর্কে বিস্তারিত জানবো।

Layer 4: Application Layer

TCP/IP মডেলের সবচেয়ে উপরের লেয়ার হলো Application Layer। OSI মডেলের উপরের তিনটি লেয়ারের (Application, Presentation, Session) কাজ এই একটি লেয়ারই করে দেয়। ইউজার সরাসরি এই লেয়ারের মাধ্যমেই নেটওয়ার্কের সাথে যুক্ত হয়।

  • প্রধান কাজ: সফটওয়্যার অ্যাপ্লিকেশনগুলোকে নেটওয়ার্ক সার্ভিস প্রদান করা এবং ডাটাকে এনকোড/ডিকোড করা।
  • ব্যবহৃত প্রোটোকল (Protocols):
    • ওয়েব ব্রাউজিংয়ের জন্য: HTTP, HTTPS
    • ফাইল ট্রান্সফারের জন্য: FTP, TFTP
    • ইমেইলের জন্য: SMTP, POP3, IMAP
    • নাম রেজল্যুশনের জন্য: DNS
  • রিয়েল-লাইফ উদাহরণ: আপনি যখন ক্রোম ব্রাউজারে youtube.com লিখেন, তখন এই লেয়ারের HTTP/HTTPS প্রোটোকল আপনার রিকোয়েস্টটি তৈরি করে।

Layer 3: Transport Layer

Application Layer থেকে আসা ডাটাকে নির্ভরযোগ্যভাবে গন্তব্যে পৌঁছানোর দায়িত্ব এই লেয়ারের। ডাটা বড় হলে এটি তাকে ছোট ছোট টুকরোতে (যাকে Segment বলা হয়) ভাগ করে।

  • প্রধান কাজ: Segmentation, Connection Multiplexing এবং Error Checking। এই লেয়ারে ডাটার সাথে একটি Port Number যুক্ত করা হয়, যাতে রিসিভার বুঝতে পারে ডাটাটি কোন অ্যাপ্লিকেশনে পাঠাতে হবে।
  • প্রধান দুটি প্রোটোকল:
    • TCP (Transmission Control Protocol): এটি অত্যন্ত রিলায়েবল (Reliable)। ডাটা ঠিকমতো পৌঁছেছে কি না তার একটি প্রাপ্তিস্বীকার বা Acknowledgment (ACK) এটি রিসিভ করে। ওয়েব ব্রাউজিং বা ইমেইলে এটি ব্যবহৃত হয়।
    • UDP (User Datagram Protocol): এটি আনরিলায়েবল কিন্তু অত্যন্ত দ্রুতগতির। ডাটা পৌঁছালো কি না, তা এটি চেক করে না। লাইভ ভিডিও স্ট্রিমিং (যেমন: Zoom Call, YouTube Live) বা গেমিংয়ে এটি ব্যবহৃত হয়।

Layer 2: Internet Layer

এই লেয়ারটি OSI মডেলের Network Layer-এর সমতুল্য। এটি পুরো TCP/IP মডেলের প্রাণকেন্দ্র। Transport Layer থেকে আসা Segment-গুলোকে এখানে Packet-এ রূপান্তর করা হয় এবং এদের গায়ে সোর্স ও ডেস্টিনেশনের IP Address যুক্ত করা হয়।

  • প্রধান কাজ: লজিক্যাল অ্যাড্রেসিং (IP Addressing) এবং রাউটিং (Routing)। অর্থাৎ, ডাটা প্যাকেটকে সবচেয়ে সেরা এবং দ্রুততম রাস্তা দিয়ে গন্তব্যে পৌঁছানো।
  • ব্যবহৃত প্রোটোকল:
    • IP (Internet Protocol): IPv4 এবং IPv6—যার মাধ্যমে প্রতিটি ডিভাইসকে চেনা যায়।
    • ICMP (Internet Control Message Protocol): এটি এরর মেসেজ এবং ট্রাবলশুটিংয়ের জন্য কাজ করে (যেমন ping কমান্ড)।
  • ডিভাইস: রাউটার (Router) মূলত এই লেয়ারে কাজ করে।

Layer 1: Network Access Layer (বা Link Layer)

এটি হলো TCP/IP মডেলের সবচেয়ে নিচের লেয়ার। OSI মডেলের Data Link এবং Physical Layer-এর কাজ এটি একাই করে। Internet Layer থেকে আসা প্যাকেটগুলোকে এখানে Frame-এ রূপান্তর করে এর সাথে MAC Address যুক্ত করা হয়। এরপর সেগুলোকে Bit (0 এবং 1) হিসেবে ফিজিক্যাল মিডিয়ামের ওপর দিয়ে পাঠানো হয়।

  • প্রধান কাজ: হার্ডওয়্যার অ্যাড্রেসিং (MAC) এবং ফিজিক্যাল সিগন্যাল (যেমন ইলেকট্রিক্যাল পালস, লাইট বা রেডিও ওয়েভ) ট্রান্সমিট করা।
  • প্রোটোকল এবং প্রযুক্তি: Ethernet, Wi-Fi (802.11), ARP (Address Resolution Protocol)।
  • ডিভাইস: Switch, Hub, Network Interface Card (NIC) এবং ক্যাবল।

৩. TCP/IP নেটওয়ার্ক ট্রাবলশুটিং (CLI Commands)

একজন নেটওয়ার্ক অ্যাডমিনিস্ট্রেটর হিসেবে আপনাকে এই লেয়ারগুলোর বিভিন্ন সমস্যা Command Line Interface (CLI) দিয়ে ট্রাবলশুট করতে হবে। নিচে বাস্তব জীবনের কিছু উদাহরণ দেওয়া হলো:

Transport Layer চেক করা (Netstat):

আপনার পিসিতে কোন কোন TCP বা UDP পোর্ট খোলা আছে এবং কোন আইপির সাথে কানেকশন তৈরি হয়েছে, তা দেখতে নিচের কমান্ডটি ব্যবহার করা হয়:

C:\> netstat -an

ব্যাখ্যা: আউটপুটে যদি ESTABLISHED লেখা দেখেন, তার মানে আপনার ডিভাইসটি সার্ভারের সাথে সফলভাবে যুক্ত আছে।

Internet Layer চেক করা (Ping):

IP Address ঠিকমতো কাজ করছে কি না বা রাউটিংয়ে কোনো সমস্যা আছে কি না, তা ICMP প্রোটোকল ব্যবহার করে চেক করা হয়:

C:\> ping 8.8.8.8

Network Access Layer চেক করা (ARP):

আপনার পিসি কোন IP Address-এর বিপরীতে কোন MAC Address স্টোর করে রেখেছে (যাকে ARP Cache বলা হয়), তা দেখার জন্য:

C:\> arp -a

টিপস: অনেক সময় IP কনফ্লিক্ট বা ডুপ্লিকেট আইপি সমস্যা সমাধানের জন্য এই কমান্ডটি ম্যাজিকের মতো কাজ করে।

Conclusion (উপসংহার)

TCP/IP Model হলো আধুনিক ইন্টারনেটের ব্লুপ্রিন্ট (Blueprint)। OSI Model আমাদের নেটওয়ার্কিং থিওরি বুঝতে সাহায্য করে, কিন্তু রিয়েল-ওয়ার্ল্ড নেটওয়ার্কিং, ফায়ারওয়াল কনফিগারেশন, রাউটিং এবং ট্রাবলশুটিং সম্পূর্ণভাবে TCP/IP প্রোটোকল স্যুটের ওপর নির্ভরশীল। Application Layer-এর প্রোটোকল থেকে শুরু করে Network Access Layer-এর হার্ডওয়্যার পর্যন্ত প্রতিটি ধাপের স্পষ্ট ধারণা থাকা একজন প্রফেশনাল আইটি বা নেটওয়ার্ক ইঞ্জিনিয়ারের জন্য অত্যন্ত জরুরি।
