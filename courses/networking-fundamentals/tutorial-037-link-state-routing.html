Link State Routing: Concepts, Mechanisms and Deep Dive

27 April, 2026 Networking Concepts Jobaer Hossain

ডায়নামিক রাউটিং এর জগতে Link State Routing একটি অত্যন্ত শক্তিশালী এবং আধুনিক পদ্ধতি। এটি মূলত Distance Vector Routing (যেমন RIP) এর সীমাবদ্ধতাগুলো দূর করার জন্য তৈরি করা হয়েছিল। বড় এবং জটিল নেটওয়ার্কগুলোতে যেখানে দ্রুত ডাটা ট্রান্সমিশন এবং নেটওয়ার্ক টপোলজি পরিবর্তনের সাথে সাথে দ্রুত আপডেট হওয়া প্রয়োজন, সেখানে Link State Routing এর কোনো বিকল্প নেই।

আজকের এই বিস্তারিত টিউটোরিয়ালে আমরা শিখবো Link State Routing কীভাবে কাজ করে, এর অ্যালগরিদম এবং প্রফেশনাল নেটওয়ার্কিংয়ে এর ব্যবহার নিয়ে গভীর আলোচনা।

১. Link State Routing কী?

Link State Routing হলো এমন একটি পদ্ধতি যেখানে প্রতিটি রাউটার পুরো নেটওয়ার্কের একটি পূর্ণাঙ্গ মানচিত্র বা Topology Map নিজের কাছে রাখে। একটি রাউটার শুধুমাত্র তার প্রতিবেশীদের তথ্য জানে না, বরং সে জানে পুরো নেটওয়ার্কে আর কয়টি রাউটার আছে এবং তারা একে অপরের সাথে কীভাবে যুক্ত।

এই পদ্ধতিতে প্রতিটি রাউটার তার সরাসরি যুক্ত লিংকের অবস্থা (Link State) যেমন- লিংকটি আপ আছে কি না, এর Bandwidth কত এবং Delay কত—এই তথ্যগুলো একটি স্পেশাল প্যাকেটের মাধ্যমে পুরো নেটওয়ার্কে ছড়িয়ে দেয়।

রিয়েল-লাইফ সিনারিও (The Full City Map Analogy)

Distance Vector: এটি অনেকটা এমন যে আপনি রাস্তায় একজনকে জিজ্ঞেস করলেন "নীলক্ষেত কোন দিকে?" সে বলল "ডানে যান"। আপনি জানেন না ডানে আসলে কী আছে।

Link State: এটি অনেকটা আপনার হাতে থাকা Google Maps এর মতো। আপনার কাছে পুরো শহরের ম্যাপ আছে। আপনি নিজেই দেখতে পাচ্ছেন কোন কোন রাস্তা খোলা আছে, কোথায় জ্যাম বেশি এবং কোন রাস্তা দিয়ে গেলে সবচেয়ে দ্রুত পৌঁছানো যাবে। Link State Routing এ প্রতিটি রাউটার তার কাছে থাকা ম্যাপ দেখে নিজেই সিদ্ধান্ত নেয় শ্রেষ্ঠ পথ কোনটি।

২. Link State Routing কীভাবে কাজ করে? (৫টি মূল ধাপ)

একটি Link State রাউটার মূলত নিচের পাঁচটি ধাপ অনুসরণ করে তার Routing Table তৈরি করে:

  1. Discovery: রাউটার তার প্রতিটি ইন্টারফেসে একটি HELLO প্যাকেট পাঠিয়ে তার সরাসরি যুক্ত প্রতিবেশীদের (Neighbors) খুঁজে বের করে।
  2. Link Cost Measurement: রাউটার প্রতিটি লিংকের Cost বা দূরত্ব মেপে নেয়। সাধারণত 100Mbps লিংকের চেয়ে 1Gbps লিংকের Cost কম হয় (যা বেশি ভালো)।
  3. LSP Creation: রাউটার একটি Link State Packet (LSP) তৈরি করে যাতে তার নিজের নাম এবং তার সাথে যুক্ত সব লিংকের তথ্য থাকে।
  4. Flooding: রাউটার এই LSP টিকে নেটওয়ার্কের সব রাউটারের কাছে পাঠিয়ে দেয়। একে বলা হয় Flooding। এভাবে প্রতিটি রাউটার সবার তথ্য পেয়ে যায়।
  5. Dijkstra's Algorithm: সব রাউটারের তথ্য পাওয়ার পর প্রতিটি রাউটার নিজের মেমোরিতে একটি Link State Database (LSDB) তৈরি করে এবং SPF (Shortest Path First) বা Dijkstra's Algorithm চালিয়ে গন্তব্যে যাওয়ার শ্রেষ্ঠ পথটি বের করে।

৩. Dijkstra's Algorithm এবং SPF Tree

Link State Routing এর হৃৎপিণ্ড হলো Dijkstra's Algorithm। এটি প্রতিটি গন্তব্যের জন্য একটি Tree তৈরি করে যেখানে বর্তমান রাউটারটি থাকে একদম শুরুতে (Root)। এরপর সে হিসাব করে প্রতিটি নোডে পৌঁছাতে মোট কত Cost ব্যয় হবে। যে পথের Cost সবচেয়ে কম, সেটিকেই Routing Table এ যুক্ত করা হয়।

৪. Link State Routing এর সুবিধা ও অসুবিধা

সুবিধা (Pros) অসুবিধা (Cons)
Fast Convergence নেটওয়ার্কে কোনো পরিবর্তন হলে এটি খুব দ্রুত সব রাউটারকে জানিয়ে দেয়।
Loop-Free যেহেতু সবার কাছে পুরো ম্যাপ থাকে, তাই রাউটিং লুপ হওয়ার সম্ভাবনা প্রায় শূন্য।
Scalability বড় বড় এন্টারপ্রাইজ নেটওয়ার্কের জন্য এটি অত্যন্ত কার্যকর।
Resource Intensive রাউটারের CPU এবং RAM অনেক বেশি ব্যবহার করে কারণ একে জটিল অ্যালগরিদম চালাতে হয়।
Complex Configuration এটি কনফিগার এবং ট্রাবলশুট করা সাধারণ প্রোটোকলের চেয়ে কঠিন।

৫. পপুলার Link State Protocols

নেটওয়ার্কিং জগতে দুটি প্রধান Link State প্রোটোকল ব্যবহৃত হয়:

  • OSPF (Open Shortest Path First): এটি সবচেয়ে জনপ্রিয় এবং বহুল ব্যবহৃত আইজিপি (IGP)। এটি IP নেটওয়ার্কে ব্যাপকভাবে ব্যবহৃত হয়।
  • IS-IS (Intermediate System to Intermediate System): এটি সাধারণত বড় বড় সার্ভিস প্রোভাইডার বা আইএসপি (ISP) নেটওয়ার্কে ব্যবহৃত হয়।

৬. Practical CLI Configuration (OSPF on MikroTik)

Link State প্রোটোকল হিসেবে OSPF কনফিগার করার একটি সাধারণ উদাহরণ নিচে দেওয়া হলো।

১. OSPF ইনস্ট্যান্স তৈরি করা:

/routing ospf instance add name=default router-id=1.1.1.1

২. এরিয়া এবং ইন্টারফেস টেমপ্লেট যুক্ত করা:

/routing ospf area add name=backbone-area instance=default area-id=0.0.0.0
/routing ospf interface-template add networks=192.168.10.0/24 area=backbone-area

ব্যাখ্যা: এখানে ১৯২.১৬৮.১০.০ নেটওয়ার্কটিকে ওএসপিএফ এর মাধ্যমে অন্য রাউটারদের কাছে প্রচার করা হচ্ছে।

৭. Troubleshooting টিপস

  • Neighbor Adjacency: যদি দেখেন রুট আসছে না, তবে সবার আগে /routing ospf neighbor print কমান্ড দিয়ে চেক করুন আপনার পাশের রাউটারের সাথে Full Adjacency তৈরি হয়েছে কি না।
  • Area ID Mismatch: ওএসপিএফ এ দুটি রাউটার যদি ভিন্ন এরিয়াতে থাকে এবং মাঝখানে Backbone Area (Area 0) না থাকে, তবে তারা ডাটা বিনিময় করবে না।
  • MTU Mismatch: অনেক সময় লিংকের MTU (Maximum Transmission Unit) এক না হলে ওএসপিএফ নে বারবার আপ-ডাউন (Flapping) করে। উভয় পাশে MTU চেক করুন।
  • LSDB Sync: যদি কোনো রাউটারে ভুল ম্যাপ দেখায়, তবে মেমোরি থেকে LSDB ফ্ল্যাশ বা রিফ্রেশ করে দেখতে পারেন।

উপসংহার (Conclusion)

Link State Routing আধুনিক নেটওয়ার্কিংয়ের একটি অবিচ্ছেদ্য অংশ। এটি তার Fast Convergence এবং Accuracy এর জন্য বিশ্বজুড়ে সমাদৃত। যদিও এটি রাউটারের হার্ডওয়্যার রিসোর্স কিছুটা বেশি ব্যবহার করে, কিন্তু এর মাধ্যমে প্রাপ্ত নেটওয়ার্ক স্ট্যাবিলিটি এবং পারফরম্যান্স অতুলনীয়। একজন প্রফেশনাল নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে Link State Routing এর মেকানিজম এবং Dijkstra's Algorithm এর ওপর দখল থাকা আপনাকে ক্যারিয়ারে অনেক দূর এগিয়ে নিয়ে যাবে।
