Address Resolution Protocol (ARP): A Deep Dive Guide

27 April, 2026 Networking Concepts Jobaer Hossain

কম্পিউটার নেটওয়ার্কিংয়ে আমরা যখন একে অপরের সাথে যোগাযোগ করি, তখন আমরা সাধারণত IP Address ব্যবহার করি। কিন্তু পর্দার আড়ালে ডাটা প্যাকেট আদান-প্রদান করার জন্য ফিজিক্যাল অ্যাড্রেস বা MAC Address এর প্রয়োজন হয়। আইপি অ্যাড্রেসকে এই ম্যাক অ্যাড্রেসে রূপান্তর করার জাদুকরী প্রোটোকলটির নামই হলো Address Resolution Protocol বা সংক্ষেপে ARP।

আজকের এই টিউটোরিয়ালে আমরা ARP-এর মেকানিজম, এর প্রকারভেদ এবং নেটওয়ার্ক ট্রাবলশুটিংয়ে এর গুরুত্ব নিয়ে বিস্তারিত আলোচনা করবো।

১. ARP কী এবং এটি কেন প্রয়োজন?

OSI Model অনুযায়ী, Network Layer (Layer 3) আইপি অ্যাড্রেস বোঝে, কিন্তু Data Link Layer (Layer 2) শুধুমাত্র ম্যাক অ্যাড্রেস চেনে। ডাটা যখন এক পিসি থেকে অন্য পিসিতে যায়, তখন তাকে লোকাল নেটওয়ার্কে পৌঁছানোর জন্য ম্যাক অ্যাড্রেস জানতেই হয়।

সহজ কথায়, ARP হলো একটি ডিরেক্টরি সার্ভিস যা লজিক্যাল অ্যাড্রেসকে (IP) ফিজিক্যাল অ্যাড্রেসে (MAC) ম্যাপ করে।

রিয়েল-লাইফ সিনারিও (The Classroom Analogy)

ধরুন একটি ক্লাসরুমে শিক্ষক (Sender) একটি নির্দিষ্ট ছাত্রকে (Destination) ডাকতে চান।

  • শিক্ষক জানেন ছাত্রটির নাম (IP Address), কিন্তু তিনি জানেন না ছাত্রটি কোথায় বসে আছে।
  • শিক্ষক ক্লাসে চিৎকার করে বললেন, "যার নাম রহিম, সে তার হাত তোলো" (ARP Request/Broadcast)।
  • পুরো ক্লাসের সবাই এটি শুনল, কিন্তু শুধুমাত্র রহিম হাত তুলল এবং বলল "আমি এখানে" (ARP Reply/Unicast)।
  • শিক্ষক এখন রহিমকে চিনতে পারলেন এবং তার কাছে ডাটা পাঠাতে পারলেন।

২. ARP কীভাবে কাজ করে? (The 4-Step Process)

ARP কমিউনিকেশন মূলত চারটি ধাপের মাধ্যমে সম্পন্ন হয়:

  1. ARP Request (Broadcast): যখন হোস্ট A জানে হোস্ট B-এর আইপি কিন্তু ম্যাক জানে না, তখন সে একটি ARP Request প্যাকেট পুরো নেটওয়ার্কে Broadcast করে।
  2. The Inspection: নেটওয়ার্কের প্রতিটি ডিভাইস এই প্যাকেটটি পায় এবং চেক করে যে প্যাকেটে থাকা আইপি অ্যাড্রেসটি তার নিজের কি না।
  3. ARP Reply (Unicast): যার আইপি মিলে যায়, সে তার ম্যাক অ্যাড্রেস সহ একটি ARP Reply হোস্ট A-কে পাঠায়। এটি Unicast বা সরাসরি যোগাযোগ।
  4. ARP Cache Update: হোস্ট A এই ম্যাক অ্যাড্রেসটি পাওয়ার পর তা নিজের ARP Cache Table-এ জমা রাখে যাতে ভবিষ্যতে আর বারবার রিকোয়েস্ট পাঠাতে না হয়।

৩. ARP-এর বিভিন্ন প্রকারভেদ

নেটওয়ার্কের পরিস্থিতি অনুযায়ী ARP বিভিন্ন ধরনের হতে পারে:

ARP Type Description Use Case
Proxy ARP একটি রাউটার যখন অন্য নেটওয়ার্কের ডিভাইসের হয়ে ARP রিপ্লাই দেয়। ডিফল্ট গেটওয়ে কনফিগার করা না থাকলে ব্যবহৃত হয়।
Gratuitous ARP একটি ডিভাইস যখন কারো রিকোয়েস্ট ছাড়াই নিজের আইপি/ম্যাক প্রচার করে। আইপি কনফ্লিক্ট চেক করতে এবং গেটওয়ে রিডান্ডেন্সি নিশ্চিত করতে।
Inverse ARP ম্যাক অ্যাড্রেস জানা থাকলে আইপি খুঁজে বের করতে ব্যবহৃত হয়। ফ্রেম রিলে নেটওয়ার্কে ব্যবহৃত হয়।
Reverse ARP (RARP) এটি পুরনো প্রোটোকল, ম্যাক থেকে আইপি পেতে ব্যবহৃত হতো। এখন DHCP এর মাধ্যমে এটি প্রতিস্থাপিত হয়েছে।

৪. ARP Table দেখা এবং ম্যানেজমেন্ট (CLI)

আপনার ডিভাইস বর্তমানে কার কার ম্যাক অ্যাড্রেস জানে তা দেখার জন্য নিচের কমান্ডগুলো ব্যবহার করতে পারেন:

১. Windows-এ ARP Table দেখা:

C:\> arp -a

২. Linux-এ ARP Table দেখা:

$ ip neigh show  অথবা  $ arp -n

৩. MikroTik-এ ARP লিস্ট দেখা:

/ip arp print

৫. ARP Security: ARP Spoofing বা Poisoning

ARP প্রোটোকলটি অনেক পুরনো হওয়ায় এতে বিল্ট-ইন সিকিউরিটি নেই। অ্যাটাকাররা ভুল ARP রিপ্লাই পাঠিয়ে আপনার ট্রাফিক তাদের পিসিতে নিয়ে যেতে পারে। একে ARP Spoofing বলা হয়।

  • সমস্যা: হ্যাকার আপনার পিসিকে বলবে "আমিই তোমার রাউটার", আর রাউটারকে বলবে "আমিই সেই ইউজার পিসি"। ফলে সব ডাটা হ্যাকারের পিসি হয়ে যাবে (Man-in-the-middle attack)।
  • সমাধান: Static ARP ব্যবহার করা, সুইচে DAI (Dynamic ARP Inspection) এনাবল করা এবং নেটওয়ার্ক মনিটরিং টুল ব্যবহার করা।

MikroTik-এ Static ARP তৈরি করার কমান্ড:

/ip arp add address=192.168.88.10 mac-address=00:0C:42:01:02:03 interface=ether1

৬. Troubleshooting টিপস

  • IP Conflict: যদি দুটি ডিভাইসে একই আইপি থাকে, তবে ARP Table বারবার ফ্ল্যাকচুয়েট করবে। Gratuitous ARP লগ চেক করে এটি সমাধান করা যায়।
  • Gateway Unreachable: যদি গেটওয়ে পিং না পান, তবে arp -a দিয়ে দেখুন গেটওয়ের ম্যাক অ্যাড্রেসটি 'Incomplete' দেখাচ্ছে কি না। যদি তাই হয়, তবে ক্যাবল বা লেয়ার ২ সংযোগ চেক করুন।
  • Clear ARP Cache: অনেক সময় পুরনো ম্যাক অ্যাড্রেস ক্যাশে জমা থাকলে সংযোগে সমস্যা হয়। সেক্ষেত্রে ক্যাশ ডিলিট করে দিতে পারেন: arp -d * (Windows)।

উপসংহার (Conclusion)

Address Resolution Protocol (ARP) হলো নেটওয়ার্কিংয়ের সেই নিঃশব্দ কর্মী যা ছাড়া আইপি কমিউনিকেশন অচল। এটি লজিক্যাল এবং ফিজিক্যাল লেয়ারের মধ্যে সেতুবন্ধন তৈরি করে। একজন নেটওয়ার্ক অ্যাডমিনিস্ট্রেটর হিসেবে ARP-এর কার্যপদ্ধতি এবং এর সিকিউরিটি ঝুঁকিগুলো সম্পর্কে সম্যক ধারণা থাকা অত্যন্ত জরুরি। এটি আপনাকে শুধুমাত্র কানেক্টিভিটি ইস্যু সমাধান করতেই সাহায্য করবে না, বরং আপনার নেটওয়ার্ককে সাইবার অ্যাটাক থেকেও সুরক্ষিত রাখবে।
