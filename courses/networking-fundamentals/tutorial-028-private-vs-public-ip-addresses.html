Public vs Private IP Address: A Technical Deep Dive

27 April, 2026 Networking Concepts Jobaer Hossain

ইন্টারনেট বা লোকাল নেটওয়ার্কে প্রতিটি ডিভাইসের একটি অনন্য পরিচয় থাকে, যাকে আমরা IP Address বলি। কিন্তু আপনি কি জানেন যে আপনার স্মার্টফোনের ভেতরে যে আইপি অ্যাড্রেসটি দেখায়, সেটি ইন্টারনেটের কাছে আপনার আসল পরিচয় নয়? এখানেই চলে আসে Public এবং Private IP Address এর ধারণা।

আজকের এই টিউটোরিয়ালে আমরা শিখবো এই দুই ধরনের আইপির পার্থক্য কী, কেন এদের আলাদা করা হয়েছে এবং কীভাবে এরা NAT (Network Address Translation) প্রযুক্তির মাধ্যমে একসাথে কাজ করে।

১. Public IP Address কী?

Public IP Address হলো এমন একটি ঠিকানা যা সরাসরি ইন্টারনেটের সাথে যুক্ত। এটি বিশ্বজুড়ে অনন্য (Unique)। একটি Public IP দিয়ে পৃথিবীর যেকোনো প্রান্ত থেকে ওই ডিভাইসটিকে খুঁজে পাওয়া সম্ভব। এটি আপনার ISP (Internet Service Provider) আপনাকে প্রদান করে।

  • ইন্টারনেটে ডাটা আদান-প্রদানের জন্য এটি বাধ্যতামূলক।
  • এটি IANA (Internet Assigned Numbers Authority) দ্বারা নিয়ন্ত্রিত হয়।
  • আপনার বাসার Router ইন্টারনেটের সাথে যে আইপি দিয়ে কথা বলে, সেটিই হলো Public IP।

২. Private IP Address কী?

Private IP Address হলো আপনার নিজের নেটওয়ার্কের (Home or Office LAN) ভেতরের পরিচয়। এটি ইন্টারনেটে দৃশ্যমান নয়। আপনার রাউটারের সাথে যুক্ত মোবাইল, ল্যাপটপ বা প্রিন্টারগুলো এই আইপি ব্যবহার করে একে অপরের সাথে কথা বলে।

  • এটি ইন্টারনেটে সরাসরি কাজ করে না।
  • একই Private IP পৃথিবীর কোটি কোটি লোকাল নেটওয়ার্কে পুনরায় ব্যবহার করা যেতে পারে।
  • এটি সাধারণত DHCP (Dynamic Host Configuration Protocol) এর মাধ্যমে রাউটার থেকে ডিভাইসগুলো পেয়ে থাকে।

রিয়েল-লাইফ সিনারিও (The Office Analogy)

একটি বড় অফিসের কথা চিন্তা করুন:

  • Public IP: এটি আপনার অফিসের প্রধান ফোন নম্বর (উদাঃ +৮৮০২-XXXXXXX)। বাইরের যে কেউ এই নম্বরে কল করে অফিসে পৌঁছাতে পারে।
  • Private IP: এটি আপনার অফিসের ভেতরের ইন্টারকম নম্বর (উদাঃ 101, 102)। আপনি এক ডেস্ক থেকে অন্য ডেস্কে কথা বলতে এই নম্বর ব্যবহার করেন। কিন্তু বাইরের কেউ সরাসরি এই ইন্টারকম নম্বরে কল দিতে পারবে না। তাকে আগে মেইন নম্বরে কল করতে হবে।

৩. Private IP রেঞ্জ (RFC 1918)

Private IP হিসেবে আপনি যেকোনো আইপি ব্যবহার করতে পারবেন না। এর জন্য নির্দিষ্ট কিছু ব্লক বা রেঞ্জ সংরক্ষিত করা হয়েছে:

Class Private IP Range Default Mask
Class A 10.0.0.0 – 10.255.255.255 255.0.0.0
Class B 172.16.0.0 – 172.31.255.255 255.240.0.0
Class C 192.168.0.0 – 192.168.255.255 255.255.0.0

৪. কেন এই বিভাজন প্রয়োজন?

IPv4 সিস্টেমে মাত্র ৪.৩ বিলিয়ন আইপি সম্ভব। বর্তমান বিশ্বে ডিভাইসের সংখ্যা আইপির সংখ্যার চেয়ে অনেক বেশি। এই অভাব পূরণ করতেই Public এবং Private IP এর সিস্টেম আনা হয়েছে।

  • IP Conservation: একটি মাত্র Public IP ব্যবহার করে একটি অফিসের ১০০০ জন ইউজার ইন্টারনেট চালাতে পারে।
  • Security: যেহেতু Private IP ইন্টারনেটে দেখা যায় না, তাই হ্যাকাররা সরাসরি আপনার লোকাল পিসিতে অ্যাটাক করতে পারে না।

৫. NAT (Network Address Translation) কীভাবে কাজ করে?

যখন আপনার লোকাল পিসি (Private IP) থেকে কোনো রিকোয়েস্ট ইন্টারনেটে যায়, তখন Router তার Private IP-কে Public IP-তে রূপান্তর করে দেয়। একেই বলা হয় NAT।

প্রক্রিয়া:

  1. আপনার ল্যাপটপ (192.168.1.10) থেকে Google-কে একটি রিকোয়েস্ট পাঠানো হলো।
  2. রাউটার এই প্যাকেটটি পায় এবং সোর্স আইপি হিসেবে তার নিজের Public IP বসিয়ে দেয়।
  3. Google ওই Public IP-তে রিপ্লাই দেয়।
  4. রাউটার তার NAT Table চেক করে দেখে এই রিকোয়েস্টটি ল্যাপটপ থেকে এসেছিল, তখন সে প্যাকেটটি ল্যাপটপে পৌঁছে দেয়।

৬. Troubleshooting ও প্রফেশনাল টিপস

নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে আপনার আইপি চেক করার পদ্ধতি জানা জরুরি:

১. Private IP দেখা (Windows/Linux CLI):

C:\> ipconfig (Windows)
$ hostname -I (Linux)

২. Public IP দেখা:

যেকোনো ব্রাউজারে গিয়ে গুগলে সার্চ করুন "what is my ip"। যে রেজাল্টটি আসবে সেটিই আপনার Public IP।

কমন ট্রাবলশুটিং সিনারিও:

  • Cisco/MikroTik রাউটারে ইন্টারনেটের সমস্যা: যদি দেখেন পিসি লোকাললি পিং (Ping) পাচ্ছে কিন্তু ইন্টারনেটে পাচ্ছে না, তবে চেক করুন আপনার রাউটারে Masquerade (NAT) কনফিগার করা আছে কি না।
  • IP Conflict: যদি আপনার লোকাল নেটওয়ার্কে কেউ ম্যানুয়ালি একই Private IP বসায়, তবে নেটওয়ার্কে সমস্যা হবে। সবসময় DHCP ব্যবহার করা ভালো।

MikroTik NAT কনফিগারেশন কমান্ড (Masquerade):

/ip firewall nat add chain=srcnat action=masquerade out-interface=ether1

উপসংহার (Conclusion)

Public এবং Private IP Address এর পার্থক্য বোঝা নেটওয়ার্কিং ক্যারিয়ারের প্রথম ধাপ। Public IP আমাদের ইন্টারনেটে দৃশ্যমান করে, আর Private IP আমাদের লোকাল নেটওয়ার্ককে সুরক্ষিত এবং সাশ্রয়ী করে। এই দুইয়ের মেলবন্ধনই হলো আজকের আধুনিক ইন্টারনেট। একজন আইটি স্পেশালিস্ট হিসেবে এই কনসেপ্টটি আপনার আইপি অ্যাড্রেসিং এবং নেটওয়ার্ক ডিজাইন দক্ষতাকে অনেক গুণ বাড়িয়ে দেবে।
