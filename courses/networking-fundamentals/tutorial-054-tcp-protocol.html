What is Transmission Control প্রোটোকল (TCP)?

March 09, 2026 Networking Jobaer Hossain

TCP (Transmission Control Protocol) হলো এমন একটি প্রোটোকল যার মাধ্যমে ইন্টারনেটে এক কম্পিউটার থেকে অন্য কম্পিউটারে নির্ভরযোগ্যভাবে (reliably) ডেটা আদান-প্রদান করা হয়। এটি কানেকশন ওরিয়েন্টেড প্রোটোকল যা ডেটাকে ছোট ছোট অংশে বিভক্ত করে সঠিক ক্রমানুসারে (right order) গন্তব্যে পৌঁছাতে সাহায্য করে এবং তথ্য হারানোর হার কমিয়ে দেয়।

Features of TCP (TCP-এর প্রধান বৈশিষ্ট্যসমূহ):

  • এটি OSI মডেলের Transport Layer (Layer 4)-এ কাজ করে।
  • ডেটা স্থানান্তরের পূর্বে প্রেরক ও প্রাপক ডিভাইসে যোগাযোগ স্থাপন করে যা কানেকশন ওরিয়েন্টেড প্রোটোকল হিসেবে পরিচিত।
  • Reliability: প্রতিটি প্যাকেট গন্তব্যে পৌঁছালে তার প্রাপ্তি স্বীকার বা একনলেজমেন্ট (ACK) প্রদান করে।
  • Full-Duplex: একই সময়ে প্রেরক ও প্রাপক উভয়ই ডেটা আদান-প্রদান করতে পারে।

TCP Header Format (টিসিপি হেডার ফরম্যাট)

প্রতিটি TCP সেগমেন্টের শুরুতে একটি হেডার থাকে যা প্যাকেটের পরিচয় এবং এটি কীভাবে প্রসেস করা হবে তা নির্ধারণ করে। নিচে এর বিভিন্ন অংশ আলোচনা করা হলো:

TCP Header (20 Bytes minimum)

Source Port (16 bits)
Destination Port (16 bits)
Sequence Number (32 bits)
Acknowledgment Number (32 bits)
HLEN (4b)
Rsvd (6b)
Flags (6b)
Window Size (16b)
Checksum (16 bits)
Urgent Pointer (16 bits)
Options and Padding (0-40 Bytes)

Connection Establishment (Three-Way Handshake)

TCP সেশন চালুর জন্য ৩-স্টেপ কমিউনিকেশন প্রসেস ব্যবহার করে যা থ্রি-ওয়ে হ্যান্ডশেক নামে পরিচিত:

CLIENT SERVER
1. SYN (কানেকশন রিকোয়েস্ট)
2. SYN-ACK (রিকোয়েস্ট রিসিভ)
3. ACK (কানেকশন সম্পন্ন)

Working of TCP (কাজের প্রক্রিয়া)

  1. Segmenting: বড় তথ্যকে ছোট ছোট প্যাকেটে বিভক্ত করা হয়।
  2. Routing via IP: প্রতিটি প্যাকেটকে সেরা পথ দিয়ে গন্তব্যে পাঠানো হয়।
  3. Reassembly: গন্তব্যে পৌঁছানোর পর প্যাকেটগুলো আবার সঠিক ক্রমানুসারে সাজানো হয়।
  4. Error Control: যদি কোনো প্যাকেট পথে হারিয়ে যায়, তবে সেটির জন্য পুনরায় রিকোয়েস্ট করা হয়।

Disadvantages of TCP (সীমাবদ্ধতাসমূহ)

  • Slow Speed: প্রতিটি প্যাকেটের একনলেজমেন্ট প্রয়োজন হওয়ায় এটি UDP এর তুলনায় কিছুটা ধীরগতির।
  • বেশি Overhead: ২০-৪০ বাইটের হেডার এবং অনবরত কমিউনিকেশন চেক করার জন্য ব্যান্ডউইডথ বেশি লাগে।
  • No Multi-cast: এটি মূলত ওয়ান-টু-ওয়ান কমিউনিকেশন পছন্দ করে, তাই বড় ব্রডকাস্টের জন্য কার্যকর নয়।
