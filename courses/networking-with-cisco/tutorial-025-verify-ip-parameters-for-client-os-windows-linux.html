Verify IP Parameters for Client OS (Windows, Linux)

17 May, 2026 CCNA Tutorials Jobaer Hossain

একটি কম্পিউটার নেটওয়ার্ক বা এন্টারপ্রাইজ ইনফ্রাস্ট্রাকচার তৈরি করার পর সবচেয়ে সাধারণ যে কাজটি একজন নেটওয়ার্ক ইঞ্জিনিয়ার বা আইটি সাপোর্ট প্রফেশনালকে প্রতিদিন করতে হয়, তা হলো—Client OS-এর নেটওয়ার্ক কানেক্টিভিটি ভেরিফাই এবং ট্রাবলশুট করা। কোনো ইউজার যদি কমপ্লেন করেন যে তিনি ইন্টারনেট এক্সেস করতে পারছেন না বা লোকাল সার্ভার পাচ্ছেন না, তখন প্রথম পদক্ষেপই হয় ওই ইউজারের ডিভাইসের IP Parameters সুনির্দিষ্টভাবে পরীক্ষা করা।

আজকের এই ডিপ-ডাইভ টিউটোরিয়ালে আমরা শিখবো কীভাবে অত্যন্ত জনপ্রিয় দুটি ওএস—Windows এবং Linux-এ নিখুঁতভাবে IP Parameters (IP Address, Subnet Mask, Default Gateway, এবং DNS) চেক, ভেরিফাই এবং ট্রাবলশুট করতে হয়।

১. ৪টি কোর IP Parameters কী এবং কেন এগুলো গুরুত্বপূর্ণ?

যেকোনো Client OS-কে নেটওয়ার্কে সফলভাবে ডেটা আদান-প্রদান করতে হলে ৪টি কোর প্যারামিটার সঠিকভাবে কনফিগার হতে হয়:

  1. IP Address: নেটওয়ার্কের ভেতরে ডিভাইসের নিজস্ব অনন্য লজিক্যাল আইডেন্টিটি।
  2. Subnet Mask: ডিভাইসটি কোন লোকাল নেটওয়ার্কের (Subnet) অংশ এবং তার লোকাল বাউন্ডারি কতটুকু, তা নির্ধারণ করে।
  3. Default Gateway: লোকাল নেটওয়ার্কের বাউন্ডারি পার হয়ে বাইরের অন্য কোনো নেটওয়ার্কে বা ইন্টারনেটে ডেটা পাঠাতে যে ইন্টারমিডিয়েট রাউটার আইপি ব্যবহার করা হয়, সেটিই হলো গেটওয়ে।
  4. DNS Server: Domain Name System সার্ভার মানুষের পড়ার উপযোগী নামকে (যেমন: google.com) মেশিনের উপযোগী আইপি অ্যাড্রেসে (যেমন: 8.8.8.8) অনুবাদ করে।

রিয়েল-লাইফ সিনারিও (The Office Workstation Problem)

ধরুন, আপনার অফিসের একজন গ্রাফিক ডিজাইনারের পিসিতে লোকাল নেটওয়ার্কের ফাইল শেয়ারিং কাজ করছে, কিন্তু তিনি ব্রাউজারে কোনো ওয়েবসাইট লোড করতে পারছেন না। একজন প্রফেশনাল হিসেবে আপনি সরাসরি তার পিসির IP Parameters ভেরিফাই করবেন। যদি দেখেন তার IP এবং Gateway ঠিক আছে কিন্তু DNS Server-এর ঘরটি ফাঁকা বা ভুল আইপি দেওয়া, তবে আপনি মুহূর্তেই সমস্যাটি সনাক্ত ও সমাধান করে ফেলতে পারবেন।

২. Windows Client OS-এ IP Parameters ভেরিফিকেশন

মাইক্রোসফট উইন্ডোজ ওএস-এ গ্রাফিক্যাল ইন্টারফেস (GUI) এবং কমান্ড লাইন (CLI)—উভয় পদ্ধতিতেই এই প্যারামিটারগুলো নিখুঁতভাবে চেক করা যায়।

ক. Windows CLI (Command Prompt / PowerShell) পদ্ধতি

সবচেয়ে দ্রুত এবং প্রফেশনাল পদ্ধতি হলো CMD ব্যবহার করা। এটি ওপেন করে নিচের কমান্ডটি রান করুন:

C:\> ipconfig

ব্যাখ্যা: এই বেসিক কমান্ডটি আপনার পিসির সব ফিজিক্যাল এবং ওয়্যারলেস অ্যাডাপ্টারের IP Address, Subnet Mask এবং Default Gateway প্রদর্শন করবে।

তবে আপনি যদি DNS এবং DHCP সার্ভারসহ সম্পূর্ণ ডিটেইলস দেখতে চান, তবে আপনাকে এক্সটেন্ডেড কমান্ডটি ব্যবহার করতে হবে:

C:\> ipconfig /all

খ. Windows GUI (Graphical User Interface) পদ্ধতি

কমান্ড লাইনে স্বাচ্ছন্দ্য বোধ না করলে গ্রাফিক্যালি চেক করার ধাপগুলো নিচে দেওয়া হলো:

  • Step 1: কিবোর্ড থেকে Windows Key + R প্রেস করে রান বক্সে লিখুন ncpa.cpl এবং এন্টার দিন। এতে Network Connections উইন্ডোটি ওপেন হবে।
  • Step 2: আপনার সক্রিয় কানেকশনটির (Ethernet বা Wi-Fi) ওপর রাইট ক্লিক করে Status-এ যান।
  • Step 3: নতুন পপ-আপ উইন্ডো থেকে Details... বাটনে ক্লিক করুন। এখানে আপনার ওএস-এর যাবতীয় আইপি প্যারামিটার এক নজরে দেখতে পাবেন।

৩. Linux Client OS-এ IP Parameters ভেরিফিকেশন

আধুনিক লিনাক্স ডিস্ট্রিবিউশনগুলোতে (যেমন: Ubuntu, Rocky Linux, Fedora, Debian) নেটওয়ার্ক স্ট্যাক ম্যানেজ করার জন্য কিছু স্ট্যান্ডার্ড টুলস এবং কনফিগারেশন ফাইল থাকে।

ক. IP Address এবং Subnet Mask চেক করা

লিনাক্সে পুরনো ifconfig কমান্ডের পরিবর্তে আধুনিক iproute2 প্যাকেজের কমান্ড ব্যবহার করা বেস্ট প্র্যাকটিস:

$ ip addr show

বা সংক্ষেপে:

$ ip a

ব্যাখ্যা: এটি আপনার লিনাক্স ডিভাইসের সব ইন্টারফেস (যেমন: eth0, wlan0) এবং তাদের সাথে অ্যাসাইন করা আইপি প্রদর্শন করবে। আইপির পাশে থাকা স্ল্যাশ নোটেশন (যেমন: /24) মূলত সাবনেট মাস্ক নির্দেশ করে।

খ. Default Gateway ভেরিফাই করা

আপনার লিনাক্স ক্লায়েন্টটি কোন রাউটার বা গেটওয়ে দিয়ে বাইরের ট্রাফিক পাস করছে তা দেখার কমান্ড:

$ ip route show

বা সংক্ষেপে:

$ ip r

আউটপুটের প্রথম লাইনেই দেখতে পাবেন default via 192.168.1.1 dev eth0, যার মানে 192.168.1.1 হলো আপনার ডিফল্ট গেটওয়ে।

গ. DNS Server প্যারামিটার ভেরিফাই করা

লিনাক্স সিস্টেমে কনফিগার করা DNS সার্ভারের আইপিগুলো একটি নির্দিষ্ট টেক্সট ফাইলে জমা থাকে। এটি টার্মিনালে রিড করার জন্য নিচের কমান্ডটি দিন:

$ cat /etc/resolv.conf

আউটপুট ভেরিফিকেশন: ফাইলে nameserver 8.8.8.8 এর মতো লাইন দেখতে পাবেন, যা নির্দেশ করে সিস্টেমটি কোন ডিএনএস সার্ভার কোয়েরি করছে।

৪. Windows এবং Linux কমান্ডের তুলনামূলক ছক

সহজে মনে রাখার জন্য উভয় ওএস-এর প্রয়োজনীয় ট্রাবলশুটিং ও ভেরিফিকেশন কমান্ডের একটি তুলনামূলক রেফারেন্স টেবিল নিচে দেওয়া হলো:

Target Parameter Windows (CMD/PowerShell) Linux (Terminal)
All IP Parameters Summary ipconfig ip a && ip r ডিভাইসের আইপি এবং গেটওয়ের কুইক ভিউ।
Deep Network Details ipconfig /all nmcli device show MAC Address, DHCP Lease এবং DNS এর সম্পূর্ণ বিবরণ।
Routing Table / Gateway route print ip route show সিস্টেমের ইন্টারনাল রাউটিং পাথ ও গেটওয়ে চেক।
DNS Resolver View nslookup google.com cat /etc/resolv.conf ডিএনএস রিজলভার সঠিকভাবে কাজ করছে কি না তা নিশ্চিত করা।

৫. APIPA (Automatic Private IP Addressing) এবং এরর সনাক্তকরণ

আইপি প্যারামিটার ভেরিফাই করার সময় একটি বিশেষ টেকনিক্যাল কন্ডিশন মাথায় রাখা জরুরি, যাকে বলা হয় APIPA।

উইন্ডোজ বা লিনাক্স ক্লায়েন্ট যদি DHCP মোডে কনফিগার করা থাকে এবং নেটওয়ার্কের DHCP সার্ভার থেকে আইপি পেতে ব্যর্থ হয়, তবে ওএস নিজে থেকেই একটি বিশেষ রেঞ্জের আইপি অ্যাসাইন করে নেয়। রেঞ্জটি হলো: 169.254.0.0 — 169.254.255.255।

ক্রিটিক্যাল ট্রাবলশুটিং লজিক

যদি ipconfig বা ip a দেওয়ার পর দেখেন ক্লায়েন্ট পিসির আইপি শুরু হয়েছে 169.254.x.x দিয়ে, তবে বুঝবেন পিসির ক্যাবল বা ওয়াই-ফাই কানেকশন সচল থাকলেও সেটি আপনার রাউটার বা ডিএইচসিপি সার্ভারের সাথে যোগাযোগ করতে পারছে না। এটি কোনো ভ্যালিড ইন্টারনেট বা লোকাল কর্পোরেট আইপি নয়।

৬. নেটওয়ার্ক কানেক্টিভিটি ভেরিফিকেশন ও ট্রাবলশুটিং স্টেপস

প্যারামিটার চেক করার পর কানেক্টিভিটি শতভাগ নিশ্চিত করতে ধাপে ধাপে নিচের টেকনিক্যাল স্টেপগুলো অনুসরণ করুন:

ধাপ ১: লোকাল TCP/IP স্ট্যাক ভেরিফাই করা (Loopback Test)

আপনার ওএস-এর ইন্টারনাল নেটওয়ার্ক কার্ডের সফটওয়্যার ড্রাইভার ঠিক আছে কি না তা পরীক্ষা করতে লুপব্যাক আইপিতে পিং করুন।

# Windows & Linux উভয় ওএস-এর জন্য সাধারণ কমান্ড
ping 127.0.0.1

ব্যাখ্যা: এখানে যদি রিপ্লাই সাকসেসফুল আসে, তবে বুঝবেন আপনার ওএস-এর নেটওয়ার্কিং সফটওয়্যার লেয়ার সম্পূর্ণ ঠিক আছে।

ধাপ ২: Default Gateway-তে পিং করা

আপনার পিসি থেকে ফিজিক্যাল ক্যাবল বা সুইচের মাধ্যমে রাউটার পর্যন্ত ডেটা পৌঁছাচ্ছে কি না তা পরীক্ষা করুন।

ping 192.168.1.1

ব্যাখ্যা: (এখানে আপনার গেটওয়ে আইপি বসাবেন) যদি পিং ড্রপ করে, তবে ক্যাবল ড্যামেজ, লুজ কানেকশন বা ভুল সাবনেট মাস্কের কারণে এই সমস্যা হতে পারে।

ধাপ ৩: পাবলিক আইপিতে পিং করা (ইন্টারনেট চেক)

রাউটার পার হয়ে বাইরের ইন্টারনেট দুনিয়ায় আপনার পিসির ডেটা প্যাকেট যাচ্ছে কি না তা নিশ্চিত হোন।

ping 8.8.8.8

ব্যাখ্যা: 8.8.8.8 হলো গুগলের পাবলিক ডিএনএস। এখানে পিং সফল হওয়ার অর্থ হলো আপনার পিসির আইপি, মাস্ক এবং গেটওয়ে একদম নিখুঁতভাবে কাজ করছে এবং ইন্টারনেটের সাথে সংযোগ আছে।

ধাপ ৪: DNS নাম রেজল্যুশন টেস্ট করা

সবশেষে ডোমেইন নেম রেজল্যুশন ঠিক আছে কি না তা পরীক্ষা করুন:

nslookup tech-enablement.info

ব্যাখ্যা: যদি এই কমান্ডটি ডোমেইনের সঠিক আইপি অ্যাড্রেস বের করে দেখাতে পারে, তবে আপনার ক্লায়েন্ট ওএস-এর সমস্ত আইপি প্যারামিটার ১০০% পারফেক্ট।

৭. প্রফেশনাল ট্রাবলশুটিং টিপস

  • IP Conflict: যদি নেটওয়ার্কে একই আইপি অন্য কোনো ডিভাইসে কনফিগার করা থাকে, তবে উইন্ডোজ "IP Address Conflict" এরর দেবে। লিনাক্সে এটি চেক করার জন্য arping টুল ব্যবহার করে ডুপ্লিকেট ম্যাক অ্যাড্রেস সনাক্ত করা যায়।
  • DHCP IP Lease Renew: উইন্ডোজ ক্লায়েন্ট যদি ভুল আইপি ধরে রাখে, তবে CLI থেকে আইপি রিলিজ এবং রিনিউ করার কমান্ড: 
    C:\> ipconfig /release
C:\> ipconfig /renew
  • Linux Network Manager Restart: লিনাক্সে স্ট্যাটিক আইপি পরিবর্তনের পর নেটওয়ার্ক সার্ভিস রিস্টার্ট করার কমান্ড: 
    $ sudo systemctl restart NetworkManager

উপসংহার (Conclusion)

যেকোনো এন্টারপ্রাইজ বা লোকাল নেটওয়ার্কের ইঞ্জিনিয়ারিংয়ে Verify IP Parameters অত্যন্ত বেসিক কিন্তু সবচেয়ে শক্তিশালী ডায়াগনস্টিক স্কিল। Windows এবং Linux Client OS-এর কমান্ড লাইন টুলস যেমন ipconfig, ip a, এবং nslookup-এর ওপর নিখুঁত নিয়ন্ত্রণ থাকলে আপনি যেকোনো নেটওয়ার্ক ডাউনটাইম বা কানেক্টিভিটি ইস্যু মুহূর্তের মধ্যে সনাক্ত এবং সমাধান করে ফেলতে পারবেন। নিয়মিত এই প্যারামিটারগুলো অ্যানালাইসিস করার অভ্যাসই আপনাকে একজন দক্ষ নেটওয়ার্ক এবং সিস্টেম অ্যাডমিনিস্ট্রেটর হিসেবে গড়ে তুলবে।
