Explain Function of Network Component Router

12 May, 2026 Networking Concepts Jobaer Hossain

নেটওয়ার্কিং জগতের সবচেয়ে বুদ্ধিমান এবং অপরিহার্য হার্ডওয়্যার ডিভাইস হলো Router। আমরা যখন ইন্টারনেট ব্যবহার করি, তখন আমাদের ডাটা প্যাকেটগুলো হাজার হাজার মাইল পাড়ি দিয়ে সঠিক গন্তব্যে পৌঁছায় মূলত এই রাউটারের কারণে। এটি শুধুমাত্র একটি কানেক্টিভিটি ডিভাইস নয়, বরং এটি একটি শক্তিশালী কম্পিউটার যা ডাটা ট্রাফিক নিয়ন্ত্রণ করে।

আজকের এই টিউটোরিয়ালে আমরা আলোচনা করবো একটি রাউটারের মূল ফাংশনগুলো কী কী এবং এটি পর্দার আড়ালে কীভাবে কাজ করে।

১. Router কী?

রাউটার হলো একটি Layer 3 (Network Layer) ডিভাইস যা একাধিক ভিন্ন ভিন্ন নেটওয়ার্ককে একে অপরের সাথে যুক্ত করে। এটি আইপি অ্যাড্রেস (IP Address) ব্যবহার করে সিদ্ধান্ত নেয় যে ডাটা প্যাকেটটি কোন পথে পাঠালে দ্রুত পৌঁছাবে। সচরাচর আমাদের বাসায় যে রাউটার থাকে তা আমাদের লোকাল নেটওয়ার্ককে (LAN) ইন্টারনেটের (WAN) সাথে যুক্ত করে।

রিয়েল-লাইফ সিনারিও (The Postal System Analogy)

সহজভাবে বোঝার জন্য ডাকঘর বা কুরিয়ার সার্ভিসের কথা চিন্তা করুন:

  • আপনার পিসি বা স্মার্টফোন হলো একটি বাড়ি যেখান থেকে চিঠি (Data Packet) পাঠানো হচ্ছে।
  • রাউটার হলো ওই এলাকার প্রধান ডাকঘর।
  • ডাকঘর চিঠির উপরের ঠিকানা (Destination IP) দেখে সিদ্ধান্ত নেয় এটি কোন জেলার ডাকঘরে পাঠাতে হবে।
  • এক ডাকঘর থেকে অন্য ডাকঘরে চিঠি ভ্রমণ করে শেষ পর্যন্ত প্রাপকের কাছে পৌঁছায়। এই পুরো প্রক্রিয়াই হলো রাউটিং।

২. রাউটারের প্রধান ফাংশনসমূহ (Main Functions)

একটি রাউটার মূলত তিনটি প্রধান কাজ সম্পন্ন করে:

  1. Path Determination: রাউটার তার Routing Table পরীক্ষা করে গন্তব্যে পৌঁছানোর জন্য সবচেয়ে কম দূরত্বের বা সবচেয়ে ভালো পথটি খুঁজে বের করে।
  2. Packet Forwarding: গন্তব্য খুঁজে পাওয়ার পর রাউটার প্যাকেটটিকে তার সঠিক Interface বা পোর্টের দিকে ঠেলে দেয়।
  3. Broadcasting Control: রাউটার ব্রডকাস্ট ট্রাফিককে এক নেটওয়ার্ক থেকে অন্য নেটওয়ার্কে যেতে বাধা দেয়, ফলে নেটওয়ার্ক জ্যাম বা Broadcast Storm তৈরি হয় না।

৩. রাউটার কীভাবে কাজ করে? (Mechanism)

রাউটার যখন কোনো ডাটা প্যাকেট পায়, সে নিচের ধাপগুলো অনুসরণ করে:

  • Step 1: প্যাকেটের ভেতরে থাকা গন্তব্য আইপি অ্যাড্রেস চেক করে।
  • Step 2: তার Routing Table-এ ওই আইপির জন্য কোনো পথ আছে কি না তা দেখে।
  • Step 3: যদি কোনো স্পেসিফিক রুট না থাকে, তবে সে প্যাকেটটিকে তার Default Gateway-তে পাঠিয়ে দেয়।
  • Step 4: রাউটার প্যাকেটের TTL (Time to Live) ভ্যালু ১ কমিয়ে দেয় যাতে প্যাকেটটি নেটওয়ার্কে লুপ তৈরি না করে।
ফিচার (Feature) রাউটারের ভূমিকা
OSI Layer এটি Layer 3 (Network Layer)-এ কাজ করে।
Addressing এটি IP Address নিয়ে কাজ করে (MAC নয়)।
Routing Protocols পথ নির্বাচনের জন্য OSPF, BGP, বা RIP ব্যবহার করে।

৪. রাউটারের ইন্টারনাল কম্পোনেন্ট

রাউটার যেহেতু একটি বিশেষ কম্পিউটার, তাই এর ভেতরেও হার্ডওয়্যার থাকে:

  • CPU: প্রোটোকল এবং অ্যালগরিদম চালানোর জন্য।
  • RAM: Routing Table এবং ARP Cache জমা রাখার জন্য।
  • NVRAM: রাউটারের কনফিগারেশন ফাইল (Startup-Config) সেভ রাখার জন্য।
  • Interfaces: যেমন WAN পোর্ট (ইন্টারনেটের জন্য) এবং LAN পোর্ট (লোকাল পিসির জন্য)।

৫. Cisco Router CLI এবং বেসিক অপারেশন

পেশাদার রাউটারে কাজ করার জন্য আমাদের কমান্ড লাইন ব্যবহার করতে হয়।

রাউটিং টেবিল চেক করার কমান্ড:

# সিসকো রাউটারে রুট দেখার জন্য
Router# show ip route

# মাইক্রোটিক রাউটারে রুট দেখার জন্য
/ip route print

বেসিক কনফিগারেশন কমান্ড (Cisco):

# ইন্টারফেসে আইপি সেট করা
Router(config)# interface gigabitEthernet 0/0
Router(config-if)# ip address 192.168.10.1 255.255.255.0
Router(config-if)# no shutdown

৬. Troubleshooting টিপস

রাউটারে কোনো সমস্যা হলে একজন ইঞ্জিনিয়ার যেভাবে চেক করেন:

  • Layer 1: ইন্টারফেসের লাইট জ্বলছে কি না এবং ক্যাবল ঠিক আছে কি না দেখুন।
  • Layer 2: ইন্টারফেসটি Up/Up স্ট্যাটাসে আছে কি না নিশ্চিত হোন।
  • Layer 3: ping এবং traceroute কমান্ড দিয়ে দেখুন ডাটা কোথায় ড্রপ হচ্ছে।

উপসংহার (Conclusion)

একটি নেটওয়ার্কের প্রাণকেন্দ্র হলো Router। এটি শুধুমাত্র ডাটা পাস করে না, বরং সঠিক দিকনির্দেশনা দিয়ে পুরো ইন্টারনেট ব্যবস্থাকে সচল রাখে। রাউটারের ফাংশন এবং এর Packet Processing মেকানিজম বোঝা একজন নেটওয়ার্ক ইঞ্জিনিয়ার হওয়ার প্রথম ধাপ। আপনি যদি CCNA বা উচ্চতর লেভেলের নেটওয়ার্কিং শিখতে চান, তবে রাউটারের প্রতিটি লজিক্যাল ফাংশন সম্পর্কে স্বচ্ছ ধারণা থাকা জরুরি। নিয়মিত ল্যাব প্র্যাকটিস এবং Routing Table অ্যানালাইসিসই আপনাকে একজন দক্ষ প্রফেশনাল হিসেবে গড়ে তুলবে।
