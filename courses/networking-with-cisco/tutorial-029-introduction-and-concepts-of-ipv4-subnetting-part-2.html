Introduction and Concepts of IPv4 Subnetting Part-2

17 May, 2026 CCNA Tutorials Jobaer Hossain

আমাদের সাবনেটিং সিরিজের প্রথম পার্টে আমরা সাবনেটিংয়ের তাত্ত্বিক ভিত্তি, এর প্রয়োজনীয়তা এবং Bit Borrowing বা বিট ধার নেওয়ার মূল লজিক নিয়ে আলোচনা করেছিলাম। আমরা জেনেছিলাম কীভাবে Host Portion থেকে বিট ধার নিয়ে নতুন Subnet তৈরি করা হয়। তাত্ত্বিক ধারণা স্পষ্ট থাকার পর, এবার আমাদের প্র্যাকটিক্যাল গাণিতিক হিসাব এবং রিয়েল-লাইফ নেটওয়ার্ক ডিজাইনে এর প্রয়োগ শেখার সময় এসেছে।

আজকের এই "পার্ট-২" টিউটোরিয়ালে আমরা একটি নির্দিষ্ট Class C নেটওয়ার্ক ব্লক নিয়ে সেটিকে ধাপে ধাপে সাবনেট করা শিখবো। একই সাথে আমরা আলোচনা করবো এমন একটি শর্টকাট টেকনিক বা Magic Number Method নিয়ে, যা ব্যবহার করে আপনারা খাতা-কলম বা বাইনারি কনভার্সন ছাড়াই মাত্র কয়েক সেকেন্ডে যেকোনো সাবনেটিংয়ের উত্তর বের করে ফেলতে পারবেন।

১. প্র্যাকটিক্যাল সাবনেটিং সিনারিও (The Scenario)

বাস্তবসম্মত একটি প্রজেক্ট দিয়ে শুরু করা যাক। ধরুন আপনি একটি মিডিয়া হাউসের নেটওয়ার্ক ডিজাইন করছেন। আপনার ইন্টারনেট সার্ভিস প্রোভাইডার (ISP) আপনাকে একটি Class C নেটওয়ার্ক ব্লক দিয়েছে: 192.168.10.0/24।

আপনার রিকোয়ারমেন্ট হলো, এই একক ব্লকটিকে ভেঙে এমনভাবে সাবনেট তৈরি করতে হবে যেন প্রতিটি সাবনেটে কমপক্ষে ৩০টি করে ভ্যালিড হোস্ট আইপি সচল থাকে।

৫টি কোর প্রশ্ন যা আমাদের সমাধান করতে হবে:

  1. আমাদের নতুন কাস্টম Subnet Mask বা CIDR ভ্যালু কত হবে?
  2. হোস্ট বিট থেকে আমাদের ঠিক কতটি বিট ধার (Borrow) করতে হবে?
  3. এই প্রক্রিয়ায় মোট কতটি নতুন Subnet তৈরি হবে?
  4. প্রতিটি সাবনেটে মোট ব্যবহারযোগ্য (Usable) আইপি সংখ্যা কত হবে?
  5. তৈরি হওয়া সাবনেটগুলোর Network ID, Usable IP Range এবং Broadcast ID-র চার্টটি কেমন হবে?

২. ধাপে ধাপে গাণিতিক সমাধান (Step-by-Step Calculation)

আসুন পার্ট-১ এ শেখা ৩টি গোল্ডেন রুলস এবং বাইনারি পজিশনাল ভ্যালু ব্যবহার করে সমস্যাটি সমাধান করি।

ধাপ ১: প্রয়োজনীয় হোস্ট বিট ($h$) নির্ধারণ করা

আমাদের প্রতিটি সাবনেটে অন্তত ৩০টি ব্যবহারযোগ্য আইপি লাগবে। আমরা জানি ভ্যালিড হোস্ট বের করার সূত্র হলো: $2^h - 2$।

এখন আমাদের $h$-এর এমন একটি মান বের করতে হবে যেন তা ৩০ বা তার চেয়ে বড় হয়:

  • যদি $h = 4$ হয়, তবে $2^4 - 2 = 16 - 2 = 14$ (রিকোয়ারমেন্ট পূরণ হয় না)।
  • যদি $h = 5$ হয়, তবে $2^5 - 2 = 32 - 2 = 30$ (রিকোয়ারমেন্ট একদম নিখুঁতভাবে পূরণ হয়)।

সুতরাং, আমাদের প্রতিটি সাবনেটের জন্য অবশিষ্টাংশ হোস্ট বিট রাখতে হবে $h = 5$।

ধাপ ২: ধার নেওয়া বিটের সংখ্যা ($n$) বের করা

আমরা জানি একটি Class C নেটওয়ার্কের শেষ ওকটেটে (৪র্থ ওকটেট) মোট ৮টি হোস্ট বিট থাকে।

যেহেতু আমরা হোস্টের জন্য ৫টি বিট ফিক্সড করে ফেলেছি, তাই বাকি বিটগুলো আমরা নেটওয়ার্কের জন্য ধার নিতে পারি:

Borrowed Bits ($n$) = Total Host Bits - Remaining Host Bits = $8 - 5 = 3$ বিট।

অর্থাৎ, আমরা হোস্টের অংশ থেকে ৩টি বিট ধার নিলাম।

ধাপ ৩: মোট তৈরি হওয়া সাবনেটের সংখ্যা বের করা

ধার নেওয়া বিটের সংখ্যা থেকে সাবনেট বের করার সূত্র হলো: $2^n$।

যেহেতু $n = 3$, তাই মোট সাবনেট সংখ্যা = $2^3 = 8$টি।

তার মানে, 192.168.10.0/24 ব্লকটিকে ভাঙলে আমরা মোট ৮টি ছোট ছোট নেটওয়ার্ক পাবো।

ধাপ ৪: নতুন কাস্টম Subnet Mask বের করা

ডিফল্ট Class C নেটওয়ার্কের মাস্ক হলো /24 (অর্থাৎ প্রথম ২৪টি বিট ১)। যেহেতু আমরা আরও ৩টি বিট ধার নিয়ে নেটওয়ার্কের সাথে যুক্ত করেছি, তাই আমাদের নতুন CIDR ভ্যালু হবে: $24 + 3 = /27$।

আসুন এই /27 কে ডেসিমাল সাবনেট মাস্কে রূপান্তর করি। এর অর্থ হলো ৪র্থ ওকটেটের প্রথম ৩টি বিট 1 এবং বাকি ৫টি বিট 0।

৪র্থ ওকটেটের বাইনারি রূপ: 1 1 1 0 0 0 0 0

আমরা জানি প্রথম ৩টি বাইনারি পজিশনের মান হলো ১২৮, ৬৪ এবং ৩২। এদের যোগ করলে পাই:

$128 + 64 + 32 = 224$।

সুতরাং, আমাদের নতুন কাস্টম Subnet Mask হলো: 255.255.255.224।

৩. শর্টকাট টেকনিক: The Magic Number Method

বাস্তব কর্মক্ষেত্রে বা পরীক্ষার হলে এত বড় বাইনারি হিসাব করার সময় আমাদের থাকে না। তাই আমরা ব্যবহার করবো Magic Number Method।

ম্যাজিক নাম্বার বের করার নিয়ম:

আপনার কাস্টম ডেসিমাল সাবনেট মাস্কের যে ওকটেটে পরিবর্তন এসেছে (এখানে ৪র্থ ওকটেট, যার মান ২২৪), সেটিকে ২৫৬ (256) থেকে বিয়োগ করতে হবে।

Magic Number = 256 - Custom Subnet Mask Value

আমাদের এই উদাহরণের জন্য: $256 - 224 = 32$।

এই 32-ই হলো আমাদের ম্যাজিক নাম্বার বা ব্লক সাইজ (Block Size)। এর মানে হলো, আমাদের তৈরি হওয়া সাবনেটগুলোর নেটওয়ার্ক আইডি ০ থেকে শুরু হয়ে ৩২ ঘর পর পর লাফিয়ে (Increment) এগিয়ে যাবে।

৪. সম্পূর্ণ সাবনেট টেবিল (The Subnetting Chart)

ম্যাজিক নাম্বার ৩২ ব্যবহার করে আসুন আমরা আমাদের তৈরি হওয়া ৮টি সাবনেটের সম্পূর্ণ রেঞ্জ চার্টটি তৈরি করে ফেলি:

Subnet # Network ID First Usable IP Last Usable IP Broadcast ID
Subnet 1 192.168.10.0 192.168.10.1 192.168.10.30 192.168.10.31
Subnet 2 192.168.10.32 192.168.10.33 192.168.10.62 192.168.10.63
Subnet 3 192.168.10.64 192.168.10.65 192.168.10.94 192.168.10.95
Subnet 4 192.168.10.96 192.168.10.97 192.168.10.126 192.168.10.127
Subnet 5 192.168.10.128 192.168.10.129 192.168.10.158 192.168.10.159
Subnet 6 192.168.10.160 192.168.10.161 192.168.10.190 192.168.10.191
Subnet 7 192.168.10.192 192.168.10.193 192.168.10.222 192.168.10.223
Subnet 8 192.168.10.224 192.168.10.225 192.168.10.254 192.168.10.255

চার্ট তৈরির টেকনিক্যাল ব্যাখ্যা:

  • Network ID: প্রথম সাবনেট সবসময় ০ থেকে শুরু হয়। পরবর্তী সাবনেট আইডি বের করতে আগের আইডির সাথে ম্যাজিক নাম্বার (৩২) যোগ করতে হয় (যেমন- ০, ৩২, ৬৪, ৯৬...)।
  • Broadcast ID: পরবর্তী সাবনেটের Network ID থেকে ১ বিয়োগ করলে আগের সাবনেটের Broadcast ID পাওয়া যায় (যেমন- দ্বিতীয় সাবনেট ৩২ হলে, প্রথমটির ব্রডকাস্ট ৩১)।
  • Usable IP Range: Network ID-র ঠিক পরের আইপি থেকে শুরু করে Broadcast ID-র ঠিক আগের আইপি পর্যন্ত হলো পিসিতে বসানোর মতো ভ্যালিড হোস্ট আইপি।

৫. Cisco Router CLI-তে সাবনেট ইন্টারফেস কনফিগারেশন

এবার আসুন আমরা আমাদের তৈরি করা চার্টের "Subnet 2" ব্লকটি একটি Cisco Router-এর গিগাবিট ইন্টারফেসে প্র্যাকটিক্যালি কনফিগার করি এবং ভেরিফাই করি। আমরা এই সাবনেটের প্রথম ভ্যালিড আইপিটি রাউটারের গেটওয়ে হিসেবে ব্যবহার করবো।

# গ্লোবাল কনফিগারেশন মোডে প্রবেশ করুন
Router> enable
Router# configure terminal

# নির্দিষ্ট ফিজিক্যাল ইন্টারফেসে প্রবেশ
Router(config)# interface gigabitEthernet 0/1

# সাবনেট মাস্কসহ কাস্টম আইপি অ্যাসাইন করা
Router(config-if)# ip address 192.168.10.33 255.255.255.224
Router(config-if)# description Gateway_For_Subnet_2
Router(config-if)# no shutdown
Router(config-if)# exit
Router(config)# exit

কনফিগারেশন ভেরিফিকেশন (Verification Commands):

পোর্টটি সঠিক মাস্ক পেয়েছে কি না এবং সচল আছে কি না তা দেখার জন্য নিচের কমান্ডটি দিন:

Router# show ip interface brief gigabitEthernet 0/1

রাউটার এই সাবনেটটিকে তার রাউটিং টেবিলে কীভাবে এন্ট্রি নিয়েছে তা দেখার কমান্ড:

Router# show ip route

আউটপুটের ভেতরের লাইনে সুনির্দিষ্টভাবে সাবনেট মাস্কসহ রুটটি দেখতে পাবেন:

192.168.10.0/27 is variably subnetted, 8 subnets, 1 masks
C    192.168.10.32/27 is directly connected, GigabitEthernet 0/1

৬. Troubleshooting ও সাধারণ সাবনেটিং ত্রুটিসমূহ

  • The "Bad Mask" Error: আপনি যদি কোনো ডিভাইসে আইপি বসানোর সময় ভুল করে সাবনেট মাস্ক 255.255.255.230 লিখে ফেলেন, তবে ওএস এরর দেবে। সাবনেট মাস্কের বাইনারি বিট সব সময় একাদিক্রমে 1 হতে হয়, মাঝখানে কোনো ০ বসতে পারে না। ভ্যালিড মাস্ক নম্বরগুলো হলো ১২৮, ১৯২, ২২৪, ২৪০, ২৪৮, ২৫২, ২৫৪ এবং ২৫৫।
  • Host Mask Mismatch on Clients: বাস্তব ক্ষেত্রে নেটওয়ার্কের সবচেয়ে বড় ঝামেলা হয় যখন একটি পিসিতে সাবনেট মাস্ক দেওয়া হয় 255.255.255.224 (/27) কিন্তু পাশের অন্য পিসিতে ভুলবশত ডিফল্ট 255.255.255.0 (/24) রেখে দেওয়া হয়। এর ফলে পিসিগুলো একই সুইচে কানেক্টেড থাকলেও একে অপরের সাথে যোগাযোগ করতে পারবে না, কারণ তাদের লোকাল নেটওয়ার্ক বাউন্ডারি আলাদা হয়ে গেছে। সবসময় পুরো সাবনেটের সব ডিভাইসে একই মাস্ক নিশ্চিত করুন।
  • Invalid Gateway Assignment: যদি আপনি কোনো পিসির আইপি সেটিংসের Default Gateway-র ঘরে ওই সাবনেটের Network ID (যেমন- ৩২) বা Broadcast ID (যেমন- ৬৩) বসিয়ে দেন, তবে লোকাল নেটওয়ার্ক আংশিক কাজ করলেও ইন্টারনেট ট্রাফিক সম্পূর্ণ ড্রপ করবে। গেটওয়ে আইপি সবসময় ভ্যালিড ইউজেবল রেঞ্জের ভেতরের হতে হবে।

উপসংহার (Conclusion)

IPv4 Subnetting Part-2-এর এই প্র্যাকটিক্যাল গাইডটির মাধ্যমে আমরা শিখলাম কীভাবে একটি নির্দিষ্ট রিকোয়ারমেন্টের ওপর ভিত্তি করে ক্লাস সি নেটওয়ার্ককে বাইনারি এবং Magic Number Method-এর সাহায্যে নিখুঁতভাবে ভেঙে ছোট ছোট সাবনেটে রূপান্তর করতে হয়। এই ম্যাজিক নাম্বারের ট্রিকটি আয়ত্ত করতে পারলে যেকোনো বড় সাবনেটিংয়ের হিসাব চোখের পলকে সমাধান করা সম্ভব। সিসিএনএ পরীক্ষা এবং নেটওয়ার্কিং ক্যারিয়ারের সফলতার জন্য এই গাণিতিক চার্ট তৈরি করতে পারা অত্যন্ত মৌলিক একটি স্কিল। নিয়মিত ল্যাব প্র্যাকটিস এবং ভিন্ন ভিন্ন আইপি ব্লক নিয়ে নিজে নিজে চার্ট তৈরি করার অভ্যাসই আপনাকে একজন দক্ষ নেটওয়ার্ক আর্কিটেক্ট হিসেবে গড়ে তুলবে।
