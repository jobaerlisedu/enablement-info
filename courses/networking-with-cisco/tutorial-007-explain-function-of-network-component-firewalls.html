Explain Function of Network Component Firewalls

12 May, 2026 Cyber Security Jobaer Hossain

সাইবার সিকিউরিটির দুনিয়ায় সবচেয়ে পরিচিত এবং অপরিহার্য একটি শব্দ হলো Firewall। আমরা যখন একটি নেটওয়ার্ক তৈরি করি, তখন সেটি ইন্টারনেটের সাথে যুক্ত হওয়ার সাথে সাথে হাজারো নিরাপত্তা ঝুঁকির মুখে পড়ে। এই ঝুঁকি বা আক্রমণ থেকে নেটওয়ার্ককে রক্ষা করার জন্য প্রথম প্রতিরক্ষা প্রাচীর হিসেবে কাজ করে ফায়ারওয়াল। এটি মূলত একটি ফিল্টারিং সিস্টেম যা নেটওয়ার্কে আসা এবং নেটওয়ার্ক থেকে যাওয়া প্রতিটি ডাটা প্যাকেটকে গভীরভাবে পরীক্ষা করে।

আজকের এই টিউটোরিয়ালে আমরা আলোচনা করবো ফায়ারওয়ালের মূল ফাংশনসমূহ, এর বিভিন্ন প্রকারভেদ এবং এটি কীভাবে একটি প্রতিষ্ঠানের ডাটা সুরক্ষিত রাখে।

১. Network Firewall কী?

ফায়ারওয়াল হলো একটি সিকিউরিটি সিস্টেম (যা হার্ডওয়্যার বা সফটওয়্যার উভয়ই হতে পারে) যা একটি বিশ্বাসযোগ্য ইন্টারনাল নেটওয়ার্ক এবং একটি অবিশ্বস্ত এক্সটারনাল নেটওয়ার্কের (যেমন ইন্টারনেট) মাঝে দেয়াল হিসেবে দাঁড়িয়ে থাকে। এটি পূর্বনির্ধারিত Security Rules এর ওপর ভিত্তি করে ইনকামিং এবং আউটগোয়িং ট্রাফিক মনিটর এবং কন্ট্রোল করে।

রিয়েল-লাইফ সিনারিও (The Security Guard Analogy)

সহজভাবে বোঝার জন্য একটি বড় অ্যাপার্টমেন্ট বা অফিসের মেইন গেটের কথা চিন্তা করুন:

  • Security Guard: ফায়ারওয়াল হলো ওই গেটের সিকিউরিটি গার্ড।
  • Visitor List: তার কাছে একটি তালিকা (Security Rules) আছে যে কারা ভেতরে ঢুকতে পারবে আর কারা পারবে না।
  • Checking: কেউ ভেতরে ঢুকতে চাইলে গার্ড তার পরিচয়পত্র পরীক্ষা করে। যদি তালিকার সাথে মিলে যায় তবেই ঢুকতে দেয়, অন্যথায় গেট থেকেই বিদায় করে দেয়।

২. ফায়ারওয়ালের প্রধান ফাংশনসমূহ (Main Functions)

একটি আধুনিক ফায়ারওয়াল মূলত নিচের কাজগুলো সম্পন্ন করে থাকে:

  1. Packet Filtering: এটি প্রতিটি প্যাকেটের Source IP, Destination IP, এবং Port Number পরীক্ষা করে সিদ্ধান্ত নেয় প্যাকেটটি পাস হবে কি না।
  2. Stateful Inspection: এটি শুধুমাত্র একটি প্যাকেট দেখে না, বরং একটি কানেকশনের পুরো ইতিহাস (State) মনে রাখে। যেমন- আপনি যদি গুগল ভিজিট করেন, তবে গুগল থেকে আসা রিপ্লাই প্যাকেটটি সে অটোমেটিক চিনতে পারে।
  3. Application Control: আধুনিক ফায়ারওয়াল (Next-Gen) নির্দিষ্ট অ্যাপ্লিকেশনের ওপর ভিত্তি করে ট্রাফিক ব্লক করতে পারে। যেমন- আপনি চাইলে পুরো ইন্টারনেট খোলা রেখে শুধু Facebook বা YouTube ব্লক করে রাখতে পারেন।
  4. NAT (Network Address Translation): ফায়ারওয়াল আপনার লোকাল নেটওয়ার্কের প্রাইভেট আইপিগুলোকে ইন্টারনেটে হাইড করে রাখে, যা সিকিউরিটির একটি বড় অংশ।

৩. ফায়ারওয়ালের প্রকারভেদ (Types of Firewalls)

প্রযুক্তির বিবর্তনের সাথে সাথে ফায়ারওয়ালও বিভিন্ন ধরনের হয়ে থাকে:

Firewall Type কাজের ধরন (Mechanism)
Packet Filtering সবচেয়ে পুরনো পদ্ধতি। এটি শুধুমাত্র IP এবং Port দেখে ট্রাফিক চেক করে। এটি দ্রুত কিন্তু কম সিকিউর।
Stateful Inspection এটি একটি কানেকশন বা সেশনের ট্র্যাকিং করে। এটি আগেরটির চেয়ে অনেক বেশি বুদ্ধিমান।
Proxy Firewall এটি ইউজার এবং সার্ভারের মাঝখানে মধ্যস্থতাকারী হিসেবে কাজ করে। ইউজার সরাসরি সার্ভারের সাথে কথা বলতে পারে না।
Next-Generation (NGFW) বর্তমানে এটি সবচেয়ে জনপ্রিয়। এতে IPS (Intrusion Prevention System) এবং Deep Packet Inspection থাকে।

৪. কেন ফায়ারওয়াল ব্যবহার করা বাধ্যতামূলক?

আপনার নেটওয়ার্কে ফায়ারওয়াল না থাকলে নিচের সমস্যাগুলো হতে পারে:

  • Unauthorized Access: হ্যাকাররা সরাসরি আপনার পিসি বা সার্ভারে ঢুকে ডাটা চুরি করতে পারবে।
  • Malware Attacks: ক্ষতিকারক ভাইরাস বা ম্যালওয়্যার আপনার নেটওয়ার্কে ছড়িয়ে পড়তে পারে।
  • Data Leakage: অফিসের গোপন ডাটা ইন্টারনেটে পাচার হয়ে যেতে পারে।
  • Bandwidth Waste: আনওয়ান্টেড ট্রাফিক আপনার নেটওয়ার্ক ব্যান্ডউইথ দখল করে রাখবে।

৫. Firewall Configuration (MikroTik CLI Example)

একটি সাধারণ ফায়ারওয়াল রুল কীভাবে কাজ করে তা বোঝার জন্য মাইক্রোটিক রাউটারের একটি উদাহরণ নিচে দেওয়া হলো।

একটি নির্দিষ্ট আইপি ব্লক করার কমান্ড:

# ১৯২.১৬৮.১০.৫০ আইপিটিকে ইন্টারনেট থেকে ব্লক করা
/ip firewall filter add chain=forward src-address=192.168.10.50 action=drop

ব্যান্ডউইথ নষ্ট করা সার্ভিস ব্লক করা (যেমন- YouTube/Social Media):

# কন্টেন্ট ফিল্টারিং এর মাধ্যমে ব্লক করা
/ip firewall filter add chain=forward content="youtube.com" action=drop

৬. Troubleshooting ও বেস্ট প্র্যাকটিস

ফায়ারওয়াল কনফিগার করার সময় নিচের বিষয়গুলো মাথায় রাখা জরুরি:

  • Least Privilege: ডিফল্টভাবে সবকিছু ব্লক রাখুন এবং শুধুমাত্র যা প্রয়োজন তা পারমিট করুন।
  • Rule Order: ফায়ারওয়াল রুলগুলো সবসময় টপ-টু-বটম (উপর থেকে নিচে) কাজ করে। তাই গুরুত্বপূর্ণ রুলগুলো উপরে রাখুন।
  • Logging: সবসময় ড্রপ হওয়া প্যাকেটের Log এনাবল রাখুন যাতে পরবর্তীতে কেউ অ্যাটাক করার চেষ্টা করলে আপনি সেটি দেখতে পারেন।
  • Updates: আপনার ফায়ারওয়াল ডিভাইসের ফার্মওয়্যার সবসময় আপ-টু-ডেট রাখুন।

উপসংহার (Conclusion)

Network Firewall হলো আপনার ডিজিটাল জগতের বডিগার্ড। এটি ছাড়া বর্তমান সময়ে একটি নেটওয়ার্ক কল্পনা করা অসম্ভব। আপনি যত ভালো ডিজাইনের নেটওয়ার্কই তৈরি করুন না কেন, সঠিক ফায়ারওয়াল পলিসি না থাকলে আপনার পুরো অবকাঠামো ঝুঁকির মুখে থাকবে। তাই একজন নেটওয়ার্ক বা সিকিউরিটি ইঞ্জিনিয়ার হিসেবে ফায়ারওয়ালের Filtering Logic এবং Next-Gen Features সম্পর্কে গভীর জ্ঞান থাকা আপনার ক্যারিয়ারের জন্য অত্যন্ত ফলপ্রসূ হবে।
