Explain Function of Network Component Servers

12 May, 2026 Networking Concepts Jobaer Hossain

কম্পিউটার নেটওয়ার্কিংয়ের জগতে Server হলো এমন একটি শক্তিশালী ডিভাইস বা প্রোগ্রাম যা অন্যান্য কম্পিউটারকে (যাদের আমরা Client বলি) বিভিন্ন রিসোর্স, ডাটা বা সার্ভিস প্রদান করে। ইন্টারনেটের প্রতিটি ওয়েবসাইট, ইমেইল সার্ভিস কিংবা অনলাইন গেম—সবকিছুর পেছনেই কাজ করে এক বা একাধিক সার্ভার। সহজ কথায়, সার্ভার হলো একটি নেটওয়ার্কের কেন্দ্রবিন্দু যা সব রিকোয়েস্ট প্রসেস করে।

আজকের এই টিউটোরিয়ালে আমরা আলোচনা করবো সার্ভারের মূল ফাংশনসমূহ, এর কাজের পদ্ধতি এবং আধুনিক আইটি ইনফ্রাস্ট্রাকচারে এর বিভিন্ন প্রকারভেদ নিয়ে।

১. Network Server কী?

সার্ভার হলো একটি উচ্চ-ক্ষমতাসম্পন্ন কম্পিউটার যা ২৪ ঘণ্টা চালু থাকে এবং নেটওয়ার্কে যুক্ত অন্যান্য ডিভাইসগুলোকে সার্ভিস দেওয়ার জন্য প্রস্তুত থাকে। হার্ডওয়্যার হিসেবে এটি একটি শক্তিশালী পিসি হতে পারে, আবার সফটওয়্যার হিসেবে এটি একটি প্রোগ্রামও হতে পারে যা নির্দিষ্ট কোনো প্রোটোকল (যেমন HTTP বা FTP) ব্যবহার করে কাজ করে।

রিয়েল-লাইফ সিনারিও (The Restaurant Analogy)

সহজভাবে বোঝার জন্য একটি রেস্টুরেন্টের কথা চিন্তা করুন:

  • Client: আপনি যখন রেস্টুরেন্টে গিয়ে খাবার অর্ডার করেন, আপনি হলেন একজন ক্লায়েন্ট।
  • Server: রেস্টুরেন্টের কিচেন বা শেফ হলো সার্ভার। সে আপনার অর্ডার (Request) নেয়, সেটি প্রস্তুত করে (Process) এবং আপনার টেবিলে খাবার পৌঁছে দেয় (Response)।
  • Network: ওয়েটার হলো নেটওয়ার্ক মাধ্যম যে আপনার এবং কিচেনের মধ্যে যোগাযোগ রক্ষা করে।

২. সার্ভারের প্রধান ফাংশনসমূহ (Main Functions)

একটি নেটওয়ার্ক সার্ভার মূলত নিচের কাজগুলো সম্পন্ন করে:

  1. Centralized Data Storage: সব গুরুত্বপূর্ণ ফাইল বা ডাটা এক জায়গায় জমা রাখে যাতে নেটওয়ার্কের অনুমোদিত ইউজাররা সেটি এক্সেস করতে পারে।
  2. Resource Sharing: এটি প্রিন্টার, ইন্টারনেট কানেকশন বা সফটওয়্যার লাইসেন্স শেয়ার করতে সাহায্য করে।
  3. Authentication: ইউজার যখন লগইন করে, সার্ভার তার ইউজারনেম এবং পাসওয়ার্ড ভেরিফাই করে নেটওয়ার্কে প্রবেশের অনুমতি দেয়।
  4. Hosting Services: ওয়েবসাইট হোস্টিং (Web Hosting) থেকে শুরু করে ইমেইল সার্ভিস ম্যানেজ করা পর্যন্ত সব কাজ সার্ভার করে।

৩. বিভিন্ন ধরনের সার্ভার (Types of Servers)

কাজের ওপর ভিত্তি করে সার্ভারকে কয়েক ভাগে ভাগ করা যায়:

Server Type কাজের ধরন (Function)
Web Server এটি ওয়েবসাইট হোস্ট করে এবং HTTP/HTTPS প্রোটোকল ব্যবহার করে ব্রাউজারে ডাটা পাঠায়।
File Server এটি নেটওয়ার্কের ভেতরে ফাইল শেয়ারিং এবং স্টোরেজ ম্যানেজমেন্টের কাজ করে।
Database Server এটি বিশাল পরিমাণ ডাটা (যেমন- ইউজার প্রোফাইল বা ট্রানজেকশন) জমা রাখে এবং কুয়েরি প্রসেস করে।
DNS Server এটি ডোমেইন নেমকে (যেমন google.com) আইপি অ্যাড্রেসে রূপান্তর করে।
Mail Server এটি ইমেইল আদান-প্রদান এবং স্টোর করার কাজ পরিচালনা করে (SMTP/IMAP/POP3)।

৪. সার্ভার কীভাবে কাজ করে? (Step-by-Step)

সার্ভার মূলত Request-Response মডেলে কাজ করে:

  • Step 1 (Request): ক্লায়েন্ট কম্পিউটার বা ব্রাউজার থেকে একটি নির্দিষ্ট রিকোয়েস্ট পাঠানো হয় (যেমন- একটি ওয়েবসাইট ভিজিট করা)।
  • Step 2 (Processing): সার্ভার রিকোয়েস্টটি পায় এবং তার ডাটাবেজ বা ফাইল সিস্টেমে প্রয়োজনীয় তথ্য খুঁজে বের করে।
  • Step 3 (Response): সার্ভার তথ্যটিকে নির্দিষ্ট ফরম্যাটে (যেমন- HTML বা JSON) ক্লায়েন্টের কাছে ফেরত পাঠায়।

৫. সার্ভার হার্ডওয়্যার বনাম সফটওয়্যার

একটি সার্ভার তৈরি করতে হলে দুটি জিনিসের সমন্বয় প্রয়োজন:

  • Hardware: এটি একটি শক্তিশালী মেশিন যার মাল্টি-কোর CPU, বিশাল পরিমাণ RAM এবং হাই-স্পিড SSD/HDD থাকে। সাধারণত এগুলো 'Rack-mount' বা 'Tower' ডিজাইনের হয়।
  • Operating System: সার্ভারে সাধারণ উইন্ডোজের বদলে বিশেষ ওএস ব্যবহৃত হয় যেমন— Windows Server, Ubuntu Server, বা Rocky Linux।

৬. বেসিক কনফিগারেশন ও অপারেশন (CLI Example)

লিনাক্স সার্ভারে কোনো সার্ভিস সচল আছে কি না তা দেখার জন্য আমাদের কমান্ড লাইন ব্যবহার করতে হয়।

সার্ভার স্ট্যাটাস চেক করার কমান্ড (Linux):

# Apache Web Server সচল আছে কি না তা দেখতে
$ systemctl status apache2

# সার্ভারের নেটওয়ার্ক পিন লিসেনিং চেক করতে
$ netstat -plntu

সার্ভার আইপি এবং কানেক্টিভিটি চেক:

# সার্ভারের আইপি অ্যাড্রেস দেখতে
$ hostname -I

# গেটওয়ের সাথে কানেক্টিভিটি চেক করতে
$ ping 8.8.8.8

৭. Troubleshooting ও মেইনটেন্যান্স টিপস

  • Overheating: সার্ভার সবসময় ঠাণ্ডা পরিবেশে (Server Room) রাখতে হয়, নাহলে হার্ডওয়্যার ফেইল করতে পারে।
  • Backups: সার্ভারের ডাটা নিয়মিত ব্যাকআপ রাখা বাধ্যতামূলক। লিনাক্সে rsync বা ক্লাউড ব্যাকআপ টুল ব্যবহার করা ভালো।
  • Security Updates: সার্ভারকে হ্যাকিং থেকে বাঁচাতে নিয়মিত ওএস এবং সার্ভিস আপডেট দিন (যেমন- apt update && apt upgrade)।
  • Downtime: যদি সার্ভার ডাউন হয়, তবে প্রথমে নেটওয়ার্ক কানেক্টিভিটি এবং তারপর সার্ভিস স্ট্যাটাস চেক করুন।

উপসংহার (Conclusion)

Servers হলো আধুনিক ইন্টারনেটের মেরুদণ্ড। একটি শক্তিশালী সার্ভার সেটআপ ছাড়া কোনো প্রতিষ্ঠান তাদের ডিজিটাল কার্যক্রম পরিচালনা করতে পারে না। আপনি যদি সিস্টেম অ্যাডমিনিস্ট্রেশন বা নেটওয়ার্ক ইঞ্জিনিয়ারিং শিখতে চান, তবে সার্ভারের ইন্টারনাল ফাংশন এবং এর Resource Management সম্পর্কে গভীর জ্ঞান থাকা অপরিহার্য। নিয়মিত প্র্যাকটিস এবং লিনাক্স কমান্ডের ওপর দখল আপনাকে একজন দক্ষ সার্ভার অ্যাডমিনিস্ট্রেটর হিসেবে গড়ে তুলবে।
